Service Monitor

// Status Dashboard · No Extra Software Needed //

A self-contained local HTML dashboard that checks the status of all your services and shows recent downloads. Two options: a static bookmark page, or a live-checking dashboard.

🐉 V1 Goal: A single dashboard.html file you can open locally (or serve from Nginx/Caddy) that shows all your services at a glance with live ping checks.
🗂️ Two Approaches
🟢 Option A — Pure HTML Dashboard (No Server)

A single dashboard.html file that uses JavaScript's fetch() to ping each service URL. Works entirely from a local file or any web server. No extra software.

Best for: Quick status checks, bookmarking, simple setup.

🔵 Option B — Homarr or Dashdot (Dedicated Dashboard App)

Installable Windows apps that provide richer dashboards with widgets, weather, Plex library stats, Minecraft player counts, etc.

Best for: A polished always-on display.

🛠️ Option A — Build Your Own Dashboard.html
How the Status Checking Works

Each service check sends a fetch() request to the service URL with a short timeout. If it resolves, the service is up. If it times out or returns an error, it's down. This works for any HTTP service on your network.

⚠️
CORS limitation: Browsers block cross-origin fetch from file:// URLs. For the cleanest experience, serve dashboard.html through a local web server (even just Python's built-in server: python -m http.server 9000 in the folder). Or use it within the index.html binder where it loads in an iframe from a file server.

Save dashboard.html to your server folder

Put it somewhere like D:\Server\Dashboard\dashboard.html.

Serve it with Python's built-in server (optional but recommended) 
# One-liner to serve the folder:
python -m http.server 9000 --directory "D:\Server\Dashboard"

Access at http://localhost:9000/dashboard.html. Make this a startup task (see below).

Create a startup task for the Python server

Create D:\Server\Dashboard\serve.bat:

@echo off
python -m http.server 9000 --directory "D:\Server\Dashboard"

Add this as a Task Scheduler startup task (same process as Minecraft startup task).

Bookmark it

Add http://[your-server-IP]:9000/dashboard.html to your browser's bookmark bar on every device.

const SERVICES = [
  { name: "Plex",            url: "http://localhost:32400/web", label: "Open Plex" },
  { name: "Calibre-Web",     url: "http://localhost:8083",      label: "Open Calibre" },

function buildCards() {
  const grid = document.getElementById('services');
  grid.innerHTML = SERVICES.map((s,i) => `
    <div class="card" id="card-${i}">
      <div class="card-title">${s.name}</div>
      <div class="status">
        <div class="dot checking" id="dot-${i}"></div>
        <span id="status-${i}">Checking...</span>
      </div>
      <a href="${s.url}" target="_blank" class="service-link">${s.label} ↗</a>
    </div>
  `).join('');
}

async function checkService(i) {
  const s = SERVICES[i];
  const dot = document.getElementById('dot-'+i);
  const stat = document.getElementById('status-'+i);
  dot.className = 'dot checking';
  stat.textContent = 'Checking...';
  try {
    const ctrl = new AbortController();
    const timer = setTimeout(() => ctrl.abort(), 3000);
    await fetch(s.url, { mode: 'no-cors', signal: ctrl.signal, cache: 'no-cache' });
    clearTimeout(timer);
    dot.className = 'dot up';
    stat.textContent = 'Online';
  } catch(e) {
    dot.className = 'dot down';
    stat.textContent = e.name === 'AbortError' ? 'Timeout' : 'Offline';
  }
}

function checkAll() {
  SERVICES.forEach((_,i) => checkService(i));
  document.getElementById('last-updated').textContent =
    'Last checked: ' + new Date().toLocaleTimeString();
}

async function loadDownloads() {
  try {
    const r = await fetch('downloads.json?t=' + Date.now());
    const data = await r.json();
    const el = document.getElementById('downloads');
    if (!data.length) { el.innerHTML = '<p style="color:#6e7a94;font-size:.83rem">No recent downloads.</p>'; return; }
    el.innerHTML = data.slice(0,20).map(d => `
      <div class="download-item">
        <div class="dl-name">${d.name}</div>
        <div class="dl-meta">${d.size || ''} · ${d.date || ''} · ${d.category || ''}</div>
      </div>
    `).join('');
  } catch(e) {
    document.getElementById('downloads').innerHTML =
      '<p style="color:#6e7a94;font-size:.83rem">downloads.json not found — see instructions above.</p>';
  }
}

buildCards();
checkAll();
loadDownloads();
setInterval(checkAll, 60000); // auto-refresh every 60s
setInterval(loadDownloads, 30000);

🔵 Option B — Homarr (Prettier Dashboard App)
What Homarr Is NATIVE WINDOWS (via Node.js)

Homarr is a polished home server dashboard with widgets for Plex, qBittorrent, Minecraft player counts, weather, and more. It has an official Windows installer.

Install Node.js 20 LTS

From nodejs.org. This is required to run Homarr on Windows without Docker.

Download and run Homarr 
npm install -g homarr-cli
homarr start

Access at http://localhost:7575.

Add as a startup service with NSSM 
nssm install Homarr "node" "C:\Users\[you]\AppData\Roaming\npm\node_modules\homarr-cli\..."
nssm start Homarr
💡
Homarr integrations: After setup, add integrations for qBittorrent (shows active downloads), Plex (shows what's playing), and Minecraft (shows online player count) from the widget configuration panel.
📥 Wiring Up Recent Downloads
qBittorrent → downloads.json

In qBittorrent: Tools → Options → Downloads → "Run external program on torrent completion":

powershell -Command "
  $f='D:\Server\Dashboard\downloads.json';
  $existing = if (Test-Path $f) { Get-Content $f | ConvertFrom-Json } else { @() };
  $new = @{ name='%N'; size='%Z'; date=(Get-Date -f 'yyyy-MM-dd HH:mm'); category='%L' };
  ($new + $existing) | Select-Object -First 50 | ConvertTo-Json | Set-Content $f
"

This appends each completed download to a JSON file (keeping the last 50) that the dashboard reads automatically.
