Minecraft Servers

// Vanilla · Modded · Auto-Start · Shutdown 2–3am //

Two separate Minecraft servers: a clean vanilla server on port 25565 and a modded server (Forge or Fabric) on port 25566. Both auto-start on boot and use the same 2–3am maintenance window.

🐉 V1 Goal: Both servers run as scheduled tasks, auto-start on boot, and share the nightly 2–3am maintenance window. The modded server uses a separate port and world folder.
⚙️ Setting Up the Minecraft Server
Step 1: Install Java

Minecraft Java Edition requires Java. For modern Minecraft (1.17+), use Java 21 LTS from adoptium.net. Download the Windows x64 installer and install it. Confirm with:

java -version
Step 2: Set Up Your Server Folder

Create a dedicated folder for the server, ideally on your internal drive:

D:\Minecraft\server\

Download the server JAR from minecraft.net (vanilla) or your preferred flavor (Paper, Fabric, etc.) into that folder.

First-time run to accept EULA:

cd D:\Minecraft\server
java -Xmx4G -Xms2G -jar server.jar nogui

It will fail and create eula.txt. Open it, change eula=false to eula=true, save. Run again — server will start.

Step 3: Create a start.bat Launch Script

Create D:\Minecraft\server\start.bat:

@echo off
title Minecraft Server
cd /d D:\Minecraft\server
java -Xmx6G -Xms2G -jar server.jar nogui
echo Server stopped. Window will close in 10 seconds.
timeout /t 10
🐉
Memory tip: -Xmx6G = maximum 6GB RAM. -Xms2G = start with 2GB. Tune these based on how much RAM your system has. Leave at least 4GB free for Windows and other services. For a gaming desktop with 16GB, -Xmx8G is fine.
Step 4: Open Firewall — Vanilla (port 25565)
netsh advfirewall firewall add rule name="Minecraft Vanilla" dir=in action=allow protocol=TCP localport=25565
netsh advfirewall firewall add rule name="Minecraft Vanilla UDP" dir=in action=allow protocol=UDP localport=25565
🔧 Modded Server (Port 25566)
Folder Structure — keep vanilla and modded separate
D:\Minecraft\
  ├── vanilla\        ← port 25565 (this server)
  │   ├── server.jar
  │   ├── start.bat
  │   └── world\
  └── modded\         ← port 25566
      ├── mods\
      ├── start.bat
      └── world\
Option A: Forge (most mods) RECOMMENDED
Download the Forge installer for your MC version

Go to files.minecraftforge.net, pick your Minecraft version → "Installer." Run it:

java -jar forge-1.20.1-47.3.0-installer.jar --installServer D:\Minecraft\modded\
Accept EULA in modded folder

Run the server once to generate eula.txt, then edit it to eula=true.

Create D:\Minecraft\modded\start.bat — note different port 
@echo off
title Minecraft Modded Server (Forge)
cd /d D:\Minecraft\modded
java -Xmx8G -Xms2G @user_jvm_args.txt ^
  @libraries/net/minecraftforge/forge/1.20.1-47.3.0/unix_args.txt nogui ^
  -- --port 25566
echo Server stopped.
timeout /t 10

The --port 25566 keeps it from conflicting with vanilla. Players connect to [your-ip]:25566.

Drop mods into D:\Minecraft\modded\mods\

Download mods from modrinth.com or CurseForge. Remember: all players must have the same client-side mods. Server-side-only mods (like Lithium, Chunky) don't need to be installed by players.

⚡
Essential performance mods for any server: Lithium (server-side, no client install needed), FerriteCore (reduces RAM usage), Starlight (faster chunk lighting). These alone can double server performance.
Option B: Fabric (lighter, better for optimization mods)
# Download installer from fabricmc.net
java -jar fabric-installer.jar server -mcversion 1.20.1 -dir D:\Minecraft\modded\

# start.bat for Fabric:
@echo off
title Minecraft Modded (Fabric)
cd /d D:\Minecraft\modded
java -Xmx8G -Xms2G -jar fabric-server-launch.jar nogui -- --port 25566
Enable RCON on Modded Server (different port)

In D:\Minecraft\modded\server.properties:

server-port=25566
enable-rcon=true
rcon.password=modded_rcon_password
rcon.port=25576      ← different from vanilla's 25575!
Open Firewall for Modded
netsh advfirewall firewall add rule name="Minecraft Modded" dir=in action=allow protocol=TCP localport=25566
netsh advfirewall firewall add rule name="Minecraft Modded UDP" dir=in action=allow protocol=UDP localport=25566
🚀 Auto-Start Both Servers on Windows Boot
Create separate Task Scheduler tasks for each server

Both servers get their own startup task. Stagger the delays so they don't both try to start at the same second.

Open Task Scheduler

Press Win + R → type taskschd.msc → Enter. Use "Create Task" (not Basic Task) for each.

Vanilla server task settings 
Name:          Minecraft Vanilla - Auto Start
General:       ☑ Run whether user is logged on or not
               ☑ Run with highest privileges
Trigger:       At startup → Delay 1 minute
Action:        D:\Minecraft\vanilla\start.bat
Start in:      D:\Minecraft\vanilla\
Settings:      ☑ Restart on failure every 1 min, up to 3 times
Modded server task settings 
Name:          Minecraft Modded - Auto Start
General:       ☑ Run whether user is logged on or not
               ☑ Run with highest privileges
Trigger:       At startup → Delay 2 minutes   ← 1 min after vanilla
Action:        D:\Minecraft\modded\start.bat
Start in:      D:\Minecraft\modded\
Settings:      ☑ Restart on failure every 1 min, up to 3 times
🌙 Scheduled 2am Shutdown & 3am Restart (Both Servers)
The Strategy

One shutdown script handles both servers — it sends a 5-minute warning to both, waits, then stops them both. One restart script starts both back up. RCON sends the graceful /stop command to each server on their separate RCON ports.

Part A: Enable RCON (already done above for both servers)

Vanilla: RCON port 25575, password vanilla_rcon_pass
Modded: RCON port 25576, password modded_rcon_pass

Install mcrcon from github.com/Tiiffi/mcrcon → extract to D:\Minecraft\tools\.

Part B: Combined Shutdown Script

Create D:\Minecraft\shutdown_all.bat:

@echo off
echo [%time%] Sending 5-minute warnings to both servers...

rem Warn vanilla server
D:\Minecraft\tools\mcrcon.exe -H localhost -P 25575 -p vanilla_rcon_pass "say [VANILLA] Server shutting down in 5 minutes for maintenance!"

rem Warn modded server  
D:\Minecraft\tools\mcrcon.exe -H localhost -P 25576 -p modded_rcon_pass "say [MODDED] Server shutting down in 5 minutes for maintenance!"

timeout /t 240

D:\Minecraft\tools\mcrcon.exe -H localhost -P 25575 -p vanilla_rcon_pass "say Server shutting down in 1 minute!"
D:\Minecraft\tools\mcrcon.exe -H localhost -P 25576 -p modded_rcon_pass "say Server shutting down in 1 minute!"

timeout /t 60

echo [%time%] Stopping both servers...
D:\Minecraft\tools\mcrcon.exe -H localhost -P 25575 -p vanilla_rcon_pass "stop"
D:\Minecraft\tools\mcrcon.exe -H localhost -P 25576 -p modded_rcon_pass "stop"

timeout /t 30

rem Backup worlds
echo [%time%] Backing up worlds...
set DATE_STR=%date:~-4,4%%date:~-7,2%%date:~0,2%
robocopy "D:\Minecraft\vanilla\world"  "E:\Backups\Minecraft\vanilla\world_%DATE_STR%" /E /NP
robocopy "D:\Minecraft\modded\world"   "E:\Backups\Minecraft\modded\world_%DATE_STR%"  /E /NP

echo [%time%] Shutdown and backup complete.
Part C: Combined Restart Script

Create D:\Minecraft\start_all.bat:

@echo off
echo [%time%] Starting vanilla server...
start "Minecraft Vanilla" /D "D:\Minecraft\vanilla" cmd /c "java -Xmx6G -Xms2G -jar server.jar nogui"

timeout /t 30

echo [%time%] Starting modded server...
start "Minecraft Modded" /D "D:\Minecraft\modded" cmd /c "start.bat"

echo [%time%] Both servers started.
Part D: Scheduled Tasks
rem Shutdown task (fires at 1:55am — gives 5min warning before 2am):
schtasks /create /tn "Minecraft - Nightly Shutdown" /sc daily /st 01:55 /tr "D:\Minecraft\shutdown_all.bat" /ru SYSTEM /f

rem Restart task (fires at 3:00am):
schtasks /create /tn "Minecraft - Nightly Restart" /sc daily /st 03:00 /tr "D:\Minecraft\start_all.bat" /ru SYSTEM /f
⚠️
Tasks set to "Run whether user is logged on or not" (SYSTEM) won't show a console window. To see output, add >> D:\Minecraft\logs\shutdown.log 2>&1 at the end of each echo line in the bat files, or check Event Viewer → Task Scheduler after a run.
💾 World Backups (While You're At It)

Add a backup step to the shutdown script. Before the stop command, after the server is quiet:

rem Add this to shutdown.bat AFTER the stop command + 30s wait:
robocopy "D:\Minecraft\server\world" "E:\Backups\Minecraft\world_%date:~-4,4%%date:~-7,2%%date:~0,2%" /E
rem Keeps dated backups. Clean old ones manually or add a cleanup script.
🔐 Access Control

Minecraft Java Edition uses its own account system (Microsoft/Mojang accounts) for authentication. You don't need SSO for who can play — that's handled by server.properties → white-list=true and the whitelist.json file. Manage the whitelist in-game with /whitelist add PlayerName or edit the JSON file directly.
