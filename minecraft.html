Automated Minecraft Server

// Auto-Start · Scheduled Shutdown 2–3am · Always Running //

A Minecraft server that starts automatically when Windows boots, shuts down gracefully at 2:00am, and restarts at 3:00am — all without any manual intervention.

🐉 V1 Goal: Minecraft server runs as a Windows scheduled task, restarts on boot, and a separate task handles the 2–3am maintenance window automatically.
⚙️ Setting Up the Minecraft Server
Step 1: Install Java

Minecraft Java Edition requires Java. For modern Minecraft (1.17+), use Java 21 LTS from adoptium.net. Download the Windows x64 installer and install it. Confirm with:

java -version
Step 2: Set Up Your Server Folder

Create a dedicated folder for the server, ideally on your internal drive:

D:\Minecraft\server\

Download the server JAR from minecraft.net (vanilla) or your preferred flavor (Paper, Fabric, etc.) into that folder.

First-time run to accept EULA:

cd D:\Minecraft\server
java -Xmx4G -Xms2G -jar server.jar nogui

It will fail and create eula.txt. Open it, change eula=false to eula=true, save. Run again — server will start.

Step 3: Create a start.bat Launch Script

Create D:\Minecraft\server\start.bat:

@echo off
title Minecraft Server
cd /d D:\Minecraft\server
java -Xmx6G -Xms2G -jar server.jar nogui
echo Server stopped. Window will close in 10 seconds.
timeout /t 10
🐉
Memory tip: -Xmx6G = maximum 6GB RAM. -Xms2G = start with 2GB. Tune these based on how much RAM your system has. Leave at least 4GB free for Windows and other services. For a gaming desktop with 16GB, -Xmx8G is fine.
Step 4: Open Firewall for Minecraft
netsh advfirewall firewall add rule name="Minecraft Server" dir=in action=allow protocol=TCP localport=25565
netsh advfirewall firewall add rule name="Minecraft Server UDP" dir=in action=allow protocol=UDP localport=25565
🚀 Auto-Start on Windows Boot
Method: Windows Task Scheduler (Most Reliable)

Using Task Scheduler means the server starts even without anyone logging in, and it runs with proper permissions.

Open Task Scheduler

Press Win + R → type taskschd.msc → Enter.

Create a new task (not basic task)

Action menu → "Create Task" (not "Create Basic Task").

General tab settings 
Name: Minecraft Server - Auto Start
☑ Run whether user is logged on or not
☑ Run with highest privileges
Configure for: Windows 10
Triggers tab → New 
Begin the task: At startup
Delay task for: 1 minute (gives Windows time to fully boot)
☑ Enabled
Actions tab → New 
Action: Start a program
Program/script: D:\Minecraft\server\start.bat
Start in: D:\Minecraft\server\
Conditions tab

Uncheck "Start the task only if the computer is on AC power" (if this is a desktop it doesn't matter, but good practice).

Settings tab 
☑ Allow task to be run on demand
☑ If the task fails, restart every: 1 minute
Attempt to restart up to: 3 times
🌙 Scheduled 2am Shutdown & 3am Restart
The Strategy

We create two scheduled tasks: one that gracefully stops the server at 2:00am (using RCON or taskkill), and one that starts it back up at 3:00am.

⚔️
Graceful vs forceful shutdown: The best approach is to use RCON (Remote Console) to send a /stop command to the server, which saves all chunks and disconnects players cleanly. If RCON isn't set up, we use taskkill as a fallback, but this can cause world corruption if the server is writing data.
Part A: Enable RCON on Your Server

In D:\Minecraft\server\server.properties, set:

enable-rcon=true
rcon.password=your_rcon_password_here
rcon.port=25575

Restart the server for changes to take effect.

Part B: Create the Shutdown Script

Install mcrcon (a tiny RCON client) — download from github.com/Tiiffi/mcrcon, extract to D:\Minecraft\tools\.

Create D:\Minecraft\shutdown.bat:

@echo off
echo Sending shutdown warning to players...
D:\Minecraft\tools\mcrcon.exe -H localhost -P 25575 -p your_rcon_password_here "say Server shutting down in 5 minutes for maintenance!"
timeout /t 240
D:\Minecraft\tools\mcrcon.exe -H localhost -P 25575 -p your_rcon_password_here "say Server shutting down in 1 minute!"
timeout /t 60
D:\Minecraft\tools\mcrcon.exe -H localhost -P 25575 -p your_rcon_password_here "stop"
echo Shutdown command sent.
timeout /t 30
echo Done.

Create D:\Minecraft\start_server.bat (used by the restart task):

@echo off
cd /d D:\Minecraft\server
start "Minecraft Server" java -Xmx6G -Xms2G -jar server.jar nogui
Part C: Create the 2am Shutdown Task

In Task Scheduler → Create Task:

Name: Minecraft - Nightly Shutdown
Trigger: Daily at 1:55 AM (sends warning before 2am)
Action: D:\Minecraft\shutdown.bat
Run whether logged on or not: ☑
Run with highest privileges: ☑
Part D: Create the 3am Restart Task
Name: Minecraft - Nightly Restart
Trigger: Daily at 3:00 AM
Action: D:\Minecraft\start_server.bat
Run whether logged on or not: ☑
Run with highest privileges: ☑
⚠️
Important: Task Scheduler tasks set to "Run whether user is logged on or not" won't open a visible console window. To see server output, log in and check the server window, or add logging to your start.bat: add >> D:\Minecraft\logs\startup.log 2>&1 to the end of the java command.
💾 World Backups (While You're At It)

Add a backup step to the shutdown script. Before the stop command, after the server is quiet:

rem Add this to shutdown.bat AFTER the stop command + 30s wait:
robocopy "D:\Minecraft\server\world" "E:\Backups\Minecraft\world_%date:~-4,4%%date:~-7,2%%date:~0,2%" /E
rem Keeps dated backups. Clean old ones manually or add a cleanup script.
🔐 Access Control

Minecraft Java Edition uses its own account system (Microsoft/Mojang accounts) for authentication. You don't need SSO for who can play — that's handled by server.properties → white-list=true and the whitelist.json file. Manage the whitelist in-game with /whitelist add PlayerName or edit the JSON file directly.
