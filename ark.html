ARK: Survival Evolved Server

// Off by Default · On-Demand · SteamCMD · Windows Native //

A self-hosted ARK: SE dedicated server that's off by default to save resources, but can be spun up from the dashboard whenever you want to play. Uses SteamCMD for installation and updates.

🦕
ARK is extremely resource-hungry. The dedicated server uses 6–12GB RAM and pins multiple CPU cores. Don't run it simultaneously with other heavy services. Keep it off by default — which is the V1 plan.
🐉 V1 Goal: ARK server installable via SteamCMD, runnable with a bat file or from the dashboard Start button, off by default, with saved worlds persisting between sessions.
⚙️ Installation via SteamCMD
Install SteamCMD

Download from Valve's SteamCMD page. Extract steamcmd.exe to C:\steamcmd\.

Download the ARK Dedicated Server

Create D:\ARK\ for the server files. Open a PowerShell prompt:

C:\steamcmd\steamcmd.exe +force_install_dir D:\ARK +login anonymous +app_update 376030 validate +quit

This downloads ~30GB of ARK server files. Run it again anytime to update.

Create a launch script

Create D:\ARK\start_ark.bat:

@echo off
title ARK: Survival Evolved Server
cd /d D:\ARK\ShooterGame\Binaries\Win64

start ShooterGameServer.exe ^
  TheIsland?listen ^
  ?SessionName="My ARK Server" ^
  ?ServerPassword=yourpassword ^
  ?ServerAdminPassword=adminpassword ^
  ?Port=7777 ^
  ?QueryPort=27015 ^
  ?MaxPlayers=10 ^
  -server ^
  -log ^
  -NoBattlEye

echo ARK server started. Check the console window.
🗺️
Map options: Replace TheIsland with ScorchedEarth_P, Aberration_P, Extinction, Genesis, or CrystalIsles. Each map has its own save directory.
Open firewall ports 
netsh advfirewall firewall add rule name="ARK Game" dir=in action=allow protocol=UDP localport=7777
netsh advfirewall firewall add rule name="ARK Query" dir=in action=allow protocol=UDP localport=27015
netsh advfirewall firewall add rule name="ARK RCON" dir=in action=allow protocol=TCP localport=27020
Test it — join via ARK client

In ARK: play → Join ARK → filter by "unofficial" → or direct connect to [your-server-IP]:7777. Enter your ServerPassword when prompted.

🎛️ Dashboard Integration (On-Demand Start/Stop)
Create Task Scheduler tasks for Start/Stop

The dashboard's control buttons call schtasks /run /tn "ark-start" etc. Create the corresponding tasks:

# In PowerShell (admin):

# Start task:
schtasks /create /tn "ark-start" /sc once /st 00:00 /tr "D:\ARK\start_ark.bat" /ru SYSTEM /f

# Stop task (graceful — saves world first):
schtasks /create /tn "ark-stop" /sc once /st 00:00 /tr "D:\ARK\stop_ark.bat" /ru SYSTEM /f

Create D:\ARK\stop_ark.bat:

@echo off
rem Send save command via RCON before killing
echo DoExit | "C:\tools\mcrcon.exe" -H localhost -P 27020 -p adminpassword
timeout /t 30
taskkill /IM ShooterGameServer.exe /F
echo ARK server stopped.
⚙️ Server Configuration
Key Config Files
D:\ARK\ShooterGame\Saved\Config\WindowsServer\
  ├── GameUserSettings.ini   (server settings, player rates, etc.)
  └── Game.ini               (advanced game settings, taming rates)

Key settings in GameUserSettings.ini:

[ServerSettings]
ServerPassword=yourpassword
ServerAdminPassword=adminpassword
MaxPlayers=10
XPMultiplier=2.0
TamingSpeedMultiplier=3.0
HarvestAmountMultiplier=2.0
PlayerCharacterWaterDrainMultiplier=0.5
PlayerCharacterFoodDrainMultiplier=0.5
DayCycleSpeedScale=1.0
DayTimeSpeedScale=1.0
NightTimeSpeedScale=3.0
RCONEnabled=True
RCONPort=27020
Keeping It Updated

Create D:\ARK\update_ark.bat and run it periodically (or add a weekly scheduled task):

@echo off
echo Stopping ARK server...
call D:\ARK\stop_ark.bat
timeout /t 30
echo Updating ARK...
C:\steamcmd\steamcmd.exe +force_install_dir D:\ARK +login anonymous +app_update 376030 validate +quit
echo Update complete. Restart ARK manually or via dashboard.
🔐 Access Control

ARK uses its own ServerPassword for entry and ServerAdminPassword for admin commands. There's no SSO integration — keep these strong and don't share them widely. The admin password lets players use console commands in-game, including destroying the whole map, so treat it like a root password.

For a whitelist (invite-only server), set AllowAnyoneBabyImprintCuddle=False and use the whitelist file:

# D:\ARK\ShooterGame\Saved\AllowedCheaterSteamIDs.txt
# Add one SteamID64 per line for admin users

# For a true whitelist (deny all not on list):
# In launch params add: ?WhiteListPlayer=1
# Then: D:\ARK\ShooterGame\Saved\PlayersJoinNoCheck.txt (one SteamID64 per line)
