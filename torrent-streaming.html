Torrent Streaming

// Real-Debrid · Zurg · Rclone · Stream on Demand · No Local Storage · PC //

A cloud-based streaming stack that lets Jellyfin play movies on demand without downloading or storing them locally. Real-Debrid caches torrents on their servers and serves them as direct HTTPS streams. Zurg exposes the Real-Debrid library as a virtual filesystem, Rclone mounts it as a local drive, and Jellyfin sees it as a normal media folder. Press play — the file streams instantly.

⚠️
Sovereignty Note

Real-Debrid is a paid third-party service (~$4/month). Unlike every other service in this Grimoire, this stack depends on an external provider — if Real-Debrid goes down or closes, the virtual library disappears. This is a deliberate tradeoff: $4/month is cheaper than the electricity and storage to download and keep everything. No genuinely self-hosted FOSS equivalent exists — the closest option would be running a personal seedbox on a VPS (~$5–10/month) and mounting it via Rclone, which costs more and requires more maintenance.

🔑 Step 1 — Real-Debrid Account
Subscribe to Real-Debrid

Go to real-debrid.com and purchase a plan. The 15-day plan is ~$3, the 180-day plan works out to ~$4/month. Payment accepts card and cryptocurrency.

Get your API key

Log in → real-debrid.com/apitoken → copy the token. You'll need this for Zurg.

🦋 Step 2 — Install Zurg

Zurg connects to Real-Debrid's API and exposes your RD library as a WebDAV server. It also handles automatic torrent addition when Radarr sends a download request.

Download Zurg for Windows

Get the latest release from github.com/debridmediamanager/zurg-testing/releases — download zurg-windows-amd64.exe. Save to D:\Server\Zurg\zurg.exe.

Create config.yml

Save as D:\Server\Zurg\config.yml:

zurg: v1
token: YOUR_REAL_DEBRID_API_TOKEN

# WebDAV server settings:
host: 0.0.0.0
port: 9999

# Organise library by type:
on_library_update: sh -c "curl http://localhost:32400/library/sections/all/refresh"

directories:
  movies:
    type: movies
    min_size_mb: 200
    only_show_the_biggest_file: true
  shows:
    type: shows
    min_size_mb: 200
    only_show_the_biggest_file: false
Run Zurg and verify WebDAV 
cd D:\Server\Zurg
.\zurg.exe

# Test WebDAV in browser:
# http://localhost:9999/dav/
# You should see movies/ and shows/ directories listed
Install Zurg as a Windows service via NSSM 
nssm install Zurg "D:\Server\Zurg\zurg.exe"
nssm set Zurg AppDirectory "D:\Server\Zurg"
nssm set Zurg DisplayName "Zurg (RealDebrid WebDAV)"
nssm set Zurg Description "Exposes Real-Debrid library as WebDAV for Rclone"
nssm set Zurg Start SERVICE_AUTO_START
nssm set Zurg AppStdout "D:\Server\Zurg\logs\zurg.log"
nssm set Zurg AppStderr "D:\Server\Zurg\logs\zurg_err.log"

mkdir "D:\Server\Zurg\logs"
nssm start Zurg
🔗 Step 3 — Mount with Rclone

Rclone mounts the Zurg WebDAV server as a local drive that Windows applications (including Jellyfin) can read as if it were a local folder.

Install Rclone for Windows

Download from rclone.org/downloads/ — get the Windows amd64 zip. Extract rclone.exe to D:\Server\Rclone\rclone.exe.

Also install WinFsp (required for Rclone to create virtual drives on Windows): winfsp.dev/rel/

Configure the Zurg WebDAV remote 
D:\Server\Rclone\rclone.exe config

# Create new remote:
# Name: zurg
# Type: webdav
# URL: http://localhost:9999/dav
# Vendor: other
# User: (blank)
# Password: (blank)
# No further options needed
Test the remote 
D:\Server\Rclone\rclone.exe ls zurg:
# Should list your Real-Debrid library contents
Mount as a virtual drive 
# Mount command — run this manually first to verify it works:
D:\Server\Rclone\rclone.exe mount zurg: D:\Debrid ^
  --vfs-cache-mode full ^
  --vfs-cache-max-size 10G ^
  --vfs-read-ahead 128M ^
  --buffer-size 256M ^
  --dir-cache-time 10s ^
  --poll-interval 10s ^
  --log-level INFO ^
  --log-file D:\Server\Rclone\logs\rclone.log

# D:\Debrid\ should now contain your RD library
# Open it in Explorer to verify
Install Rclone mount as a Windows service via NSSM 
mkdir "D:\Debrid"
mkdir "D:\Server\Rclone\logs"

nssm install RcloneDebrid "D:\Server\Rclone\rclone.exe"
nssm set RcloneDebrid AppParameters "mount zurg: D:\Debrid --vfs-cache-mode full --vfs-cache-max-size 10G --vfs-read-ahead 128M --buffer-size 256M --dir-cache-time 10s --poll-interval 10s --log-level INFO --log-file D:\Server\Rclone\logs\rclone.log"
nssm set RcloneDebrid DisplayName "Rclone Debrid Mount"
nssm set RcloneDebrid Start SERVICE_AUTO_START
nssm set RcloneDebrid AppStdout "D:\Server\Rclone\logs\rclone_out.log"

# Zurg must be running before Rclone starts:
nssm set RcloneDebrid DependOnService Zurg

nssm start RcloneDebrid
🎬 Step 4 — Add Jellyfin Library
Add a new Movies library in Jellyfin

Jellyfin Dashboard → Libraries → Add Media Library:

  • Content type: Movies
  • Display name: Debrid Library (or "Stream Library")
  • Folder: D:\Debrid\movies\
  • Preferred language: English
  • Real-time monitoring: OFF — Rclone handles refresh via Zurg's on_library_update hook
Configure Radarr's debrid download client

Settings → Download Clients → Add → Rclone (or use qBittorrent pointed at Real-Debrid via a Torrent-to-RD browser extension).

The recommended approach is the Real-Debrid Radarr integration: install the rdt-client (RealDebrid Torrent Client), which acts as a fake qBittorrent that actually sends torrents to Real-Debrid instead of downloading them locally.

# rdt-client in Docker (WSL2):
services:
  rdtclient:
    image: rogerfar/rdtclient:latest
    container_name: rdtclient
    restart: unless-stopped
    ports:
      - "6500:6500"
    volumes:
      - /mnt/d/Server/rdtclient:/data/db
      - /mnt/d/Debrid:/data/downloads  # maps to D:\Debrid
    environment:
      - PUID=1000
      - PGID=1000

In Radarr: Settings → Download Clients → Add qBittorrent → point at localhost:6500. rdt-client intercepts the magnet link, sends it to Real-Debrid, and the file appears in D:\Debrid\ via Rclone.

💡 List A vs List B — Which Goes Where
📋 List A — Own Locally (Radarr → qBittorrent)

Movies you want to keep permanently in high quality. Downloaded by the regular Radarr → qBittorrent pipeline. Stored on the PC's local drive. Appear in the main Jellyfin Movies library. 1080p quality profile, stored forever.

🧲 List B — Stream on Demand (Radarr → rdt-client → RD)

Movies you want to watch without storing. Added to Radarr's debrid profile. Radarr sends the magnet to rdt-client → Real-Debrid caches it → appears in D:\Debrid\movies\ → Jellyfin plays it. Nothing stored locally. Purely additive — manual deletion only.

🎬 The Letterboxd sync script reads two .txt files of Letterboxd list URLs and syncs them to Radarr every 24 hours. List A goes to the local-storage Radarr profile. List B goes to the debrid Radarr profile. Purely additive — nothing is ever removed automatically.
📁 Config Files

Two plain text files, one URL per line. Save them at:

D:\Server\Letterboxd\lists_local.txt     ← List A: own locally (→ qBittorrent)
D:\Server\Letterboxd\lists_debrid.txt    ← List B: stream on demand (→ rdt-client)
Example: lists_local.txt
https://letterboxd.com/yourname/list/films-to-own/
https://letterboxd.com/yourname/list/family-classics/
https://letterboxd.com/yourname/watchlist/
Example: lists_debrid.txt
https://letterboxd.com/yourname/list/watch-once/
https://letterboxd.com/yourname/list/curious-about/

Letterboxd watchlists use the URL pattern /username/watchlist/. Regular lists use /username/list/list-name/. Both work — Letterboxd exposes both as RSS feeds.

🐍 The Sync Script
Install dependencies 
pip install requests feedparser
Create the script at D:\Server\Letterboxd\letterboxd_sync.py 
#!/usr/bin/env python3
"""
letterboxd_sync.py
==================
Reads Letterboxd list RSS feeds and adds movies to Radarr.
  List A (lists_local.txt)  → Radarr local quality profile (1080p, stored forever)
  List B (lists_debrid.txt) → Radarr debrid quality profile (1080p, stream via RD)

Run manually or via Task Scheduler (daily at 2am).
"""

import feedparser, requests, sys
from pathlib import Path

# ── Config ──────────────────────────────────────────────────────────────────
RADARR_URL     = "http://localhost:7878"
RADARR_API_KEY = "YOUR_RADARR_API_KEY"   # Radarr → Settings → General → API Key

# Radarr quality profile IDs — get these from:
# GET http://localhost:7878/api/v3/qualityprofile
# Common IDs: 1=Any, 4=HD-1080p, 5=Ultra-HD
PROFILE_LOCAL  = 4   # 1080p → qBittorrent → local storage
PROFILE_DEBRID = 4   # 1080p → rdt-client → Real-Debrid

# Radarr root folder paths — get from:
# GET http://localhost:7878/api/v3/rootfolder
ROOT_LOCAL  = "D:\\Media\\Movies"        # permanent local library
ROOT_DEBRID = "D:\\Debrid\\movies"       # virtual RD drive

# List files (one Letterboxd URL per line):
SCRIPT_DIR    = Path(__file__).parent
LIST_LOCAL    = SCRIPT_DIR / "lists_local.txt"
LIST_DEBRID   = SCRIPT_DIR / "lists_debrid.txt"

TMDB_BASE = "https://api.themoviedb.org/3"

# ── Helpers ─────────────────────────────────────────────────────────────────

def radarr_get(endpoint):
    r = requests.get(f"{RADARR_URL}/api/v3/{endpoint}",
                     headers={"X-Api-Key": RADARR_API_KEY}, timeout=15)
    r.raise_for_status()
    return r.json()

def radarr_post(endpoint, payload):
    r = requests.post(f"{RADARR_URL}/api/v3/{endpoint}",
                      json=payload,
                      headers={"X-Api-Key": RADARR_API_KEY}, timeout=15)
    return r

def get_existing_tmdb_ids():
    movies = radarr_get("movie")
    return {m["tmdbId"] for m in movies}

def letterboxd_rss_url(list_url: str) -> str:
    """Convert a Letterboxd list URL to its RSS feed URL."""
    url = list_url.rstrip("/")
    return url + "/rss/"

def fetch_titles_from_list(list_url: str) -> list[dict]:
    """Parse a Letterboxd RSS feed and return list of {title, year} dicts."""
    rss_url = letterboxd_rss_url(list_url)
    feed = feedparser.parse(rss_url)
    movies = []
    for entry in feed.entries:
        title = entry.get("letterboxd_filmtitle") or entry.get("title", "")
        year  = entry.get("letterboxd_filmyear")
        if title:
            movies.append({"title": title, "year": int(year) if year else None})
    return movies

def lookup_tmdb(title: str, year: int | None) -> dict | None:
    """Look up a movie on TMDB via Radarr's lookup endpoint."""
    params = {"term": f"{title} {year}" if year else title}
    results = radarr_get(f"movie/lookup?term={requests.utils.quote(params['term'])}")
    if results:
        return results[0]
    return None

def add_to_radarr(tmdb_result: dict, quality_profile_id: int, root_folder: str) -> bool:
    """Add a movie to Radarr. Returns True if added, False if already exists."""
    payload = {
        "title":            tmdb_result["title"],
        "tmdbId":           tmdb_result["tmdbId"],
        "qualityProfileId": quality_profile_id,
        "rootFolderPath":   root_folder,
        "monitored":        True,
        "minimumAvailability": "released",
        "addOptions": {
            "searchForMovie": True
        }
    }
    r = radarr_post("movie", payload)
    if r.status_code == 201:
        return True
    elif r.status_code == 400 and "already" in r.text.lower():
        return False   # already in Radarr
    else:
        print(f"    [WARN] Unexpected response {r.status_code}: {r.text[:200]}")
        return False

def process_list_file(list_file: Path, profile_id: int, root_folder: str,
                      existing_ids: set, label: str):
    if not list_file.exists():
        print(f"[SKIP] {list_file.name} not found")
        return

    urls = [u.strip() for u in list_file.read_text().splitlines()
            if u.strip() and not u.strip().startswith("#")]

    print(f"\n── {label} ({len(urls)} lists) ──────────────────────────────")

    for list_url in urls:
        print(f"  Fetching: {list_url}")
        try:
            movies = fetch_titles_from_list(list_url)
        except Exception as e:
            print(f"  [ERR] Failed to fetch RSS: {e}")
            continue

        for movie in movies:
            try:
                result = lookup_tmdb(movie["title"], movie.get("year"))
                if not result:
                    print(f"    [MISS] Not found on TMDB: {movie['title']}")
                    continue
                if result["tmdbId"] in existing_ids:
                    continue   # already in Radarr, skip silently
                added = add_to_radarr(result, profile_id, root_folder)
                if added:
                    existing_ids.add(result["tmdbId"])
                    print(f"    [ADD ] {result['title']} ({result.get('year', '?')})")
            except Exception as e:
                print(f"    [ERR] {movie['title']}: {e}")

# ── Main ────────────────────────────────────────────────────────────────────

if __name__ == "__main__":
    print("Letterboxd → Radarr sync starting...")
    try:
        existing = get_existing_tmdb_ids()
        print(f"Radarr has {len(existing)} movies already.")
    except Exception as e:
        print(f"[FATAL] Cannot reach Radarr: {e}")
        sys.exit(1)

    process_list_file(LIST_LOCAL,  PROFILE_LOCAL,  ROOT_LOCAL,  existing, "List A — Local Storage")
    process_list_file(LIST_DEBRID, PROFILE_DEBRID, ROOT_DEBRID, existing, "List B — Debrid Stream")

    print("\nSync complete.")

D:\Debrid\ uses zero disk space — it is a virtual filesystem backed by Real-Debrid's servers. The Rclone VFS cache (--vfs-cache-max-size 10G) uses up to 10GB of your main drive as a read-ahead buffer during playback — this is temporary and auto-purged.

🛠️ Common Issues
D:\Debrid\ is empty or shows no files
Check that Zurg is running (nssm status Zurg) and responding at http://localhost:9999/dav/. Then check Rclone is running (nssm status RcloneDebrid). Rclone depends on Zurg — if Zurg starts slowly, Rclone may have failed. Restart RcloneDebrid after Zurg is confirmed running.
Movie in Jellyfin but won't play — buffering immediately
Real-Debrid hasn't cached this torrent yet, or the torrent is not a cached "instant" item. Check Real-Debrid's "Downloads" page to see if the item is fully cached. Zurg's only_show_the_biggest_file: true setting should prevent broken files from appearing, but occasionally a bad torrent gets through.
Letterboxd sync script failing — "Cannot reach Radarr"
Verify Radarr is running and the API key in the script is correct. Run manually: python D:\Server\Letterboxd\letterboxd_sync.py. Check that RADARR_URL is set to http://localhost:7878.
Movies showing as "Missing" in Radarr debrid profile
rdt-client may not have received the search request, or the torrent indexer didn't find a match. Check rdt-client at http://localhost:6500 for pending/failed items. Verify Prowlarr has YTS and other movie indexers configured.
Rclone mount disappears after PC restart
NSSM service startup order issue. Make sure RcloneDebrid has DependOnService Zurg set, and both services are set to SERVICE_AUTO_START. WinFsp must also be running — verify it starts automatically in Services (services.msc).
