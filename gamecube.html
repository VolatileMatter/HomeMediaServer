Remote GameCube Emulation

// Dolphin · Sunshine/Moonlight · Play From Anywhere //

Stream Dolphin GameCube/Wii emulation from your desktop to any device anywhere in the world, using a controller connected to the remote device. Not browser-native — uses low-latency game streaming (like GeForce Now, but yours).

🐉
The right architecture: GameCube games need ~60fps with low latency — pure browser streaming (like Parsec-in-browser) has too much delay for fast games. The best approach is Sunshine + Moonlight: Sunshine runs on your server and streams Dolphin, Moonlight is a native client app on the remote device. It's designed exactly for this and works over LAN and internet.
🐉 V1 Goal: Sunshine streaming server on your desktop, Moonlight client on remote devices. Connect a controller to the remote device — it transmits inputs back. Single PIN or SSO-protected.
🏗️ Architecture Overview
ComponentWhere It RunsWhat It Does
DolphinYour desktop serverEmulates GameCube/Wii games. Runs headless or windowed.
SunshineYour desktop serverGame streaming host. Captures Dolphin's output, encodes it with GPU acceleration, streams to clients.
MoonlightRemote device (PC, Mac, Android, iOS, TV)Receives the stream, displays it, forwards controller input back to Sunshine.
ControllerRemote deviceUSB or Bluetooth gamepad connected to the remote device — inputs are forwarded as if connected to your server.
🎮
Controller compatibility: Any XInput controller works over Moonlight — Xbox controllers, PS4/PS5 (via DS4Windows), Switch Pro Controller (via BetterJoy). The controller connects to the remote device, and Moonlight forwards inputs to your server exactly as if the controller was physically plugged in.
🐬 Step 1: Set Up Dolphin
Download Dolphin

From dolphin-emu.org/download. Get the latest stable version. Extract to D:\Emulators\Dolphin.

Add your game ISOs/ROMs

In Dolphin: Config → Paths → Add → point to your GameCube ISO folder (e.g. D:\Games\GameCube). You must own the games you dump.

Configure for streaming performance

Graphics Settings → set Backend to Direct3D 12 (best performance on Windows). V-Sync: ON. These settings ensure Dolphin renders at a consistent 60fps for clean streaming.

Configure controller input for Moonlight

Dolphin → Controllers → Gamecube Port 1 → Standard Controller. Click Configure → for each button, select "XInput/0/Gamepad" (the virtual XInput device that Moonlight creates when a remote controller connects). Map all buttons.

💡
Pre-made Dolphin GameCube profiles: Search "Dolphin XInput GameCube profile" — there are community-made profiles that handle the button mapping automatically for Xbox-style controllers.
Test Dolphin locally first

Launch a game, make sure it runs at 60fps, and that a locally-connected controller works. Resolve any performance issues before adding the streaming layer.

☀️ Step 2: Install & Configure Sunshine
What Sunshine Is

Sunshine is an open-source, self-hosted game streaming server. It's the server-side replacement for Nvidia GameStream. Works with any GPU (AMD, Intel, or Nvidia). Streams via the RTSP protocol at up to 120fps in 4K.

Download and install Sunshine

From github.com/LizardByte/Sunshine. Download the .exe installer for Windows. Install — it creates a system tray icon and a web admin UI.

Access the Sunshine admin UI

Open https://localhost:47990 (note: HTTPS, self-signed cert — accept the browser warning). Set up your admin username and password on first launch.

Add Dolphin as a streaming app

In Sunshine web UI → Applications → Add:

Name:         GameCube (or name of specific game)
Cmd:          "D:\Emulators\Dolphin\Dolphin.exe"
Working Dir:  D:\Emulators\Dolphin
Image:        (optional: drag a cover art image)

# For launching a specific game directly:
Cmd: "D:\Emulators\Dolphin\Dolphin.exe" -e "D:\Games\GameCube\WindWaker.iso"

You can add one entry per game, or just add Dolphin itself and let the user pick from the game list.

Configure streaming quality

Configuration → Encoder settings:

Resolution:  1920x1080 (or your monitor res)
FPS:         60
Bitrate:     20–50 Mbps (higher = better quality, more bandwidth)
Encoder:     nvenc (Nvidia), AMF (AMD), or software (any GPU)
Open required firewall ports 
netsh advfirewall firewall add rule name="Sunshine HTTP" dir=in action=allow protocol=TCP localport=47989
netsh advfirewall firewall add rule name="Sunshine HTTPS" dir=in action=allow protocol=TCP localport=47990
netsh advfirewall firewall add rule name="Sunshine Video" dir=in action=allow protocol=UDP localport=47998-48000
netsh advfirewall firewall add rule name="Sunshine Control" dir=in action=allow protocol=TCP localport=47998-48010
Set Sunshine to start on Windows boot

Sunshine installs as a Windows Service by default. Verify in Services (services.msc) → "Sunshine" should be set to Automatic.

Pair Moonlight with your server (LAN)

Open Moonlight → Add Host → enter your server's local IP (e.g. 192.168.1.100). Moonlight will show a 4-digit PIN. Enter this PIN in the Sunshine admin UI under "PIN Pair." Done — they're paired.

Connect a controller to the remote device

Plug in a USB controller, or pair a Bluetooth gamepad to the phone/PC/TV. Moonlight forwards all controller input to Sunshine, which injects it into Windows as a virtual XInput device that Dolphin sees.

Launch a game

In Moonlight, click your server → you'll see your Dolphin app(s). Click one → the stream starts. Full 1080p60 gameplay, controller working, from anywhere.

🌍 Remote Access Over the Internet
Option A: Tailscale (Recommended — Easiest) RECOMMENDED

Install Tailscale on both your server and the remote device. In Moonlight, add the server's Tailscale IP (e.g. 100.x.x.x) as the host instead of the local IP. Works instantly, no port forwarding, works through NAT and firewalls.

⚡
Bandwidth requirement: Game streaming needs a good internet connection on both ends. At 20 Mbps bitrate, you need ~25 Mbps upload on your home connection and ~25 Mbps download on the remote device. For best results, the remote device should be on WiFi 5GHz or Ethernet.
Option B: Port Forward

Forward the Sunshine ports (47989, 47990, 47998–48010) in your router to your server IP. Then add your home's public IP as a host in Moonlight. Pair with the PIN as normal. This works but exposes ports to the internet — change the Sunshine admin password to something very strong.

🔐 Authentication & Access Control
Single PIN (V1 — Simplest)

Moonlight pairing uses a one-time PIN. Once a device is paired, it connects without a PIN. To unpair a device: Sunshine admin UI → Clients → revoke the device. This is suitable for personal use.

SSO / Reverse Proxy Authentication (Advanced)

Sunshine's admin UI (port 47990) can be put behind a reverse proxy with authentication. Add a Caddy rule:

sunshine.yourdomain.com {
  forward_auth http://authentik:9000/outpost.goauthentik.io/...
  reverse_proxy https://localhost:47990 {
    transport http { tls_insecure_skip_verify }
  }
}

This puts Authentik (from the SSO tab) in front of the Sunshine admin UI. The actual streaming protocol (Moonlight) handles its own authentication via device pairing and doesn't go through the proxy.

🔧 Common Issues
IssueFix
Controller inputs not working in DolphinMake sure Dolphin controller profile is set to XInput, not "Background Input" only. In Sunshine: enable "Virtual Gamepad."
High latency over internetLower bitrate to 10–15 Mbps. Enable "Optimize game settings" in Moonlight. Use 720p if needed.
Black screen / stream won't startSunshine needs to run as the logged-in user's session, not a service account. Ensure a user is logged into Windows (not just at the lock screen).
Audio not transmittingSunshine → Audio → select your main audio output device. In Dolphin, ensure audio output isn't set to a virtual device.
Dolphin drops frames during streamPin Dolphin to high-performance cores in Task Manager → right-click → Set Affinity. Lower game's resolution multiplier in Dolphin graphics settings.
