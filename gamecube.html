Remote GameCube / Wii Emulation

// Dolphin · Sunshine · Moonlight · Stream From Anywhere //

Stream Dolphin GameCube/Wii emulation from your desktop to any device, using a controller connected to the remote device. Low-latency game streaming via Sunshine+Moonlight.

🐉
Architecture: Dolphin runs on your server. Sunshine captures and encodes the video. Moonlight (on any device) receives the stream and sends controller input back. The remote player sees and plays the game as if they were sitting at your PC — but from anywhere.
🐉 V1 Goal: Dolphin + Sunshine running on your server. Moonlight on remote devices. Single PIN pairing. Tailscale for internet access. Controller plugged into remote device works immediately.

🐬 Step 1: Set Up Dolphin
Download Dolphin

From dolphin-emu.org. Get the latest stable. Extract to D:\Emulators\Dolphin\.

Add your ISO library

Config → Paths → Add → point to D:\Emulation\GameCube\ and D:\Emulation\Wii\. ROMs are .iso, .rvz (compressed), or .gcz format.

Configure for streaming performance

Graphics → Backend: Direct3D 12 (Windows). V-Sync: On. This ensures stable 60fps output for Sunshine to encode.

Configure GameCube controller for XInput (Moonlight)

Dolphin → Controllers → GameCube Port 1 → Standard Controller → Configure. Set each input to XInput/0/Gamepad — this is the virtual controller Moonlight creates when a remote controller connects. Map all buttons to the corresponding XInput buttons.

💡
For a pre-made XInput profile: Dolphin → Controllers → Load Profile → download a community XInput→GameCube mapping from the Dolphin wiki. Much faster than manual mapping.
Test locally first

Launch a game, confirm 60fps, confirm a locally connected controller works. Resolve any performance issues before adding Sunshine.

☀️ Step 2: Sunshine — Game Streaming Host FOSS
Install Sunshine

From github.com/LizardByte/Sunshine — download the Windows .exe installer. It installs as a Windows Service (auto-starts with Windows).

Access the Sunshine admin UI

Navigate to https://localhost:47990 (accept the self-signed cert warning). Set your admin username and password on first launch.

Add games as Sunshine applications

Applications → Add New:

# Launch a specific game directly:
Name:        Wind Waker
Cmd:         "D:\Emulators\Dolphin\Dolphin.exe" -e "D:\Emulation\GameCube\WindWaker.rvz"
Working Dir: D:\Emulators\Dolphin

# Or add Dolphin itself as a launcher (browse to any game):
Name:        Dolphin (GameCube/Wii)
Cmd:         "D:\Emulators\Dolphin\Dolphin.exe"

Add one entry per game for a clean game library in Moonlight, or just add the Dolphin launcher.

Configure encoding quality

Configuration tab in Sunshine web UI:

Resolution:  1920x1080
FPS:         60
Bitrate:     20–50 Mbps (higher = better quality, more bandwidth)
Encoder:     nvenc (Nvidia), amf (AMD), or vaapi (Intel/any)
Open required firewall ports 
netsh advfirewall firewall add rule name="Sunshine HTTPS" dir=in action=allow protocol=TCP localport=47990
netsh advfirewall firewall add rule name="Sunshine Stream" dir=in action=allow protocol=UDP localport=47998-48000
netsh advfirewall firewall add rule name="Sunshine Control" dir=in action=allow protocol=TCP localport=47998-48010

Pair Moonlight with your server (LAN)

Open Moonlight → Add Host → enter your server's local IP. Moonlight shows a 4-digit PIN. Enter this PIN in Sunshine web UI → PIN Pairing. Done — device is paired permanently.

Connect a controller to the remote device

USB or Bluetooth gamepad on the client device. Moonlight forwards all inputs to Sunshine, which injects them as XInput for Dolphin to read.

Launch a game

In Moonlight → your server → click any game → stream starts. Full 1080p60 gameplay with working controller.

🌍 Remote Access Over the Internet
Option A: Tailscale (Recommended) RECOMMENDED

Install Tailscale on server and remote device. Add your server's Tailscale IP in Moonlight instead of the local IP. Works from anywhere, no port forwarding, no home IP exposure.

⚡
Bandwidth needed: 20–50 Mbps upload on home connection + 20–50 Mbps download on remote device. 5GHz WiFi or Ethernet on both ends. Playable on 10 Mbps with quality settings reduced to 720p/30 Mbps bitrate.
Option B: Direct Port Forward

Forward Sunshine's ports on your router to your server IP. Add your home's public IP in Moonlight. Works but exposes ports publicly — use a strong admin password in Sunshine.

🔧 Troubleshooting
IssueFix
Controller not working in DolphinEnsure Dolphin controller profile is set to XInput/0. Enable "Background Input" in Dolphin controller settings.
Black screen on stream startSunshine needs a logged-in Windows session (not lock screen). Disable sleep/hibernate in power settings.
High latency over internetLower bitrate to 15 Mbps, try 720p, check Tailscale route (direct vs relay — relay adds latency).
Dolphin drops frames during streamLower internal resolution in Dolphin graphics settings. Sunshine encoding + Dolphin rendering share GPU.
Audio not transmittedSunshine → Configuration → Audio: select your main audio output device.
