Infinite / Eternal Jukebox

// Self-Hosted · YouTube Input · Endless Remixing //

The Infinite Jukebox analyzes a song's beat structure and loops it endlessly by jumping between similar-sounding points in the track. You paste a YouTube link, it downloads the audio, analyzes it, and plays it forever.

🐉 V1 Goal: A local web server at port 5050 where you paste a YouTube URL and get back an infinite, seamlessly looping version of that song.
🧠 Background: What the Eternal Jukebox Is

The original Eternal Jukebox was made by Paul Lamere at The Echo Nest (later Spotify). It used Spotify's audio analysis API to find "branch points" in a song — moments that sound similar enough that you can jump from one to another without it sounding jarring, creating an endless remix.

Spotify deprecated the public Echo Nest API, so the original service mostly stopped working. There are several community forks and reimplementations that work locally using open audio analysis libraries.

🔧 Option A — infinite-jukebox-web (Recommended) FOSS · Node.js
What This Uses

This implementation uses librosa (Python audio analysis) to find beat/segment data locally (no Spotify API needed), combined with a Node.js web server and a browser-based visualizer. You paste a YouTube URL and it downloads, analyzes, and plays.

Create D:\Server\Jukebox\jukebox_server.py:

from flask import Flask, request, jsonify, send_from_directory
import subprocess, os, sys, uuid, threading
from Remixatron import InfiniteJukebox

app = Flask(__name__, static_folder='static')
AUDIO_DIR = 'D:/Server/Jukebox/audio'
os.makedirs(AUDIO_DIR, exist_ok=True)

jobs = {}

def process_youtube(job_id, url):
    jobs[job_id] = {'status': 'downloading', 'progress': 0}
    # Download audio
    audio_path = f'{AUDIO_DIR}/{job_id}.mp3'
    subprocess.run([
        'yt-dlp', '-x', '--audio-format', 'mp3',
        '-o', audio_path, url
    ], check=True)
    jobs[job_id]['status'] = 'analyzing'
    # Analyze with Remixatron
    jukebox = InfiniteJukebox(filename=audio_path, start_beat=1, do_async=False)
    jobs[job_id] = {
        'status': 'ready',
        'audio': f'/audio/{job_id}.mp3',
        'beats': jukebox.beats,
        'segments': jukebox.segments
    }

@app.route('/api/submit', methods=['POST'])
def submit():
    data = request.json
    url = data.get('url')
    if not url: return jsonify({'error': 'no url'}), 400
    job_id = str(uuid.uuid4())[:8]
    t = threading.Thread(target=process_youtube, args=(job_id, url))
    t.daemon = True
    t.start()
    return jsonify({'job_id': job_id})

@app.route('/api/status/<job_id>')
def status(job_id):
    return jsonify(jobs.get(job_id, {'status': 'unknown'}))

@app.route('/audio/<filename>')
def serve_audio(filename):
    return send_from_directory(AUDIO_DIR, filename)

@app.route('/')
def index():
    return send_from_directory('static', 'index.html')

if __name__ == '__main__':
    app.run(port=5050, host='0.0.0.0')

🎵 How the Infinite Loop Actually Works

The algorithm works in three steps:

Beat detection

Librosa analyzes the audio file and finds every beat — the timestamp of each "tap" in the music. It also segments the song into sections (intro, verse, chorus, etc.).

Similarity analysis

For each beat, it calculates how similar it is to every other beat using timbre and loudness features. If beat 24 sounds a lot like beat 96 (e.g. both are at the start of a chorus), those become "jump candidates."

Infinite playback

As the song plays, at each beat the player randomly decides whether to continue to the next beat or jump to a similar beat elsewhere. This creates an endless, coherent-sounding remix that never exactly repeats.

🎛️ Audio Filters & Effects (Browser-Side DSP)
Architecture: Web Audio API

All filters run in the browser using the Web Audio API — no server changes needed. The jukebox plays audio through a chain of AudioNode processors that you can enable, disable, and tweak in real time. The filter chain sits between the audio source and the speakers.

AudioSource → GainNode → [Filters] → AnalyserNode → AudioContext.destination
                          ↑
              BiquadFilterNode (highpass/lowpass/EQ)
              ConvolverNode (reverb)
              StereoPannerNode
              DynamicsCompressorNode

🎸 VST2 / VST3 Support — Feasibility Assessment
Can browser-based audio use VST plugins?

Not directly — browsers can't load native VST2/VST3 DLLs. However, there are two paths forward:

Path A: Server-Side VST Processing (Recommended)

The jukebox's infinite-loop audio is processed server-side through a VST host (like JUCE or Carla), then streamed to the browser as a modified audio stream. Users pick their VST preset from the UI; the server applies the DSP.

How: Use Carla (free, FOSS VST host for Windows/Linux) as a subprocess in the Flask server. Route jukebox audio through Carla's FX rack before serving it.

# Carla on Windows (carla.kx.studio):
# 1. Load your VST2/VST3 plugin in Carla's rack
# 2. Route jukebox audio output through Carla
# 3. Capture Carla's output with sounddevice/pyaudio
# 4. Re-encode and serve as audio stream
Path B: Web-Based VST Ports

Some popular VST effects have been ported to WebAssembly and run directly in the browser — no server needed. Projects like wam-community and Web Audio Modules (WAM) offer browser-native audio plugins.

Quality and selection are improving but still limited vs real VST libraries. Better for simple effects (reverb, chorus, delay) than complex synths.

Check: webaudiomodules.com

⚠️
This is an advanced feature. Server-side VST routing requires audio I/O configuration on Windows (ASIO drivers or WASAPI loopback), which is complex. The Web Audio API filters above cover 80% of use cases without any of this complexity. Implement VST support after the basic jukebox is working well.
⏺️ Record Clip — Download What You're Hearing
Browser-side recording via MediaRecorder API

The browser's MediaRecorder API can capture the Web Audio output directly — no server storage needed. The recording happens entirely in the browser, and when the user stops, it downloads as a WebM/MP3 file immediately.

<script>
let mediaRecorder = null;
let recordedChunks = [];
let recInterval = null;
let recSeconds = 0;
const MAX_SECONDS = 3600; // 1 hour

function toggleRecording() {
  if (mediaRecorder && mediaRecorder.state === 'recording') {
    stopRecording();
  } else {
    startRecording();
  }
}

async function startRecording() {
  // Capture the Web Audio output as a media stream
  // (requires connecting a MediaStreamDestination to the audio chain)
  const dest = ctx.createMediaStreamDestination();
  // Connect your last audio node to dest instead of (or in addition to) ctx.destination:
  // lastAudioNode.connect(dest);

  recordedChunks = [];
  mediaRecorder = new MediaRecorder(dest.stream, { mimeType: 'audio/webm;codecs=opus' });
  mediaRecorder.ondataavailable = e => { if (e.data.size > 0) recordedChunks.push(e.data); };
  mediaRecorder.onstop = downloadRecording;
  mediaRecorder.start(1000); // collect data every second

  recSeconds = 0;
  document.getElementById('rec-btn').textContent = '⏹ Stop Recording';
  document.getElementById('rec-btn').style.color = '#4caf7d';
  document.getElementById('rec-status').textContent = 'Recording…';

  recInterval = setInterval(() => {
    recSeconds++;
    const h = Math.floor(recSeconds/3600), m = Math.floor((recSeconds%3600)/60), s = recSeconds%60;
    document.getElementById('rec-timer').textContent =
      [h,m,s].map(x=>String(x).padStart(2,'0')).join(':');
    if (recSeconds >= MAX_SECONDS) stopRecording();
  }, 1000);
}

function stopRecording() {
  if (mediaRecorder) mediaRecorder.stop();
  clearInterval(recInterval);
  document.getElementById('rec-btn').textContent = '⏺ Start Recording';
  document.getElementById('rec-btn').style.color = '#e05c5c';
  document.getElementById('rec-status').textContent = 'Processing…';
  document.getElementById('rec-timer').textContent = '';
}

function downloadRecording() {
  const blob = new Blob(recordedChunks, { type: 'audio/webm' });
  const url  = URL.createObjectURL(blob);
  const a    = document.createElement('a');
  const date = new Date().toISOString().slice(0,19).replace(/[T:]/g,'-');
  a.href     = url;
  a.download = `jukebox-recording-${date}.webm`;
  a.click();
  URL.revokeObjectURL(url);
  recordedChunks = [];
  document.getElementById('rec-status').textContent = 'Download started!';
  setTimeout(() => { document.getElementById('rec-status').textContent = ''; }, 3000);
}
</script>
