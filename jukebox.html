Infinite / Eternal Jukebox

// Self-Hosted · YouTube Input · Endless Remixing //

The Infinite Jukebox analyzes a song's beat structure and loops it endlessly by jumping between similar-sounding points in the track. You paste a YouTube link, it downloads the audio, analyzes it, and plays it forever.

🐉 V1 Goal: A local web server at port 5050 where you paste a YouTube URL and get back an infinite, seamlessly looping version of that song.
🧠 Background: What the Eternal Jukebox Is

The original Eternal Jukebox was made by Paul Lamere at The Echo Nest (later Spotify). It used Spotify's audio analysis API to find "branch points" in a song — moments that sound similar enough that you can jump from one to another without it sounding jarring, creating an endless remix.

Spotify deprecated the public Echo Nest API, so the original service mostly stopped working. There are several community forks and reimplementations that work locally using open audio analysis libraries.

🔧 Option A — infinite-jukebox-web (Recommended) FOSS · Node.js
What This Uses

This implementation uses librosa (Python audio analysis) to find beat/segment data locally (no Spotify API needed), combined with a Node.js web server and a browser-based visualizer. You paste a YouTube URL and it downloads, analyzes, and plays.

Create a YouTube-to-Jukebox wrapper script

Create D:\Server\Jukebox\jukebox_server.py:

from flask import Flask, request, jsonify, send_from_directory
import subprocess, os, sys, uuid, threading
from Remixatron import InfiniteJukebox

app = Flask(__name__, static_folder='static')
AUDIO_DIR = 'D:/Server/Jukebox/audio'
os.makedirs(AUDIO_DIR, exist_ok=True)

jobs = {}

def process_youtube(job_id, url):
    jobs[job_id] = {'status': 'downloading', 'progress': 0}
    # Download audio
    audio_path = f'{AUDIO_DIR}/{job_id}.mp3'
    subprocess.run([
        'yt-dlp', '-x', '--audio-format', 'mp3',
        '-o', audio_path, url
    ], check=True)
    jobs[job_id]['status'] = 'analyzing'
    # Analyze with Remixatron
    jukebox = InfiniteJukebox(filename=audio_path, start_beat=1, do_async=False)
    jobs[job_id] = {
        'status': 'ready',
        'audio': f'/audio/{job_id}.mp3',
        'beats': jukebox.beats,
        'segments': jukebox.segments
    }

@app.route('/api/submit', methods=['POST'])
def submit():
    data = request.json
    url = data.get('url')
    if not url: return jsonify({'error': 'no url'}), 400
    job_id = str(uuid.uuid4())[:8]
    t = threading.Thread(target=process_youtube, args=(job_id, url))
    t.daemon = True
    t.start()
    return jsonify({'job_id': job_id})

@app.route('/api/status/<job_id>')
def status(job_id):
    return jsonify(jobs.get(job_id, {'status': 'unknown'}))

@app.route('/audio/<filename>')
def serve_audio(filename):
    return send_from_directory(AUDIO_DIR, filename)

@app.route('/')
def index():
    return send_from_directory('static', 'index.html')

if __name__ == '__main__':
    app.run(port=5050, host='0.0.0.0')
Create the frontend UI

Create D:\Server\Jukebox\static\index.html — a minimal page with a YouTube URL input and the Remixatron beat visualizer. The visualizer shows the song's beat graph and animates the playback position jumping between loop points.

A complete starter UI is available at the Remixatron GitHub page — copy the web/ folder into your static/ directory.

Run as a startup service 
# Using NSSM:
nssm install Jukebox python "D:\Server\Jukebox\jukebox_server.py"

# Access at http://localhost:5050
⚠️
Audio analysis is slow: Librosa analyzing a 4-minute song takes 15–60 seconds depending on your CPU. The UI should show a "processing" spinner while it works. Results are cached, so the same song analyzed once is instant next time.

Remixatron (Most Feature-Complete)

github.com/drensin/Remixatron is the most actively maintained infinite jukebox engine. It has a web UI and REST API. The YouTube integration is one step of yt-dlp download + one step of Remixatron analysis.

🎵 How the Infinite Loop Actually Works

The algorithm works in three steps:

Beat detection

Librosa analyzes the audio file and finds every beat — the timestamp of each "tap" in the music. It also segments the song into sections (intro, verse, chorus, etc.).

Similarity analysis

For each beat, it calculates how similar it is to every other beat using timbre and loudness features. If beat 24 sounds a lot like beat 96 (e.g. both are at the start of a chorus), those become "jump candidates."

Infinite playback

As the song plays, at each beat the player randomly decides whether to continue to the next beat or jump to a similar beat elsewhere. This creates an endless, coherent-sounding remix that never exactly repeats.
