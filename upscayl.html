Upscayl — AI Image Upscaler

// Docker · WSL2 · Port 8094 · Authentik forward_auth · GPU Recommended //

Upscayl Web is a browser-based AI image upscaler. Upload an image, pick a scale factor and AI model, and get back a high-resolution version — all processed locally on the server. No images are stored permanently; everything stays in the browser session.

🖼️ Upscayl processes images entirely in the browser/server — nothing is uploaded to external services. Images are not stored on disk after processing; they exist only in your browser session. There is no per-user data folder for this service.
📁 User Data Storage → SSO TAB

Upscayl Web is stateless. Images are processed in-memory and delivered back to the browser. Nothing is written to D:\Users\. If users want to keep upscaled images, they download them directly from the browser.

💡
No backup, quota, or symlink setup needed for this service. The only persistent data is the Docker image itself.
⚙️ GPU Requirement Note

Upscayl Web uses AI upscaling models (Real-ESRGAN, etc.) which run significantly faster on a GPU. On CPU only, upscaling a single image can take 30–120 seconds depending on size and scale factor. The Docker image supports GPU acceleration via NVIDIA Container Toolkit.

With NVIDIA GPU

Install the NVIDIA Container Toolkit in WSL2. Add the deploy section to the compose file (see below). Upscaling a 4× image takes 3–10 seconds.

CPU-only (default)

Works without any GPU setup. Slower but functional. Fine for occasional use. Remove or skip the deploy section.

⚠️
Upscayl uses the same GPU as Sunshine. If someone is currently game-streaming via Sunshine, upscaling will compete for GPU resources and may cause stream stuttering. Communicate with family members before running heavy upscale jobs.
🐳 Deployment — Docker in WSL2
Create the compose file 
mkdir -p ~/services/upscayl && cd ~/services/upscayl

cat > docker-compose.yml << 'EOF'
services:
  upscayl:
    image: ghcr.io/upscayl/upscayl-web:latest
    container_name: upscayl
    restart: unless-stopped
    ports:
      - "8094:80"
    # No volumes needed — images are not persisted
    # Uncomment below for NVIDIA GPU acceleration:
    # deploy:
    #   resources:
    #     reservations:
    #       devices:
    #         - driver: nvidia
    #           count: 1
    #           capabilities: [gpu]
EOF
Start the container 
docker compose up -d
docker compose logs upscayl   # confirm started

Upscayl Web is now running at http://localhost:8094.

Open Windows Firewall for port 8094 
netsh advfirewall firewall add rule name="Upscayl" dir=in action=allow protocol=TCP localport=8094
Auto-start on Windows boot

Task Scheduler → Create Task → Action:

wsl -d Ubuntu -- bash -c "cd ~/services/upscayl && docker compose up -d"

Trigger: At startup, delay 2 minutes. Run whether logged on or not: ☑

🔐 Authentik — forward_auth Gate
Caddy configuration

Upscayl has no built-in auth. Protect it with Caddy forward_auth. Note that Upscayl requires specific security headers for the browser-side processing to work correctly:

# Add to your Caddyfile:
upscayl.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:8094

    # Required headers for Upscayl's browser-side WASM/SharedArrayBuffer:
    header {
        Cross-Origin-Embedder-Policy "require-corp"
        Cross-Origin-Opener-Policy "same-origin"
    }
}
⚠️
The COOP/COEP headers are required. Without Cross-Origin-Embedder-Policy: require-corp and Cross-Origin-Opener-Policy: same-origin, the browser-side AI processing will fail with a SharedArrayBuffer error.
