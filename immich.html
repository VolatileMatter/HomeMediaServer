Immich

// Self-Hosted Google Photos · Port · Docker WSL2 · Authentik OIDC //

Immich is a dedicated self-hosted photo and video backup solution — the closest thing to Google Photos available. It handles auto-backup from mobile, face recognition, AI-powered smart search, map view, shared albums, and duplicate detection. All runs locally; no data leaves the server.

⚠️
WSL2 + Docker required. Immich is Linux/Docker only. On Windows, run it inside WSL2 via Docker Compose. It won't interfere with Windows services — it's fully isolated in WSL2. If you've already set up RomM, WSL2 and Docker are already installed.
⚙️ Installation WSL2 + Docker
Confirm WSL2 + Docker are installed

See the RomM tab for WSL2 + Docker setup. If RomM is already running, skip this — you're good.

Create Immich directory and download official compose file 
mkdir ~/immich && cd ~/immich
wget -O docker-compose.yml https://github.com/immich-app/immich/releases/latest/download/docker-compose.yml
wget -O .env https://github.com/immich-app/immich/releases/latest/download/example.env
Edit the .env file 
# Key values to set in .env:
UPLOAD_LOCATION=/mnt/d/Photos/Immich    # maps to D:\Photos\Immich on Windows
DB_PASSWORD=your_strong_password_here
IMMICH_VERSION=release

The UPLOAD_LOCATION is where all uploaded photos land. Point it at your large media drive so photos don't fill up WSL2's virtual disk.

Create the D:\Photos\Immich folder on Windows first 
# In PowerShell:
New-Item -ItemType Directory -Force "D:\Photos\Immich"
Start Immich 
cd ~/immich
docker-compose up -d

# First startup takes 2–3 minutes while it initialises the database.
# Access at http://localhost:
Create the admin account 🔐

Navigate to http://localhost: — the first visit shows a signup page. Create your admin account. Then add users via Administration → Users → Create User.

Open Windows Firewall for port 
netsh advfirewall firewall add rule name="Immich" dir=in action=allow protocol=TCP localport=
Auto-start on Windows boot via Task Scheduler 🔐 
# In Task Scheduler → Create Task:
# Trigger: At startup, delay 3 minutes
# Action:  wsl -d Ubuntu -- bash -c "cd ~/immich && docker-compose up -d"

📚 Importing an Existing Photo Library
Option A — External Library (non-destructive, recommended first)

Point Immich at your existing photos folder without moving files. Immich indexes and displays them without copying. In Immich: Administration → External Libraries → Create, set the import path to /mnt/d/Photos (the WSL2 path to D:\Photos).

⚠️
External library photos are read-only in Immich — you can view them but the mobile app backup goes to UPLOAD_LOCATION instead. Run a scan after adding to index the files.
Option B — Upload via mobile app

Install the Immich mobile app, point it at your server, and enable Background Backup. The app will upload all existing camera roll photos on first sync, then auto-backup new ones as they're taken.

Option C — Bulk upload via CLI
# From WSL2, use the Immich CLI to bulk-import a folder:
npx @immich/cli upload --server http://localhost: --key YOUR_API_KEY /mnt/d/Photos/OldLibrary/

Generate an API key in Immich: Account Settings → API Keys → New API Key.

✨ Key Features
📅 Photo Timeline

Immich displays all photos in a chronological timeline sorted by date taken (from EXIF data), just like Google Photos. Swipe or scroll through years, months, and days. Thumbnail grid updates in real-time as new photos are backed up.

👤 Face Recognition & People Albums

Immich automatically detects and groups faces. After initial processing (which can take hours on a large library), you'll see a People section in the sidebar. Tap a face to name the person and browse all their photos. Runs entirely on the server CPU — no cloud involved.

🔍 Smart Search (CLIP)

Type "beach sunset 2023" or "dog in snow" and Immich returns semantically matching photos using the CLIP AI model. It understands the content of images, not just their filenames or tags. No photo tagging required. Runs locally.

🗺️ Map View

Any photo with GPS EXIF data appears on an interactive map. Navigate the map to browse photos by location. Useful for travel photos — see exactly where every shot was taken.

📁 Albums & Sharing

Create named albums and add photos to them manually or by date range. Share albums with other Immich users on the server, or generate a public share link (read-only, no account required) for external guests.

🔁 Duplicate Detection

Immich detects duplicate photos across your library automatically. Review duplicates via Administration → Jobs → Duplicate Detection. Saves space and keeps the library clean without manual review.

🎞️ Video Support

Videos are fully supported — Immich auto-transcodes them for smooth browser streaming at any bitrate. Original files are preserved. Videos appear in the timeline alongside photos.

📚 External Library

Point Immich at an existing folder of photos (e.g. D:\Photos\Archive) without moving files. Immich indexes them for search and display. Files stay in place — Immich just reads them.

How to update Immich

Immich does not auto-update. Check the Immich releases page periodically, especially after any major version bump.

Pull the latest images and restart 
cd ~/immich
docker-compose pull
docker-compose up -d
Verify the update

Open and check Administration → About for the new version number. Test SSO login in an incognito window. Record the confirmed working version in versions.json.

⚠️
Never skip major versions. Immich database migrations run sequentially. If you skip from v1.x to v1.x+5, always update one major version at a time to avoid migration failures.

⚡ Known Breakpoints
🔴 Database migration failure on update
If you skip multiple major versions, the Postgres migration chain can break. Always update incrementally. If this happens, restore from a database backup and re-run the migration from the correct version.
🟡 WSL2 network changes break the Task Scheduler auto-start
After Windows updates, WSL2 networking occasionally resets, causing the startup task to fail silently. Check if Immich is running by visiting http://[server-ip]:. If it's down, run cd ~/immich && docker-compose up -d manually in WSL2.
🟡 ML models stop processing (face recognition / smart search stall)
The ML container occasionally hangs, particularly after host sleep/hibernate. Restart it with docker-compose restart immich-machine-learning from ~/immich.
🎯 How Immich Achievements Work

Immich does not have a native webhook system. Achievements are triggered by polling the Immich API — the achievement server periodically queries Immich's REST API for stats (upload counts, album count, etc.) and fires triggers when thresholds are crossed.

# Immich API — get asset stats for a user:
GET http://localhost:/api/assets/statistics
Headers:
  x-api-key: USER_API_KEY

# Example response:
{
  "images": 1042,
  "videos": 38,
  "total":  1080
}

Generate a per-user API key via Account Settings → API Keys → New API Key in the Immich web UI. Store these in the achievement server config so it can poll each user's stats.

👤 Per-User Data
What Immich stores per user

All per-user data (photo library, albums, favourites, face assignments, shared links, API keys) is stored inside Immich's Postgres database (Docker volume immich_pgdata). The actual photo files in D:\Photos\Immich are physically separated but logically tied to the owning user in the database.

💡
All per-user metadata is in the Postgres database. No per-user folder in is needed for Immich. Back up the database volume and the \Immich folder to fully preserve all user data.

🛠️ Common Issues
❌ Immich is unreachable — containers aren't running
Check if the containers are up:
cd ~/immich && docker-compose ps

If any containers are stopped or restarting:
docker-compose logs --tail=50

Try a full restart:
docker-compose down && docker-compose up -d
❌ Database migration failed after update
Check the Postgres container logs:
docker-compose logs immich_postgres --tail=100

If migration failed, roll back by restoring your most recent immich_backup.sql backup, then update one version at a time rather than skipping. Do not run the containers with a partially-migrated database.
🟡 SSO login redirects back to login page without signing in
The redirect URI in Authentik must exactly match the one Immich sends. In Authentik admin () → Applications → Immich → Edit Provider → verify the Redirect URI is:
/auth/login

Also check that the Issuer URL in Immich Administration → OAuth matches .
🟡 Face recognition / smart search not processing
The ML container may have hung (common after host sleep):
docker-compose restart immich-machine-learning

Then go to Administration → Jobs and manually queue a Smart Search or Face Detection job to resume processing.
🟡 Mobile backup not running in background
On iOS, Immich background backup requires the app to have Background App Refresh enabled (Settings → Immich → Background App Refresh = On). On Android, ensure Immich is excluded from battery optimisation. Also confirm the server URL in the app resolves correctly from outside the LAN if using an external URL.
🟢 Thumbnails are missing or broken after moving the upload folder
If you change UPLOAD_LOCATION in .env, update the Docker volume mount in docker-compose.yml to match, then regenerate thumbnails: Administration → Jobs → Generate Thumbnails → Run All. This can take a while on a large library.
