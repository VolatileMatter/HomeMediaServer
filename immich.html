// Immich vs Nextcloud Photos · Self-Hosted Google Photos //

Two options for self-hosted photo storage: Immich (dedicated photo app, closest to Google Photos) or Nextcloud's built-in Photos app. Here's how they compare and why Immich wins for most use cases.

🐉
Recommendation: Use Immich. If you want an experience that feels like Google Photos — beautiful timeline, face albums, "On This Day", map view, smart search — Immich is the only self-hosted option that genuinely delivers it. Nextcloud Photos is functional but it's a photo viewer bolted onto a file sync tool. The setup complexity difference is minor.

That said, if you only need "a place to store and browse photos" and you already have Nextcloud running, the built-in Photos app might be enough. Use Immich if you want to actually replace Google Photos.
🐉 Recommendation: Run Immich. Install after Nextcloud is up. They serve different purposes — Nextcloud for files/documents, Immich for photos/videos.
⚠️
WSL2 / Docker required. Immich is officially Linux/Docker only. On Windows, the cleanest path is WSL2 + Docker Compose. This also means Immich won't interfere with your Windows services at all — it's isolated in WSL2.
⚙️ Installation via WSL2 + Docker WSL2
Ensure WSL2 + Docker are installed

See the RomM tab for WSL2 + Docker install steps. If you set up RomM first, this is already done.

Create Immich directory and download official compose file 
mkdir ~/immich && cd ~/immich
wget -O docker-compose.yml https://github.com/immich-app/immich/releases/latest/download/docker-compose.yml
wget -O .env https://github.com/immich-app/immich/releases/latest/download/example.env
Edit the .env file 
# Key settings in .env:
UPLOAD_LOCATION=/mnt/d/Photos/Immich    # Windows D:\Photos\Immich
DB_PASSWORD=strongpassword
IMMICH_VERSION=release

The UPLOAD_LOCATION points to your Windows storage drive via the WSL2 mount. Photos uploaded to Immich land here.

Start Immich 
docker-compose up -d

# First startup takes 2-3 minutes while it initializes the database.
# Access at http://localhost:2283
Create admin account at http://localhost:2283

First visit shows a signup page. Create your admin account. Then add users via Administration → Users → Create User.

Auto-start on Windows boot 
# Add to Task Scheduler (At startup, delay 3 minutes):
# Action: wsl -d Ubuntu -- bash -c "cd ~/immich && docker-compose up -d"
Open firewall 
netsh advfirewall firewall add rule name="Immich" dir=in action=allow protocol=TCP localport=2283

In the mobile app: Settings → Server URL → enter http://[server-ip]:2283. Log in with your Immich account. Enable "Background Backup" — the app will automatically upload new photos as they're taken.

✨ Key Features
Face Recognition & People Albums

Immich automatically groups photos by faces. After initial processing (can take hours on a large library), you'll see "People" in the UI — tap a face to name the person and see all their photos. Processing uses the CPU and runs in the background.

Smart Search

Type "beach sunset 2023" and Immich finds matching photos using CLIP (AI image understanding). No need to tag photos manually. Runs locally — no data leaves your server.

Album Sharing

Create albums and share them with other Immich users on your server, or generate a public share link (read-only, no account needed) for external guests.

External Library

Point Immich at an existing folder of photos without moving them. Administration → External Libraries → add your D:\Photos folder. Immich indexes and displays them without copying.

Store photos on your large media HDD (e.g. D:\Photos\Immich). The Immich database (metadata, thumbnails, ML models) is ~5GB and stays in WSL2's home directory.

🔐 SSO Integration → SSO TAB

Immich supports OAuth2/OIDC natively. Administration → OAuth → enable → set your Google OAuth or Authentik OIDC endpoint. Users then see "Sign in with Google" on the login page.
