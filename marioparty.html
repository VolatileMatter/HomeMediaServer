Mario Party Group Play

// Netplay · 4 Players · Code-Based Sessions · N64 & GCN //

Host a multiplayer Mario Party session where up to 4 players connect using a code, each with their own controller. This requires a netplay-capable emulator setup — not simple browser emulation.

🎲
Honest complexity warning: True 4-player emulator netplay is technically challenging. Input lag is real. N64 netplay via Mupen64Plus works best on fast internet (low ping between players). This is a best-effort setup, not a turnkey solution.
🐉 V1 Goal: Host an N64 or GameCube Mario Party session where players join via a room code using Parsec or NetPlay, each connecting their own controller. You host on your server, they connect from their PC.
🧠 How This Works: Two Approaches
Option A: Parsec Arcade (Easiest) RECOMMENDED

Parsec is a game streaming app. You run the emulator on your server, share a "game room" link or code, and up to 4 friends connect. Their keyboard/controller input is sent to your machine, and the video is streamed back.

How it feels: Like playing split-screen together — you're all looking at the same screen, same instance, same game. Latency is 30–80ms typically. Perfect for Mario Party since it's turn-based.

Option B: Emulator NetPlay (Harder, Lower Latency)

Mupen64Plus/RetroArch supports proper netplay where each player runs their own emulator and they sync game state over the internet. More complex, requires port forwarding, can have sync issues.

Best for: Platformers, fighting games. Mario Party works fine with Parsec.

🎮 Option A: Parsec Arcade Setup (Recommended)
Install Parsec on your server

Download from parsec.app/downloads. Create a free Parsec account. Install on your server — this is the host.

Install Project64 or RetroArch (N64)

For N64 Mario Party (1, 2, 3): install Project64 3.0 (free, from project64-emulator.com). Load your Mario Party ROM.

For GameCube Mario Party 4–7: use Dolphin (already in the GameCube tab).

Configure controllers in the emulator

In Project64: set up 4 controller slots. With Parsec, each remote player's controller input maps to a separate slot. In Project64 input settings, set each player's controller to use a different XInput device (XInput/0, XInput/1, etc.).

🎮
Virtual controller tip: Install ViGEmBus (free) on Windows — it creates virtual XInput controllers for remote players. Parsec uses it automatically. Each friend who connects gets a virtual controller slot.
Host a Parsec Arcade room

In Parsec → Arcade → Host Game. Set:

Game:        (browse to your emulator EXE, or the already-running window)
Max guests:  3 (you + 3 friends = 4 players)
Room link:   Share this code/link with your 3 friends

Guests install Parsec for free and join via the link. They connect their own controller (USB/Bluetooth) to their own PC — Parsec forwards those inputs to your server.

Friends connect via the link

Friends go to the shared Parsec link, install Parsec if they haven't, and join the session. They see your emulator screen. Each person's controller controls the corresponding player in the emulator.

🏆
Best Mario Party games for this setup: Mario Party 2 (N64) is widely considered the best of the N64 era. Mario Party 4 (GCN) is the best GameCube entry. Both work great over Parsec since turns take seconds to execute and latency is barely noticeable.
🎮 Option B: RetroArch NetPlay (N64)
Install RetroArch on all players' PCs

Download from retroarch.com. Install the Mupen64Plus-Next core from the core downloader.

Host sets up a NetPlay session

On your server: RetroArch → Load Core → Mupen64Plus-Next → Load Content → (your Mario Party ROM) → NetPlay → Start NetPlay Host:

Port:        55435 (default, open in firewall)
Password:    set a room password
Server Mode: Server (you host)
Spectators:  Disabled
Open firewall port 
netsh advfirewall firewall add rule name="RetroArch NetPlay" dir=in action=allow protocol=TCP localport=55435
netsh advfirewall firewall add rule name="RetroArch NetPlay UDP" dir=in action=allow protocol=UDP localport=55435
Friends connect

Friends load the same ROM in RetroArch → NetPlay → Connect to NetPlay Host → enter your public IP (or domain) and port 55435. They need the same ROM file (same hash) to join.

⚠️
ROM sync: All players need an identical ROM file (same region, no modifications). If someone has a different dump, they'll get a sync error. Share the exact same file.
🔐 Access Control

For Parsec: the room link is the password — don't share it publicly. You can also set a PIN in Parsec Arcade settings that guests must enter.

For RetroArch NetPlay: set a room password in NetPlay settings. Players must enter it to join.

Neither approach requires SSO — the session is controlled by who has the room link/password, which you share directly with your friends.

💾 Save Data Notes

Mario Party is primarily a session-based multiplayer game — each match is self-contained and results aren't typically saved between sessions. The N64 games use a battery-save for unlocking boards, and the GameCube games use a memory card.

For GameCube games played via Dolphin: the memory card save is managed by the same Dolphin symlink system as all other GameCube games. When you start a session, the session manager points Dolphin's memory card at D:\Users\[your-username]\emulation\dolphin\gc\. This preserves any unlocked boards or minigames across sessions.

For N64 games played via Project64 or Parsec: saves are stored locally on the host machine. These are shared (the host's unlocks are available for all sessions), which is appropriate since Mario Party unlocks are server-wide content, not per-user data. No symlink needed.

See SSO tab → User Data Storage for the full architecture.
