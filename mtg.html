⚔️ XMage — Full MTG Client with AI Opponents
🐉
XMage is a full-featured MTG client. You can play against AI (Mage AI or Monte Carlo) or against other humans on your LAN. It handles rules enforcement automatically — perfect for testing decks without arguing about interactions.
Step 1: Install Java 17 LTS

XMage requires Java 17. Download from Adoptium Temurin 17 — the Windows installer, x64. Run it with all defaults.

Step 2: Download the XMage launcher

From xmage.today → Download → download XMageLauncher.exe. Place it in D:\Games\MTG\XMage\. Run it once — it will download the full client and server automatically.

Step 3: Configure the server

In XMage Launcher → Config tab:

Server address:   localhost
Server port:      17171
Max game threads: 4 (or half your CPU cores)
Deck folder:      D:\Games\MTG\Decks\mine
                  D:\Games\MTG\Decks\enemy  ← for AI opponent decks

Check "Start server automatically" to launch the server when you open XMage.

Step 4: Set up AI game

XMage Launcher → Start Server → Start Client → Connect to localhost:17171 → New Game → vs Computer → select your deck from D:\Games\MTG\Decks\mine. The AI picks from D:\Games\MTG\Decks\enemy.

Step 5: LAN multiplayer (optional)

Others on your network can connect to [server-ip]:17171 using their own XMage client. Expose via Tailscale for friends outside the LAN. No account or internet required.

Deck file format (.dck)

XMage uses .dck format. Each deck is a text file:

# Example Commander deck (commander_dragons.dck):
NAME:Dragon Tribal
COMMANDER:Scion of the Ur-Dragon
1 Scion of the Ur-Dragon
4 Dragon's Hoard
3 Dragonspeaker Shaman
2 Crucible of Fire
4 Lightning Bolt
# ... etc

Decks saved from the Deck Manager above are exported in this format automatically to D:\Games\MTG\Decks\mine\ or D:\Games\MTG\Decks\enemy\.

🔥 Forge — Single-Player AI with Draft and Sealed
🐉
Forge is a self-contained single-player MTG client with an excellent AI. Best for: testing decks solo, draft/sealed simulation, quest mode, and playing formats XMage handles less well. Both Forge and XMage read from the same deck folders via symlinks.
Step 1: Download Forge

From github.com/Card-Forge/forge/releases → latest release → download forge-gui-desktop-[version]-jar-with-dependencies.jar (or the Windows installer if available). Place in D:\Games\MTG\Forge\.

Step 2: Launch Forge and download card images 
java -jar forge-gui-desktop-*.jar

On first launch: Preferences → Content Management → Download Card Images. This downloads from Scryfall (~3–10 GB). Let it run in the background.

Step 3: Point Forge's deck folder at the shared deck location

Forge stores decks in %UserProfile%\AppData\Roaming\Forge\decks\ by default. After creating the symlink (see Symlinks tab), Forge reads from D:\Games\MTG\Decks\mine automatically. No configuration needed if the symlink is set up first.

Step 4: Configure AI opponent decks

Forge's built-in AI uses its own prebuilt decks by default. To make it use your enemy library, place decks in Forge's AI deck folder — the symlink handles this:

D:\Games\MTG\Forge\res\decks\constructed\  ← symlink target for enemy decks
Step 5: Create a desktop shortcut 
# Create D:\Games\MTG\launch_forge.bat:
@echo off
cd /d "D:\Games\MTG\Forge"
java -Xmx2048m -jar forge-gui-desktop-*.jar %*
Forge deck format (.dck)

Forge also uses the .dck format — exactly the same as XMage. This is why one set of deck files works for both programs.

[metadata]
Name=Dragon Tribal

[main]
4 Dragon's Hoard
3 Dragonspeaker Shaman
2 Crucible of Fire
4 Lightning Bolt

[sideboard]
2 Dragon Tempest
🔗 Shared Deck Folder via Symlinks
💡
Both XMage and Forge need to read the same deck files. Instead of copying decks to two places, use a single shared folder on your data drive and create Windows symlinks (junction points) pointing each app to that folder. When you export a deck from the Deck Manager, it writes to one place and both apps see it instantly.
Folder structure
D:\Games\MTG\
  ├── Decks\
  │   ├── mine\        ← YOUR decks (playable in XMage + Forge as your decks)
  │   │   ├── dragon_tribal.dck
  │   │   └── mono_red_burn.dck
  │   └── enemy\       ← OPPONENT decks (XMage AI + Forge AI pick from here)
  │       ├── control_blue.dck
  │       └── aggro_white.dck
  ├── XMage\           ← XMage installation
  └── Forge\           ← Forge installation
Create the symlinks (run PowerShell as Administrator)
# First, create the master deck folders if they don't exist:
New-Item -ItemType Directory -Force -Path "D:\Games\MTG\Decks\mine"
New-Item -ItemType Directory -Force -Path "D:\Games\MTG\Decks\enemy"

# ── XMage symlinks ──
# XMage reads player decks from its own deck folder by default.
# Point XMage's deck folder to our shared mine/ folder:
$xmage_decks = "$env:LOCALAPPDATA\XMage\Decks"
if (Test-Path $xmage_decks) { Remove-Item $xmage_decks -Recurse }
New-Item -ItemType Junction -Path $xmage_decks -Target "D:\Games\MTG\Decks\mine"
Write-Host "✓ XMage player decks → D:\Games\MTG\Decks\mine"

# XMage AI opponent decks (computer decks folder):
$xmage_ai = "$env:LOCALAPPDATA\XMage\PlayerDecks\computer"
if (Test-Path $xmage_ai) { Remove-Item $xmage_ai -Recurse }
New-Item -ItemType Junction -Path $xmage_ai -Target "D:\Games\MTG\Decks\enemy"
Write-Host "✓ XMage AI decks → D:\Games\MTG\Decks\enemy"

# ── Forge symlinks ──
# Forge player decks:
$forge_decks = "$env:APPDATA\Forge\decks\constructed"
if (Test-Path $forge_decks) { Remove-Item $forge_decks -Recurse }
New-Item -ItemType Junction -Path $forge_decks -Target "D:\Games\MTG\Decks\mine"
Write-Host "✓ Forge player decks → D:\Games\MTG\Decks\mine"

# Forge AI decks (constructed AI opponent pool):
$forge_ai = "D:\Games\MTG\Forge\res\decks\constructed"
if (Test-Path $forge_ai) { Move-Item $forge_ai "$forge_ai.bak" }
New-Item -ItemType Junction -Path $forge_ai -Target "D:\Games\MTG\Decks\enemy"
Write-Host "✓ Forge AI decks → D:\Games\MTG\Decks\enemy"

Write-Host "`n✓ All symlinks created. Both XMage and Forge now share the same deck files."
⚠️
Run this script after installing both XMage and Forge (so their default deck folders exist to be replaced). If Forge hasn't been run yet, its AppData folder may not exist — launch it once first, then run this script.
Deck file export from the Deck Manager

Deck metadata is stored in browser localStorage for the UI. When you click Export .dck, the file is written to D:\Users\[username]\mtg\decks\ — the user's personal folder. The D:\Games\MTG\Decks\mine symlink (managed by session_manager.py) always points there, so XMage and Forge see the right decks automatically:

# D:\Server\MTG\deck_server.py
from flask import Flask, request, jsonify
import os

app = Flask(__name__)
USERS_BASE = 'D:/Users'
ENEMY_DIR  = 'D:/Games/MTG/Decks/enemy'   # shared AI opponent pool

@app.route('/export', methods=['POST'])
def export():
    data     = request.json
    deck     = data['deck']
    username = request.headers.get('X-authentik-username', 'anonymous')

    # Enemy decks → shared folder; My decks → user's personal folder
    folder = ENEMY_DIR if data.get('enemy') else f'{USERS_BASE}/{username}/mtg/decks'
    os.makedirs(folder, exist_ok=True)

    safe_name = "".join(c for c in deck['name'] if c.isalnum() or c in ' _-').strip()
    filename  = f"{safe_name}.dck"
    content   = f"NAME:{deck['name']}\n"
    if deck.get('commander'):
        content += f"COMMANDER:{deck['commander']}\n"
    content += deck.get('cards', '# Paste card list here\n')

    with open(os.path.join(folder, filename), 'w') as f:
        f.write(content)
    return jsonify({'ok': True, 'file': filename, 'folder': folder})

# D:\Games\MTG\Decks\mine is a symlink → D:\Users\[username]\mtg\decks\
# session_manager.py updates this symlink on every Sunshine/session start.
# XMage and Forge always read from the symlink path — no app reconfiguration needed.

if __name__ == '__main__':
    app.run(port=8099, host='localhost')
