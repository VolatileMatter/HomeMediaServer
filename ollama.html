Private AI — Ollama

// 100% Local · No Logs · No Internet · Claude-like Models //

Run large language models entirely on your own hardware. No data ever leaves your server. Slow compared to cloud AI, but completely private. Accessible from any device via a web UI.

🐉 V1 Goal: Ollama running as a Windows service, serving a model like Llama 3 or Mistral, with Open WebUI providing a polished chat interface accessible from any browser on your network.

Practical recommendation: If your gaming PC has an Nvidia GPU with 8GB+ VRAM, run Llama 3.1 8B — it runs fast on the GPU and gives good results. If you have 16GB+ VRAM (RTX 3090, 4090, etc.), try Llama 3.1 70B quantized (Q4) — much smarter, still runs at usable speed. Without a good GPU, any model will be very slow (CPU-only).
About Speed

On a mid-range gaming GPU (RTX 3060/3070), expect:

  • 7B/8B models: 20–60 tokens/second — fast, usable in real time
  • 13B models: 10–25 tokens/second — a bit slower, still fine
  • 70B models (quantized): 2–8 tokens/second — noticeably slow but functional
  • CPU only: 1–5 tokens/second for any model — usable for short prompts only
⚙️ Step 1: Install Ollama FOSS · Windows Native
Download and install Ollama

From ollama.com/download/windows. The Windows installer registers Ollama as a system service that starts automatically. After install, Ollama runs at http://localhost:11434.

Pull your first model

Open a PowerShell or Command Prompt window:

# Recommended first model (8GB VRAM / 8GB RAM):
ollama pull llama3.1

# If you have 16GB+ VRAM, try the bigger quantized version:
ollama pull llama3.1:70b-instruct-q4_K_M

# Ultra-lightweight (works on anything with 4GB RAM):
ollama pull phi3

# Great instruction-following (6GB VRAM):
ollama pull mistral

Models are stored in C:\Users\[you]\.ollama\models\. They can be moved to a different drive — see below.

Test it from the command line 
ollama run llama3.1
# You'll get a prompt: >>> 
# Type anything, hit Enter. Type /bye to exit.
Move models to a larger drive (optional)

Models are large (4–40GB each). Move the storage to your big internal drive:

# Set environment variable OLLAMA_MODELS before running Ollama:
# System Properties → Environment Variables → New System Variable:
# Name:  OLLAMA_MODELS
# Value: D:\AI\OllamaModels

# Then re-pull your models — they'll download to the new location.
# Or move the existing folder and set the variable.
Allow network access (for web UI and remote devices)

By default Ollama only listens on localhost. Set it to accept connections from your LAN:

# System Properties → Environment Variables → New System Variable:
# Name:  OLLAMA_HOST
# Value: 0.0.0.0:11434

# Then restart the Ollama service:
# Task Manager → Services → Ollama → Restart
# Or: net stop ollama && net start ollama
# Open firewall:
netsh advfirewall firewall add rule name="Ollama" dir=in action=allow protocol=TCP localport=11434
⚠️
Security: Don't expose port 11434 to the internet — it has no built-in auth. Access it via Tailscale or put the Open WebUI behind Caddy with authentication (see below).
🌐 Step 2: Open WebUI — Chat Interface FOSS
What Open WebUI Is

Open WebUI is a polished, ChatGPT-like web interface for Ollama. It runs in any browser, supports multiple models, has conversation history, file uploads, system prompt customization, and per-user accounts. It's the main way users on your server will interact with the AI.

Install Node.js 20 LTS (if not already)

From nodejs.org.

Install Open WebUI via pip (Python) 
pip install open-webui

# Run it:
open-webui serve --port 3000

Access at http://localhost:3000. On first launch, create an admin account. It auto-detects Ollama at localhost:11434.

Run Open WebUI as a Windows Service 
# Using NSSM:
nssm install OpenWebUI "python" "-m open_webui serve --port 3000"
nssm set OpenWebUI AppEnvironmentExtra OLLAMA_BASE_URL=http://localhost:11434
nssm start OpenWebUI
Open firewall 
netsh advfirewall firewall add rule name="Open WebUI" dir=in action=allow protocol=TCP localport=3000
🤖
Open WebUI features worth knowing: It supports multiple users with separate conversation histories. It can be set to require login. You can create custom "Model" presets with specific system prompts (e.g. a "Research Assistant" persona with certain instructions baked in). It also supports RAG (Retrieval Augmented Generation) — you can upload documents and the AI will answer questions about them.
🔒 Privacy Confirmation
What "No Logs" Actually Means Here
  • Ollama itself does not log conversations or send any data to external servers
  • Open WebUI stores conversation history in a local SQLite database on your server — no external calls
  • The model runs entirely in RAM/VRAM on your machine — inference is 100% local
  • Zero internet required after the initial model download — you can firewall the entire thing after setup

To verify: block Ollama and Open WebUI in Windows Firewall after setup and they still work perfectly — because they only talk to each other on localhost.

🌍 Remote Access

The cleanest approach: put Open WebUI behind Caddy with a login requirement, exposed via Tailscale — add a Caddyfile block for ai.ixo.lol with Authentik forward_auth. Users go to ai.ixo.lol, log in, and chat — all traffic stays within your infrastructure.

# Caddyfile entry:
ai.ixo.lol {
    reverse_proxy localhost:3000
    # Open WebUI has its own login — no need for Caddy basicauth
}

Open WebUI's own user accounts system is sufficient for access control — users must have an account to use it.
