Seedbox + Automated TV

// Always Behind VPN · Auto TV Downloads · Other Services Unaffected //

A torrent client that routes ALL traffic through a VPN, but only the torrent client — not Plex, Minecraft, or your other services. Plus Sonarr for automated TV episode downloading.

🐉
Legal note: This guide assumes you're downloading content you have the right to download (Linux ISOs, your own rips, public domain content, or content you've purchased). The VPN-binding technique itself is a legitimate privacy tool.
🐉 V1 Goal: qBittorrent bound to a VPN interface so if the VPN drops, torrents stop. Sonarr watches RSS feeds for your TV shows and automatically adds them to qBittorrent. Other services unaffected.
🧠 How VPN Binding Works

The trick is binding qBittorrent to the VPN's network interface. When the VPN is connected, traffic flows through it. If the VPN disconnects, qBittorrent has no interface to use and stops sending/receiving — it doesn't fall back to your real IP. Other apps (Plex, Minecraft, etc.) keep using your normal internet connection.

💡
Why this is better than a VPN killswitch: A killswitch blocks ALL internet on VPN drop — including your Plex streams and Minecraft server. Interface binding only isolates the torrent client, so everything else keeps working.

Install VPN with Dedicated Interface

Install your VPN provider's Windows client. Connect to a server. Open Network & Internet Settings → Change adapter options. You should see a new network adapter (often named "Mullvad" or "ProtonVPN" or "TAP-Windows Adapter"). Note the name — you'll need it.

⚠️
Keep the VPN client running: Set the VPN client to launch on Windows startup so it's always connected before qBittorrent tries to use it.
⬇️ Step 2: Install & Configure qBittorrent
Download qBittorrent

From qbittorrent.org. Free, open source. Install normally.

Bind to VPN interface

Tools → Options → Advanced → "Network interface" → select your VPN adapter from the dropdown (e.g., "Mullvad" or "ProtonVPN TAP"). This is the kill-switch binding.

Also set "Optional IP address to bind to" to the VPN's IP if your provider gives you a static VPN LAN IP.

Enable the Web UI

Tools → Options → Web UI → ☑ Enable web user interface. Set port 8090. Set a strong username/password. This is how Sonarr communicates with qBittorrent.

URL to access: http://localhost:8090
Configure download paths 
Default save path: D:\Downloads\Complete
Temp path:         D:\Downloads\Incomplete

In the Categories section, add a "tv-sonarr" category pointing to D:\Downloads\Complete\TV.

Open firewall for qBittorrent Web UI 
netsh advfirewall firewall add rule name="qBittorrent WebUI" dir=in action=allow protocol=TCP localport=8090
Set qBittorrent to launch on startup

Tools → Options → Behavior → ☑ "Start qBittorrent on Windows startup." It will minimize to tray.

Verify VPN Binding Is Working

With qBittorrent running and VPN connected, add a test torrent (a Linux ISO). You should see traffic. Now disconnect the VPN — qBittorrent should show all torrents as "Stalled" or "Error" and stop transferring. Reconnect VPN → torrents resume. That's working correctly.

📺 Step 3: Sonarr — Automated TV Show Downloads
What Sonarr Does

Sonarr monitors RSS feeds (indexers) for new episodes of TV shows you've added to its library. When a matching episode appears, it automatically sends it to qBittorrent. After download completes, it moves the file to your Plex media folder and renames it correctly.

Download and install Sonarr for Windows

From sonarr.tv. The installer sets up Sonarr as a Windows Service. Access at http://localhost:8989.

Add qBittorrent as a Download Client

Settings → Download Clients → + → qBittorrent:

Host:     localhost
Port:     8090
Username: [your qbit username]
Password: [your qbit password]
Category: tv-sonarr

Click Test — should show a green checkmark.

Add Indexers (RSS feed sources)

Settings → Indexers → + → Add your indexers. Sonarr supports both public indexers (like EZTV, free) and private trackers. For public indexers without Jackett:

EZTV:     https://eztv.re/ezrss.xml
1337x:    Use Jackett (see below)
(Optional) Add Jackett for More Indexers

Jackett is a Windows service that translates many torrent sites into Sonarr-compatible RSS feeds. Download from github.com/Jackett/Jackett. Install → runs at http://localhost:9117. Add trackers in Jackett, then add them to Sonarr using the Torznab feed URL Jackett provides.

Configure Root Folders

Settings → Media Management → Root Folders → Add → D:\Media\TV (your Plex TV folder). Sonarr will move completed downloads here and rename them.

Add TV Shows

Series → Add Series → Search for your show → select quality profile → Add. Sonarr will immediately search for existing episodes and start monitoring for new ones automatically.

📺
Radarr for movies: Radarr is the movie equivalent of Sonarr — same concept, same setup process, just for movies. Both are from the same development team and integrate identically with qBittorrent.
🎬 Step 4: Connect to Plex

In Sonarr: Settings → Connect → + → Plex Media Server:

Host:     localhost
Port:     32400
Auth Token: (get this from Plex: sign in to plex.tv → account → 
             open dev tools → any API call → copy X-Plex-Token value)

Check "Update Library." Now when Sonarr downloads a new episode and moves it to the Plex folder, it automatically tells Plex to refresh that section — the episode appears in Plex within seconds.
