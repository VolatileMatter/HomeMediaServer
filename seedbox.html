Purchase and install Mullvad

Buy at mullvad.net — you get an account number, no email required. Download the Windows client and install it.

Configure Mullvad settings for torrenting

Open Mullvad → Settings:

VPN Protocol:       WireGuard (faster, more stable)
Auto-connect:       ☑ On (connects on Windows startup)
Launch on startup:  ☑ On
Kill switch:        ☑ On (belt-and-suspenders, though we also bind qBit)
LAN access:         ☑ Allow (critical — lets your other services reach the LAN)
⚠️
"Allow LAN" must be ON. Without it, Plex, Nextcloud, Minecraft etc. won't be reachable on your local network when Mullvad is connected. This setting makes Mullvad tunnel only internet traffic, not LAN traffic.
Enable Port Forwarding in Mullvad

In the Mullvad app → Settings → VPN Settings → Enable port forwarding. Note the port number assigned — you'll enter this in qBittorrent as the listening port. This allows peers to connect to you for better seeding speeds.

💡
Port forwarding is only available on certain Mullvad servers (most but not all). Choose a server that supports it — the Mullvad app shows which ones have a port forward icon.
Find the Mullvad network adapter name

Once connected, open Settings → Network & Internet → Change adapter options. Look for the new adapter — it will be named something like "Mullvad" or "WireGuard Tunnel". Note the exact name — needed for qBittorrent binding.

# Or via PowerShell:
Get-NetAdapter | Where-Object {$_.InterfaceDescription -like "*Mullvad*" -or $_.Name -like "*Mullvad*"}
⬇️ Step 2: Install & Configure qBittorrent
Download and install qBittorrent

From qbittorrent.org — free, open source, best Windows torrent client. FOSS

Bind to the Mullvad network interface

Tools → Options → Advanced tab:

Network interface:             [select Mullvad adapter from dropdown]
Optional IP address to bind:   (leave blank unless you have a static VPN IP)

This is the kill switch: if Mullvad disconnects, that adapter disappears and qBittorrent stops all transfers immediately.

Set the listening port to your Mullvad forwarded port

Tools → Options → Connection:

Port used for incoming connections: [your Mullvad forwarded port number]
☑ Use UPnP / NAT-PMP: UNCHECK this (the port is already forwarded via Mullvad)
Configure download paths 
Default save path:    D:\Downloads\Complete
Keep incomplete:      ☑ On
Incomplete directory: D:\Downloads\Incomplete

Add a Category for each automated service:

  • Category tv-sonarr → save path D:\Downloads\Complete\TV
  • Category radarr → save path D:\Downloads\Complete\Movies
Enable the Web UI (for Sonarr/Radarr integration) 
Tools → Options → Web UI:
☑ Enable web user interface
Port:     8090
Username: [set a strong username]
Password: [set a strong password]
Bind to localhost only: ☑ (don't expose to internet)
Start qBittorrent with Windows & verify VPN binding

Tools → Options → Behavior → ☑ Start qBittorrent on Windows startup (minimized to tray).

Test the binding: Add a test torrent. Confirm it downloads. Disconnect Mullvad. The torrent should immediately stall. Reconnect — it resumes. ✓

📡 Step 3: Prowlarr — Unified Indexer Manager FOSS
What Prowlarr Does

Prowlarr replaces Jackett as the indexer manager. It integrates directly with Sonarr and Radarr — add an indexer once in Prowlarr and it automatically syncs to all your *arr apps. No more configuring Jackett feeds in each app separately.

Download and install Prowlarr for Windows

From github.com/Prowlarr/Prowlarr. Runs as a Windows Service. Access at http://localhost:9696.

Add indexers in Prowlarr

Indexers → Add Indexer → search for public indexers (EZTV, RARBG mirrors, 1337x, etc.) or your private tracker. Prowlarr handles the RSS feeds automatically.

Connect Prowlarr to Sonarr and Radarr

Prowlarr → Settings → Apps → Add → Sonarr:

Prowlarr Server: http://localhost:9696
Sonarr Server:   http://localhost:8989
API Key:         (from Sonarr: Settings → General → API Key)

Repeat for Radarr. All indexers in Prowlarr now appear automatically in both apps.

📺 Step 4: Sonarr — Automated TV Downloads FOSS
Install Sonarr

From sonarr.tv — Windows installer, runs as a service. Access at http://localhost:8989.

Add qBittorrent as download client

Settings → Download Clients → + → qBittorrent:

Host:     localhost
Port:     8090
Username: [your qbit username]
Password: [your qbit password]
Category: tv-sonarr
Set root folder to your Plex TV directory

Settings → Media Management → Root Folders → D:\Media\TV

Add TV shows

Series → Add Series → search → select quality profile → Add Series. Sonarr monitors for new episodes and auto-downloads them. After download, it moves to the Plex folder and renames correctly.

Connect to Plex for auto-refresh

Settings → Connect → + → Plex Media Server → enter localhost:32400 and your Plex token. Now when Sonarr adds an episode, Plex refreshes that section automatically.

🎬 Step 5: Radarr — Automated Movie Downloads FOSS

Radarr is identical to Sonarr but for movies. Install from radarr.video, access at http://localhost:7878. Setup is identical — point it at qBittorrent (category: radarr) and root folder D:\Media\Movies. Add movies, set quality profiles, done.

The Wanted → Missing view lets you manually trigger a search for any movie in your list that hasn't been found yet.

🔐 Security Notes
  • All *arr apps should be localhost only — never expose ports 8989, 7878, 9696, 8090 to the internet
  • Access them via Tailscale when remote, or add them behind Caddy with auth if needed
  • Set API keys on all apps (they generate one by default) — don't share these
  • qBittorrent Web UI: keep "Bypass auth for localhost" unchecked
