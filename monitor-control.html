Dashboard Setup Guide

// How to Deploy This Dashboard & Make It Publicly Viewable //
🏗️ Architecture: Two Dashboards
🌍 Public Status Page (read-only)

A stripped-down version of this dashboard — service status lights, recent downloads, player counts. No controls. Served publicly at status.yourdomain.com. File: monitor-public.html.

Anyone on the internet can see if your services are up, without being able to touch anything.

🔒 Private Control Panel (this page)

Full controls for starting/stopping services. Only accessible from your local network (or Tailscale). File: monitor-control.html — this page. Caddy rule blocks external access.

⚙️ Enabling Live System Stats
Option A: Local Stats Agent (Python script)

Create a tiny Python script that serves system stats as JSON, then the dashboard reads it. Install psutil:

pip install psutil flask

# stats_server.py
from flask import Flask, jsonify
import psutil, time
app = Flask(__name__)

@app.route('/stats')
def stats():
    d = psutil.disk_usage('D:\\')
    return jsonify({
        'cpu': psutil.cpu_percent(interval=1),
        'ram': psutil.virtual_memory().percent,
        'disk_used': d.percent,
        'disk_free_gb': round(d.free / 1e9, 1),
        'uptime': int(time.time() - psutil.boot_time())
    })

if __name__ == '__main__':
    app.run(port=9001)

Run this as a startup task. The dashboard will fetch http://localhost:9001/stats every 30 seconds.

🎛️ Service Start/Stop Controls

The Start/Stop buttons call a local control endpoint. Create control_server.py:

from flask import Flask, request, jsonify
import subprocess, os

app = Flask(__name__)
ALLOWED_SERVICES = ['minecraft', 'ark', 'ss13', 'audiobookshelf', 'calibre']

@app.route('/control/<service>/<action>', methods=['POST'])
def control(service, action):
    if service not in ALLOWED_SERVICES:
        return jsonify({'error': 'unknown service'}), 400
    if action == 'start':
        subprocess.Popen(f'schtasks /run /tn "{service}-start"', shell=True)
    elif action == 'stop':
        subprocess.Popen(f'schtasks /run /tn "{service}-stop"', shell=True)
    elif action == 'restart':
        subprocess.Popen(f'schtasks /run /tn "{service}-restart"', shell=True)
    return jsonify({'status': 'ok', 'service': service, 'action': action})

if __name__ == '__main__':
    app.run(host='127.0.0.1', port=9002)  # localhost only!
⚠️
Security: control_server.py binds to 127.0.0.1 only — not accessible from the network. Caddy should also block /control/ routes from external access.
🌐 Serving the Public Status Page

In your Caddy config (Caddyfile), serve the public page and block the control endpoints:

status.yourdomain.com {
    root * D:\Server\Dashboard
    file_server
    # Rewrite all requests to the public page
    rewrite * /monitor-public.html
    # Block control endpoint from internet
    @control path /control/*
    respond @control 403
}

# Internal dashboard — only via Tailscale or LAN IP
192.168.1.100, 100.x.x.x {
    root * D:\Server\Dashboard
    file_server
}
