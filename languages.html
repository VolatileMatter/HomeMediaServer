Language Learning

// LinguaCafe · Anki · LibreLingo · EN · ES · RU //

Self-hosted language learning suite: LinguaCafe for reading-based vocab acquisition, AnkiWeb-compatible flashcard server for SRS review, and LibreLingo for structured courses.

🐉 V1 Goal: LinguaCafe on port 8090 serving English, Spanish, Russian. Anki-web on port 8091 with per-user progress. LibreLingo on port 8092.
🗣️ LinguaCafe — Reading-Based Vocab FOSS
What LinguaCafe Is

LinguaCafe is a self-hosted tool inspired by LingQ. You import texts in your target language, and it tracks which words you know. As you read, you look up unfamiliar words, mark them, and they get added to your review queue. Supports 15+ languages including Spanish and Russian.

Installation on Windows (Docker via WSL2) REQUIRES WSL2

LinguaCafe is Docker-only officially. Run it under WSL2 (the lightweight Linux layer built into Windows 10).

Enable WSL2 and install Ubuntu 
wsl --install
# Reboot, then open Ubuntu from Start menu
Install Docker inside WSL2 
sudo apt update
sudo apt install -y docker.io docker-compose
sudo usermod -aG docker $USER
newgrp docker
Set up LinguaCafe 
mkdir ~/linguacafe && cd ~/linguacafe

# Download the compose file:
wget https://raw.githubusercontent.com/simjanos-dev/LinguaCafe/main/docker-compose.yml

# Configure .env — set your admin password:
echo "LINGUACAFE_ADMIN_PASSWORD=yourpassword" > .env

docker-compose up -d
Enable Spanish and Russian language packs

Access LinguaCafe at http://localhost:8090. Go to Settings → Languages → enable Español and Русский. Each language requires downloading a model from within the UI (NLP models for tokenization, ~200–500MB each).

Open firewall and auto-start 
netsh advfirewall firewall add rule name="LinguaCafe" dir=in action=allow protocol=TCP localport=8090

# Auto-start WSL2 + Docker on Windows boot:
# Create a Task Scheduler task:
# Action: wsl -d Ubuntu -e bash -c "cd ~/linguacafe && docker-compose up -d"
# Trigger: At startup, delay 2 minutes
📖
Per-user accounts: LinguaCafe has a built-in user system. Each user has their own word knowledge database. SSO via Google OAuth can be added through Authentik forward auth.
🃏 Anki — Spaced Repetition Flashcards FOSS
Architecture: AnkiWeb-Compatible Self-Hosted Server

The self-hosted option for Anki sync is anki-sync-server (official Anki project). It lets users sync their Anki decks to your server instead of ankiweb.net, with per-user progress stored locally.

Users still use the regular Anki desktop/mobile app — they just change the sync server URL to yours.

Install anki-sync-server 
pip install anki

# Run the sync server:
python -m anki.syncserver

# It starts on port 8091 by default.
# Set environment variable for data storage:
ANKI_SYNC_PATH = D:\Server\AnkiSync
Create user accounts 
# Set via environment variables before starting:
SYNC_USER1=alice:alicepassword
SYNC_USER2=bob:bobpassword

# Or use the newer config approach:
# In a .env file or Windows environment variables
Run as a Windows service with NSSM 
nssm install AnkiSync python "-m anki.syncserver"
nssm set AnkiSync AppEnvironmentExtra "SYNC_USER1=alice:password ANKI_SYNC_PATH=D:\Server\AnkiSync"
nssm start AnkiSync
Configure Anki clients to use your server

In the Anki desktop app: Tools → Preferences → Network → Self-hosted sync server → enter http://[server-ip]:8091/. Users log in with the username/password you created above.

For AnkiDroid (Android): Settings → Advanced → Custom sync server → same URL.

Open firewall 
netsh advfirewall firewall add rule name="Anki Sync" dir=in action=allow protocol=TCP localport=8091
🃏
Adding language decks: Users download community decks from ankiweb.net/shared/decks into their local Anki app, then sync to your server. Popular decks: "Spanish 5000 Most Common Words," "Russian Core 2000," "Duolingo Spanish" etc. Once synced, their progress lives on your server.
🌱 LibreLingo — Structured Courses FOSS
What LibreLingo Is

LibreLingo is a Duolingo-like FOSS alternative. It serves structured language courses with exercises, listening practice, and progress tracking. The courses are community-created and the server is self-hostable.

Install LibreLingo 
pip install librelingo-tools librelingo-yaml-loader

# Or run the web app directly:
git clone https://github.com/LibreLingo/LibreLingo
cd LibreLingo
pip install -r requirements.txt
python manage.py runserver 0.0.0.0:8092
Add Spanish and Russian courses

Download community course files from the LibreLingo course repository:

# Spanish for English speakers:
git clone https://github.com/LibreLingo/Spanish-from-English

# Russian for English speakers (check repo for current state):
# https://github.com/LibreLingo/

Note: LibreLingo is still growing — the Spanish course is the most complete. Russian is in development. Check current availability at librelingo.app.

Run as a service 
nssm install LibreLingo python "manage.py runserver 0.0.0.0:8092"
nssm set LibreLingo AppDirectory "D:\Server\LibreLingo"
nssm start LibreLingo

netsh advfirewall firewall add rule name="LibreLingo" dir=in action=allow protocol=TCP localport=8092
⚠️
LibreLingo maturity: LibreLingo is actively developed but less mature than LinguaCafe. If the course content for Russian isn't ready yet, consider using Mnemosyne (another FOSS Anki-like) or simply relying on LinguaCafe + Anki for Russian until LibreLingo catches up.
🗺️ How the Three Tools Work Together
ToolBest forLanguages
LinguaCafeReading comprehension, word acquisition from real textsEN, ES, RU + 12 others
AnkiVocabulary SRS review, memorizing grammar rules, user-curated decksAny (user-supplied decks)

Typical learning flow: LibreLingo for structure → LinguaCafe to read real content → Anki to drill vocabulary that keeps slipping.

🌐 Remote Access

All three are web apps. Add them to your Caddyfile:

languages.yourdomain.com {
    reverse_proxy localhost:8090  # LinguaCafe
}
anki.yourdomain.com {
    reverse_proxy localhost:8091  # Anki sync
}
lingo.yourdomain.com {
    reverse_proxy localhost:8092  # LibreLingo
}

Or access all via Tailscale using internal ports directly.
