Pxls — Collaborative Pixel Canvas

// Docker · WSL2 · Port 8095 · PostgreSQL + Redis · Authentik OAuth2 //

Pxls is a collaborative pixel art canvas — think r/place but private, just for your family. Each authenticated user gets a cooldown-limited pixel placement. The canvas persists indefinitely. Supports OAuth2 login so users sign in with their Authentik (Google) account.

🎨 Pxls uses native OAuth2 — users log in with their Authentik Google account directly. No Caddy forward_auth proxy needed. Pxls handles authentication itself.
📁 User Data Storage → SSO TAB

Pxls is a shared communal service — all users collaborate on the same canvas. Data is stored in the common folder rather than per-user:

D:\Users\common\pxls\
  config\      ← pxls.conf configuration file
  backups\     ← automatic canvas backups
  db\          ← PostgreSQL data
  redis\       ← Redis session data

See the SSO tab → User Data Storage for the full architecture. The common folder is not per-user quota-tracked.

🔐 Authentik — Native OAuth2 Setup (do this first)
Step 1 — Create the Authentik OAuth2 Application
# Authentik admin → Applications → Create:
Name:       Pxls
Slug:       pxls
Launch URL: https://pxls.yourdomain.com

# Create an OAuth2/OIDC Provider:
Name:                Pxls Provider
Authorization flow:  default-provider-authorization-explicit-consent
Client type:         Confidential
Redirect URI:        https://pxls.yourdomain.com/auth/authentik/callback
Scopes:              openid, profile, email
# Save the Client ID and Client Secret — you'll need them below
🐳 Deployment — Docker in WSL2
Create data directories (Windows) 
New-Item -ItemType Directory -Force "D:\Users\common\pxls\config"
New-Item -ItemType Directory -Force "D:\Users\common\pxls\backups"
New-Item -ItemType Directory -Force "D:\Users\common\pxls\db"
New-Item -ItemType Directory -Force "D:\Users\common\pxls\redis"
Create pxls.conf

Save as D:\Users\common\pxls\config\pxls.conf:

server {
  host: "0.0.0.0"
  port: 4567
}

oauth {
  # Authentik OAuth2 credentials from Authentik Application above
  useOauth: true

  authentik {
    id: "CLIENT_ID_FROM_AUTHENTIK"
    secret: "CLIENT_SECRET_FROM_AUTHENTIK"
    # Your Authentik server's OpenID config URL:
    wellKnown: "https://auth.yourdomain.com/application/o/pxls/.well-known/openid-configuration"
  }
}

# Canvas settings
canvasCode: "family"
width: 1000
height: 1000
# Cooldown in seconds between pixel placements
cooldown: 300

# Board backup
backup {
  enabled: true
  interval: "1h"
  maxBackups: 24
}
Create the compose file 
mkdir -p ~/services/pxls && cd ~/services/pxls

# Create .env file with the database password:
echo "DB_PASSWORD=$(openssl rand -hex 24)" > .env

cat > docker-compose.yml << 'EOF'
services:
  pxls:
    image: ghcr.io/pxlsspace/pxls:latest
    container_name: pxls
    restart: unless-stopped
    ports:
      - "8095:4567"
    volumes:
      - /mnt/d/Users/common/pxls/config:/app/config
      - /mnt/d/Users/common/pxls/backups:/app/backups
    environment:
      - DB_URL=jdbc:postgresql://db:5432/pxls
      - DB_USER=pxls
      - DB_PASS=${DB_PASSWORD}
      - REDIS_HOST=redis
    depends_on:
      - db
      - redis

  db:
    image: postgres:15-alpine
    restart: unless-stopped
    volumes:
      - /mnt/d/Users/common/pxls/db:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=pxls
      - POSTGRES_USER=pxls
      - POSTGRES_PASSWORD=${DB_PASSWORD}

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    volumes:
      - /mnt/d/Users/common/pxls/redis:/data
EOF
Start the stack 
docker compose up -d
docker compose logs -f pxls   # wait for startup

Pxls is now running at http://localhost:8095. Open it and verify the OAuth2 login button appears.

Open Windows Firewall for port 8095 
netsh advfirewall firewall add rule name="Pxls" dir=in action=allow protocol=TCP localport=8095
Auto-start on Windows boot

Task Scheduler → Create Task → Action:

wsl -d Ubuntu -- bash -c "cd ~/services/pxls && docker compose up -d"

Trigger: At startup, delay 2 minutes. Run whether logged on or not: ☑

🌐 Caddy Configuration

Pxls authenticates via OAuth2 itself — Caddy just does TLS and proxying, no forward_auth needed:

# Add to your Caddyfile:
pxls.yourdomain.com {
    reverse_proxy localhost:8095
}
💡
Pxls uses WebSockets for real-time canvas updates. Caddy handles WebSocket proxying automatically — no extra configuration needed.
⚙️ Canvas Administration
Making a user an admin

The first user to log in via OAuth2 is not automatically an admin. Grant admin rights via the database:

# In WSL2, connect to the PostgreSQL container:
docker exec -it pxls-db-1 psql -U pxls -d pxls

# Grant admin to a user (replace 'alice' with Authentik username):
UPDATE users SET role = 'admin' WHERE username = 'alice';
\q
Adjusting cooldown and canvas size

Edit D:\Users\common\pxls\config\pxls.conf and restart the container:

docker compose restart pxls
🔧 Updating
cd ~/services/pxls
docker compose pull
docker compose up -d
