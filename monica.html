Monica

// Personal CRM · Port · Docker WSL2 · Authentik OIDC · AI Inbox //

Monica is a personal relationship manager — track contacts, life events, reminders, and notes about the people you care about. On this server it also has an AI-powered inbox: drop a .txt or .md file into your inbox folder and Ollama automatically extracts contact data and creates Monica entries.

Monica is a PHP/Docker application backed by MySQL. Lightweight and stateless in compute. The AI inbox feature delegates to Ollama on the PC — so Monica can run on the Pi while still calling the PC's Ollama endpoint over the Tailscale network.

📁 Step 1 — Create Directory Structure 🔐 ADMIN ONLY
Per-user folder layout

Monica uses an isolated storage structure per user. Create these directories before deploying the container. Repeat for each user you'll be adding to the monica-users group.

🔐 Create base directories for each user (PowerShell) 
# Replace {NAME} with the actual username (e.g. alice, bob)
$name = "alice"

New-Item -ItemType Directory -Force "D:\Users\$name\monica\storage"
New-Item -ItemType Directory -Force "D:\Users\$name\monica\db"
New-Item -ItemType Directory -Force "D:\Users\$name\inbox"
New-Item -ItemType Directory -Force "D:\Users\$name\inbox\processed"
New-Item -ItemType Directory -Force "D:\Users\$name\backups"

# Set permissions so WSL2 container user (1000:1000) can write:
# In WSL2 terminal:
# chmod -R 755 /mnt/d/Users/$name/monica/
# chmod -R 755 /mnt/d/Users/$name/inbox/

🐳 Step 2 — Deploy Monica via Docker DOCKER WSL2
Architecture

Monica uses the official monicahq/monica:fpm image. It runs alongside an nginx container (serving the PHP-FPM app) and a MySQL container. Each user's database and storage are bind-mounted from their personal directory on the Windows host.

🔐 Create the compose file 
mkdir -p ~/services/monica && cd ~/services/monica

cat > docker-compose.yml << 'EOF'
services:

  monica-db:
    image: mysql:8.0
    container_name: monica-db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: changeme_root
      MYSQL_DATABASE: monica
      MYSQL_USER: monica
      MYSQL_PASSWORD: changeme_monica
    volumes:
      - /mnt/d/Users/common/monica/db:/var/lib/mysql
    networks:
      - monica-net

  monica:
    image: monicahq/monica:fpm
    container_name: monica
    restart: always
    depends_on:
      - monica-db
    environment:
      APP_ENV: production
      APP_KEY: base64:REPLACE_WITH_GENERATED_KEY
      APP_URL: https://monica.yourdomain.com
      DB_HOST: monica-db
      DB_PORT: 3306
      DB_DATABASE: monica
      DB_USERNAME: monica
      DB_PASSWORD: changeme_monica
      MAIL_MAILER: log
    volumes:
      - /mnt/d/Users/common/monica/storage:/var/www/html/storage
    networks:
      - monica-net

  monica-nginx:
    image: nginx:alpine
    container_name: monica-nginx
    restart: always
    depends_on:
      - monica
    ports:
      - "8100:80"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf:ro
      - /mnt/d/Users/common/monica/storage:/var/www/html/storage:ro
    networks:
      - monica-net

networks:
  monica-net:
    driver: bridge
EOF
🔐 Create the nginx config 
cat > nginx.conf << 'EOF'
server {
    listen 80;
    root /var/www/html/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass monica:9000;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires max;
        log_not_found off;
    }
}
EOF

🤖 Step 4 — AI Inbox Watchdog
🧠 The inbox watchdog monitors userdata/{NAME}/inbox/ for new .txt or .md files. When a file appears, it sends the content to Ollama (llama3.1 or mistral) asking it to extract contact information, then creates or updates the contact in Monica via the API. The file is then moved to inbox/processed/.
💡
Use case: You've just met someone at an event. You jot notes in a text file: "Met Sarah Chen at the conference. Works at Acme Corp as a lead engineer. Loves hiking. Birthday is March 4th." Drop it in your inbox — Monica creates the contact automatically.
Requirements
  • Ollama must be running with llama3.1 or mistral pulled
  • Monica API token generated (Settings → API → Create token)
  • Python 3.10+ on the server
  • The watchdog script runs as a Windows Service via NSSM
🔐 Install Python dependencies 
pip install watchdog requests --break-system-packages
🔐 Create the watchdog script

Save as D:\Scripts\monica_inbox_watchdog.py:

"""
Monica Inbox Watchdog
Watches userdata/{user}/inbox/ for .txt/.md files,
sends content to Ollama for contact extraction,
creates contact in Monica via API, then moves file to processed/.
"""
import os, time, shutil, json, requests
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler

USERDATA_ROOT = r"D:\Users"
OLLAMA_URL    = "http://localhost:11434/api/generate"
OLLAMA_MODEL  = "llama3.1"   # or "mistral"
MONICA_URL    = "http://localhost:8100"

# Map each username to their Monica API token
# Generate tokens: Monica → Settings → API → Create token
USER_TOKENS = {
    "alice": "YOUR_ALICE_MONICA_TOKEN",
    "bob":   "YOUR_BOB_MONICA_TOKEN",
}

EXTRACTION_PROMPT = """
Extract contact information from the following note.
Return ONLY a JSON object with these fields (use null if not found):
{
  "first_name": "",
  "last_name": "",
  "company": "",
  "job_title": "",
  "birthday_day": null,
  "birthday_month": null,
  "birthday_year": null,
  "notes": "",
  "email": null,
  "phone": null
}

Note:
{NOTE_TEXT}
"""

def extract_contact(text):
    payload = {
        "model": OLLAMA_MODEL,
        "prompt": EXTRACTION_PROMPT.replace("{NOTE_TEXT}", text),
        "stream": False
    }
    r = requests.post(OLLAMA_URL, json=payload, timeout=120)
    r.raise_for_status()
    raw = r.json()["response"].strip()
    # Strip markdown fences if present
    if raw.startswith("```"):
        raw = raw.split("```")[1]
        if raw.startswith("json"):
            raw = raw[4:]
    return json.loads(raw)

def create_monica_contact(contact, token):
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json",
        "Accept": "application/json"
    }
    payload = {
        "first_name": contact.get("first_name") or "Unknown",
        "last_name":  contact.get("last_name")  or "",
        "company":    contact.get("company")     or "",
        "job_title":  contact.get("job_title")   or "",
    }
    r = requests.post(f"{MONICA_URL}/api/contacts", json=payload, headers=headers, timeout=30)
    r.raise_for_status()
    contact_id = r.json()["data"]["id"]

    # Add note if present
    if contact.get("notes"):
        note_payload = {"contact_id": contact_id, "body": contact["notes"]}
        requests.post(f"{MONICA_URL}/api/notes", json=note_payload, headers=headers)

    return contact_id

class InboxHandler(FileSystemEventHandler):
    def __init__(self, username, token):
        self.username = username
        self.token    = token
        self.inbox    = os.path.join(USERDATA_ROOT, username, "inbox")
        self.done     = os.path.join(self.inbox, "processed")

    def on_created(self, event):
        if event.is_directory:
            return
        path = event.src_path
        if not (path.endswith(".txt") or path.endswith(".md")):
            return
        time.sleep(1)  # let the file finish writing
        try:
            with open(path, "r", encoding="utf-8") as f:
                text = f.read()
            print(f"[{self.username}] Processing: {os.path.basename(path)}")
            contact = extract_contact(text)
            cid = create_monica_contact(contact, self.token)
            print(f"[{self.username}] Created Monica contact ID {cid}: {contact.get('first_name')} {contact.get('last_name')}")
            dest = os.path.join(self.done, os.path.basename(path))
            shutil.move(path, dest)
            print(f"[{self.username}] Moved to processed/")
        except Exception as e:
            print(f"[{self.username}] ERROR processing {path}: {e}")

if __name__ == "__main__":
    observer = Observer()
    for user, token in USER_TOKENS.items():
        inbox = os.path.join(USERDATA_ROOT, user, "inbox")
        os.makedirs(os.path.join(inbox, "processed"), exist_ok=True)
        handler = InboxHandler(user, token)
        observer.schedule(handler, inbox, recursive=False)
        print(f"Watching {inbox} for user {user}")
    observer.start()
    try:
        while True:
            time.sleep(10)
    except KeyboardInterrupt:
        observer.stop()
    observer.join()

💾 Backups

Monica's data lives in two places: the MySQL database and the storage folder (uploaded files, avatars). Both need to be backed up.

Database backup (run periodically)

# Optionally compress:

⚠️ Known Breakpoints
🔴 Migrations fail after update
Always run docker exec -it monica php artisan migrate --force after pulling a new image. Monica frequently adds columns between releases. If the app is showing a 500 error after update, a pending migration is almost always the cause.
⚠️ Watchdog stops processing files
Check in order:
  • Is Ollama running? → curl http://localhost:11434/api/tags
  • Is the watchdog service running? → Get-Service MonicaInboxWatchdog
  • Is the model still pulled? → ollama list
  • Check logs: Get-Content D:\Logs\monica_watchdog.log -Tail 50
  • Stale file locks: move the stuck file out and back in after restarting the watchdog
⚠️ Ollama returns malformed JSON from the extraction prompt
Mistral is more reliable for structured JSON extraction than llama3.1 in some cases. Try switching OLLAMA_MODEL = "mistral" in the watchdog script. Alternatively, the note text may be too short or ambiguous — add more detail to the input file. The script strips markdown fences but if Ollama adds additional commentary, the JSON parse will fail — check the log for the raw output.
⚠️ SSO login redirect loop
Usually a mismatch between APP_URL in docker-compose.yml and the configured redirect URI in Authentik. Both must be exactly . Rebuild the container after any environment change: docker compose up -d --force-recreate monica
💡 Per-user storage (multi-tenancy)
Monica is not natively multi-tenant — each user's data lives in the same database. The userdata/{NAME}/monica/ folder structure is used for the inbox and backups, but all Monica contacts/data are stored in a shared DB keyed by user account. If you need complete data isolation between users, you'd need separate Monica stacks per user.

🛠️ Troubleshooting
🔴 Monica shows a 500 error after update
Almost always a pending migration. Run: docker exec -it monica php artisan migrate --force. If that fails, check docker logs monica for a more specific PHP error.
⚠️ SSO login fails / redirect error
Check in order:
  • Authentik provider → Redirect URIs matches exactly: /auth/callback
  • APP_URL in docker-compose.yml matches exactly (no trailing slash)
  • Restart Monica after any environment change: docker compose up -d --force-recreate monica
  • Check Authentik logs for the specific rejection reason
⚠️ AI inbox note processed but no contact created in Monica
The Ollama extraction may have succeeded but the Monica API call failed. Check D:\Logs\monica_watchdog.log for the specific API error. Common causes: API token expired or was regenerated, Monica is unreachable on port , or the extracted JSON had a null first_name that Monica rejected.
⚠️ Container fails to start — MySQL permission error
The WSL2 container needs write access to the bind-mounted DB directory. From WSL2: chmod -R 777 /mnt/d/Users/common/monica/db/. On first run, if the db/ directory has Windows-only permissions MySQL can't initialize.
💡 Inbox file is stuck (not being processed)
The watchdog only reacts to file creation events. If a file was already there when the watchdog started, it won't process it. Move the file out and back in (copy → delete → paste) to trigger the creation event, or restart the watchdog service: Restart-Service MonicaInboxWatchdog.
