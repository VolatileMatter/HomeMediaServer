Navidrome Music Server

// Self-Hosted Spotify · Subsonic API · Per-User Playlists //

Navidrome is the best FOSS self-hosted music server. Stream your entire music library from any device, with per-user play history, playlists, and favorites. Compatible with dozens of client apps.

🐉 V1 Goal: Navidrome serving your music library from an internal drive, accessible via browser at port 4533, and via mobile apps like DSub or Symfonium.
⚙️ Installation on Windows FOSS · Windows Native
Download Navidrome

From github.com/navidrome/navidrome/releases — download the latest navidrome_X.X.X_windows_amd64.zip. Extract to D:\Server\Navidrome\.

Create a configuration file

Create D:\Server\Navidrome\navidrome.toml:

MusicFolder    = "D:/Music"
DataFolder     = "D:/Server/Navidrome/data"
Port           = 4533
LogLevel       = "info"
EnableTranscoding = true
DefaultTheme   = "Dark"
EnableSharing  = true  # allows public share links for playlists
Run Navidrome 
cd D:\Server\Navidrome
navidrome.exe --configfile navidrome.toml

Access at http://localhost:4533. First login creates the admin account.

Install as a Windows Service (NSSM) 
nssm install Navidrome "D:\Server\Navidrome\navidrome.exe" "--configfile D:\Server\Navidrome\navidrome.toml"
nssm set Navidrome AppDirectory "D:\Server\Navidrome"
nssm start Navidrome
Open firewall 
netsh advfirewall firewall add rule name="Navidrome" dir=in action=allow protocol=TCP localport=4533
📁 Music Library Organization

Navidrome reads standard music folder structures. Best practice:

D:\Music\
  ├── Artist Name\
  │   ├── Album Name (Year)\
  │   │   ├── 01 - Track Title.mp3
  │   │   ├── 02 - Track Title.flac
  │   │   └── cover.jpg
  │   └── Another Album (Year)\
  └── Another Artist\
      └── ...

Navidrome reads ID3 tags embedded in the files. If tags are missing or wrong, use MusicBrainz Picard (free, FOSS) to bulk-tag your library.

🎵
Supported formats: MP3, FLAC, AAC, OGG, OPUS, WAV, AIFF, WMA, M4A. FLAC is ideal for quality; MP3/AAC for compatibility. Navidrome transcodes on-the-fly for devices that can't handle lossless.
📱 Client Apps

Navidrome uses the Subsonic API, so it works with any Subsonic/Airsonic-compatible app:

FeishinWindows/Mac/Linux desktopBest desktop client, FOSS

SubstreameriOSBest iOS client
Navidrome Web UIAny browserBuilt-in, decent
SonixdDesktop (Electron)Spotify-like UI, FOSS

To connect any of these: set the server URL to http://[server-ip]:4533, username, and password. Done.

👥 Per-User Features

Each Navidrome user gets their own:

  • Play history — every song played is logged, with scrobbling to Last.fm optional
  • Starred songs/albums — per-user favorites
  • Playlists — private and shareable playlists
  • Play counts — "most played" statistics per user

Admin panel lets you create, modify, and delete user accounts. Roles can be set (regular user vs admin vs disabled).

🔐 SSO Integration → SSO TAB

Navidrome doesn't natively support Google OAuth or OIDC (yet). For V1, use Navidrome's own username/password system. For Tier 3 SSO, put Navidrome behind Authentik's forward auth proxy — Authentik handles the login and passes the username header to Navidrome.

🌍 Remote Access
# Caddyfile:
music.yourdomain.com {
    reverse_proxy localhost:4533
}

Or via Tailscale. Once accessible remotely, mobile apps will stream your music over mobile data — even in your car.
