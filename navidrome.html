Navidrome Music Server

// Self-Hosted Spotify · Subsonic API · Port · Windows Native · Caddy Forward Auth //

Navidrome is a FOSS self-hosted music streaming server. Stream your entire library from any device, with per-user play history, playlists, and favorites. Compatible with every Subsonic-compatible client app.

⚙️ Step 1 — Install Navidrome FOSS · Windows Native
🔐 Download Navidrome

From github.com/navidrome/navidrome/releases — download the latest navidrome_X.X.X_windows_amd64.zip. Extract to D:\Server\Navidrome\.

🔐 Create a configuration file

Create D:\Server\Navidrome\navidrome.toml:

MusicFolder       = "D:/Music"
DataFolder        = "D:/Server/Navidrome/data"
Port              = 
LogLevel          = "info"
EnableTranscoding = true
DefaultTheme      = "Dark"
EnableSharing     = true
🔐 Test run Navidrome 
cd D:\Server\Navidrome
navidrome.exe --configfile navidrome.toml

Access at http://localhost:. First login creates the admin account. Confirm the music library scans correctly, then stop the process (Ctrl+C) before installing as a service.

🔐 Install as a Windows Service (NSSM) 
nssm install Navidrome "D:\Server\Navidrome\navidrome.exe" "--configfile D:\Server\Navidrome\navidrome.toml"
nssm set Navidrome AppDirectory "D:\Server\Navidrome"
nssm start Navidrome
🔐 Open the firewall 
netsh advfirewall firewall add rule name="Navidrome" dir=in action=allow protocol=TCP localport=
📁 Step 2 — Organize Your Music Library

Navidrome reads standard music folder structures and ID3 tags embedded in files. Recommended layout:

D:\Music\
  ├── Artist Name\
  │   ├── Album Name (Year)\
  │   │   ├── 01 - Track Title.mp3
  │   │   ├── 02 - Track Title.flac
  │   │   └── cover.jpg
  │   └── Another Album (Year)\
  └── Another Artist\
      └── ...

If tags are missing or wrong, use MusicBrainz Picard (free, FOSS) to bulk-tag your library before the first scan.

🎵
Supported formats: MP3, FLAC, AAC, OGG, OPUS, WAV, AIFF, WMA, M4A. FLAC is ideal for quality; MP3/AAC for compatibility. Navidrome transcodes on-the-fly for devices that can't handle lossless formats.
🔐 Step 3 — SSO via Caddy Forward Auth
How it works

Navidrome has no native OIDC support. Caddy's forward_auth directive protects it transparently — every request to is validated against Authentik before reaching Navidrome. The authenticated username is passed via the X-authentik-username header and Navidrome uses it to auto-create or log in the matching local user.

🔐 Configure Authentik outpost (if not already done)

In → Applications → Create a new Application for Navidrome. Use Provider type: Proxy Provider (Forward Auth — Single Application). Set the external host to .

🔐 Add Navidrome to your Caddyfile

Open and add:

 {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:
}

Reload Caddy: caddy reload --config

🔐 Enable reverse proxy headers in Navidrome

Add to navidrome.toml so Navidrome trusts the username header from Caddy:

ReverseProxyUserHeader = "X-authentik-username"
ReverseProxyWhitelist  = "127.0.0.1/32"

Restart the service: Restart-Service Navidrome

Test SSO end-to-end

Open in an incognito window. You should be redirected to the Authentik login page, and after logging in, be dropped directly into Navidrome as your username. Navidrome will auto-create the account on first login.

⚠️
Subsonic app auth: Mobile and desktop Subsonic clients (DSub, Symfonium, Feishin, etc.) bypass the Caddy SSO layer because they use the /rest/ API endpoint. They authenticate directly with a Navidrome username and password — create per-user Navidrome accounts for each person who needs app access. The web UI at always goes through SSO.
👥 Step 4 — Create User Accounts

Each family member needs a Navidrome account. When they log in via the web UI, Navidrome auto-creates the account using their Authentik username. For app access, you'll need to manually create accounts in the Navidrome admin panel.

# Navidrome admin panel:
http://localhost:/app/#/config

Create an account for each user. They can then use these credentials in any Subsonic-compatible app. Each user gets their own play history, starred songs, playlists, and play counts.

👥 Per-User Features

Each Navidrome user gets their own isolated data — nothing is shared between users unless they explicitly share a playlist:

  • Play history — every track played is logged; optional scrobbling to Last.fm
  • Starred songs / albums / artists — per-user favorites
  • Playlists — private playlists and shareable public links
  • Play counts — "most played" statistics visible per user
  • Ratings — 1–5 star ratings, per user

javascript:(function(){
  const titleEl  = document.querySelector('[class*="song-title"],[class*="songTitle"]');
  const artistEl = document.querySelector('[class*="artist"]');
  if (!titleEl) { alert('No track detected — make sure a song is playing'); return; }
  const query = encodeURIComponent((artistEl?.textContent||'') + ' ' + titleEl.textContent);
  window.open('http://localhost:/?yt_query=' + query, '_blank');
})();

How to update Navidrome

Navidrome is a single binary — updates are manual. No auto-update.

Check current version 
# In the Navidrome web UI:
# Settings (gear icon) → About → version string shown there

# Or check the binary directly:
D:\Server\Navidrome\navidrome.exe --version
Download new release

From github.com/navidrome/navidrome/releases — download the new navidrome_X.X.X_windows_amd64.zip.

Perform the update 
Stop-Service Navidrome

# Replace the binary (keep your .toml and data folder):
Copy-Item navidrome.exe D:\Server\Navidrome\navidrome.exe -Force

Start-Service Navidrome
After confirming the update works — record the version

Open \versions.json → find "navidrome" → fill in version and checkedDate. This gives you a known-good baseline if a future update breaks something.

⚠️
Before updating: Check the Navidrome release notes for database migration warnings. Major version bumps occasionally include schema changes that are one-way — back up D:\Server\Navidrome\data\navidrome.db before proceeding.
⚠️ Potential Breakpoints
🔴 Database corruption after unclean shutdown
If the server loses power while Navidrome is writing to navidrome.db, SQLite can be left in a corrupt state. Service starts but returns errors on all API calls. Fix: stop the service, restore from your last good backup of D:\Server\Navidrome\data\navidrome.db, restart.
🟡 SSO breaks after domain change
If you change your domain, update the Authentik proxy provider's external host URL in , then update the Caddyfile hostname at . The ReverseProxyWhitelist in navidrome.toml only whitelists IPs, not domains — that doesn't need changing.
🟡 Subsonic apps stop working after Caddy config changes
Subsonic API clients hit /rest/ endpoints directly. If forward_auth accidentally blocks these routes, app logins fail. Ensure the Caddy forward_auth block doesn't interfere with /rest/ paths — Navidrome handles its own Subsonic auth for those routes.
🟢 Library not updating after adding new files
Navidrome scans the library on a schedule (default: every minute). If new files aren't appearing, trigger a manual rescan from the admin panel at http://localhost:/app/#/config → Library → Scan.

🎯 How Navidrome Achievements Work

Navidrome has no native webhook system. Achievements are driven by the Last.fm scrobbling integration — Navidrome scrobbles every track played to Last.fm, and the achievement server polls Last.fm's API to check conditions. Alternatively, the achievement server can poll Navidrome's own API directly (no Last.fm required).

💡
Simpler approach: Poll the Navidrome API for play counts and history. The Subsonic API endpoint /rest/getNowPlaying and /rest/getUser return enough data to trigger most achievement conditions without needing Last.fm.

🗃️ What's Stored Where
D:\Server\Navidrome\data\

Everything Navidrome manages itself — no per-user folder in is needed:

  • navidrome.db — SQLite database: all users, play history, playlists, starred items, ratings, play counts
  • cache/ — transcoded audio chunks and album art cache (safe to delete; will be rebuilt)
  • navidrome.log — application log
💡
All per-user data is in navidrome.db. No per-user folder in is needed for this service. Back up just the database file to preserve all users' history, playlists, and stars.
D:\Music\ (your library)

Navidrome reads but never writes to your music library. Files stay exactly as you placed them. Navidrome only reads ID3 tags and file metadata — it does not rename, move, or modify your audio files.

🛠️ Common Issues
❌ Service won't start / crashes immediately
Usually a config parse error in navidrome.toml or the data folder not existing yet.
  • Run the binary manually to see the error: D:\Server\Navidrome\navidrome.exe --configfile D:\Server\Navidrome\navidrome.toml
  • Confirm D:\Server\Navidrome\data\ exists and is writable
  • Confirm D:\Music\ exists (Navidrome will error if the MusicFolder is missing)
❌ All pages return errors after a crash / power loss
SQLite database may be corrupted. Stop the service, replace navidrome.db with your last good backup, then restart. To check without restoring: sqlite3 D:\Server\Navidrome\data\navidrome.db "PRAGMA integrity_check;" — any result other than ok means corruption.
⚠️ SSO login redirects to Authentik but returns an error
Almost always a provider misconfiguration in Authentik.
  • In → check the Navidrome proxy provider's external host is exactly
  • Ensure the Authentik outpost is deployed and healthy
  • Reload Caddy after any Caddyfile change: caddy reload --config
⚠️ Subsonic app can't connect (DSub / Feishin / Symfonium)
Subsonic clients authenticate directly via the /rest/ API — they don't use SSO.
  • Server URL should be http://[server-LAN-ip]: for local access, or for external
  • Username and password must be the Navidrome account credentials, not the Google/Authentik credentials
  • Create the user account first in the Navidrome admin panel if it doesn't exist yet
⚠️ New music files not showing up
Navidrome scans on a schedule (default every 60 seconds). If files still don't appear after a few minutes, trigger a manual scan: admin panel → http://localhost:/app/#/config → Library → Scan Now. If they still don't appear, check that the files have valid ID3 tags — Navidrome will skip files it can't parse.
💡 Album art missing for some albums
Navidrome looks for art in this order: embedded album art in the file tags → cover.jpg / folder.jpg in the album directory → MusicBrainz fetch. For best results, embed cover art in the files using MusicBrainz Picard, or place a cover.jpg in each album folder. Delete the cache at D:\Server\Navidrome\data\cache\ and restart to force a refresh.
