Navidrome Music Server

// Self-Hosted Spotify · Subsonic API · Per-User Playlists //

Navidrome is the best FOSS self-hosted music server. Stream your entire music library from any device, with per-user play history, playlists, and favorites. Compatible with dozens of client apps.

🐉 V1 Goal: Navidrome serving your music library from an internal drive, accessible via browser at port 4533, and via mobile apps like DSub or Symfonium.
⚙️ Installation on Windows FOSS · Windows Native
Download Navidrome

From github.com/navidrome/navidrome/releases — download the latest navidrome_X.X.X_windows_amd64.zip. Extract to D:\Server\Navidrome\.

Create a configuration file

Create D:\Server\Navidrome\navidrome.toml:

MusicFolder    = "D:/Music"
DataFolder     = "D:/Server/Navidrome/data"
Port           = 4533
LogLevel       = "info"
EnableTranscoding = true
DefaultTheme   = "Dark"
EnableSharing  = true  # allows public share links for playlists
Run Navidrome 
cd D:\Server\Navidrome
navidrome.exe --configfile navidrome.toml

Access at http://localhost:4533. First login creates the admin account.

Install as a Windows Service (NSSM) 
nssm install Navidrome "D:\Server\Navidrome\navidrome.exe" "--configfile D:\Server\Navidrome\navidrome.toml"
nssm set Navidrome AppDirectory "D:\Server\Navidrome"
nssm start Navidrome
Open firewall 
netsh advfirewall firewall add rule name="Navidrome" dir=in action=allow protocol=TCP localport=4533
📁 Music Library Organization

Navidrome reads standard music folder structures. Best practice:

D:\Music\
  ├── Artist Name\
  │   ├── Album Name (Year)\
  │   │   ├── 01 - Track Title.mp3
  │   │   ├── 02 - Track Title.flac
  │   │   └── cover.jpg
  │   └── Another Album (Year)\
  └── Another Artist\
      └── ...

Navidrome reads ID3 tags embedded in the files. If tags are missing or wrong, use MusicBrainz Picard (free, FOSS) to bulk-tag your library.

🎵
Supported formats: MP3, FLAC, AAC, OGG, OPUS, WAV, AIFF, WMA, M4A. FLAC is ideal for quality; MP3/AAC for compatibility. Navidrome transcodes on-the-fly for devices that can't handle lossless.

👥 Per-User Features

Each Navidrome user gets their own:

  • Play history — every song played is logged, with scrobbling to Last.fm optional
  • Starred songs/albums — per-user favorites
  • Playlists — private and shareable playlists
  • Play counts — "most played" statistics per user

Admin panel lets you create, modify, and delete user accounts. Roles can be set (regular user vs admin vs disabled).

🔐 Authentik SSO via Caddy Forward Auth
Navidrome + Authentik (via Caddy, no native OIDC needed)

Navidrome doesn't support OIDC natively, but Caddy's forward_auth directive protects it transparently. When a user visits music.yourdomain.com, Caddy first validates their Authentik session. If valid, it passes their username as the X-authentik-username header to Navidrome.

# Caddyfile entry for Navidrome:
music.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:4533
}

# After this, unauthenticated visitors are redirected to auth.yourdomain.com
# Authenticated visitors reach Navidrome directly with their username in a header.
Auto-create Navidrome users from Authentik header

Navidrome can auto-create users from the header with this config in navidrome.toml:

# navidrome.toml additions for Authentik forward auth:
ReverseProxyUserHeader = "X-authentik-username"
ReverseProxyWhitelist  = "127.0.0.1/8"

# This means: when Navidrome receives a request with X-authentik-username header
# from localhost (Caddy), it automatically creates or logs in that user.
# No separate Navidrome password needed — Authentik handles all auth.
🎶
Result: Users visit music.yourdomain.com, are redirected to sign in with Google (if not already signed in to Authentik), then land directly in Navidrome with their own playlists, history, and favorites — no second login.
🎵 "Open in Jukebox" Button Integration
Add an "Open in Jukebox" button to Navidrome via Caddy JS injection

Caddy can inject a small JavaScript snippet into every Navidrome page response. This snippet detects when a track is playing and adds a button that sends it to your Infinite Jukebox.

# In Caddyfile, add response modification:
music.yourdomain.com {
    forward_auth localhost:9000 { ... }  # as above

    reverse_proxy localhost:4533 {
        # Inject JS snippet into HTML responses:
        @html header Content-Type text/html*
        header_up Accept-Encoding ""  # disable compression so we can modify HTML
    }

    # Inject the button script before :
    @html_resp header Content-Type *text/html*
    route @html_resp {
        reverse_proxy localhost:4533 {
            header_up Accept-Encoding ""
        }
    }
}

Since Caddy's response modification is limited, the simpler approach is a browser extension or a bookmarklet that users install once:

// Navidrome → Jukebox bookmarklet
// Drag this to your bookmark bar:
javascript:(function(){
  const titleEl = document.querySelector('[class*="song-title"],[class*="songTitle"]');
  const artistEl = document.querySelector('[class*="artist"]');
  if (!titleEl) { alert('No track detected — make sure a song is playing'); return; }
  const query = encodeURIComponent((artistEl?.textContent||'') + ' ' + titleEl.textContent);
  const jukeboxUrl = 'http://localhost:5050/?yt_query=' + query;
  window.open(jukeboxUrl, '_blank');
})();

Alternatively, add a button via a userscript (Tampermonkey/Greasemonkey) that all your users install:

// ==UserScript==
// @name         Navidrome → Jukebox
// @match        https://music.yourdomain.com/*
// ==/UserScript==
(function() {
  'use strict';
  // Wait for player to appear
  const observer = new MutationObserver(() => {
    const player = document.querySelector('.player-bar, [class*="playerBar"]');
    if (player && !document.getElementById('jukebox-btn')) {
      const btn = document.createElement('button');
      btn.id = 'jukebox-btn';
      btn.textContent = '🎵 Jukebox';
      btn.title = 'Open current track in Infinite Jukebox';
      btn.style.cssText = 'margin-left:8px;background:rgba(200,168,75,.2);border:1px solid rgba(200,168,75,.4);color:#c8a84b;padding:4px 10px;border-radius:3px;cursor:pointer;font-size:.8rem';
      btn.onclick = () => {
        const title = document.querySelector('[class*="songTitle"],[class*="song-title"]')?.textContent || '';
        const artist = document.querySelector('[class*="artistName"],[class*="artist-name"]')?.textContent || '';
        window.open(`http://localhost:5050/?query=${encodeURIComponent(artist + ' ' + title)}`, '_blank');
      };
      player.appendChild(btn);
    }
  });
  observer.observe(document.body, { childList: true, subtree: true });
})();
💡
On the Jukebox side, add a ?query URL parameter handler that auto-submits the YouTube search query when the page loads — so clicking the button in Navidrome instantly starts loading that song in the Jukebox.
