Stirling-PDF — PDF Toolkit

// Docker · WSL2 · Port 8090 · Built-in Auth · Authentik OIDC //

Stirling-PDF is a self-hosted web app with 50+ PDF operations: merge, split, compress, convert, OCR, rotate, watermark, sign, redact, and more. No data is sent to external services — all processing happens on your server.

🐉 Stirling-PDF has its own built-in login system. Use it in front of Authentik OIDC so users sign in with their Google account like everything else. The built-in admin account is only for initial setup.
📁 User Data Storage → SSO TAB

Stirling-PDF processes files on-the-fly — no input files are stored permanently. Persistent data is configuration and OCR training data only:

D:\Users\common\stirling\
  trainingData\    ← Tesseract OCR language data (tessdata)
  extraConfigs\    ← app settings, user DB
  logs\            ← troubleshooting logs

This is shared infrastructure data, not per-user content. No per-user folder is needed. See the SSO tab → User Data Storage for the full architecture.

💡
Files uploaded for processing are held in memory/temp only. They are never written to disk permanently. Stirling-PDF is safe to use for sensitive documents.
🐳 Deployment — Docker in WSL2
Create data directories (Windows) 
New-Item -ItemType Directory -Force "D:\Users\common\stirling\trainingData"
New-Item -ItemType Directory -Force "D:\Users\common\stirling\extraConfigs"
New-Item -ItemType Directory -Force "D:\Users\common\stirling\logs"
Create the compose file 
mkdir -p ~/services/stirling-pdf && cd ~/services/stirling-pdf

cat > docker-compose.yml << 'EOF'
services:
  stirling-pdf:
    image: frooodle/s-pdf:latest
    container_name: stirling-pdf
    restart: always
    ports:
      - "8090:8080"
    volumes:
      - /mnt/d/Users/common/stirling/trainingData:/usr/share/tessdata
      - /mnt/d/Users/common/stirling/extraConfigs:/configs
      - /mnt/d/Users/common/stirling/logs:/logs
    environment:
      # Security
      - DOCKER_ENABLE_SECURITY=true
      - SECURITY_ENABLELOGIN=true
      - SECURITY_INITIALLOGIN_USERNAME=admin
      - SECURITY_INITIALLOGIN_PASSWORD=changeme   # CHANGE ON FIRST LOGIN

      # Features
      - LANGS=en_US,en_GB
      - SYSTEM_DEFAULTLOCALE=en-US
      - INSTALL_BOOK_AND_ADVANCED_HTML_OPS=true

      # Performance
      - JAVA_TOOL_OPTIONS=-Xms512m -Xmx2g
EOF
Start the container 
docker compose up -d
docker compose logs -f stirling-pdf   # wait for "Started" message (30–60 sec)
First login — change admin password immediately

Open http://localhost:8090. Log in as admin / changeme. Go to Settings → Account → Change Password. Store the new password in Vaultwarden.

Open Windows Firewall for port 8090 
netsh advfirewall firewall add rule name="Stirling-PDF" dir=in action=allow protocol=TCP localport=8090
Auto-start on Windows boot

Task Scheduler → Create Task → Action:

wsl -d Ubuntu -- bash -c "cd ~/services/stirling-pdf && docker compose up -d"

Trigger: At startup, delay 2 minutes. Run whether logged on or not: ☑

🔐 Authentik — OIDC Login
Configure Stirling-PDF to use Authentik OIDC

Stirling-PDF supports OpenID Connect natively. This replaces its internal login with Authentik SSO so users log in with Google like everything else.

# In Stirling-PDF Settings → Single Sign-On:
# (Or add to docker-compose environment:)
- SECURITY_OAUTH2_ENABLED=true
- SECURITY_OAUTH2_ISSUER=https://auth.yourdomain.com/application/o/stirling-pdf/
- SECURITY_OAUTH2_CLIENTID=[client-id-from-authentik]
- SECURITY_OAUTH2_CLIENTSECRET=[client-secret-from-authentik]
- SECURITY_OAUTH2_SCOPES=openid,profile,email
- SECURITY_OAUTH2_USERASROLE=true    # map Authentik groups to Stirling roles
Authentik Application entry
# Authentik admin → Applications → Create:
Name:           Stirling-PDF
Slug:           stirling-pdf
Launch URL:     https://pdf.yourdomain.com
Provider:       OAuth2/OIDC Provider
  Redirect URI: https://pdf.yourdomain.com/login/oauth2/code/oidc
  Scopes:       openid, profile, email
  Signing Key:  [your Authentik signing key]
Access:         authentik-family + authentik-admins groups
Caddy configuration
# Add to your Caddyfile:
# Note: forward_auth is NOT needed here — Stirling handles login via OIDC itself.
# Caddy just terminates TLS and proxies.
pdf.yourdomain.com {
    reverse_proxy localhost:8090 {
        header_up X-Forwarded-Proto https
    }
}

# Important: set client_max_body_size equivalent in Caddy:
pdf.yourdomain.com {
    request_body {
        max_size 100MB    # essential for large PDFs
    }
    reverse_proxy localhost:8090
}
🔠 OCR Language Support
Adding additional OCR languages (Tesseract)

English OCR data is included. To add other languages, download .traineddata files and place them in the tessdata folder:

# In WSL2 terminal:
# Download from: https://github.com/tesseract-ocr/tessdata
wget https://github.com/tesseract-ocr/tessdata/raw/main/fra.traineddata \
  -O /mnt/d/Users/common/stirling/trainingData/fra.traineddata

# No container restart needed — Tesseract reads the folder at OCR time.
FileLanguage
eng.traineddataEnglish (included)
fra.traineddataFrench
deu.traineddataGerman
spa.traineddataSpanish
jpn.traineddataJapanese
🔧 Updating
cd ~/services/stirling-pdf
docker compose pull
docker compose up -d

Check the Stirling-PDF release notes before updating — environment variable names occasionally change between major versions. The extraConfigs volume preserves your settings across updates.
