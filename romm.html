RomM — ROM Library Manager

// Browse & Download ROMs · Web Interface · Port 8098 //

RomM (ROM Manager) is a self-hosted web application that organizes your ROM collection with cover art, metadata, and a beautiful browser interface. Users can browse your collection and download ROMs to play locally.

🐉 V1 Goal: RomM running on port 8098 via WSL2/Docker, serving your ROM library at romm.yourdomain.com with per-user access and download capability.
🐉
Legal note: Distributing ROMs is legally grey even if you own the originals. RomM is designed for personal use — serving your own collection to yourself across devices, or to a closed group of trusted people. Don't make it publicly accessible on the internet.
🏗️ Architecture

RomM is Docker-only officially, but runs cleanly in WSL2. It provides:

  • Web gallery with cover art fetched from IGDB/MobyGames/Screenscraper
  • Filter by platform (GBA, N64, PS1, etc.), genre, year
  • Download individual ROMs from the browser
  • Multi-user accounts with per-user download history
  • REST API (for integration with EmulatorJS later)
⚙️ Installation via WSL2 WSL2 + Docker
Enable WSL2 and install Ubuntu (if not already done) 
wsl --install
# Reboot, then open Ubuntu from Start Menu
Install Docker inside WSL2 
sudo apt update && sudo apt install -y docker.io docker-compose
sudo usermod -aG docker $USER
newgrp docker
Create RomM config directory 
mkdir -p ~/romm/{config,resources,redis-data}
cd ~/romm
Create docker-compose.yml 
cat > docker-compose.yml << 'EOF'
version: "3"
services:
  romm:
    image: rommapp/romm:latest
    container_name: romm
    environment:
      - DB_PASSWD=strongpassword
      - ROMM_AUTH_SECRET_KEY=change-this-to-a-long-random-string
      - IGDB_CLIENT_ID=your-igdb-client-id
      - IGDB_CLIENT_SECRET=your-igdb-client-secret
    volumes:
      # Point at your Windows ROM folder via WSL2 path:
      - /mnt/d/Emulation:/romm/library
      - ~/romm/resources:/romm/resources
      - ~/romm/config:/romm/config
    ports:
      - "8098:8080"
    depends_on:
      - romm-db
    restart: unless-stopped

  romm-db:
    image: mariadb:latest
    environment:
      - MYSQL_ROOT_PASSWORD=strongrootpassword
      - MYSQL_DATABASE=romm
      - MYSQL_USER=romm
      - MYSQL_PASSWORD=strongpassword
    volumes:
      - ~/romm/mysql-data:/var/lib/mysql
    restart: unless-stopped
EOF
💡
Windows path in WSL2: Your D:\Emulation folder is accessible at /mnt/d/Emulation inside WSL2. RomM reads directly from this path — no copying needed.
Get IGDB API credentials (free, for cover art)

Go to dev.twitch.tv/console → create a free app → get Client ID and Client Secret. These let RomM fetch cover art and metadata from IGDB.

Start RomM 
docker-compose up -d
# Access at http://localhost:8098
# First-time setup: create admin account
Auto-start on Windows boot via Task Scheduler 
# Create startup task:
# Action: wsl -d Ubuntu -u [your-wsl-username] -e bash -c "cd ~/romm && docker-compose up -d"
# Trigger: At startup, delay 2 minutes

# Or create a startup bat file:
# wsl -d Ubuntu -- bash -c "cd ~/romm && docker-compose up -d >> ~/romm/startup.log 2>&1"
Open firewall for port 8098 
netsh advfirewall firewall add rule name="RomM" dir=in action=allow protocol=TCP localport=8098
📁 ROM Library Structure

RomM expects a specific folder structure — platform folders named by their platform ID:

D:\Emulation\
  ├── gba\          ← Game Boy Advance
  │   ├── Pokemon Emerald.gba
  │   └── Fire Emblem.gba
  ├── n64\          ← Nintendo 64
  │   └── Mario Party 2.z64
  ├── nds\          ← Nintendo DS
  ├── gc\           ← GameCube
  ├── wii\          ← Wii
  ├── xbox360\      ← Xbox 360
  ├── psx\          ← PlayStation 1
  └── snes\         ← Super Nintendo

After starting RomM and pointing it at this folder, go to the admin panel → Scan Library. It will automatically fetch cover art, descriptions, genres, and release years for every ROM.

👥 Users & Downloads

RomM has built-in user management. Admin panel → Users → create accounts for your users. Each user can:

  • Browse the full ROM library with cover art
  • Download ROMs to their device (plays in their local emulator or EmulatorJS)
  • See their download history

You can restrict download access per user, or set the server to require login before browsing. Keep this behind Tailscale or Caddy with auth — don't expose publicly.

🔗 Integration with EmulatorJS

RomM has a REST API. The EmulatorJS game picker can optionally call RomM's API to list available ROMs instead of hardcoding them. This means you manage your ROM library in one place (RomM) and the web emulator automatically stays up to date.

# RomM API example — list all GBA ROMs:
GET http://localhost:8098/api/roms?platform=gba

# Response includes download URLs, metadata, cover art URLs, etc.
