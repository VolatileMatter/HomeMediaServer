Tunarr

// Fake Cable TV · Port 8200 · Docker WSL2 · No Auth Required //

Tunarr creates fake cable TV channels from your Jellyfin library. Each channel runs a 24-hour schedule of TV episodes, movies, and filler content — exposed as M3U + XMLTV so Jellyfin Live TV sees it as a real cable provider. Nineteen channels, each with its own genre identity, recurring events, and rotating film library.

📺 Tunarr is running on port 8200. Jellyfin Live TV is pointed at the M3U and XMLTV endpoints below. 19 channels defined. Filler sync is on a weekly Sunday 3am Task Scheduler job.

🐳 Docker Compose
services:
  tunarr:
    image: ghcr.io/chrisbenincasa/tunarr:latest
    container_name: tunarr
    restart: unless-stopped
    ports:
      - "8200:8000"
    environment:
      - TZ=Australia/Sydney
    volumes:
      - ./data:/home/node/app/.tunarr
      - /mnt/d/Media:/media:ro   # read-only mount of media drive
    networks:
      - proxy

networks:
  proxy:
    external: true
📺 Jellyfin Live TV Integration
1
Open Jellyfin Admin → Dashboard → Live TV
2
Add Tuner Device → Type: HD Homerun → URL: http://localhost:8200
3
Add TV Guide Data → Type: XMLTV → Path: http://localhost:8200/api/xmltv.xml
4
Refresh Guide → Dashboard → Live TV → Refresh Guide. All 19 channels appear.
5
Map channels → Guide Data Mapping → assign each Tunarr channel to its guide entry.

🕐 How Tunarr Schedules Content
Time Slot Types
TypeBehaviour
TV EpisodeSequential — plays next unwatched episode, wraps
Movie Slot3-hour block — one film from pool, rotating
Filler PadFills time between a short slot and next fixed slot
RandomPicks randomly from a Jellyfin library or collection
FlexEmpty — Tunarr fills with channel filler automatically
Pad Mode (Recommended Setting)

Set each slot to Pad mode. When a TV episode ends early (e.g. a 22-minute episode in a 30-minute slot), Tunarr automatically fills the gap with a filler clip before the next slot starts.

This prevents the schedule from drifting and keeps content feeling like real TV with ad breaks replaced by ambient clips.

📅 Slot Timing Conventions Used in This Setup
Slot lengthContent typeNotes
3 hoursFeature filmMost films are under 2.5hrs; pad fills remainder
30–60 minTV episode (standard)Sitcoms 22–30min, dramas 42–60min
90 minTV episode (prestige)GoT, Westworld, Sherlock
5–10 minFiller clipMax duration set per filler playlist
12am–5amOvernight blockLow-rated films (under 70% audience score) + filler
⚙️ Timezone Note
⚠️
TZ=Australia/Sydney is set in the Docker container. All schedules are authored in Sydney time. If you change the TZ variable, all time slots shift. Do not change it without re-authoring every channel schedule.
🎞 Filler System Overview

Filler clips are short videos (under 10 minutes) that Tunarr automatically inserts to pad gaps between scheduled content. Each channel has its own curated filler playlist stored in D:\Media\Filler\[channel]\.

FileLocationPurpose
filler_sync.pyD:\Scripts\Downloads YouTube filler via yt-dlp and splits clips
filler_config.jsonD:\Scripts\Config for each filler playlist — URL, split mode, clip length
Filler clipsD:\Media\Filler\[channel]\Actual MP4 files served to Tunarr
🔪 Split Modes
scene — PySceneDetect

For compilation videos. Detects scene transitions and splits at each cut. Use for AMV compilations, toy commercial compilations, trailer reels. Discards transitions shorter than 4 seconds (scene_min_seconds: 4).

chunk — Fixed ffmpeg segments

For ambient/continuous footage. Splits into fixed-length chunks. Use for NASA footage, castle drone footage, rain ambience, lo-fi music streams. Set chunk_seconds to desired clip length.

none — Download as-is

No splitting. Use for shorts, stand-up clips, and content already the right length (<10 minutes).

⚙️ filler_config.json structure
{
  "playlists": [
    {
      "name":              "comedy-standups",
      "folder":            "D:\\Media\\Filler\\comedy",
      "url":               "https://www.youtube.com/playlist?list=...",
      "max_duration":      300,       // max clip seconds kept
      "split_mode":        "none",    // none | chunk | scene
      "chunk_seconds":     60,        // used when split_mode=chunk
      "scene_min_seconds": 4,         // used when split_mode=scene
      "scene_max_seconds": 120,       // discard scenes over this length
      "scene_threshold":   30.0       // PySceneDetect sensitivity
    },
    {
      "name":              "nasa-footage",
      "folder":            "D:\\Media\\Filler\\scifi",
      "url":               "https://www.youtube.com/playlist?list=...",
      "split_mode":        "chunk",
      "chunk_seconds":     90,
      "max_duration":      120
    }
  ]
}
🕒 Automatic Sync Schedule

Task Scheduler job runs python D:\Scripts\filler_sync.py every Sunday at 3am. After downloads complete, the script hits the Jellyfin API to trigger a library rescan so new filler clips appear immediately.

# Manual run:
python D:\Scripts\filler_sync.py

# Force re-download all (ignore cache):
python D:\Scripts\filler_sync.py --force
🔄 Updating Tunarr
# Pull latest image and restart:
cd ~/services/tunarr
docker compose pull
docker compose down
docker compose up -d
docker compose logs -f --tail=50

⚠️ After updating, verify channel schedules are intact. Tunarr updates have occasionally reset filler assignments.

💾 Backup

The entire Tunarr configuration lives in ~/services/tunarr/data/ — this is what needs to be backed up.

# Backup Tunarr config:
cp -r ~/services/tunarr/data ~/backups/tunarr-$(date +%Y%m%d)

# Restore from backup:
docker compose down
cp -r ~/backups/tunarr-YYYYMMDD ~/services/tunarr/data
docker compose up -d
