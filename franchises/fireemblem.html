⚙️ Setup for In-Browser Games
Same architecture as Pokémon and Castlevania libraries

The Flask save server, EmulatorJS frontend, and per-user save architecture is identical. Extend the same server.py:

FIRE_EMBLEM_ROMS = 'D:/Emulation/FireEmblem'
# Saves go directly to D:/Users/[username]/emulation/fireemblem/
# username = request.headers.get('X-authentik-username', 'anonymous')
# save_dir = f'D:/Users/{username}/emulation/fireemblem'
# See SSO tab → User Data Storage for full architecture

@app.route('/fe/roms/<path:filename>')
def serve_fe_rom(filename):
    if 'user' not in session: return redirect('/login')
    return send_from_directory(FIRE_EMBLEM_ROMS, filename)

# Save IDs: fe-blazing, fe-sacred, fe-shadow-dragon, etc.

🎮 GameCube / Wii Games via Dolphin + Sunshine
Path of Radiance & Radiant Dawn

These run in Dolphin (GameCube/Wii emulator). Use the Sunshine streaming setup from the GameCube tab. Add each game as a Sunshine application with a pre-launch command that sets the user's symlinks:

# In Sunshine → Applications → Add:
Name:    Fire Emblem: Path of Radiance
Cmd:     "D:\Emulators\Dolphin\Dolphin.exe" -e "D:\Emulation\FireEmblem\gc\path-of-radiance.iso"
# Pre-launch (sets symlinks for this user before Dolphin opens):
Detached: curl -s -X POST http://localhost:9004/session/start -H "Content-Type: application/json" -d "{\"username\":\"%%SUNSHINE_CLIENT_NAME%%\"}"

Name:    Fire Emblem: Radiant Dawn
Cmd:     "D:\Emulators\Dolphin\Dolphin.exe" -e "D:\Emulation\FireEmblem\wii\radiant-dawn.iso"

Save files for Dolphin are managed by the Session Manager (port 9004). When a Sunshine session starts, it calls the session manager which re-points the Dolphin save folder symlink at D:\Users\[username]\emulation\dolphin\. Each user's Fire Emblem memory card and save data is completely isolated. See the SSO tab → User Data Storage for the full symlink architecture.

💾 User Data Storage → SSO TAB

GBA/NDS games (EmulatorJS): Saves go directly to D:\Users\[username]\emulation\fireemblem\ via the Authentik username header. Each user's progress is isolated.

GCN/Wii games (Dolphin via Sunshine): The Session Manager (port 9004) re-points Dolphin's save folder symlink to D:\Users\[username]\emulation\dolphin\ when a Sunshine session starts. The Sunshine pre-launch command triggers this automatically.

# Sunshine pre-launch (set before Dolphin opens):
curl -s -X POST http://localhost:9004/session/start ^
  -H "Content-Type: application/json" ^
  -d "{\"username\":\"%%SUNSHINE_CLIENT_NAME%%\"}"

All save data lives in D:\Users\[username]\ and counts toward Nextcloud quota. See SSO tab → User Data Storage.
