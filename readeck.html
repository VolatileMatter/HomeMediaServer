Readeck — Bookmark & Read-Later

// Docker · WSL2 · Per-User Instances · Port Range 8087–8096 · Authentik forward_auth //

Readeck is a self-hosted read-later and bookmark service. It fetches and stores a clean, readable copy of any article you save — no paywalls, no tracking, no ads. Each user gets their own isolated container and data folder.

🐉 Each user gets their own Readeck container on a dedicated port. This gives full data isolation — one user's bookmarks are completely invisible to all others. The trade-off is one container per user rather than a single multi-user app.
📁 User Data Storage → SSO TAB
D:\Users\[username]\readeck\    ← all bookmarks, article snapshots, metadata
  articles\                     ← fetched article content
  db.sqlite3                    ← bookmark database

Each user's Readeck data lives in their own folder and counts toward their Nextcloud quota. See the SSO tab → User Data Storage for the full architecture.

⚠️
WSL2 path mapping: The Docker volume mount uses the WSL2 path. Map D:\Users\alice\readeck as /mnt/d/Users/alice/readeck in the compose file. See the pattern below.
🐳 Deployment — Per-User Docker Containers
Port Assignment — one port per user
UserPortSubdomain
alice8087read-alice.yourdomain.com
bob8088read-bob.yourdomain.com
Add more users in sequence8089, 8090…read-[name].yourdomain.com
💡
Alternatively, run a single shared Readeck instance on port 8087 if your family is comfortable sharing bookmarks. Readeck has its own internal user accounts — Authentik then acts as a gate in front of the whole instance.
Create user data directories (Windows) 
# Run once per user:
New-Item -ItemType Directory -Force "D:\Users\alice\readeck"
New-Item -ItemType Directory -Force "D:\Users\bob\readeck"
Create the compose file 
mkdir -p ~/services/readeck && cd ~/services/readeck

cat > docker-compose.yml << 'EOF'
services:
  readeck-alice:
    image: codeberg.org/readeck/readeck:latest
    container_name: readeck-alice
    environment:
      - READECK_ALLOWED_HOSTS=read-alice.yourdomain.com
      - READECK_LOG_LEVEL=info
    volumes:
      - /mnt/d/Users/alice/readeck:/data
    ports:
      - "8087:8000"
    restart: unless-stopped

  readeck-bob:
    image: codeberg.org/readeck/readeck:latest
    container_name: readeck-bob
    environment:
      - READECK_ALLOWED_HOSTS=read-bob.yourdomain.com
      - READECK_LOG_LEVEL=info
    volumes:
      - /mnt/d/Users/bob/readeck:/data
    ports:
      - "8088:8000"
    restart: unless-stopped
EOF
Start containers 
docker compose up -d
docker compose logs   # confirm both started
Open Windows Firewall ports 
netsh advfirewall firewall add rule name="Readeck-Alice" dir=in action=allow protocol=TCP localport=8087
netsh advfirewall firewall add rule name="Readeck-Bob"  dir=in action=allow protocol=TCP localport=8088
Auto-start on Windows boot

Task Scheduler → Create Task → Action:

wsl -d Ubuntu -- bash -c "cd ~/services/readeck && docker compose up -d"

Trigger: At startup, delay 2 minutes. Run whether logged on or not: ☑

🔐 Authentik — Per-User forward_auth
Caddy configuration — one block per user
# Add to your Caddyfile:
read-alice.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    # Only allow alice through — block all other authenticated users
    @notAlice not header X-authentik-username alice
    respond @notAlice "Forbidden" 403
    reverse_proxy localhost:8087
}

read-bob.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    @notBob not header X-authentik-username bob
    respond @notBob "Forbidden" 403
    reverse_proxy localhost:8088
}

This ensures even a logged-in family member cannot reach another user's bookmarks.

Authentik Application entries

📱 Mobile — Share Sheet Integration
Saving articles from a phone

Readeck exposes a simple bookmark API. You can save any URL from a mobile share sheet using a Shortcut (iOS) or intent (Android).

# iOS Shortcut — "Save to Readeck" action:
# URL: POST https://read-alice.yourdomain.com/api/bookmarks
# Body (JSON): {"url": "[Shortcut Input]"}
# Headers:
#   Content-Type: application/json
#   Authorization: Bearer [YOUR_API_TOKEN]

# Get your API token from Readeck → Settings → API Tokens
💡
Generate the API token from within your Readeck instance after first login. Go to Settings → API Tokens → Create token. Each user generates their own token from their own instance.
