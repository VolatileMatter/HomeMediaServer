How blocks are enforced

The control server writes to Windows Firewall via netsh and appends to a Caddy blocklist file. Add this to your Caddyfile:

@blocked remote_ip 203.0.113.45 198.51.100.22
respond @blocked "Access Denied" 403

For dynamic updates without restarting Caddy, use the Caddy admin API:

curl -X POST http://localhost:2019/config/apps/http/servers/srv0/routes \
  -H "Content-Type: application/json" \
  -d '{"match":[{"remote_ip":{"ranges":["1.2.3.4"]}}],"handle":[{"handler":"static_response","status_code":403}]}'
🏆 Achievements
🎮 RetroAchievements (retro games)

EmulatorJS supports RetroAchievements natively. Each user enters their own RA username and API token in the game UI. Achievements are tracked on retroachievements.org per user account.

// In your game launcher page:
EJS_retroArchCheevos = true;
EJS_cheevosUsername = userRAName;  // from user profile
EJS_cheevosToken = userRAToken;    // from RA settings page

Users create free accounts at retroachievements.org and enter their credentials once.

📚 Reading & Listening Achievements

Audiobookshelf → Settings → Notifications → add webhook on "Audiobook Finished" → POST to http://localhost:9003/api/achievement/book.

Calibre-Web → when user marks a book "read," fires to the same endpoint.

@app.route('/api/achievement/book', methods=['POST'])
def book_ach():
    d = request.json
    grant(d['user'], 'BOOK_FINISH', d.get('title',''))
    return jsonify({'ok':True})

⚙️ SSO Tier 2 — Google OAuth

Get Client ID + Secret from console.cloud.google.com. Configure per app:

  • Calibre-Web: Admin → config → ☑ Enable Google OAuth
  • Audiobookshelf: Settings → Auth → OpenID Connect (Issuer: accounts.google.com)
  • Nextcloud: Apps → Social Login → Custom OIDC
  • Emulation server: Add authlib Google flow
⚙️ SSO Tier 3 — Authentik (One Page for All)
cd ~/authentik  # (in WSL2)
wget https://goauthentik.io/docker-compose.yml
docker-compose up -d
# Admin: http://localhost:9000/if/admin/

All apps redirect to Authentik's login page. Configure Google as a social login source in Authentik. All users log in with Google once and access everything.
