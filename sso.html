Single Sign-On (SSO)

// Google OAuth · UN/PW · Authentik Self-Hosted SSO //

Three tiers: per-app usernames/passwords (simplest), Google OAuth per-app (easy, uses existing Gmail), or a self-hosted SSO like Authentik for one login to rule them all.

🐉 V1 Strategy: Start with per-app username/password (already built into each service). Then upgrade to Google OAuth per service. Then optionally centralize with Authentik.

2️⃣ Tier 2 — Google OAuth Per App ("Sign in with Google")
Step 0: Set Up Google OAuth Credentials (One-Time)

All apps share the same Google API credential setup process.

Go to Google Cloud Console

console.cloud.google.com → Create a new project (name it "Home Server SSO" or similar).

Configure OAuth Consent Screen

APIs & Services → OAuth consent screen → External → Fill in app name ("Home Server"), your email, no logo needed. Add your Gmail address under "Test users" (required for external apps).

Create OAuth Client ID

APIs & Services → Credentials → Create Credentials → OAuth Client ID → Web application. You'll add redirect URIs per app (see below). Save your Client ID and Client Secret.

📚 Calibre-Web + Google OAuth
Add redirect URI in Google Console 
http://[your-server-IP]:8083/google/authorized
In Calibre-Web Admin → Edit app configuration 
☑ Enable Google OAuth
Google Client ID:     [your-client-id]
Google Client Secret: [your-client-secret]
Link existing users to Google accounts

Admin → Users → Edit user → "Link Google account" — they click this while signed into Google to link their account. After that they can use "Sign in with Google."

🎧 Audiobookshelf + OpenID Connect (Google)
Add redirect URI in Google Console 
http://[your-server-IP]:13378/auth/openid/callback
In ABS Admin → Settings → Authentication 
☑ OpenID Connect Authentication
Issuer URL:     https://accounts.google.com
Client ID:      [your-client-id]
Client Secret:  [your-client-secret]
Auto Launch:    off (shows button on login page)
☁️ Nextcloud + Google Login App
Install the "Social Login" app

Admin → Apps → search "Social Login" → Install.

Add redirect URI in Google Console 
http://[your-server-IP]/nextcloud/index.php/apps/sociallogin/custom_oidc/google
Configure Social Login in Nextcloud

Admin → Social Login → Add "Custom OpenID Connect" provider:

Name:          google
Title:         Sign in with Google
Authorize URL: https://accounts.google.com/o/oauth2/auth
Token URL:     https://oauth2.googleapis.com/token
Display Name:  name
Username:      email
Client ID:     [your-client-id]
Client Secret: [your-client-secret]
Scope:         openid email profile
3️⃣ Tier 3 — Authentik Self-Hosted SSO (Advanced)
⚡
Note: Authentik officially only supports Linux/Docker. On Windows 10 natively, you'd run it under WSL2 (Windows Subsystem for Linux). If you've had Docker issues in the past, WSL2 alone is often more reliable — it's just a lightweight Linux VM built into Windows.
What Authentik Gives You

A single login page at (e.g.) http://auth.homeserver.local. All your apps redirect here for authentication. Users log in once, and all apps trust the Authentik session. Supports MFA, app-level access control, audit logs.

Installing Authentik via WSL2 (No Full Docker Required)
Enable WSL2 
wsl --install
# Reboot when prompted
# This installs Ubuntu as the default WSL distro
Inside WSL2 Ubuntu, install Authentik 
sudo apt update && sudo apt install -y docker.io docker-compose
mkdir ~/authentik && cd ~/authentik
wget https://goauthentik.io/docker-compose.yml
# Edit .env with your settings
docker-compose up -d

Authentik will be accessible at http://localhost:9000 from Windows.

Configure each app to use Authentik as OIDC provider

In Authentik, create an Application + Provider for each service (Calibre-Web, ABS, Nextcloud). Each gets a client ID and secret. Configure each app exactly as in Tier 2, but using your Authentik URL instead of Google's.

💡 Alternative: Authelia (Lighter Weight)

Authelia is simpler than Authentik. It works as a middleware in front of your apps (via Caddy/Nginx) and adds a login gate with optional MFA. It doesn't replace app-level auth but adds a unified front door. Also runs under WSL2.
