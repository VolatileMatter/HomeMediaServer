Single Sign-On (SSO)

// Per-User Progress · Google OAuth · Authentik · Emulation Saves //

SSO for apps where individual users need their own tracked progress: Calibre-Web (reading history), Audiobookshelf (listening progress), Nextcloud (personal files), and emulated game saves.

💡
Plex is excluded here — Plex has its own account system (plex.tv) and handles multi-user watch history natively. Share your server by inviting friends via Settings → Manage Library Access.
🐉 V1 Strategy: Give each user their own local username/password in each app. Then upgrade to Google OAuth per service. Then optionally centralize with Authentik for one login page.
🗺️ Why Per-User Matters for These Apps
AppWhy users need separate accounts
Calibre-WebReading progress, bookmarks, "currently reading" shelf
AudiobookshelfListening position, finished chapters, podcast subscriptions
NextcloudPersonal files, private calendars, separate storage quotas
Pokémon / Castlevania / Fire EmblemIndividual game save files — crucial, can't share
JukeboxHistory / favorites (optional; a simple shared password may be fine)
1️⃣ Tier 1 — Per-App Username & Password (V1)
Calibre-Web User Setup

Admin panel → User Management → Add new user. Set per-user permissions:

Read only:   can browse and download books
Upload:      can add new books
Edit books:  can modify metadata
Admin:       full control

Each user's reading progress (currently reading, bookmarks, read history) is tied to their account.

Audiobookshelf User Setup

Settings → Users → Add User. Set per-user library access — you can restrict a user to only see specific libraries (e.g. a child's account that only sees kids' audiobooks).

Each user's listening position is stored server-side — they can switch from laptop to phone and resume exactly where they left off.

Nextcloud User Setup

Admin → Users → New User → set username, email, initial password, storage quota. Each user gets their own private file storage, separate from other users.

Emulation (Pokémon / Castlevania / Fire Emblem)

The emulation Flask server has its own username/password system (see the respective tabs). Users log in, and their saves are stored in saves/[username]/ directories. Each user's game progress is completely separate.

# Add users via Python one-liner:
python -c "
import json
from werkzeug.security import generate_password_hash
users = json.load(open('users.json')) if os.path.exists('users.json') else {}
users['newuser'] = {'password': generate_password_hash('their-password')}
json.dump(users, open('users.json','w'))
print('User added.')
"
2️⃣ Tier 2 — Google OAuth ("Sign in with Google")
One-Time: Set Up Google OAuth Credentials
Create a project in Google Cloud Console

console.cloud.google.com → Create Project → name it "Home Server SSO"

Configure OAuth Consent Screen

APIs & Services → OAuth consent screen → External → fill in app name, your email. Under "Test users," add every Gmail address that should have access. (While in "Testing" mode, only listed users can authenticate.)

Create OAuth Client ID

APIs & Services → Credentials → Create Credentials → OAuth Client ID → Web application. Save the Client ID and Client Secret.

📚 Calibre-Web + Google

Add redirect URI in Google Console: http://[server-ip]:8083/google/authorized

In Calibre-Web Admin → Edit app configuration:

☑ Enable Google OAuth Login
Google Client ID:     [paste client id]
Google Client Secret: [paste client secret]

Each user links their Google account once: Admin → Users → Edit user → "Link Google account." After that, they see "Sign in with Google" on the login page.

🎧 Audiobookshelf + OpenID Connect (Google)

Add redirect URI: http://[server-ip]:13378/auth/openid/callback

In ABS Admin → Settings → Authentication:

☑ OpenID Connect Authentication
Issuer URL:     https://accounts.google.com
Client ID:      [paste]
Client Secret:  [paste]
Button Text:    Sign in with Google

🎮 Emulation Server + Google

The Flask-based emulation server can be extended with Google OAuth using the authlib library:

pip install authlib requests

# In server.py, replace the login route with Google OAuth:
from authlib.integrations.flask_client import OAuth

oauth = OAuth(app)
google = oauth.register('google',
    client_id='YOUR_GOOGLE_CLIENT_ID',
    client_secret='YOUR_GOOGLE_CLIENT_SECRET',
    server_metadata_url='https://accounts.google.com/.well-known/openid-configuration',
    client_kwargs={'scope': 'openid email profile'}
)

@app.route('/login/google')
def google_login():
    return google.authorize_redirect(url_for('google_callback', _external=True))

@app.route('/login/google/callback')
def google_callback():
    token = google.authorize_access_token()
    user_info = token['userinfo']
    email = user_info['email']
    # Use email as the username for save directory lookup
    session['user'] = email.split('@')[0]  # or use full email
    return redirect('/')

After this, each user's saves are stored in saves/[gmail-prefix]/ — completely separate.

3️⃣ Tier 3 — Authentik (One Login to Rule Them All)
What Authentik Provides

A single login page. Users authenticate once, and every app trusts that session. Supports Google as an upstream "social login" — so users can still use their Gmail, but through Authentik instead of per-app.

Runs via Docker on Linux or WSL2 on Windows. See the Linux tab for WSL2 setup.

# Install via WSL2 + Docker Compose:
wget https://goauthentik.io/docker-compose.yml
# Edit .env: set AUTHENTIK_SECRET_KEY, database passwords
docker-compose up -d

# Access admin at http://localhost:9000/if/admin/
Connecting Authentik to Each App

For each app, create an "Application" and "OAuth2/OIDC Provider" in Authentik. The app then uses Authentik's OIDC endpoints instead of Google's directly. The process is identical to Tier 2 per app — just replace Google's URLs with your Authentik URL.

Issuer URL:     http://localhost:9000/application/o/[app-name]/
Client ID:      (from Authentik app config)
Client Secret:  (from Authentik app config)

Authentik can itself be configured to use Google as a "social login" upstream, so users click "Sign in with Google" on the Authentik page and all apps get the Google identity.
