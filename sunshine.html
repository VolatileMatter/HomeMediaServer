☀️ Step 1 — Install Sunshine
Download and install Sunshine

Get the latest Windows installer from github.com/LizardByte/Sunshine. Run the .exe installer — it registers itself as a Windows service that auto-starts with Windows.

Access the Sunshine admin UI

Open https://localhost:47990 in a browser on the server. Accept the self-signed certificate warning. Set your admin username and password on first launch.

🔐
The Sunshine admin UI is LAN only — never expose port 47990 externally. Use a strong password anyway.
Configure encoding quality

In the Sunshine web UI → Configuration tab:

Resolution:  1920x1080
FPS:         60
Bitrate:     20–50 Mbps
Encoder:     nvenc (Nvidia) / amf (AMD) / vaapi (Intel)
Open required firewall ports 
netsh advfirewall firewall add rule name="Sunshine HTTPS" dir=in action=allow protocol=TCP localport=47990
netsh advfirewall firewall add rule name="Sunshine Stream UDP" dir=in action=allow protocol=UDP localport=47998-48000
netsh advfirewall firewall add rule name="Sunshine Control TCP" dir=in action=allow protocol=TCP localport=47998-48010

Pair each device with your server

Open Moonlight → Add Host → enter your server's LAN IP. Moonlight displays a 4-digit PIN. Enter it in Sunshine web UI → PIN Pairing. The device is then permanently paired.

💡
Name each client after its user's Authentik username (e.g. "alice", "bob"). Sunshine exposes this as %%SUNSHINE_CLIENT_NAME%% in pre-launch commands, which the Session Manager uses to route save data to the correct user folder.
🎮 Step 3 — Add Apps to Sunshine

Each game or emulator is added as a Sunshine "Application." Users see all apps in Moonlight and can click any one to start streaming it directly.

Add each app in Sunshine Web UI → Applications → Add New

📋 What Is Sunshine?

Sunshine is a self-hosted implementation of NVIDIA's GameStream protocol — the same protocol used by Moonlight clients. It captures the server's display, encodes it using the GPU (NVENC/AMF/VAAPI), and streams it to Moonlight over the network. Input (keyboard, mouse, controller) is forwarded back from client to server in real time.

The result: any device running Moonlight can play any game that runs on the server, with near-zero-latency control at 1080p60 or higher — even over WiFi on the same network.

🖥️ Single-Seat Limitation

Sunshine can only stream one session at a time. If Alice is streaming, Bob cannot start a new Sunshine session until Alice disconnects. This is a hardware/protocol limitation, not a configuration issue.

Browser-based games (EmulatorJS, Flash Arcade) are completely independent of Sunshine — multiple people can use those simultaneously. Minecraft, ARK, and other server games are also unaffected.

🌊 User Flow: From Moonlight to Playing
  1. Open Moonlight on your device
  2. Select the server (paired once — stays paired permanently)
  3. Browse the app list — all Sunshine apps appear as tiles with cover art
  4. Click any app to start streaming instantly
  5. For Sims 2 and emulators: a session picker page may appear first in the browser to select your user — then the game launches automatically
  6. When done, close Moonlight or press the quit shortcut — the session ends and saves are committed to your user folder

🖼️ Adding Cover Art

In the Sunshine app editor, upload a cover image for each game. This is what Moonlight displays in the app grid. Source art from steamgriddb.com — they have custom art for GOG, emulator games, and non-Steam titles. Portrait format (600×900) works best in Moonlight's grid view.

🛠️ Common Issues
Black screen when connecting via Moonlight
Sunshine requires a logged-in Windows desktop session — not the lock screen or login prompt. Ensure the server is logged in and not sleeping. Disable sleep/hibernate in Windows power settings.
Controller input not working in the game
Moonlight forwards input as a virtual XInput controller. Ensure the game or emulator is configured for XInput. In Dolphin: Controllers → Standard Controller → XInput/0/Gamepad. In most PC games, XInput works by default.
High latency or stuttering stream
Lower the Sunshine bitrate to 15–20 Mbps and try 720p. Check if another GPU task (Plex hardware transcode, Ollama) is running simultaneously. For best results, use 5GHz WiFi or wired Ethernet on both ends.
Can't connect — Moonlight can't find the server
Ensure the server's LAN IP hasn't changed (set a static LAN IP or DHCP reservation via your router). Confirm the firewall rules are in place. Try adding the IP manually in Moonlight → Add Host.
Audio not transmitted to client
In Sunshine web UI → Configuration → Audio: select the correct output device. The device must be the one Windows is actively routing audio to. If using HDMI or a virtual audio device, select that specifically.
Save data ended up in the wrong user's folder
The Session Manager (port 9004) matches saves by Moonlight client name. Verify the Moonlight client is named exactly as the Authentik username (lowercase, no spaces). Re-pair the device if the name needs to change.
