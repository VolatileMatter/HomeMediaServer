🔐 Step 1 — Create a Dedicated stream-user Windows Account

Sunshine must run as a low-privilege account, not your admin account. If a user escapes the game, they land in a locked-down sandbox with no access to your files.

Create the account (PowerShell as Administrator) 
$pw = ConvertTo-SecureString "ChooseAStrongPassword!" -AsPlainText -Force
New-LocalUser -Name "stream-user" -Password $pw `
  -FullName "Stream User" -Description "Sunshine streaming sandbox"
# Do NOT add to Administrators group
Grant access only to emulator + game folders 
$folders = @("D:\Emulators","D:\Emulation","D:\Games","D:\Users")
foreach ($f in $folders) {
    $acl = Get-Acl $f
    $rule = New-Object System.Security.AccessControl.FileSystemAccessRule(
        "stream-user","ReadAndExecute,Modify",
        "ContainerInherit,ObjectInherit","None","Allow")
    $acl.SetAccessRule($rule)
    Set-Acl $f $acl
}
Configure Sunshine service to run as stream-user

Open services.msc → Sunshine → Properties → Log On tab → "This account" → enter stream-user and password → OK → Restart service.

⚠️
After changing the service account, re-test pairing — the new account needs to accept the Sunshine service agreement on first launch.
🎮 Step 2 — Install Playnite as the Kiosk Shell

Playnite is the game launcher users see when they connect. It runs fullscreen as the shell replacement for stream-user — replacing Windows Explorer entirely. When a game exits, users return to Playnite, not the desktop.

Install Playnite

Download from playnite.link. Install to D:\Server\Playnite\ under the stream-user profile.

Configure Playnite for fullscreen kiosk mode 
# In Playnite Settings (under stream-user):
# General: Start in fullscreen mode = ON, Close to tray = OFF
# Input:   Enable gamepad control in fullscreen = ON
Set Playnite as the shell for stream-user

Log in as stream-user, then run in a command prompt:

reg add "HKCU\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" ^
  /v Shell /t REG_SZ ^
  /d "D:\Server\Playnite\Playnite.FullscreenApp.exe" /f
💡
Test by switching to stream-user interactively (Switch User). You should see Playnite fullscreen immediately — no taskbar, no desktop.
Block Alt+Tab and system hotkeys via AutoHotkey 
; D:\Server\Playnite\block-hotkeys.ahk
#NoEnv
#Persistent
!Tab::return        ; Alt+Tab
^Escape::return     ; Ctrl+Esc
LWin::return        ; Windows key
RWin::return        ; Windows key
#d::return          ; Win+D show desktop
#e::return          ; Win+E Explorer
!F4::return         ; Alt+F4 on desktop

; Add shortcut to:
; C:\Users\stream-user\AppData\Roaming\Microsoft\Windows
;   \Start Menu\Programs\Startup\
⚠️
AutoHotkey must be installed system-wide so stream-user can run it without admin rights.
☀️ Step 3 — Install & Harden Sunshine
Download and install Sunshine

Get the latest Windows installer from github.com/LizardByte/Sunshine. Run the .exe installer — it registers as a Windows service that auto-starts with Windows.

Access the admin UI and set critical security options

Open https://localhost:47990 → accept the self-signed cert → set a strong admin password.

# Sunshine Web UI -> Configuration -> General:
terminate_on_app_close = true   # CRITICAL - set this first
Resolution:  1920x1080
FPS:         60
Bitrate:     20-50 Mbps
Encoder:     nvenc (Nvidia) / amf (AMD) / vaapi (Intel)
🔐
terminate_on_app_close = true is the single most important security setting. Without it, closing Playnite leaves users with live desktop access.
Add Playnite as the single Sunshine application 
# Sunshine Web UI -> Applications -> Add New:
Name:        Game Library
Command:     "D:\Server\Playnite\Playnite.FullscreenApp.exe"
Working Dir: D:\Server\Playnite\

# Pre-launch (detached) - session manager for per-user saves:
curl -s -X POST http://localhost:9004/session/start ^
  -H "Content-Type: application/json" ^
  -d "{\"username\":\"%%SUNSHINE_CLIENT_NAME%%\"}"

# Undo command (detached) - cleanup on session end:
curl -s -X POST http://localhost:9004/session/end
💡
Moonlight shows one tile: "Game Library". All games are managed inside Playnite. No individual games are added as separate Sunshine apps.
Open required firewall ports 
netsh advfirewall firewall add rule name="Sunshine HTTPS" dir=in action=allow protocol=TCP localport=47990
netsh advfirewall firewall add rule name="Sunshine Stream UDP" dir=in action=allow protocol=UDP localport=47998-48000
netsh advfirewall firewall add rule name="Sunshine Control TCP" dir=in action=allow protocol=TCP localport=47998-48010

📋 What Is Sunshine?

Sunshine is a self-hosted implementation of NVIDIA's GameStream protocol — the same protocol used by Moonlight clients. It captures the server's display, encodes it using the GPU (NVENC/AMF/VAAPI), and streams it to Moonlight over the network. Input (keyboard, mouse, controller) is forwarded back from client to server in real time.

The result: any device running Moonlight can play any game that runs on the server, with near-zero-latency control at 1080p60 or higher — even over WiFi on the same network.

🖥️ Single-Seat Limitation

Sunshine can only stream one session at a time. If Alice is streaming, Bob cannot start a new Sunshine session until Alice disconnects. This is a hardware/protocol limitation, not a configuration issue.

Browser-based games (EmulatorJS, Flash Arcade) are completely independent of Sunshine — multiple people can use those simultaneously. Minecraft, ARK, and other server games are also unaffected.

🛡️ Optional: AppLocker Whitelist

AppLocker restricts which executables stream-user can run. Belt-and-suspenders defense — even if AHK fails, stream-user cannot launch cmd.exe or powershell.exe.

# gpedit.msc -> Computer Configuration -> Windows Settings ->
# Security Settings -> Application Control Policies -> AppLocker
# Executable Rules - Allow for stream-user only:
#   D:\Server\Playnite\*.exe
#   D:\Emulators\**\*.exe
#   D:\Games\**\*.exe
#   C:\Windows\System32\curl.exe
#   C:\Program Files\AutoHotkey\*.exe

Set-Service -Name AppIDSvc -StartupType Automatic
Start-Service AppIDSvc
⚠️
AppLocker requires Windows 10/11 Pro or Enterprise. Test thoroughly — strict rules can break emulators that spawn sub-processes.

In the Sunshine app editor, upload a cover image for each game. This is what Moonlight displays in the app grid. Source art from steamgriddb.com — they have custom art for GOG, emulator games, and non-Steam titles. Portrait format (600×900) works best in Moonlight's grid view.

🛠️ Common Issues
Black screen when connecting via Moonlight
Sunshine requires a logged-in Windows desktop session — not the lock screen or login prompt. Ensure the server is logged in and not sleeping. Disable sleep/hibernate in Windows power settings.
Controller input not working in the game
Moonlight forwards input as a virtual XInput controller. Ensure the game or emulator is configured for XInput. In Dolphin: Controllers → Standard Controller → XInput/0/Gamepad. In most PC games, XInput works by default.
High latency or stuttering stream
Lower the Sunshine bitrate to 15–20 Mbps and try 720p. Check if another GPU task (Plex hardware transcode, Ollama) is running simultaneously. For best results, use 5GHz WiFi or wired Ethernet on both ends.
Can't connect — Moonlight can't find the server
Ensure the server's LAN IP hasn't changed (set a static LAN IP or DHCP reservation via your router). Confirm the firewall rules are in place. Try adding the IP manually in Moonlight → Add Host.
Audio not transmitted to client
In Sunshine web UI → Configuration → Audio: select the correct output device. The device must be the one Windows is actively routing audio to. If using HDMI or a virtual audio device, select that specifically.
Save data ended up in the wrong user's folder
The Session Manager (port 9004) matches saves by Moonlight client name. Verify the Moonlight client is named exactly as the Authentik username (lowercase, no spaces). Re-pair the device if the name needs to change.
User can see the desktop after closing a game
Verify terminate_on_app_close = true in Sunshine Configuration. Also check that Playnite is stream-user's shell: HKCU\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon → Shell should point to Playnite.FullscreenApp.exe. See the Security tab checklist.
Playnite does not appear fullscreen — desktop visible briefly on connect
Playnite startup may be slow. Change the Sunshine command to cmd.exe /c timeout /t 3 && "D:\Server\Playnite\Playnite.FullscreenApp.exe". Or configure Playnite to auto-start at stream-user login so it is already running when Moonlight connects.
Alt+Tab not blocked — user can task-switch out of a game
The AutoHotkey blocker is not running. Check stream-user's Startup folder for the AHK shortcut. Verify AutoHotkey is installed at C:\Program Files\AutoHotkey\. Log in as stream-user interactively and check Task Manager for an AutoHotkey process.
Sunshine service fails to start after switching to stream-user account
stream-user needs "Log on as a service" rights. Open secpol.msc → Local Policies → User Rights Assignment → "Log on as a service" → add stream-user. Also ensure stream-user has Execute permissions on the Sunshine install directory.
