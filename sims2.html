🏠

The Sims 2

// Sunshine Streaming · Per-User Saves · Shared + Private Worlds · Port 47990 //

The Sims 2 Ultimate Collection running on the server, streamed via Sunshine to Moonlight. Multiple users each get private save data and optional personal mod folders. One shared communal neighborhood is available to all. One person plays at a time.

⚠️ Single-seat service — Sunshine supports one active stream at a time. Only one person can play at once. This is a hardware/architecture limitation, not a bug.
🗺️ Architecture Overview

One shared game install. One shared mod collection (you maintain it). Per-user save data and optional personal mod folders. A single Active\ symlink is re-pointed at session start to whoever is playing — the game never knows anything unusual is happening.

🐉
Core principle: The game always reads from Documents\EA Games\The Sims 2\. That path is a permanent junction to C:\Sims2\Active\. The launcher script changes what Active\ points to before the game opens. The game sees a normal save folder every time.
🌍 World Types

Personal World — user's private Neighborhoods and Storytelling folders. Only they can see and edit it.

Shared World — a communal neighborhood any user can load. Always uses the shared mod folder. No mod choice is offered when this is selected.

📦 Mod Folder Types

Shared Mods — the server's curated collection in C:\Sims2\Mods\Shared\. Recommended for most users.

Personal Mods — the user's own folder in C:\Sims2\Mods\Users\[name]\. They manage it themselves. Can be seeded from Shared on first use.

🔧 Part 1 — Modern Compatibility Fixes

Apply all of these before setting up the multi-user architecture. They are system-wide and benefit every user.

Install The Sims 2 Ultimate Collection via EA App

Free from the EA App. Includes all expansions and has the best compatibility with modern fixes. Do not use a disc-based install. The executable you'll be patching is Sims2EP9.exe in the install directory.

Run Graphics Rules Maker

The most important fix. Sims 2 has a hardcoded GPU allowlist from 2004 and runs in degraded software mode on any modern GPU without this patch. Search for "Graphics Rules Maker" — it's a free tool. Point it at your Sims 2 install directory and run it. It patches Graphics Rules.sgr and Video Cards.sgr automatically, unlocking full texture quality, shadows, and high resolution.

💡
Also set your desired resolution in Graphics Rules Maker. Set it to the server monitor's native resolution — Sunshine streams at whatever resolution the game runs at.
Apply the 4GB Patch to Sims2EP9.exe

Sims 2 is a 32-bit app limited to 2 GB RAM by default. With a large mod folder, this causes crashes. Search for "4GB patch exe tool". Apply it to Sims2EP9.exe. This raises the memory ceiling to 4 GB and dramatically improves stability with heavy mod loads.

# Path to patch:
C:\Sims2\Game\Sims2EP9.exe
Install the Smooth Patch mod

Sims 2 was designed for ~30 fps and behaves incorrectly at high framerates — game time fast-forwards, animations glitch, physics break. The Smooth Patch mod (community-made, available on mod sites) caps and corrects frame pacing. Install it as a .package file into the shared mod folder.

Install DXVK (optional but recommended)

Sims 2 uses DirectX 9. DXVK (a DirectX-to-Vulkan translation layer) improves frame consistency and reduces stuttering on modern hardware. Download the latest DXVK release from GitHub. Copy d3d9.dll into the Sims 2 game directory alongside Sims2EP9.exe. The game does not know it is there.

Disable intro movies

Delete or rename all files inside TSData/Res/Movies/Startup. This skips the EA/Maxis splash screens on every launch — important when users are repeatedly launching via the picker UI.

Set Windows compatibility flags on the executable

Right-click Sims2EP9.exe → Properties → Compatibility tab:

☑  Run as Administrator
☑  Disable fullscreen optimizations
    Override high DPI scaling behaviour: Application

Create the permanent Documents junction

Sims 2 reads user data from Documents\EA Games\The Sims 2\ by default. Create a permanent junction from that path to C:\Sims2\Active\. You only ever change what Active\ points to at launch time — the game never knows anything unusual is happening.

# Run once to create the permanent junction (run as Administrator):
# First, delete the existing EA Games folder if it exists
Remove-Item "$env:USERPROFILE\Documents\EA Games\The Sims 2" -Force -Recurse -ErrorAction SilentlyContinue

# Create the permanent junction
New-Item -ItemType Junction -Path "$env:USERPROFILE\Documents\EA Games\The Sims 2" -Target "C:\Sims2\Active"
⚠️
Do this before first launch. If the game has already run, there will be an existing folder at that path. Back it up first, then delete it and create the junction.
⚙️ Part 3 — Per-User config.json
Create this file at C:\Sims2\Worlds\Users\[username]\config.json
{
  "mod_folder": "shared",       // "shared" or "personal"
  "mod_folder_locked": false,   // becomes true after first launch
  "world_has_saves": false      // becomes true after first launch
}
🚨
Why mod_folder_locked matters: Once a user has played, their save data references objects and Sims that exist in their current mod folder. Switching mod folders after saves exist causes missing content — Sims lose appearance, objects disappear, neighborhoods can corrupt. This flag drives the warning shown in the picker UI when a user attempts to change their mod choice after playing.
🚀 Part 4 — PowerShell Launcher Script
Save as C:\Sims2\launch.ps1

This script runs before the game launches. It reads the user's config, sets up the correct symlinks for world and mods, locks the config, then starts the game.

param(
  [string]$User,
  [string]$WorldType   # "personal" or "shared"
)

$activeDir  = "C:\Sims2\Active"
$docsPath   = "$env:USERPROFILE\Documents\EA Games\The Sims 2"
$userConfig = "C:\Sims2\Worlds\Users\$User\config.json"

# Load user config
$config = Get-Content $userConfig | ConvertFrom-Json

# Remove existing Active junction
Remove-Item $activeDir -Force -Recurse -ErrorAction SilentlyContinue
Remove-Item $docsPath  -Force -Recurse -ErrorAction SilentlyContinue

# Point world junction
if ($WorldType -eq "shared") {
  $worldTarget = "C:\Sims2\Worlds\Shared"
} else {
  $worldTarget = "C:\Sims2\Worlds\Users\$User"
}
New-Item -ItemType Junction -Path $activeDir -Target $worldTarget

# Point docs junction back to Active
New-Item -ItemType Junction -Path $docsPath -Target $activeDir

# Point mods junction (inside Active\Downloads)
if ($config.mod_folder -eq "shared") {
  $modsTarget = "C:\Sims2\Mods\Shared"
} else {
  $modsTarget = "C:\Sims2\Mods\Users\$User"
}
$activeDownloads = "C:\Sims2\Active\Downloads"
Remove-Item $activeDownloads -Force -ErrorAction SilentlyContinue
New-Item -ItemType Junction -Path $activeDownloads -Target $modsTarget

# Lock the config now that they are launching
$config.mod_folder_locked = $true
$config.world_has_saves   = $true
$config | ConvertTo-Json | Set-Content $userConfig

# Launch the game
Start-Process "C:\Sims2\Game\Sims2EP9.exe"

Add these routes to the existing Flask server. The picker identifies the logged-in user via Authentik SSO, presents the world and mod options, checks whether Sunshine is already in use, then calls the launcher script.

The UI is a two-step flow for personal world selection. Shared world always uses shared mods — no mod choice is shown for it.

💡
New user — "My Own Mod Folder" selected for the first time: Offer a choice to start empty or copy shared mods as their baseline. The copy may take a few minutes — show a progress indicator.
5.2 — Sunshine Active-Session Check

Before showing any launch buttons, poll Sunshine's local API to check whether someone is already streaming. If active, grey out the buttons and show a message — do not name who is playing.

def sunshine_is_active():
    try:
        r = requests.get('http://localhost:47990/api/clients', timeout=1)
        return len(r.json()['clients']) > 0
    except:
        return False

5.3 — Mod Config Change Warning

If a user tries to change their mod folder after mod_folder_locked is true, redirect them to a warning page. Do not allow this change silently.

🚨
The warning page must include all of the following:
  • Heading: WARNING — This May Break Your Game
  • Explanation that saves reference their current mod folder
  • Statement that missing mods cause corrupted Sims, missing objects, broken neighborhoods
  • Statement that this is NOT reversible without a backup, and no backup system exists on this server
  • A text input requiring the user to type CONFIRM exactly before the proceed button activates
  • A clearly visible cancel link that returns them safely to the picker
5.4 — Calling the Launcher from Flask
import subprocess, json
from flask import request, redirect

@app.route('/sims2/launch', methods=['POST'])
def sims2_launch():
    username  = request.headers.get('X-authentik-username', '')
    worldtype = request.form.get('worldtype', 'personal')  # "personal" or "shared"

    if not username:
        return redirect('/login')
    if sunshine_is_active():
        return "Server in use", 409

    subprocess.Popen([
        'powershell', '-File', r'C:\Sims2\launch.ps1',
        '-User', username,
        '-WorldType', worldtype
    ])
    return redirect('/sims2/launching')
💡
The username comes from the X-authentik-username header injected by Authentik's forward_auth proxy. The picker does not need to ask the user who they are — Authentik already knows.

💾 User Data Storage → SSO TAB

Sims 2 save data lives entirely within each user's folder under C:\Sims2\Worlds\Users\[username]\. This folder structure is separate from D:\Users\ (the standard user data root used by other services) due to Sims 2's specific junction architecture. See the SSO tab → User Data Storage for the full cross-service architecture.

C:\Sims2\Worlds\Users\alice\
  Neighborhoods\        ← all of Alice's Sim families and lots
  Storytelling\         ← screenshots and story content
  config.json           ← mod choice and lock state

C:\Sims2\Mods\Users\alice\
  Packages\             ← Alice's personal .package mod files (if using personal mods)
  Overrides\
⚠️
Backup note: No automated backup exists for Sims 2 save data. This is noted explicitly in the mod-change warning page. Users should be aware that save data is not recoverable if corrupted by a mod folder switch.
👋 Part 7 — Adding a New User
# Run once per new user (replace "alice" with their Authentik username):

$user = "alice"

# Create world folders
New-Item -ItemType Directory -Force "C:\Sims2\Worlds\Users\$user\Neighborhoods"
New-Item -ItemType Directory -Force "C:\Sims2\Worlds\Users\$user\Storytelling"

# Create personal mod folder
New-Item -ItemType Directory -Force "C:\Sims2\Mods\Users\$user\Packages"
New-Item -ItemType Directory -Force "C:\Sims2\Mods\Users\$user\Overrides"

# Create config.json
@{ mod_folder="shared"; mod_folder_locked=$false; world_has_saves=$false } |
  ConvertTo-Json | Set-Content "C:\Sims2\Worlds\Users\$user\config.json"

Then add the user to Authentik so they can log in via SSO. Their first launch via the picker UI initialises everything else.
