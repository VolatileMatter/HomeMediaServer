Security

// Hardening · Firewall · VPN · Exposure Reduction //

A home server connected to the internet is a real attack surface. This page covers the practical security steps needed to keep it safe without making it unusable.

🐉 Priority order: (1) Strong passwords + firewall, (2) Reverse proxy with HTTPS, (3) Fail2ban or similar, (4) Domain + DNS, (5) SSO. Don't skip straight to SSO — the basics matter most.
🎯 Threat Model: What Are You Actually Protecting Against?

For a home server, your realistic threats are:

  • Automated bots scanning the internet for exposed admin panels and weak passwords (very common, happens within hours of opening a port)
  • Credential stuffing — people trying known username/password combos
  • Exposed sensitive services — Minecraft RCON, qBittorrent WebUI, Plex without auth
  • Accidental data exposure — misconfigured Nextcloud making files public

You're probably not facing targeted nation-state attacks. Simple, consistent security measures stop 99% of threats.

🔥 Step 1: Windows Firewall (Do This First)
Only Open Ports You Need

The Windows Firewall blocks inbound connections by default. Only open ports for services you actively want accessible. This guide has added rules per-service — audit what you've opened:

# See all inbound allow rules:
netsh advfirewall firewall show rule name=all dir=in action=allow | findstr "Rule Name"

# Delete a rule you no longer need:
netsh advfirewall firewall delete rule name="Rule Name Here"

🔒 Step 2: HTTPS Everywhere via Caddy (Reverse Proxy)
Why a Reverse Proxy?

Instead of exposing each service on its own port, a reverse proxy (Caddy) sits in front of everything. It handles HTTPS certificates automatically, hides internal ports, and gives you one place to add auth rules. Install Caddy from caddyserver.com — it's a single binary.

# Example Caddyfile — serves all your services under subdomains:
# Place at C:\caddy\Caddyfile, run: caddy run --config Caddyfile

books.yourdomain.com {
    reverse_proxy localhost:8083
    # Optional: add basic auth
    basicauth {
        admin $2a$14$... # bcrypt hash, generate with: caddy hash-password
    }
}

audio.yourdomain.com {
    reverse_proxy localhost:13378
}

cloud.yourdomain.com {
    reverse_proxy localhost:80
}

status.yourdomain.com {
    root * D:\Server\Dashboard
    file_server
    rewrite * /monitor-public.html
}

# All of the above are Tailscale-only — no public exposure needed.
# Only exception: port 8096 (Jellyfin) forwarded for Smart TV apps.
💡
Free TLS certificates: Caddy automatically gets free HTTPS certificates from Let's Encrypt for any domain you control. No manual certificate management needed.
🚫 Step 3: Fail2Ban Equivalent for Windows
IPBan (FOSS, Windows Native) FOSS

IPBan monitors Windows Event Log for failed login attempts (RDP, SSH, etc.) and automatically blocks IPs after too many failures. Download from github.com/DigitalRuby/IPBan.

# Run installer, then start as a service:
sc start IPBan

# Default: blocks IPs after 5 failed attempts in 10 minutes
# Config file: C:\Program Files\IPBan\ipban.config.xml

For Caddy/web app brute force protection, Caddy's rate_limit plugin adds request rate limiting per IP on any endpoint.

🔑 Step 4: Strong Passwords & SSH Keys
Password Rules for Every Service
  • Every service gets a unique, 20+ character password
  • Use Vaultwarden (self-hosted Bitwarden) to store them — see below
  • Admin accounts should never be named "admin" — bots guess this first
  • Enable 2FA (TOTP) on every service that supports it (Nextcloud, Authentik)
Vaultwarden — Self-Hosted Password Manager FOSS

Vaultwarden is a lightweight, self-hosted Bitwarden-compatible server. Run it as a Windows service via NSSM. All your passwords stored on your own hardware, accessible from the Bitwarden browser extension.

# Download Vaultwarden Windows binary from:
# github.com/dani-garcia/vaultwarden/releases
# Extract to D:\Server\vaultwarden\
# Run: vaultwarden.exe
# Access at http://localhost:8082
# Use NSSM to run as service
🌐 Step 5: Don't Expose Services Directly — Use Tailscale
Tailscale — All Services (Primary)

All services run behind Tailscale. Users install Tailscale once, you approve their device, and they access every service at its *.ixo.lol subdomain. WireGuard-encrypted, zero open ports, free with no bandwidth cap. See the Network tab for full setup.

Port Forward — Jellyfin TV Only (Exception)

The single deliberate exception: port 8096 is forwarded on the router for Smart TV Jellyfin apps (LG WebOS, Tizen, Roku, Fire TV) that cannot install Tailscale. Protected by Jellyfin's own auth. All other services remain Tailscale-only. See the Network tab → Jellyfin TV Carve-out.

🔐 Step 6: Keep Windows Updated

Run Windows Update regularly. Most home server compromises happen through unpatched vulnerabilities, not sophisticated attacks. Settings → Windows Update → Advanced → enable "Receive updates for other Microsoft products." Also keep all your service software (Caddy, qBittorrent, Nextcloud, etc.) updated — subscribe to their GitHub release pages for notifications.
