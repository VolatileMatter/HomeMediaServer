📡 Uptime Kuma — Service Monitor FOSS · Node.js
Port: 3001 · Is it a good addition?

Yes — highly recommended. Uptime Kuma is a polished, actively maintained status monitoring dashboard. It goes beyond the basic ping checks in the custom dashboard by:

  • Monitoring HTTP/HTTPS endpoints, TCP ports, DNS records, databases
  • Sending alerts via email, Telegram, Discord, Pushover, ntfy, and 90+ other integrations
  • Tracking uptime history over time (99.9% vs 97% uptime charts)
  • Providing a public status page you can share with users
  • Monitoring response times with graphs

It complements (not replaces) the custom dashboard — Uptime Kuma is for historical monitoring and alerts; the custom dashboard is for real-time control.

Install Node.js 20 LTS (if not already)

From nodejs.org.

Install Uptime Kuma 
npm install -g uptime-kuma
uptime-kuma --port 3001 --data-dir D:\Server\UptimeKuma

Access at http://localhost:3001. Create admin account on first visit.

Run as a Windows Service (NSSM) 
nssm install UptimeKuma node "%AppData%\npm\node_modules\uptime-kuma\server\server.js"
nssm set UptimeKuma AppParameters "--port 3001 --data-dir D:\Server\UptimeKuma"
nssm start UptimeKuma
Add monitors for every service

In Uptime Kuma → Add New Monitor → HTTP(s):

Plex:           http://localhost:32400/web    (HTTP)
Calibre-Web:    http://localhost:8083          (HTTP)
Audiobookshelf: http://localhost:13378         (HTTP)
Navidrome:      http://localhost:4533          (HTTP)
Nextcloud:      http://localhost/nextcloud     (HTTP)
Jellyfin:       http://localhost:8096          (HTTP)
qBittorrent:    TCP localhost:8090             (TCP port)
Minecraft Van:  TCP localhost:25565            (TCP port)
Minecraft Mod:  TCP localhost:25566            (TCP port)
Kavita Comics:  http://localhost:5000          (HTTP)
RomM:           http://localhost:8098          (HTTP)
Configure notification alerts

Settings → Notifications → Add. Recommended: ntfy (push notifications, free, self-hostable) or Discord webhook (if you have a server Discord). You'll get a push notification on your phone when any service goes down.

Open firewall 
netsh advfirewall firewall add rule name="Uptime Kuma" dir=in action=allow protocol=TCP localport=3001
📡
Public status page: Uptime Kuma can generate a public status page (like statuspage.io but free) at a custom URL. Configure it at Settings → Status Pages → Add Status Page. Expose at status.yourdomain.com alongside or instead of the custom monitor-public.html.
✅ Vikunja — Task Manager FOSS · Windows Binary
Port: 3456 · What Vikunja Is

Vikunja is a self-hosted Todoist/Notion-style task manager. It supports projects, tasks with subtasks, due dates, priorities, labels, and team sharing. Great for managing server projects, D&D campaign notes, or anything that needs organized task tracking. Single binary, runs natively on Windows.

Download Vikunja

From vikunja.io/docs/installing → Windows binary. Extract vikunja.exe to D:\Server\Vikunja\.

Create config file

Create D:\Server\Vikunja\config.yml:

service:
  jwtsecret: "change-this-to-a-long-random-string"
  frontendurl: "http://[your-server-ip]:3456/"
  interface: ":3456"

database:
  type: "sqlite"
  path: "D:/Server/Vikunja/vikunja.db"

files:
  basepath: "D:/Server/Vikunja/files/"
Run and install as service 
cd D:\Server\Vikunja
vikunja.exe  # test run first

# Then install via NSSM:
nssm install Vikunja "D:\Server\Vikunja\vikunja.exe"
nssm set Vikunja AppDirectory "D:\Server\Vikunja"
nssm start Vikunja

netsh advfirewall firewall add rule name="Vikunja" dir=in action=allow protocol=TCP localport=3456
Access and create account

Open http://localhost:3456. Register your admin account. You can enable or disable public registration in config.

SSO Integration

Vikunja supports OpenID Connect. In config.yml:

auth:
  openidconnect:
    enabled: true
    redirecturl: "http://[server]:3456/auth/openid/"
    providers:
      - name: "Google"
        authurl: "https://accounts.google.com"
        clientid: "your-client-id"
        clientsecret: "your-client-secret"
🍲 Mealie — Recipe Manager WSL2
Port: 9925 · What Mealie Is

Mealie is a self-hosted recipe manager and meal planner. Import recipes from any URL (it scrapes the recipe from cooking sites automatically), organize them into collections, plan weekly meals, and generate shopping lists. Runs via Docker in WSL2.

Create Mealie directory in WSL2 
mkdir ~/mealie && cd ~/mealie
Create docker-compose.yml 
cat > docker-compose.yml << 'EOF'
version: "3.7"
services:
  mealie:
    image: ghcr.io/mealie-recipes/mealie:latest
    container_name: mealie
    ports:
      - "9925:9000"
    environment:
      - ALLOW_SIGNUP=false
      - [email protected]
      - DEFAULT_PASSWORD=changeme
      - BASE_URL=http://[server-ip]:9925
      - DB_ENGINE=sqlite
    volumes:
      - ~/mealie/data:/app/data
    restart: unless-stopped
EOF
docker-compose up -d
Access and set up

Open http://localhost:9925. Log in with the default credentials you set → change the password → create user accounts for your household.

Import your first recipe

Click "Create Recipe" → "Scrape a Website" → paste any recipe URL from AllRecipes, NYT Cooking, Serious Eats, etc. Mealie extracts ingredients, steps, and photos automatically.

Auto-start on Windows boot 
# Add to Task Scheduler (At startup, delay 3 minutes):
wsl -d Ubuntu -- bash -c "cd ~/mealie && docker-compose up -d"

# Open firewall:
netsh advfirewall firewall add rule name="Mealie" dir=in action=allow protocol=TCP localport=9925
🍲
Meal planning + shopping list: Mealie's meal planner lets you drag recipes into a weekly calendar. It automatically generates a consolidated shopping list from all planned meals. Export as PDF or share with household members via the app.
SSO Integration

Mealie supports OpenID Connect in newer versions. Set via environment variables:

environment:
  - OIDC_AUTH_ENABLED=true
  - OIDC_SIGNUP_ENABLED=false
  - OIDC_CONFIGURATION_URL=https://accounts.google.com/.well-known/openid-configuration
  - OIDC_CLIENT_ID=your-client-id
  - OIDC_CLIENT_SECRET=your-client-secret
