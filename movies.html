📦 Transitioning Away from Plex
Why switch to Jellyfin?

Jellyfin is 100% free and open-source, requires no plex.tv cloud account for remote access, includes hardware transcoding at no cost, has zero telemetry by default, and integrates natively with Authentik SSO — something Plex cannot do. The migration is non-destructive and reversible.

plex2jellyfin — migrate watch history
# Install:
pip install plex2jellyfin

# Run migration (exports Plex watch history → imports to Jellyfin):
plex2jellyfin --plex-url http://localhost:32400 \
              --plex-token YOUR_PLEX_TOKEN \
              --jellyfin-url http://localhost:8096 \
              --jellyfin-api-key YOUR_JELLYFIN_KEY \
              --migrate watched

To find your Plex token: open Plex Web → F12 → Network tab → any API request → look for X-Plex-Token header.

💡
Run both simultaneously during transition. Plex (port 32400) and Jellyfin (port 8096) can run at the same time pointing at the same media folders — neither modifies the files. Run Jellyfin in parallel for a few weeks, then shut Plex down when you're confident.
⚙️ Step 1 — Install Jellyfin (Windows Native)
Download and install Jellyfin

Go to jellyfin.org/downloads/server → choose Windows. Run the installer as administrator. It registers itself as a Windows Service automatically — no Docker, no XAMPP, no dependencies required.

Default install path: C:\Program Files\Jellyfin\Server\
Data path:            C:\ProgramData\Jellyfin\Server\
Confirm the service is running 
# In PowerShell (admin):
Get-Service -Name JellyfinServer
# Should show: Running

# Set to start automatically on boot:
Set-Service -Name JellyfinServer -StartupType Automatic
Open firewall port 
netsh advfirewall firewall add rule name="Jellyfin" ^
  dir=in action=allow protocol=TCP localport=8096
Complete first-time setup wizard at http://localhost:8096

Create an admin account → choose language → add media libraries:

  • Movies library: D:\Media\Movies
  • TV Shows library: D:\Media\TV

These are the same folders Plex uses — no copying required. Jellyfin scans and builds its own metadata database independently.

🎮 Step 2 — Enable Hardware Transcoding (Free)

Unlike Plex, hardware transcoding is completely free in Jellyfin. This dramatically reduces CPU load when multiple users stream different resolutions.

Dashboard → Playback → Transcoding

Select the hardware acceleration type that matches your GPU:

GPUSelect this option
NVIDIA GTX/RTXNVENC
Intel (integrated/Arc)Intel QuickSync (QSV)
AMD RadeonAMF
No GPUSoftware (CPU only — acceptable for 1–2 users)
Verify it's working

Start playing a video that requires transcoding (mismatched format or resolution) → Dashboard → Active Streams → verify the stream shows the GPU encoder name (e.g. h264_nvenc) instead of libx264.

🔐 Step 3 — SSO Integration (Authentik OIDC)
🔑 This is how Jellyfin connects to your Authentik instance so users log in with their Google account — the same login they use for every other service.
Part A — Install the SSO-Auth plugin in Jellyfin
Dashboard → Plugins → Catalog → search "SSO"

Install the SSO-Auth plugin (by 9p4). After installation, restart the Jellyfin service:

Restart-Service JellyfinServer
Dashboard → Plugins → SSO-Auth → Configure

Add an OIDC provider with these settings:

Provider Name:       Authentik
OID Endpoint:        https://auth.yourdomain.com/application/o/jellyfin/
Client ID:           (from Authentik — see Part B)
Client Secret:       (from Authentik — see Part B)
Enabled:             ✅
Enable Authorization: ✅
Roles:               jellyfin-users   ← Authentik group name
Admin Roles:         jellyfin-admins  ← Authentik group name (optional)
Default Provider:    ✅ (skip local login, go straight to Authentik)
Part B — Create the Jellyfin application in Authentik
Authentik Admin → Applications → Providers → Create → OAuth2/OpenID Provider 
Name:                Jellyfin
Authorization flow:  (your default authorization flow)
Client type:         Confidential
Client ID:           (auto-generated — copy this into Jellyfin)
Client Secret:       (auto-generated — copy this into Jellyfin)
Redirect URIs:       https://jellyfin.yourdomain.com/sso/OID/redirect/Authentik
Signing Key:         (your Authentik signing key)
Create the Application 
Authentik Admin → Applications → Applications → Create
Name:           Jellyfin
Slug:           jellyfin
Provider:       (the provider you just created)
Launch URL:     https://jellyfin.yourdomain.com
Assign users / groups

In the Application's Policy/Binding tab, add a group binding for the Authentik group that should have Jellyfin access (e.g. family or jellyfin-users). Users not in this group will be denied access even if they have an Authentik account.

Part C — Caddy reverse proxy config

Jellyfin does NOT use Caddy forward_auth — SSO is handled natively by the SSO-Auth plugin. Caddy only needs to proxy the traffic:

# In your Caddyfile:
jellyfin.yourdomain.com {
    reverse_proxy localhost:8096
}
💡
No forward_auth needed. Jellyfin's SSO-Auth plugin handles authentication via OIDC directly. Users who hit jellyfin.yourdomain.com will see a "Sign in with Authentik" button and are redirected through your Authentik login flow automatically.
Part D — Test the login flow
Open an incognito window → go to jellyfin.yourdomain.com

You should see the Jellyfin login page with a "Sign in with Authentik" button. Click it → you're redirected to Authentik → authenticate with Google → redirected back to Jellyfin as a logged-in user.

Verify user was auto-created

Dashboard → Users → confirm the SSO user appeared with the correct username (should match their Authentik username). If the user doesn't appear, check the SSO-Auth plugin logs.

👥 Step 4 — User Access

With SSO enabled, new users are auto-provisioned in Jellyfin on their first login via Authentik. You don't need to manually create accounts. However, you may want to set default permissions for auto-created users:

Dashboard → Users → (click auto-created user) → set:
  ✅ Allow media playback
  ✅ Allow audio playback
  ✅ Allow subtitle management
  ❌ Allow media management (uploading/deleting — leave off for regular users)
  ❌ Allow live TV access (unless you have a tuner)
⚠️
Keep at least one local admin account. If Authentik goes down, you'll want a local Jellyfin admin account as a fallback. Create one with a strong password and store it in Vaultwarden. Don't use SSO for it.

🔄 Updates
How to update Jellyfin

Jellyfin does not auto-update. Check for new releases periodically (especially if something breaks — updates often contain fixes).

Check current version 
Dashboard → Admin → About → version shown at top

Or check GitHub: github.com/jellyfin/jellyfin/releases

Download the new installer and run it

From jellyfin.org/downloads/server. The installer handles stopping the service, replacing binaries, and restarting — your config and library database are preserved.

# Or via PowerShell (stops service, updates, restarts):
Stop-Service JellyfinServer
# (run installer manually)
Start-Service JellyfinServer
Update plugins separately

Dashboard → Plugins → check each installed plugin for updates. The SSO-Auth plugin is particularly important to keep current as Jellyfin updates can change the plugin API.

⚠️
Check plugin compatibility before updating Jellyfin. Major version updates (e.g. 10.8 → 10.9) can break plugins. Check the SSO-Auth plugin's GitHub issues for compatibility notes before upgrading.
⚠️ Potential Breakpoints — Watch For These
🔴 SSO-Auth plugin breaks after a Jellyfin update
The SSO-Auth plugin is a third-party plugin and may lag behind Jellyfin releases. If users can't log in after a Jellyfin update, check if a compatible version of SSO-Auth is available. In the meantime, users can use the local admin account as a fallback. Always keep a local admin account active.
🟡 Authentik OIDC redirect URI changes
If you ever change your domain, the Redirect URI in both Authentik (the provider config) and the SSO-Auth plugin must be updated simultaneously. Mismatched URIs cause a silent redirect_uri_mismatch error. Keep a note of the exact URI format: https://jellyfin.yourdomain.com/sso/OID/redirect/Authentik
🟡 Hardware transcoding stops working after a Windows or driver update
GPU driver updates occasionally reset NVENC/QSV settings. After any major driver or Windows update, verify transcoding still works by starting a stream that requires transcode and checking the Active Sessions dashboard. Re-enable the hardware acceleration setting if it was reset.
🟡 Library scan stops finding new media
If new files added to D:\Media\ don't appear in Jellyfin, check that the folder structure matches Jellyfin's naming conventions. Movies must be in their own subfolder: D:\Media\Movies\Movie Name (Year)\Movie Name (Year).mkv. TV must follow: D:\Media\TV\Show Name\Season 01\Show Name S01E01.mkv.
🟢 Media database becomes slow / large over time
Jellyfin's SQLite database can grow over time with cached metadata, images, and activity logs. Dashboard → Administration → Clean Up Logs, and periodically run Dashboard → Libraries → Manage Libraries → Clean Library to remove orphaned entries. No data loss occurs from these operations.

🎯 How Jellyfin Achievements Work

Jellyfin has a built-in Webhooks plugin that fires HTTP POST requests when users take actions (play, pause, finish, etc.). These webhooks post to the achievement server at port 9003, which checks the trigger against definitions.json and grants XP/achievements.

User finishes a movie in Jellyfin
  → Jellyfin fires webhook → POST http://localhost:9003/api/achievement/trigger
    { "username": "alice", "trigger": "movie_finished", "metadata": { "title": "..." } }
  → Achievement server grants "First Watch" achievement + 10 XP
  → Stored in D:\Users\alice\achievements.json
  → Appears in homepage community feed
🔌 Webhook Setup
Step 1 — Install the Webhooks plugin
Dashboard → Plugins → Catalog → search "Webhooks" → Install → Restart Jellyfin
Step 2 — Configure webhook endpoints
Dashboard → Plugins → Webhooks → Add Webhook

Name:          Achievement Server
URL:           http://localhost:9003/api/achievement/trigger
Events:        ✅ Playback Start
               ✅ Playback Stop
               ✅ Item Marked Played  ← most reliable "finished" signal
               ✅ User Created

Template (JSON body):
{
  "username": "{{NotificationUsername}}",
  "trigger":  "{{Event}}",
  "metadata": {
    "itemId":    "{{ItemId}}",
    "title":     "{{Name}}",
    "mediaType": "{{ItemType}}",
    "year":      "{{Year}}",
    "seriesName":"{{SeriesName}}"
  }
}
💡
The NotificationUsername variable in Jellyfin webhooks returns the Jellyfin username — which should match the Authentik username if SSO is set up correctly. Verify this is consistent when testing.

👤 Per-User Data
What Jellyfin stores per user

Unlike most services on this server, Jellyfin keeps all per-user data inside its own database (jellyfin.db) rather than in D:\Users\[username]\. This includes:

  • Watch history and playback progress
  • Continue Watching queue
  • Personal ratings and favorites
  • Subtitle and audio stream preferences
  • User-level parental control settings
💡
No per-user folder needed. Jellyfin's database handles all user data internally. The media files in D:\Media\ are shared and read-only — Jellyfin never writes to them. Backup C:\ProgramData\Jellyfin\Server\data\jellyfin.db regularly to preserve all watch history.
How much does each user's data take up?

Jellyfin's per-user data is extremely lightweight — a few kilobytes of database rows per user, per show or movie. Even a power user who has watched hundreds of items will have under 5 MB of personal data stored in the database. The bulk of Jellyfin's storage is metadata (artwork, NFO files) which is shared across all users.

# Simple backup script (run via Task Scheduler, daily):
# Backs up the Jellyfin database to D:\Backups\Jellyfin\
$src = "C:\ProgramData\Jellyfin\Server\data\jellyfin.db"
$dst = "D:\Backups\Jellyfin\jellyfin-$(Get-Date -f yyyy-MM-dd).db"

🛠️ Common Issues
❌ "Sign in with Authentik" button doesn't appear on the login page
The SSO-Auth plugin isn't installed or isn't configured. Check:
  • Dashboard → Plugins → confirm SSO-Auth is listed and enabled
  • Dashboard → Plugins → SSO-Auth → confirm the provider is configured and "Enabled" checkbox is ticked
  • Restart Jellyfin after any plugin config change: Restart-Service JellyfinServer
❌ SSO login redirects to Authentik but then shows "redirect_uri_mismatch" error
The redirect URI in Authentik doesn't match what Jellyfin is sending. Check the exact URI format: 
Required format: https://jellyfin.yourdomain.com/sso/OID/redirect/Authentik
                                                              ↑ must match your provider name exactly (case-sensitive)
Update both: Authentik provider → Redirect URIs, and SSO-Auth plugin config.
❌ Jellyfin isn't accessible from outside the network
Check in order:
  • Is the Jellyfin service running? → Get-Service JellyfinServer
  • Is Caddy running and proxying correctly? → check jellyfin.yourdomain.com in browser
  • Is the Cloudflare DNS record pointing to your server's public IP?
  • Is port 443 open on your router and forwarded to the server running Caddy?
Note: Jellyfin's port 8096 does NOT need to be forwarded — Caddy handles all external traffic on 443.
⚠️ Video stutters or buffers — especially for remote users
Almost always a transcoding issue. Check Dashboard → Active Streams during playback:
  • Direct Play = client can play the file natively (best — no server load)
  • Direct Stream = container remux only (good — minimal CPU)
  • Transcode = full re-encode (check that hardware transcoding is active — codec should show h264_nvenc not libx264)
If transcoding to CPU (libx264), re-enable hardware acceleration: Dashboard → Playback → Transcoding.
⚠️ New movies/shows added to D:\Media\ don't appear in Jellyfin
  • Trigger manual scan: Dashboard → Libraries → [library] → Scan Library
  • Check folder naming matches Jellyfin's expected format: Movie Name (Year)\Movie Name (Year).mkv for movies
  • Check for hidden files or permissions issues on the new files (Jellyfin must have read access)
  • Enable realtime monitoring: Dashboard → Libraries → [library] → check "Enable realtime monitoring"
⚠️ Metadata / artwork is wrong for a movie or show
Right-click the item → Identify → search by name or IMDB/TMDB ID → select the correct match → Save. If artwork is wrong, right-click → Edit Images → Search for artwork to pick from the database.
❌ Jellyfin service won't start after an update
Check the log for errors: 
Get-Content "C:\ProgramData\Jellyfin\Server\log\jellyfin-$(Get-Date -f yyyyMMdd).log" -Tail 50
Common causes: a plugin is incompatible with the new version (remove it and retry), or the database needs migration (usually done automatically — check for error messages about schema).
💡 How to generate a support bundle for debugging
Dashboard → Help → click Generate Support Bundle. This creates a ZIP with logs, config info, and system details. Don't share it publicly as it contains server paths and usernames.
