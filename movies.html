📦 Transitioning Away from Plex
Why switch to Jellyfin?

Jellyfin is 100% free and open-source, requires no plex.tv cloud account, includes hardware transcoding at no cost, has zero telemetry by default, and integrates natively with Authentik SSO — something Plex cannot do. The migration is non-destructive and reversible.

⚙️ Step 1 — Install Jellyfin (Windows Native)
Download and install Jellyfin

Go to jellyfin.org/downloads/server → choose Windows. Run the installer as administrator. It registers itself as a Windows Service automatically — no Docker, no XAMPP, no extra dependencies.

Default install path:  C:\Program Files\Jellyfin\Server\
Data path:             C:\ProgramData\Jellyfin\Server\
Confirm the service is running and set to auto-start 
# In PowerShell (admin):
Get-Service -Name JellyfinServer        # should show: Running
Set-Service -Name JellyfinServer -StartupType Automatic
Open firewall port 
netsh advfirewall firewall add rule name="Jellyfin" dir=in action=allow protocol=TCP localport=
Complete first-time setup wizard at http://localhost:

Create an admin account (strong password, store in Vaultwarden — don't use SSO for the admin account) → choose language → add media libraries:

  • Movies library:
  • TV Shows library:

These are the same folders Plex uses — no copying needed. Jellyfin scans and builds its own metadata database independently.

🎮 Step 2 — Enable Hardware Transcoding (Free)

Unlike Plex, hardware transcoding is completely free in Jellyfin. Enable it to prevent CPU bottlenecks when multiple users are streaming simultaneously.

Go to Dashboard → Playback → Transcoding

Select your hardware acceleration type:

  • NVIDIA GPU: select NVENC — best performance
  • Intel CPU/iGPU: select Intel QuickSync (QSV)
  • AMD GPU: select AMF
Enable the recommended codecs

Check: H.264 (8-bit), H.264 (10-bit), H.265/HEVC (8-bit), H.265 (10-bit), AV1 (if supported), VP9. Leave audio codecs at defaults.

Verify hardware transcoding is working

Start a stream that requires transcoding → Dashboard → Active Streams. The codec should show h264_nvenc (or equivalent), not libx264. If it shows libx264, hardware transcoding isn't engaged.

🔄 How to Update Jellyfin
Last confirmed working version:

Jellyfin does not auto-update. Check releases periodically, especially after something breaks — updates often contain fixes.

Back up the database first 
Stop-Service JellyfinServer
Copy-Item "C:\ProgramData\Jellyfin\Server\data\jellyfin.db" `
  "E:\Backups\Jellyfin\jellyfin-pre-update-$(Get-Date -f yyyy-MM-dd).db"
Download and run the new installer

From jellyfin.org/downloads/server. The installer stops the service, replaces binaries, and restarts. Config and library database are preserved.

Update plugins separately

Dashboard → Plugins → check each installed plugin for updates. The SSO-Auth plugin is especially important to keep current — major Jellyfin version updates can change the plugin API and break SSO.

Verify the update 
Get-Service JellyfinServer   # confirm Running
# Open http://localhost: — confirm version in Dashboard → About
# Test SSO login with a non-admin user account
# Test a transcode stream — confirm hw codec still shows in Active Streams
⚠️
Check plugin compatibility before major version updates. Major version bumps (e.g. 10.8 → 10.9) can break plugins. Check the SSO-Auth plugin GitHub for a compatible release before upgrading Jellyfin. If SSO breaks after an update, the admin password fallback gets you back in.
⚠️ Known Breakpoints
  • SSO-Auth plugin / Jellyfin version mismatch: Major Jellyfin updates sometimes break the SSO-Auth plugin. Always check plugin compatibility before updating. Fallback: admin password login still works.
  • Hardware transcoding after driver updates: GPU driver updates can occasionally break NVENC/QSV. If transcoding suddenly uses CPU (libx264), go to Dashboard → Playback → Transcoding and re-save the hardware acceleration settings to force a re-detect.
  • New media not appearing after adding files: Jellyfin's realtime filesystem monitoring can miss new files added by Radarr/Sonarr. If content doesn't appear within 30 minutes, trigger a manual scan.
  • Database corruption after power loss: The SQLite database (jellyfin.db) can corrupt if the server loses power mid-write. Maintain daily automated backups — see the Data Storage tab.

🏆 How Achievements Are Triggered

Jellyfin fires webhook events when a user stops playback. The achievement server receives these, checks definitions.json, and unlocks matching achievements. Opt-out is respected — the server silently skips opted-out users.

User finishes watching a movie / episode
  → Jellyfin Webhooks plugin fires PlaybackStop event
  → POST http://localhost:/api/achievement/trigger
    { "username": "alice", "trigger": "jellyfin_watched", "metadata": {...} }
  → Achievement server checks opt-out → checks definitions.json
  → Grants achievement + XP if criteria met
  → Stored in \alice\achievements.json
  → definitions.json at: \definitions.json

🤖 Feeding the AI Recommendation Engine

Movies and episodes watched to completion (85%+ of runtime) are reliable taste signals for the recommendation engine. Partially-watched items (<30%) are negative signals and should be recorded as "signal":"abandoned". The recommendation engine uses this to avoid suggesting similar content.

Note: The Jellyfin webhook fires on every playback stop, including pauses. The rec_server.py endpoint should check the PlaybackPositionTicks metadata against the total item duration to determine the actual percentage watched before deciding the signal quality.

Record to rec_server.py
# POST http://localhost:/api/history/record
{
  "user":   "username",
  "type":   "movie",         # or "episode" for TV
  "id":     "jellyfin_item_id",
  "title":  "Movie Title",
  "signal": "finished",      # or "abandoned"
  "metadata": {
    "percent_complete": 92,
    "genre": ["action", "sci-fi"]
  }
}

👤 Per-User Data
What Jellyfin stores per user (Pattern A — all in service DB)

Unlike most services on this server, Jellyfin stores all per-user data inside its own database (jellyfin.db) — not in \[username]\. This includes:

  • Watch history and playback progress (resume points)
  • Continue Watching queue
  • Personal ratings and favorites
  • Audio track and subtitle preferences
  • Parental control settings (per-user)
💡
No per-user folder needed. The media files in are shared and read-only — Jellyfin never writes to them. Back up C:\ProgramData\Jellyfin\Server\data\jellyfin.db daily to preserve all watch history.
Storage footprint per user

Jellyfin's per-user data is extremely lightweight — a few kilobytes of database rows per user per item. A power user with hundreds of watched items will have under 5 MB of personal data in the database. The bulk of Jellyfin's storage is the shared metadata cache (artwork, NFO files).

🛠️ Common Issues
❌ Jellyfin service won't start after an update
Check the daily log for startup errors: 
Get-Content "C:\ProgramData\Jellyfin\Server\log\jellyfin-$(Get-Date -f yyyyMMdd).log" -Tail 50
Common causes:
  • Incompatible plugin after a major version update — remove the offending plugin from C:\ProgramData\Jellyfin\Server\plugins\ and retry
  • Database schema migration failed — check for "migration" errors in the log. Restore from last known-good backup if needed.
❌ "Sign in with Authentik" button doesn't appear on the login page
The SSO-Auth plugin isn't installed, isn't configured, or Jellyfin needs a restart after plugin config changes.
  • Dashboard → Plugins → confirm SSO-Auth is listed and the Enabled checkbox is ticked
  • Dashboard → Plugins → SSO-Auth → confirm the Authentik provider is configured
  • Restart after any plugin config change: Restart-Service JellyfinServer
  • If the button appears but login fails: check the redirect URI in Authentik matches exactly (case-sensitive)
❌ SSO login shows "redirect_uri_mismatch" error in Authentik
The redirect URI in Authentik doesn't match what Jellyfin is sending. The exact format must be: 
https://jellyfin./sso/OID/redirect/Authentik
                                                  ↑ must match your provider name exactly (case-sensitive)
Update both: Authentik provider → Redirect URIs, and the SSO-Auth plugin config in Jellyfin.
⚠️ Video stutters or buffers — especially for remote users
Almost always a transcoding issue. Check Dashboard → Active Streams during playback:
  • Direct Play = client plays natively (best — zero server load)
  • Direct Stream = container remux only (good — minimal CPU)
  • Transcode = re-encode. Codec should show h264_nvenc not libx264. If it shows libx264, hardware transcoding is off — re-enable: Dashboard → Playback → Transcoding.
⚠️ New movies / shows added to the library don't appear in Jellyfin
  • Trigger a manual scan: Dashboard → Libraries → [library] → Scan Library
  • Check folder naming: Movie Name (Year)\Movie Name (Year).mkv for movies; Show Name\Season 01\Show Name S01E01.mkv for TV
  • Check file permissions — Jellyfin must have read access to the new files
  • Enable realtime monitoring: Dashboard → Libraries → [library] → check "Enable realtime monitoring"
⚠️ Metadata or artwork is wrong for a specific movie or show
Right-click the item → Identify → search by name or IMDB/TMDB ID → select the correct match → Save. If artwork is wrong, right-click → Edit Images → search for artwork to pick from the database.
💡 Jellyfin not accessible from outside the network
Check in order:
  • Is Jellyfin running? → Get-Service JellyfinServer
  • Is Caddy running and proxying correctly? → check in browser
  • Is Tailscale running on the client device? Check system tray or phone status bar.
  • Is the DNS record for pointing at the hub Tailscale IP?
  • Is Caddy running? → Get-Service Caddy or check the Caddy service in services.msc
  • Smart TV users (LG WebOS, Tizen, Roku, Fire TV): connect to instead — TV apps cannot use Tailscale and use Jellyfin-native passwords, not Google SSO. Port 8096 must be forwarded on your router.
💡 How to generate a support bundle for debugging

🔌 Tool 1 — LetterboxdSync Plugin (Gizmo091 fork) RECOMMENDED
What it is

A Jellyfin plugin that automatically syncs your watch history from Jellyfin to your Letterboxd diary. This is the currently maintained fork — the original by danielveigasilva was archived in November 2025 after last being pushed in March 2025. The awesome-jellyfin list updated its reference to this fork in March 2026.

Works by checking which films you've marked as watched in Jellyfin and logging them to your Letterboxd diary on a 24-hour schedule. Optionally syncs Jellyfin favorites as Letterboxd likes. One Letterboxd account per Jellyfin user.

🔌 Tool 2 — jellyfin-plugin-letterboxd (builtbyproxy) NEWER / FEWER STARS

Good to watch but not the first choice for a set-and-forget install.

🐳 Tool 3 — letterboxd-jellyfin Docker Sidecar (MathisVerstrepen)
What it is

A Python Docker container that syncs your Letterboxd watchlist into Jellyfin and Radarr. It scrapes your watchlist, adds unwatched films to a Jellyfin collection, queues missing films in Radarr for download, and auto-removes films from the collection once you've watched them in Jellyfin. Runs continuously on a schedule you set.

This is the tool to use if you want "add to Letterboxd watchlist → Radarr downloads it → Jellyfin shows it in a personal collection → marks done after watching." A complete pipeline.

⚠️
This tool scrapes Letterboxd watchlists. Unlike the Letterboxd RSS sync in the torrent-streaming page (which uses public RSS feeds), this scrapes your watchlist pages. If you use large proxy lists or run it very frequently, Letterboxd may rate-limit you. For personal use (1–2 users, default 10-minute interval), this is fine.
Generate a Jellyfin API key 🔐

Dashboard → API Keys → Add API Key → name it "letterboxd-jellyfin". Copy the key.

Create config.yaml

Clone the repo or grab config.yaml.example from GitHub and save to D:\Server\LetterboxdJellyfin\config.yaml:

system:
  sync_interval: 30  # minutes

jellyfin:
  url: "http://localhost:8096"
  api_key: "YOUR_API_KEY_HERE"   # from Dashboard → API Keys

radarr:
  url: "http://localhost:7878"
  api_key: "YOUR_RADARR_API_KEY"
  root_folder_path: "D:\\Media\\Movies"
  quality_profile_id: 4          # get from GET /api/v3/qualityprofile

letterboxd:
  max_concurrent_requests: 5

users:
  - letterboxd_username: "yourusername"
    jellyfin_username: "yourusername"
    collection_name: "Watchlist"
Run via Docker Compose (on The Muscle / WSL2) 
version: "3.8"
services:
  letterboxd-jellyfin:
    image: ghcr.io/mathisverstrepen/letterboxd-jellyfin:latest
    container_name: letterboxd-jellyfin
    restart: unless-stopped
    volumes:
      - D:/Server/LetterboxdJellyfin:/app/config

Note: api_key here is the Jellyfin API key you generated — nothing to do with your Authentik/OIDC token. The container talks to Jellyfin's REST API at port 8096 directly.

🔑 Generating Jellyfin API Keys for These Tools

All non-plugin tools need a Jellyfin API key. Create one per tool so you can revoke them individually:

🔐 Create API key (admin only) 
1. Open Jellyfin admin: http://localhost:8096 (or jellyfin.ixo.lol on LAN)
2. Dashboard → API Keys → + Add API Key
3. Name it descriptively: "letterboxd-jellyfin", "auto-collections", etc.
4. Copy the generated key immediately — it's only shown once

These keys authenticate directly to Jellyfin's REST API. They are completely separate from your Authentik/OIDC session. Revoking a key here (Dashboard → API Keys → 🗑) immediately breaks any tool using it — useful if a key is compromised.

💡 Recommended Setup for This Server
For auto-diary (set and forget)

Install Tool 1 (Gizmo091 fork) as a Jellyfin plugin. Enter each user's Letterboxd credentials once. Films log to Letterboxd automatically every 24 hours. No Docker, no scripts, no maintenance beyond credential refresh if Letterboxd changes its site.

For watchlist → Radarr pipeline

The RSS-based sync in torrent-streaming.html → Letterboxd Sync tab already covers this for List A (local) and List B (debrid). For a richer Jellyfin-collection view of the watchlist, add Tool 5 (Jellyfin-Auto-Collections) pointed at your Letterboxd watchlist URL.

For one-shot history export

If you ever want to bulk-import your entire Jellyfin history to Letterboxd (e.g. when first setting up), use Tool 4 (zianwar CLI) as a one-time run on The Muscle. Don't schedule it — it opens a browser window.

Browser convenience

Install Tool 6 (Firefox extension) on any browser used to browse Jellyfin. Zero setup, zero maintenance. Nice for checking ratings without leaving Jellyfin.
