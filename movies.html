You currently run Plex. This tab covers getting it stable on proper storage, and — if you ever want to switch — a full comparison with Jellyfin and a step-by-step migration guide.

🐉 V1: Keep Plex running on internal storage. Jellyfin migration is optional — read the comparison first. Both can run simultaneously during a transition period.
🟡 Plex — Current Setup
Getting Plex Stable on Internal Storage
Update library paths after storage migration

Settings → Libraries → edit each library → change the folder path from the old USB letter (e.g. F:\Media\TV) to the new internal path (e.g. D:\Media\TV). Plex re-matches metadata without re-downloading it.

Confirm Plex Media Server auto-starts

Services (services.msc) → "Plex Media Server" → set to Automatic. Test by rebooting.

Invite users via plex.tv

Settings → Manage Library Access → Invite by email. Each user gets their own watch history, continue watching, and recommendations. No SSO needed — Plex accounts handle this natively.

Plex Hardware Transcoding (Optional)

Plex Free only software-transcodes, which is CPU-heavy. Plex Pass ($40/yr or $120 lifetime) unlocks hardware transcoding via your GPU — dramatically reduces CPU load when multiple people stream different resolutions. If you have a modern GPU (GTX 1000+ or RX 400+), this is worth it.

Settings → Troubleshooting → Generate Support Bundle
(to verify hardware transcoding is working after enabling Plex Pass)

🟢 Jellyfin Setup — Windows Native FOSS · No Docker
Port: 8096

Jellyfin runs natively on Windows with its own installer. No XAMPP, no dependencies. It installs as a Windows Service.

Download Jellyfin installer

From jellyfin.org/downloads/server — choose Windows. Run the installer as admin. It registers as a Windows Service automatically.

First-time setup at http://localhost:8096

Create admin account → select language → add media libraries (point to your existing D:\Media\Movies, D:\Media\TV folders — the same ones Plex uses). Jellyfin can run alongside Plex on different ports without conflicts.

Open firewall 
netsh advfirewall firewall add rule name="Jellyfin" dir=in action=allow protocol=TCP localport=8096
Enable hardware transcoding (free!)

Dashboard → Playback → Transcoding → enable "Intel QuickSync / Nvidia NVENC / AMD AMF" depending on your GPU. This is free in Jellyfin — no subscription required.

Add users

Dashboard → Users → Add User. Each user gets separate watch history, "Continue Watching," and playback progress. Unlike Plex, users don't need a cloud account — just a username and password on your server.

Jellyfin + SSO (Authentik / Google OAuth)

Install the SSO Authentication Plugin from the Jellyfin plugin catalog. This adds OpenID Connect support, letting users log into Jellyfin with Google or via your Authentik instance. See the SSO tab for Authentik setup.

# In Jellyfin: Dashboard → Plugins → Catalog → search "SSO"
# Install "SSO-Auth" plugin → restart → configure OIDC provider

What transfers and what doesn't

Plex2Jellyfin migration tool
# Install Python tool:
pip install plex2jellyfin

# Run migration (exports Plex watch history and imports to Jellyfin):
plex2jellyfin --plex-url http://localhost:32400 \
              --plex-token YOUR_PLEX_TOKEN \
              --jellyfin-url http://localhost:8096 \
              --jellyfin-api-key YOUR_JELLYFIN_KEY \
              --migrate watched

Get your Plex token: open Plex web → F12 → Network tab → any API request → look for X-Plex-Token header.

💡
Run both simultaneously during transition. Plex and Jellyfin can run at the same time pointing at the same media folders — neither modifies the files. Run Jellyfin in parallel for a few weeks until you're confident, then shut down Plex.
