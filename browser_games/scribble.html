🎨 V1 Goal: Scribble.rs running in Docker, accessible at scribble. via Tailscale, gated by Authentik forward_auth. A full round of Pictionary playable from two different devices.
🐳 Step 1 — Docker Compose

Scribble.rs ships a Docker image. Add it to your stack:

scribble:
  image: ghcr.io/scribble-rs/scribble.rs:latest
  ports:
    - "8880:8080"
  environment:
    - PORT=8080
  restart: unless-stopped

The app listens on 8080 inside the container, mapped to 8880 on the host. Adjust the host port if there's a conflict.

🌐 Step 2 — Caddy Reverse Proxy
scribble.{$DOMAIN} {
  forward_auth authentik:9000 {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy scribble:8080
}

Register it as an Application + Provider in Authentik. Users who are on Tailscale and have an Authentik session go straight to the Scribble.rs lobby.

🎨 What Is This?

Scribble.rs is a self-hostable web Pictionary. One player gets a word and draws it on a shared canvas; other players type guesses into a chat. The faster you guess, the more points you earn. The drawer also gets points for each correct guess.

Lobbies support 2–12 players, configurable round counts, drawing time, and language. Custom word lists let you replace the built-in dictionary entirely — useful for themed sessions. The UI is clean and minimal, and the whole game works in a standard browser tab.

The server is written in Go, ships as a tiny Docker image or a single static binary, and has no database — all state lives in memory. Rooms are ephemeral and vanish when the last player disconnects.

🔌 Protocol

Drawing strokes are synced over WebSocket. The server is the authority on scores, turn order, and the word list. All players must be able to reach the server — Tailscale handles this for all enrolled devices. No external relay.
