📁 Step 1 — Get the Web Build

Mindustry provides an official web build on GitHub Releases. Download the latest Mindustry.zip and extract it:

cd D:\Server\BrowserGames
# Download from GitHub Releases:
# https://github.com/Anuken/Mindustry/releases/latest
# Look for Mindustry.zip (the HTML/web build)

Expand-Archive -Path Mindustry.zip -DestinationPath mindustry
💡
The web build is in the GitHub Releases assets, not the repo source. Look for Mindustry.zip — not the desktop JAR. Entry point is index.html at the root of the extracted folder.

🔐 Step 3 — Access Control

Served under the Browser Games hub. Inherits the hub's forward_auth — Tailscale + Authentik SSO required.

🐳 Optional — Dedicated Multiplayer Server

For multiplayer co-op, Mindustry supports a headless dedicated server. Create ~/services/mindustry/docker-compose.yml:

services:
  mindustry:
    image: tyrannas/mindustry-server:latest
    container_name: mindustry
    restart: unless-stopped
    ports:
      - "6567:6567/tcp"
      - "6567:6567/udp"
    volumes:
      - mindustry_data:/config

volumes:
  mindustry_data:
⚠️
Browser clients cannot connect to the dedicated server via WebSocket — the native Mindustry client is required for multiplayer. Browser play is singleplayer only.
✅ Mindustry browser saves are stored in the browser's IndexedDB. Progress persists across sessions in the same browser.

🎮 What Is This?

Mindustry is a factory-building tower defence RTS by Anuken. The core loop is building resource extraction and processing chains — drills, conveyors, smelters, and fabricators — to supply turrets with ammunition and repair units. Enemy waves attack your core at intervals; survival requires both a functional factory and a well-designed defence grid.

The game has a substantial campaign spanning multiple planets, a sandbox mode, and a custom map editor. The automation depth rivals dedicated factory games like Factorio, scaled to fit alongside the real-time combat element. Strategy involves optimising throughput, managing power grids, and adapting defences to different enemy types.

The browser build via LibGDX's HTML5 backend is fully functional for singleplayer. Multiplayer co-op requires the native desktop client connecting to a dedicated server — browser clients cannot join dedicated servers.

📖 How to Play
  • Start with the Campaign to follow the tutorial and story missions
  • Place drills on ore deposits to extract resources onto conveyors
  • Route conveyors to smelters and fabricators to process raw materials
  • Supply turrets with the correct ammunition type via conveyors or containers
  • Defend against waves — reinforce weak points between attacks
  • Unlock new units and buildings by completing campaign objectives

🔧 Dedicated Server Management
# Check status:
docker compose -f ~/services/mindustry/docker-compose.yml ps

# View logs (server console):
docker compose -f ~/services/mindustry/docker-compose.yml logs -f

# Send server command (e.g. load a map):
docker exec -it mindustry sh -c "echo 'host ' > /proc/1/fd/0"

# Update server:
docker compose -f ~/services/mindustry/docker-compose.yml pull
docker compose -f ~/services/mindustry/docker-compose.yml up -d
💡
Players connect via the native client: Multiplayer → Add Server → enter the server's Tailscale IP and port 6567.
