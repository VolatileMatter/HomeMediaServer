⚠️ Architecture Note — C Backend Requires Compilation
⚠️
Gamble-Plinko is the only game in the hub with a C backend. The server must be compiled from source using GCC before it can run. It uses libmicrohttpd for HTTP serving. This is more involved than a Docker pull — plan for a 20–30 minute setup including dependency installation and compilation. All credits are virtual only.
🔧 Step 1 — Install Dependencies (WSL2)
sudo apt update
sudo apt install -y gcc libmicrohttpd-dev make
📁 Step 2 — Clone & Compile
cd ~/services
git clone https://github.com/Carrtik/Gamble-Plinko.git plinko
cd plinko

# Compile the C server (check Makefile or README for exact command):
make
# or if no Makefile:
gcc -o plinko-server server.c -lmicrohttpd
💡
Check the repo README for the exact compile command and any additional linker flags. The -lmicrohttpd flag links the HTTP library. If the build fails, verify libmicrohttpd-dev is installed and the header paths are correct.
🐳 Step 3 — Run as a Service

Wrap the compiled binary in a Docker container for consistent management, or run it directly under NSSM in WSL2:

# Option A — run directly:
./plinko-server --port 8760

# Option B — Docker (create a Dockerfile if the repo doesn't include one):
# FROM ubuntu:22.04
# RUN apt-get update && apt-get install -y libmicrohttpd-dev
# COPY plinko-server /usr/local/bin/
# CMD ["plinko-server", "--port", "8080"]

🎮 What Is This?

Plinko is a probability demonstration game popularised by The Price Is Right. A ball is dropped from the top of a triangular grid of pins. At each pin, the ball randomly deflects left or right. After bouncing through all rows of pins, it lands in one of several buckets at the bottom, each carrying a different multiplier. The bucket distribution follows a binomial distribution — centre buckets are most likely, edges least likely.

What makes this implementation interesting technically is the C backend using libmicrohttpd — a lightweight embedded HTTP server library. The physics simulation runs client-side in the browser via HTML5 Canvas, while the C server handles virtual credit state and game validation. It's the most unusual technology stack in the entire hub.

The visual appeal of watching the ball bounce unpredictably between pins is genuinely compelling — each drop feels uncertain even though the statistics are well-understood.

📖 How to Play
  • Choose a bet size from your virtual credit balance
  • Drop a ball from the top of the pin board — click or tap to release
  • Watch the ball bounce through the pins via physics simulation
  • The bucket it lands in determines your multiplier — centre is safest, edges pay most
  • Repeat until you're satisfied or the variance claims your balance
