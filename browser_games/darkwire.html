🐳 Step 1 — Docker Compose

Create ~/services/darkwire/docker-compose.yml:

services:
  darkwire:
    image: darkwire/darkwire.io:latest
    container_name: darkwire
    restart: unless-stopped
    ports:
      - "8800:3000"
docker compose -f ~/services/darkwire/docker-compose.yml up -d
💡
Darkwire stores no message data — there is nothing to back up. The server only relays encrypted WebSocket messages; it cannot read them.
🌐 Step 2 — Caddy Configuration

Add to :

chat.{$DOMAIN} {
  forward_auth  {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy localhost:8800
}

💬 What Is This?

Darkwire.io is an end-to-end encrypted group chat tool that runs entirely in the browser. It uses the Web Cryptography API to encrypt all messages client-side before they're relayed through the server — the server sees only ciphertext and has no ability to read conversations. Rooms are ephemeral: they exist only while at least one person is connected, and all history is discarded when the last participant leaves.

On this server, Darkwire serves a practical purpose: it gives players a place to coordinate game sessions without needing to use an external service or install an app. Scheduling a Suroi match, sharing a Quizmaster room link, or discussing an Ancient Beast strategy — all without leaving the Tailnet.

No accounts are required. Create a room, copy the URL, and share it. Anyone with the link and a Tailscale connection can join.

📖 How to Use
  • Open chat. — you'll be placed in a new room automatically
  • Copy the room URL and share it with others on the Tailnet
  • Type to chat — messages are encrypted before being sent
  • File sharing and image sending are supported in some versions
  • Leave or close the tab to disconnect; when the last person leaves, the room is gone
⚠️ Known Limitations
IssueDetail
No message historyAll messages are lost when the room closes. Not suitable for persistent communication — use this for live coordination only.
No usernamesParticipants appear with random identifiers unless the version supports custom names. Check the current UI.
Ephemeral roomsRoom URLs are permanent only while the room exists. If everyone leaves and you reload, you'll get a new room — the old one is gone.
