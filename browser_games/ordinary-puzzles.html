📁 Step 1 — Get the Web Build

Ordinary Puzzles is a React Native for Web project. It requires a build step to produce static files.

cd D:\Server\BrowserGames
git clone https://github.com/mmazzarolo/ordinary-puzzles-app.git ordinary-puzzles
cd ordinary-puzzles
npm install
npx expo export --platform web

The built output lands in dist/. Serve that folder as the game directory.

💡
Node.js 16+ required. After building, the dist/ folder is fully static — no Node.js runtime needed to serve it. Update the folder in index.json to ordinary-puzzles/dist.

🎮 What Is This?

Ordinary Puzzles is a minimalistic implementation of Picross — also known as nonograms, Hanjie, or Griddlers. The game presents a grid with numbered clues along each row and column. The numbers describe runs of filled cells in that row or column — for example, a clue of "3 1" means there's a run of 3 filled cells followed by a gap, then a run of 1 filled cell. Solving the puzzle means using logic to determine which cells to fill, ultimately revealing a pixel-art image.

Picross is one of the most elegant puzzle formats ever designed — solutions are always logically deducible without guessing (in well-constructed puzzles), and the satisfaction of crossing off a row with a complex clue is deeply rewarding. The Nintendo Picross series has been running since 1995 for exactly this reason.

Ordinary Puzzles has a clean, calm aesthetic with a built-in puzzle library spanning small grids suitable for beginners and larger, more intricate puzzles for experienced solvers.

📖 How to Play
  • Each row and column has a clue — a sequence of numbers indicating runs of filled cells
  • A clue of 3 means exactly 3 consecutive filled cells somewhere in that line
  • A clue of 2 1 means a run of 2, a gap of at least 1, then a run of 1
  • Tap/click a cell to fill it; tap again to mark it as definitely empty (X)
  • Use logic to determine where runs must or cannot be — never guess in a well-designed puzzle
  • Complete the grid to reveal the hidden pixel-art image
Controls
Tap / click cell Fill cell (first tap) / mark empty (second tap) / clear (third) Drag Fill or clear multiple cells in one gesture
⚠️ Known Limitations
IssueDetail
Build step requiredRequires npx expo export --platform web before serving. Re-run after git pull.
Progress is browser-localPuzzle completion tracked in localStorage — tied to the specific browser.
Fixed puzzle libraryThe puzzle set is baked into the build. Adding new puzzles requires editing source and rebuilding.
🏆 Ordinary Puzzles tracks completed puzzles locally. No time-based scoring or shared leaderboard.
🏆 Progress Tracking

The game marks puzzles as completed in localStorage. There is no time tracking or score — the completion itself is the reward. For competitive play among friends, comparing which puzzles have been solved or racing to completion on a specific puzzle works informally.
