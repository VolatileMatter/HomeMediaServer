⚠️ Complexity Warning
⚠️
Lila is a large, complex application. It is the entire lichess.org backend — not a simple game server. Self-hosting requires running Scala (lila), MongoDB, Redis, Stockfish (lila-fishnet), and a search engine (lila-search). Plan for a multi-hour setup and a non-trivial ongoing maintenance burden. The community maintains a Docker-based setup tool (lila-docker) that greatly simplifies this; use it rather than the raw repo.
🐳 Step 1 — lila-docker

Use the official lila-docker project — it orchestrates all required services automatically:

cd ~/services
git clone https://github.com/lichess-org/lila-docker.git lichess
cd lichess
./lila-docker setup

The setup wizard will ask which optional services to include (Stockfish analysis, opening explorer, etc.). For a minimal private server, decline optional services to reduce resource usage.

💡
lila-docker uses its own internal nginx. Point Caddy at its exposed port rather than proxying individual service ports. Check docker compose ps after setup to find which host port the frontend is exposed on.
🌐 Step 2 — Caddy Configuration

Add to once lila-docker is running and you know its exposed port:

chess.{$DOMAIN} {
  forward_auth  {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy localhost:LILA_PORT
}
⚠️
Lichess uses WebSockets heavily for real-time board updates. Caddy handles this automatically, but the lila-docker nginx may also handle WebSocket upgrades internally — check that WebSocket connections aren't being double-proxied if you encounter connection issues.
👤 Step 3 — Create Accounts

A self-hosted Lichess instance has its own account system separate from Authentik. Each player needs to register an account on the Lichess instance directly.

💡
Authentik forward_auth gates access to the subdomain (only Tailscale members can reach it), but Lichess manages its own logins internally. Players will have a Lichess username separate from their Authentik username.

♟️ What Is This?

Lila is the open-source codebase that powers lichess.org, one of the largest free chess platforms in the world. Self-hosting it gives your friend group a private chess server with the full feature set: rated and casual games, all major variants (Chess960, Crazyhouse, Three-check, King of the Hill, Antichess, and more), a puzzle trainer, a game analysis board backed by Stockfish, tournaments, and a study tool for annotating games.

The platform is entirely libre — no ads, no subscriptions, no paywalls. Running a private instance means ratings and history accumulate over time, giving the friend group an ongoing competitive record. The built-in puzzle trainer is genuinely excellent for improving tactical vision.

Be aware that Lila is a large application built for massive scale. On a home server it runs fine for a small group, but it consumes more RAM than any other game in this hub. Allocate at least 4GB to the Docker stack.

⚠️ Known Limitations
IssueDetail
Resource usageMongoDB + Redis + Scala JVM requires ~4GB RAM minimum for comfortable operation.
Separate accountsLichess has its own account system. Authentik SSO only gates network access — players still register directly on the Lichess instance.
Update complexityUpdating requires rebuilding the Docker stack. Test on a non-critical window; in-progress games may be lost during restarts.
EmailA self-hosted instance won't have email configured by default. Password resets require admin console access.
👥 All gameplay is multiplayer — 1v1 games, tournaments, and team matches. Solo use (puzzles, analysis) is also fully supported.

# Check all service status:
cd ~/services/lichess
docker compose ps

# View lila application logs:
docker compose logs -f lila

# Restart the full stack:
docker compose restart

# Update (rebuild from latest source):
git pull
docker compose up -d --build
