📋 Hub Registration

The Flash Arcade is pre-registered in browser_games/index.json as an externalHub entry. When a user clicks Play from the Browser Games lobby, it opens the Flash Arcade at flash. in a new tab rather than loading inside an iframe.

This is because the Flash Arcade is its own nginx + Docker service on a separate subdomain — it cannot be served as a sub-path of the Browser Games hub without reconfiguring the arcade's own routing.

// In browser_games/index.json — already present:
{
  "id":          "flash-games",
  "externalHub": true,
  "externalUrl": "/flash/"   // ← update to the actual subdomain URL
}
⚠️
Update externalUrl in index.json to the full Tailscale-accessible URL of the Flash Arcade: https://flash..
🔐 Access Control

The Flash Arcade subdomain (flash.) is gated by Authentik forward_auth at the Caddy reverse proxy level. Access requires:

  • Tailscale enrollment — the subdomain is only reachable from inside the Tailnet
  • Authentik SSO session — Caddy forward_auth checks for a valid session before allowing access

The X-authentik-username header is injected by Caddy and used by the arcade to route saves to the correct per-user directory. No separate login screen is shown — if you're on Tailscale and logged into Authentik, you go straight in.

💾 Per-User Save Setup

Save files (.sol) are stored per-user. When a new user is onboarded to the server, create their saves folder:

# Run on the server (in WSL2 or PowerShell, where the arcade files live)
mkdir -p userdata/{NAME}/arcade/saves
# {NAME} must exactly match their Authentik username
🐉
Username must match exactly. Caddy injects X-authentik-username verbatim. If the folder name differs from the Authentik username (capitalisation, spaces), saves will fail silently or go to the wrong user.
🎮 Adding Games to the Library

Drop a .swf file into the shared library and register it in library.json:

userdata/common/arcade/library/
  mygame.swf
  library.json    ← add an entry here

// library.json entry format:
{
  "id":        "mygame",
  "file":      "mygame.swf",
  "title":     "My Game",
  "icon":      "🎮",
  "thumbnail": "mygame.png",    // optional
  "genre":     "Arcade",
  "year":      2006,
  "author":    "Author Name",
  "notes":     "Original source: Newgrounds."
}

🎮 What Is This?

The Flash Arcade is a self-hosted preservation effort. Flash Player was officially discontinued by Adobe in January 2021, making billions of browser games, animations, and interactive toys from the early internet era inaccessible. Ruffle — an open-source Flash Player re-implementation written in Rust and compiled to WebAssembly — allows Flash content to run again without any plugins, in any modern browser.

This server's arcade collects a curated set of classic Flash games from the era of Newgrounds, Kongregate, and Flash game portals of the mid-2000s to mid-2010s. Each game is preserved locally as a .swf file, served through a private nginx container, and played via Ruffle.

Save files (.sol files — Flash's native save format) are stored server-side, separately per user. This means multiple people can share the same game library but maintain completely independent progress and high scores within each game.

💡
Ruffle compatibility: Not every Flash game works perfectly in Ruffle — it is an ongoing reverse-engineering project. ActionScript 2 games (most games from before 2008) have very good compatibility. ActionScript 3 games (2008+) have partial support. If a game doesn't load or behaves incorrectly, check the Ruffle compatibility tracker.
📖 How to Use
  • Navigate to flash. — you'll be taken straight to the arcade lobby
  • Browse the game library and click a game to launch it
  • Ruffle loads the .swf file and runs it in the browser — no plugin, no installer
  • Save files are automatically stored under your username — progress persists across sessions
  • The game runs in a dedicated Ruffle player area; use your browser's Back button or the arcade UI to return to the library
Controls
Mouse Primary input for most Flash games Arrow Keys Movement in most platformers and shooters Z / X / C Common action keys (varies per game) Esc Back to the arcade library

Controls vary entirely per game — most Flash games display their own control instructions on a title or help screen.

💾 Your Saves

Save files are stored on the server at:

userdata/YOUR_USERNAME/arcade/saves/

Your saves are completely isolated from other users. Two people can play the same game simultaneously and their saves never collide. If you want to check or back up a specific save, the files are standard .sol format (Local Shared Objects — Flash's native save format) and can be read with any SharedObject editor.

⚠️ Known Limitations
IssueDetail
Ruffle compatibilitySome Flash games use advanced ActionScript 3 features that Ruffle doesn't fully support yet. Games from before ~2008 work best. If a game doesn't load, it's usually a Ruffle compatibility issue, not a server issue.
Sound in some gamesAudio playback in Ruffle has improved significantly but some games may have missing or distorted sound — known Ruffle limitation.
Online featuresAny game that required connecting to external servers (online leaderboards, multiplayer, DRM checks) will not work. Single-player offline content works fine.
MobileFlash games were designed for mouse and keyboard. Many are unplayable on touch without a physical keyboard. Simple point-and-click games work fine on touch.
External URLsThe Flash Arcade opens in a new tab rather than embedded in the Browser Games hub iframe. This is because it runs on its own subdomain and Docker service.

⚖️ License & Preservation Note

Ruffle is open-source under the MIT and Apache 2.0 licenses. The game library consists of individually sourced .swf files. Where possible, games are sourced directly from Newgrounds or other sites where the original creator still hosts them.

Flash game preservation occupies a legal grey area similar to ROM preservation — the original creators often cannot be contacted (many studios have closed), the content is otherwise inaccessible, and the cultural value of preserving these works is significant. This server's arcade is strictly private, accessible only to enrolled friends and family, with no commercial use.

If a game creator contacts us with a takedown request, it will be honoured immediately.
