📁 Step 1 — Get the Web Build

Hauberk is written in Dart. The repo compiles to JavaScript for browser play. Check whether a prebuilt web version is available in the repo's web/ or build/ directory — if so, serve that directly. If not, a Dart SDK build is needed:

cd D:\Server\BrowserGames
git clone https://github.com/munificent/hauberk.git hauberk
cd hauberk

# If a prebuilt web/ folder exists, serve it directly.
# Otherwise, install Dart SDK and build:
# dart pub get
# dart compile js -o web/hauberk.dart.js web/main.dart
💡
Check the repo's web/ directory first — the author often commits a compiled JS build alongside the Dart source. If it exists, no Dart SDK is needed. Update folder in index.json to point at whichever directory contains the index.html.

🎮 What Is This?

Hauberk is an ASCII roguelike dungeon crawler written by Bob Nystrom — best known as the author of Game Programming Patterns and Crafting Interpreters, two of the most widely read game and language implementation books in software engineering. Hauberk is his ongoing personal roguelike project, developed over many years as an exploration of game design and Dart.

The game follows classic roguelike conventions: procedurally generated dungeons, turn-based movement and combat, items and equipment, and permadeath — when your character dies, the run is over and you start fresh. Each dungeon level is populated with monsters of escalating danger; descending deeper is the goal, but descending too quickly without proper equipment is fatal.

The code is notably clean and well-structured — Nystrom applies his own teaching to the implementation. The game is still in active development; newer versions add skills, hero classes, and expanded content over the years.

📖 How to Play
  • Move with arrow keys or numpad — each move takes one turn
  • Bump into monsters to attack them; they attack back on their turn
  • Pick up items and equipment to increase your effectiveness
  • Find stairs (>) to descend to the next dungeon level
  • Death is permanent — there is no save/load. Start a new run.
  • How deep can you go? That's the question.
