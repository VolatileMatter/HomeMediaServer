⚠️ Architecture Note
⚠️
Massive Decks requires a persistent server for game state management. It is not a static file game. The Docker image includes both the Elm frontend and the Haskell backend.
🐳 Step 1 — Docker Compose

Create ~/services/massivedecks/docker-compose.yml:

services:
  massivedecks:
    image: ghcr.io/lattyware/massivedecks:latest
    container_name: massivedecks
    restart: unless-stopped
    ports:
      - "8890:8080"
    environment:
      - MD_STORAGE_TYPE=memory
docker compose -f ~/services/massivedecks/docker-compose.yml up -d
💡
The default memory storage type is fine for a small friend group — all game state lives in RAM. If the container restarts mid-game, the game is lost. A file-based storage option exists for persistence; see the repo README for details.
🌐 Step 2 — Caddy Configuration

Add to :

decks.{$DOMAIN} {
  forward_auth  {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy localhost:8890
}

🎮 What Is This?

Massive Decks is a browser-based implementation of the Cards Against Humanity format, self-hostable and free of the limitations of commercially-run alternatives. Each player holds a hand of answer cards. A rotating "Card Czar" draws a black prompt card with a blank to fill in. Players choose the funniest card from their hand to complete the prompt; the Czar picks a winner for the round.

The game ships with the full Cards Against Humanity base set and many official expansions included. Custom decks in JSON format (or from compatible community sources) can be loaded via URL at room creation time, making themed games straightforward to set up.

It supports configurable house rules — blanks in hand, happy ending, reboot the universe, and others — and handles up to 20 players without issue. Best at 5–10 players where the pace stays lively.

📖 How to Play
  • One player creates a game, selects decks and house rules, and shares the room code
  • Each player is dealt a hand of white answer cards
  • The Card Czar flips a black prompt card — players choose their best/funniest answer
  • The Czar reads all answers aloud and picks the winner, awarding an Awesome Point
  • The Czar role rotates each round; first to the target Awesome Points wins
  • Custom decks: paste a JSON URL into the deck selection screen at room creation
