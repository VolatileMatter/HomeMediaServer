⚠️ Architecture Note
⚠️
Suroi has two components: a client (served as static files) and a game server (Node.js, handles physics, matchmaking, and game state). Both must run for the game to work. The repo provides a Docker Compose setup that runs both together.
📁 Step 1 — Clone & Build
cd ~/services
git clone https://github.com/HasangerGames/suroi.git suroi
cd suroi
💡
Suroi requires building from source — the client assets are compiled via Vite. The repo includes a Docker Compose that handles the full build and run cycle.
🐳 Step 2 — Docker Compose

The repo ships with a docker-compose.yml. Review it first, then build and start:

cd ~/services/suroi
docker compose up -d --build

Default ports from the repo's compose file:

ServiceContainer portNotes
Client (static)3000Serve the Vite-compiled frontend
Game server8000WebSocket game server
⚠️
Check the current repo's docker-compose.yml for the actual ports — they may differ from the above. The project is actively developed and defaults change between releases.
🌐 Step 3 — Caddy Configuration

Add to :

suroi.{$DOMAIN} {
  forward_auth  {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }

  # WebSocket game server
  handle /play* {
    reverse_proxy localhost:8000
  }
  handle /team* {
    reverse_proxy localhost:8000
  }

  # Static client
  handle {
    reverse_proxy localhost:3000
  }
}

🎮 What Is This?

Suroi is the open-source successor to surviv.io — a 2D top-down battle royale that was beloved for its accessibility and frantic pacing before its original developer shut it down. Suroi was built by the community to preserve and extend that experience.

The format is a classic battle royale: players drop onto an island, scavenge buildings for weapons and healing items, and a shrinking red zone forces encounters. Last player (or team) alive wins. Matches are short — typically 5 to 15 minutes depending on player count — which makes it well-suited to quick competitive game sessions.

Running a private server means matches are exclusively among your friend group, which removes the skill disparity problem of public servers and makes coordinated team games straightforward.

📖 How to Play
  • Click Play to queue for a match — the game starts when enough players are ready
  • You spawn on the island unarmed — immediately loot a nearby building
  • The red gas zone shrinks over time, forcing players toward the centre
  • Eliminate other players with ranged weapons, melee, or grenades
  • Heal with bandages, medkits, or cola; armour reduces incoming damage
  • Last player or team standing wins

⚠️ Known Limitations
IssueDetail
Min players to startThe server may wait for a minimum player count before starting a match. Configurable in server settings.
Build complexitySuroi builds from source via Vite — rebuilds take time after updates. Check the build logs if the Docker image fails.
Active developmentThe codebase changes frequently. After a git pull, always rebuild: docker compose up -d --build.
👥 Suroi requires at least 2 players to start a match. Solo testing is possible if the server's minimum player count is set to 1 in config.
