🐳 Step 1 — Clone & Docker Compose
cd ~/services
git clone https://github.com/hamzaelhibaoui/gacha_game.git rpggacha
cd rpggacha

Check if the repo includes a docker-compose.yml. If not, create one:

services:
  rpggacha:
    build: .
    container_name: rpggacha
    restart: unless-stopped
    ports:
      - "8740:5000"
    environment:
      - FLASK_ENV=production
      - DATABASE_URL=sqlite:///gacha.db
    volumes:
      - rpggacha_data:/app/instance

volumes:
  rpggacha_data:
docker compose up -d --build
💡
Flask apps often use SQLite for development. Check the repo for the actual database backend — it may use PostgreSQL for the full feature set. Adjust the DATABASE_URL and add a db service if so.

🎮 What Is This?

RPG Gacha Game is the most mechanically substantial entry in the casino/gacha section — it's not just a pull simulator but a full game with ongoing progression. Players register accounts, earn virtual currency, spend it on banner pulls to acquire characters across 5 rarity tiers, equip gear to strengthen them, assemble a team, and battle other players' teams in an asynchronous PvP arena.

The gacha mechanic serves its traditional purpose here: acquiring the characters needed to compete. Better characters have higher base stats and more powerful abilities, making the pull system feel meaningful rather than purely cosmetic. Equipment adds a second axis of character strengthening that's accessible regardless of pull luck.

For a private friend-group server this creates a persistent metagame — everyone is building their roster over time, comparing characters, and competing in the PvP rankings. It rewards both consistent play and occasional lucky pulls.

⚠️ Known Limitations
IssueDetail
Separate account systemFlask manages its own user accounts — Authentik gates network access only.
Database backup criticalAll character progress, equipment, and pull history lives in the database. Back up the Docker volume regularly.
PvP may be asynchronousDepending on implementation, PvP may be async (battle cached teams) rather than real-time. Check the repo.
👥 Multiple players share the same server with independent accounts. PvP arena lets accounts battle each other's teams.
🌐 Server Architecture
ComponentDetail
BackendPython / Flask — HTTP API and server-rendered pages
DatabaseSQLite or PostgreSQL — stores all accounts, characters, equipment, currency
PvP modelLikely async — players submit teams, battles resolve server-side against stored opponent teams
Account isolationEach player has their own account, currency, and roster — no shared state beyond PvP rankings
