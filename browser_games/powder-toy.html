📁 Step 1 — Obtain the Web Build

The Powder Toy's browser version is not a simple clone-and-serve. The main repository is a C++ application. You need the Emscripten-compiled web build specifically.

⚠️
Do not attempt to build from source unless you have an Emscripten toolchain set up. Use the official web build instead — it is already compiled and available.

Option A — Use the official web app (recommended): The Powder Toy maintains an official web version at powdertoy.co.uk/Play.html. If self-hosting isn't required, simply link to this from the hub. Users need internet access.

Option B — Self-host the built output: Check the GitHub releases for a pre-built web distribution, or ask in the Powder Toy community for a static build package. Extract to D:\Server\BrowserGames\powder-toy\.

🎮 What Is This?

The Powder Toy is the granddaddy of falling-sand games and one of the most sophisticated physics sandboxes ever created in open-source software. Originally developed in 2008 and actively maintained since, it simulates hundreds of distinct elements — not just sand and water, but metals that conduct electricity, fissile materials that undergo nuclear reactions, biological agents that spread and mutate, programmable logic elements, and much more.

The simulation runs on a pixel grid where each pixel is an individual particle with its own state, temperature, pressure, and element type. Physics interactions are modelled at a surprisingly deep level: heat conducts through metals, liquids exert pressure, gases expand and compress, explosions create shockwaves, and radiation propagates through the simulation.

Beyond the sandbox, The Powder Toy has a large community that creates and shares elaborate simulations — fully functional computers built from logic elements, working rockets, chemical reactors, and detailed pixel art dioramas. The online save browser (on the official site) has thousands of community creations.

The web version runs the full simulation engine compiled to WebAssembly. It is functionally equivalent to the desktop version, though the online save/browse functionality may be limited in the self-hosted build.

📖 How to Play
  • Select an element from the element browser (press F to open, or browse the palette bar)
  • Left-click and drag to place the selected element on the canvas
  • Right-click to erase elements
  • Watch elements interact — build contraptions, observe emergent chemistry
  • Use the HUD display to see temperature, pressure, and element properties at the cursor
  • Press Space to pause/resume the simulation
  • Press S to save (to a local file or localStorage in the web build)

⚠️ Known Limitations
IssueDetail
Online savesThe self-hosted build cannot browse or upload to the powdertoy.co.uk save server. Saves work locally (localStorage or file download). Use the official site for community simulations.
PerformanceThe Powder Toy is CPU-intensive at high particle counts. Complex simulations with thousands of active particles may run slowly. The WASM build is single-threaded — modern desktop CPUs handle it fine; older hardware may struggle.
MobileNot recommended. The element browser, HUD, and fine placement controls require mouse precision. Touch input is too coarse for comfortable play.
WASM loadThe WASM bundle is large (larger than Sandspiel). First load takes several seconds. Subsequent loads use browser cache.
Lua scriptingThe Lua console (~ key) is available in the web build but may have reduced capabilities compared to the desktop version due to sandboxing.
