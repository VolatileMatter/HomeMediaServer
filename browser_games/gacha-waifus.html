🐳 Step 1 — Clone & Docker Compose
cd ~/services
git clone https://github.com/typpi77/test-web-socket.git gacha-waifus
cd gacha-waifus

Create ~/services/gacha-waifus/docker-compose.yml if not present:

services:
  gacha:
    build: .
    container_name: gacha-waifus
    restart: unless-stopped
    ports:
      - "8750:3000"
    volumes:
      - ./data:/app/data
docker compose up -d --build
💡
The ./data volume mount persists the JSON storage files — character collections and pity counts. Keep this backed up.
🌐 Step 2 — Caddy Configuration
gacha.{$DOMAIN} {
  forward_auth  {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy localhost:8750
}

🎮 What Is This?

Gacha Waifus Overlay is a Node.js gacha system designed with streaming in mind — pull results broadcast via WebSocket in real time, making it suitable as a live overlay for stream engagement. Repurposed for a private friend group, it functions as a shared gacha game where everyone builds their own collection from a shared character pool.

The system implements genuine pity mechanics — each pull increments a counter, and the rate of pulling rare characters increases as the counter rises, with a hard guarantee after a set number of pulls. Collections are stored per-user in JSON files, and characters can be traded between players.

The JSON-based storage is simple and hackable — you can edit the character pool, rarity rates, and pull costs directly in the data files without modifying application code.

⚙️ Customisation

Because storage is JSON-based, the character pool is easy to customise. Locate the character data files in the ./data volume and edit them to add, remove, or modify characters, adjust rarity tiers, or change pull costs. No rebuild required — changes take effect on restart.

💡
This makes it practical to theme the gacha around something relevant to your friend group — franchise characters, inside jokes, server mascots. The JSON format is straightforward to extend.
