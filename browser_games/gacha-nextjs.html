⚠️ Architecture Note — Next.js + Prisma
⚠️
This is a fullstack Next.js application with Prisma ORM. It requires a build step and a database (likely PostgreSQL or SQLite via Prisma). Check the repo for a .env.example file — database connection string and any auth secrets must be set before the build.
📁 Step 1 — Clone & Configure
cd ~/services
git clone https://github.com/reyhanasta/gacha-game.git gacha-nextjs
cd gacha-nextjs

# Copy and edit the env file:
cp .env.example .env
# Edit .env — set DATABASE_URL, NEXTAUTH_SECRET, NEXTAUTH_URL
💡
Generate a strong NEXTAUTH_SECRET: openssl rand -base64 32. Set NEXTAUTH_URL to your Caddy subdomain, e.g. https://gacha2.ixo.lol.
🐳 Step 2 — Docker Compose

Check if the repo ships a docker-compose.yml. If not:

services:
  gacha:
    build: .
    container_name: gacha-nextjs
    restart: unless-stopped
    ports:
      - "8730:3000"
    env_file: .env
    depends_on:
      - db

  db:
    image: postgres:16-alpine
    container_name: gacha-nextjs-db
    restart: unless-stopped
    environment:
      - POSTGRES_DB=gacha
      - POSTGRES_USER=gacha
      - POSTGRES_PASSWORD=changeme
    volumes:
      - gacha_pg:/var/lib/postgresql/data

volumes:
  gacha_pg:
docker compose up -d --build
# Run Prisma migrations:
docker exec gacha-nextjs npx prisma migrate deploy
🌐 Step 3 — Caddy Configuration
gacha2.{$DOMAIN} {
  forward_auth  {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy localhost:8730
}

🎮 What Is This?

This is the same developer (reyhanasta) as the Wuthering Waves simulator, but this is a more general-purpose gacha game rather than a simulation of a specific title. It features multi-banner support — multiple character pools available simultaneously, each with their own featured characters and independent pity counters — which is the standard structure used by games like Genshin Impact, Honkai, and most modern gacha titles.

The daily login rewards and achievement system give players reasons to return regularly, turning it into a casual ongoing engagement game rather than a one-time pull experience. Prisma provides type-safe database access with proper migrations, making it the most maintainable backend of the gacha titles in this hub.

For a private friend group this creates a persistent collection game — everyone is pulling on different banners, comparing who pulled what, and working toward achievements at their own pace.
