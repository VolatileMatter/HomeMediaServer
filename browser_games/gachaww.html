⚠️ Architecture Note — PHP/Laravel
⚠️
This is a PHP/Laravel/Livewire application — the only PHP app in the games hub. It requires PHP 8.x, Composer, and Redis (for Livewire's real-time updates). The Docker setup below bundles these. Check the repo README for any additional environment variables or seed commands required.
🐳 Step 1 — Clone & Docker Compose
cd ~/services
git clone https://github.com/reyhanasta/wuthering-waves-simulator.git gachaww
cd gachaww

Check if the repo ships a docker-compose.yml. If not, create one:

services:
  app:
    build: .
    container_name: gachaww
    restart: unless-stopped
    ports:
      - "8770:80"
    environment:
      - APP_ENV=production
      - APP_KEY=base64:GENERATE_WITH_php_artisan_key_generate
      - REDIS_HOST=redis
    depends_on:
      - redis

  redis:
    image: redis:7-alpine
    container_name: gachaww-redis
    restart: unless-stopped

volumes:
  gachaww_data:
docker compose up -d --build
# Then run migrations if required:
docker exec gachaww php artisan migrate --force
⚠️
Generate a fresh APP_KEY before deploying: docker exec gachaww php artisan key:generate. Running without a proper key will break session handling.

🎮 What Is This?

This simulator replicates the banner and gacha pull system from Wuthering Waves, an action RPG by Kuro Games. Like most gacha games, Wuthering Waves uses a system where players spend a pull currency for random rewards, with low base rates for rare characters (5-star) but a guaranteed "pity" system that ensures a 5-star pull within a fixed number of attempts.

The simulator implements the exact mechanics: base 5-star rate, soft pity (increased rate after a threshold), hard pity (guaranteed at max), and the 50/50 system where the first 5-star on a banner has a 50% chance to be the featured character. This makes it genuinely useful for planning — if you want to calculate how many pulls you need for a guaranteed featured character with high probability, you can run thousands of simulated attempts instantly.

Built with Laravel and Livewire, the interface updates in real time as pulls execute without full page reloads.
