⚠️ Architecture Note
⚠️
Hypersomnia has two components: a dedicated game server (C++ binary or Docker) and a web client (served as static WASM + HTML files). The client loads in the browser but the game itself runs via WebAssembly. The dedicated server is required for multiplayer matches.
🐳 Step 1 — Dedicated Server

Create ~/services/hypersomnia/docker-compose.yml:

services:
  hypersomnia:
    image: hypersomnia/hypersomnia-server:latest
    container_name: hypersomnia
    restart: unless-stopped
    ports:
      - "8412:8412/udp"   # Game server UDP
      - "8820:80"          # Web client static files
docker compose -f ~/services/hypersomnia/docker-compose.yml up -d
💡
Check the official Hypersomnia Docker documentation for the current image name and recommended port configuration — these have changed between releases.
🌐 Step 2 — Caddy Configuration

Add to for the web client:

hypersomnia.{$DOMAIN} {
  forward_auth  {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy localhost:8820
}
⚠️
The game server communicates over UDP port 8412 — this bypasses Caddy entirely. UDP traffic goes directly from client to server over Tailscale. Ensure the port is reachable on the Tailscale network.

🎮 What Is This?

Hypersomnia is a top-down multiplayer shooter originally developed as a native C++ application, subsequently ported to the browser via Emscripten/WebAssembly. The WASM port brings the full native engine to the browser — physics simulation, dynamic lighting and shadows, bullet penetration through surfaces, recoil patterns, and a detailed sound model all run at near-native performance.

Gameplay revolves around team-based modes — deathmatch, bomb defusal, and others — on handcrafted maps. The top-down perspective with realistic bullet travel and light simulation creates tense, tactical engagements where cover and sight lines matter as much as aim.

For a private server on Tailscale this means a surprisingly high-quality competitive shooter playable without any client install — just a modern browser. The initial load is heavier than a typical browser game due to the WASM binary size, but subsequent loads are fast from cache.

📖 How to Play
  • Open the game URL — wait for the WASM assets to load (larger initial download)
  • Connect to the server by entering the Tailscale IP or hostname + port in the server browser
  • Select a team and spawn into the match
  • Use cover — light and shadows affect visibility; bullets penetrate thin surfaces
  • Objectives vary by mode: deathmatch (eliminate enemies), bomb (plant/defuse)

⚠️ Known Limitations
IssueDetail
Large initial loadThe WASM binary is large. First load takes 30–60 seconds depending on connection speed. Subsequent loads are cached.
UDP game serverGame server traffic bypasses Caddy over UDP. Authentik only gates the web client page load — not individual game connections.
Keyboard + mouse requiredNot playable on mobile or tablet.
SharedArrayBuffer requiredSome browsers may need COOP/COEP headers set on the web client to enable WASM threading. The Docker image should handle this, but verify if the game fails to start.
👥 Hypersomnia requires at least 2 players. The game server manages match state; the WASM client renders and handles input locally.

⚠️ Known Issues
IssueDetail
WASM threading headersIf the game loads but freezes or crashes, check that the server is sending Cross-Origin-Opener-Policy: same-origin and Cross-Origin-Embedder-Policy: require-corp headers. These are needed for SharedArrayBuffer support.
UDP through CaddyCaddy does not proxy UDP. The game server port must be directly accessible on the Tailscale network — don't put it behind Caddy's reverse_proxy.
