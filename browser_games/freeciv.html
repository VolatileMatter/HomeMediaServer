⚠️ Complexity Warning
⚠️
Freeciv-web is a large, multi-service application. It requires several Docker containers: a Java game server, a WebSocket proxy, a web client server, and supporting services. The repo ships a Docker Compose setup — use it rather than attempting to wire services manually. Plan for a 30–60 minute setup and ongoing maintenance.
🐳 Step 1 — Clone & Start
cd ~/services
git clone https://github.com/freeciv/freeciv-web.git freeciv
cd freeciv
docker compose up -d --build

The build will take several minutes — it compiles Java source and builds the web client. Wait for all containers to show as healthy before attempting to access the URL.

💡
Check the repo's README and docker-compose.yml for the exact exposed port before configuring Caddy — it may differ between releases.
🌐 Step 2 — Caddy Configuration

Add to once you know the exposed frontend port:

freeciv.{$DOMAIN} {
  forward_auth  {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy localhost:FREECIV_PORT
}
⚠️
Freeciv-web uses WebSockets for the game server connection. Caddy handles WebSocket upgrades automatically, but the internal Docker Compose networking must route WebSocket traffic correctly through the proxy service bundled with the project.

🎮 What Is This?

Freeciv is an open-source turn-based strategy game that has been in active development since 1996, inspired by the Civilization series. Freeciv-web is the HTML5 browser client for it, allowing the full game to be played without any native install. Players start with a lone settler unit, found cities, research technologies across multiple eras, build armies and infrastructure, and compete for victory conditions — domination, cultural, or technology-based.

The depth here significantly exceeds any other strategy game in this hub. A full game with 4–8 players can span dozens of sessions over weeks or months. Freeciv supports AI opponents, so a solo or mixed human/AI game is fully viable. The ruleset is configurable — you can run classic Civ1-style rules, more modern rulesets, or custom scenarios.

The server-side save system is a major advantage over in-memory games: the game can be paused, saved, and resumed the next time the group gathers. This makes it one of the few games in this hub suited to a persistent long-term campaign.

⚠️ Known Limitations
IssueDetail
Build complexityMulti-container Java build. Initial setup is the most complex after Lichess. Follow the repo README carefully.
Resource usageJava game server is memory-hungry. Allocate at least 2GB to the Docker stack for a comfortable small-group game.
Turn speedIn async multiplayer, all human players must submit their turn before the game advances. Agree on a turn deadline with your group.
Learning curveFreeciv has deep mechanics. New players should start with a solo game vs AI to learn the interface before joining a multiplayer game.
👥 Freeciv supports both real-time and turn-based async multiplayer. The async mode is ideal for a friend group spread across time zones or schedules.
