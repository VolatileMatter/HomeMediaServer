📁 Step 1 — Clone Repository
cd D:\Server
git clone https://github.com/abrenaut/posio.git posio
cd posio
🐳 Step 2 — Docker Compose

Posio is a Python application (Flask + WebSockets). The cleanest approach is a Docker container:

posio:
  build:
    context: D:\Server\posio
  ports:
    - "5050:5000"
  restart: unless-stopped

If there is no Dockerfile in the repo, create a minimal one:

FROM python:3.11-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
EXPOSE 5000
CMD ["python", "application.py"]
💡
Check the repo's requirements.txt and entry point filename — it may be app.py, server.py, or application.py.
🌐 Step 3 — Caddy Reverse Proxy
posio.{$DOMAIN} {
  forward_auth authentik:9000 {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy posio:5000
}

WebSocket connections need to pass through — Caddy handles this automatically for reverse_proxy directives.

🌍 What Is This?

Posio is a multiplayer geography game where players are shown a place name and must click on a world map to place their pin as close as possible to where that location actually is. Scoring is distance-based — the closer your pin, the more points you earn.

Rounds are fast-paced and synchronous: all players see the same prompt, and everyone's pin placement is revealed at the end of each round. The real-time leaderboard updates after every round, creating momentum and competition even in small groups.

The game is built on Python (Flask) and WebSockets, making it easy to self-host. It has no database — sessions are entirely in-memory. Questions cover world cities, countries, and geographic landmarks.

🖱️ Controls

Click or tap anywhere on the world map to place your pin. You can reposition your pin before the round timer expires. No keyboard shortcuts — entirely mouse/touch driven. Works on mobile browsers.

🔌 How Sessions Work

Posio uses WebSockets for real-time synchronisation. All players connecting to the same server instance share the same game session — there is no separate room/lobby system in the base codebase.

  • All players on posio. are in the same session
  • The server controls round timing and question order
  • Scores are calculated server-side based on pin distance
  • Sessions reset when the server restarts
⚠️ Known Limitations
