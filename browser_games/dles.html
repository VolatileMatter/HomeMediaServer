📋 Step 1 — Register in index.json

The Dles is a third-party public website, not a self-hosted game. It is registered as an externalHub entry — the same pattern as Flash Arcade. Clicking Play in the lobby opens the site in a new tab rather than loading it inside an iframe.

// In browser_games/index.json — add this entry:
{
  "id":          "dles",
  "name":        "The Dles",
  "icon":        "📅",
  "description": "600+ curated daily web games. Wordle-style puzzles covering words, geography, music, sports, video games, and more. New puzzles every day — everyone plays the same one.",
  "folder":      null,
  "entry":       null,
  "category":    "singleplayer",
  "highScores":  false,
  "players":     "1",
  "tags":        ["daily", "word", "puzzle", "trivia", "external"],
  "tech":        "SvelteKit · External Site",
  "hasPage":     true,
  "externalHub": true,
  "externalUrl": "https://dles.aukspot.com"
}
💡
Because externalHub: true, no iframe is involved. The hub calls window.open(externalUrl, '_blank'). No Caddy config, no folder, no auth gate needed — the site is public.
🔐 Step 2 — Access Control

The Dles is a public website at dles.aukspot.com. There is no server-side access control to configure — anyone with a browser can reach it. The hub entry is just a convenient bookmark with description and tags, not a gated service.

No Tailscale or Authentik gate applies. The link opens the public internet.

📁 Step 3 — No Files Needed

Nothing to copy, clone, or build. The only action is adding the JSON entry above to browser_games/index.json and placing this wiki page at browser_games/dles.html.

The wiki page is surfaced via the 📖 Wiki / Setup link on the lobby card, which loads it inside the hub's frame viewer just like any other wiki page.

🎮 What Is This?

The Dles is a curated directory of "dles" — daily web games in the Wordle tradition. A dle is a game that resets every day, gives everyone the same puzzle, and is free to play with no account needed. Wordle is the archetype but the genre has exploded into hundreds of variants spanning almost every topic imaginable.

The site itself is not a game — it's a launcher and catalogue. Each entry is a link to a third-party site where the game actually lives. The Dles keeps the list curated, removes dead links, and organises everything into categories so you can find what you're in the mood for.

Northernlion quote on the README: "They call them the dles but they are anything but."

🎯 How to Choose Which Dles to Play

The site gives you several tools to cut 600+ games down to a personal rotation you actually care about. None of this requires an account — everything is stored in your browser's localStorage and persists as long as you're on the same device and haven't cleared your browser data.

⭐ Favourites — Build Your Daily Rotation

Every game card has a star button. Starring a game adds it to your Favourites list, which appears at the top of the page for instant access. This is the main tool for building a personal rotation — star the 5–10 dles you want to check every morning and ignore everything else.

💡
Your favourites are stored in localStorage under a key like favorites. They survive page refreshes but not browser data clears. If you use the site on multiple devices, you'll need to star your favourites separately on each one — there's no account sync.
🙈 Hide — Get Rid of Games You'll Never Play

Games can be hidden so they disappear from the list entirely. If you don't care about chess, baseball, K-pop, or Doctor Who — hide those games so they stop cluttering up the view. Use the kebab/options menu on a game card to hide it.

Hidden games are stored in localStorage. You can reveal them again via the toggle at the top of the page ("Show hidden") if you change your mind.

🗂️ Category Filter — Browse by Topic

The sidebar (or top filter row on mobile) lists all 17 categories. Click a category to see only games from that group. Useful when you're in the mood for geography but not word games, or when you want to discover new games in a topic you like.

Category filtering combines with the search bar — you can search for "chess" inside the Card/Board Games category, for example.

🎲 Random Pick

A random game button is available in the toolbar. Click it to be dropped into a random dle from the full list (or from the currently filtered subset). Good for discovery when your usual rotation is done and you want to try something new.

✏️ Controlling Words in Individual Dles

Individual dle games — Wordle clones and their variants — are third-party sites. The Dles hub is a directory; it doesn't control the word that any game uses today. To influence words or word lists, you work with each game's own site or source code directly.

There are three main approaches depending on what you want to do:

① URL Parameters — Play a Specific Day or Word

Many Wordle-style games accept URL parameters to load a specific puzzle. This is the easiest way to play a past puzzle or set up a specific word for a friend.

GameHow to ControlExample
NYT Wordle Append ?d=YYYY-MM-DD to load a specific date's puzzle (unofficial — may stop working) nytimes.com/games/wordle/index.html
hello wordl (hellowordl.net) Append #[word] to the URL to set a specific word. Share that URL with a friend. hellowordl.net/#grape
Absurdle Append ?seed=[number] to replay a specific seed. Different seeds produce different adversarial games. qntm.org/files/absurdle/#42
Quordle Practice mode is available from the main menu — lets you play without affecting your streak, and the words are different from the daily. Click "Practice" on the Quordle site
💡
Not all games support URL parameter control — it varies by site. The easiest ones to share are those with a hash-based word parameter like hello wordl. Check each game's "How to Play" or GitHub README for details.
② Self-Hosting a Game with Your Own Word List

If you want full control over which words appear and when, the answer is to self-host a clone with your own word list. Many dle-style games are open source and trivially forkable. The workflow:

1
Find the game's GitHub repository (many link to it in their footer or "About" page). Fork it to your own GitHub account.
2
Locate the word list file — usually a words.js, wordlist.json, or answers.js at the root or in a src/ or data/ folder. Replace the word array with your own list.
3
If the game uses a fixed date-based index (e.g. wordList[daysSinceEpoch % wordList.length]), you can control today's word by ordering your list accordingly, or replace the date logic with a fixed index for testing.
4
Drop the forked/modified game into D:\Server\BrowserGames\[game-id]\ and register it in browser_games/index.json as a normal (non-external) game entry. It'll load inside the hub iframe, and you control the words entirely.
⚠️
Check each game's license before forking. MIT and GPL allow this freely. Some games have more restrictive terms. When in doubt, keep your fork private (it's on your LAN anyway) and don't redistribute.
③ Suggesting a New Dle to the Directory

The Dles directory itself is open source. If you discover a great daily game that isn't listed yet, you can contribute it back to the public directory. This doesn't add words — it adds a new game to the catalogue.

1
The easiest path is the suggestion form (linked from the README). Fill out the issue template with the game's name, URL, category, and description.
2
Or edit src/dles.json directly in the GitHub web editor. Copy an existing entry and update the fields — do not add an id field, it's assigned automatically on build.
3
Open a pull request. The maintainer (aukspot) reviews and merges it. The site redeploys automatically via GitHub Actions. 
// dles.json entry format — paste and fill in:
{
  "name":        "Game Name",
  "url":         "https://thegame.com",
  "category":    "Words",
  "description": "One clear sentence about what it is and how you play."
  // DO NOT add an "id" field — it's auto-assigned
}
💡
To fix a broken link: find the entry in dles.json, update the URL field, and open a PR. To remove a dead game: delete the entry entirely. The README also has a "dle suggestion" issue template that skips the need to edit JSON directly.
🏗️ Running the Dles Site Locally

If you want to run your own private version of the entire Dles directory — with your own curated list, custom categories, or hidden games baked in — you can fork and self-host the whole SvelteKit app.

git clone https://github.com/aukspot/dles.git
cd dles
npm install
npm run dev          # dev server at localhost:5173

# Build for production:
npm run build        # outputs to ./docs (GitHub Pages format)

# To add / remove / edit dles, edit:
#   src/dles.json    ← game catalogue
#   src/categories.json (if it exists) ← category definitions
⚠️
Requires Node.js. The npm run build output lands in ./docs/ for GitHub Pages compatibility — if you're self-hosting on the LAN via Caddy, point the file server at that docs/ directory.
