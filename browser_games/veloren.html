⚠️ Architecture Note — Native Client Required
⚠️
Veloren is not a browser game. Players must install the Veloren native client on their machine (Windows/Mac/Linux). The server runs in Docker on the home server; clients connect to it over Tailscale. This page is included here because Veloren is part of the self-hosted game ecosystem — but the hub "Play" button does not apply. It is registered as an externalHub entry pointing to the Veloren download page.
🐳 Step 1 — Docker Compose

Create ~/services/veloren/docker-compose.yml:

services:
  veloren:
    image: veloren/veloren-server-standalone:latest
    container_name: veloren
    restart: unless-stopped
    ports:
      - "14004:14004/tcp"
      - "14004:14004/udp"
    volumes:
      - veloren_data:/opt/veloren/userdata

volumes:
  veloren_data:
docker compose -f ~/services/veloren/docker-compose.yml up -d
💡
Veloren uses both TCP and UDP on port 14004. Ensure both protocols are forwarded. The userdata volume persists character saves and world state.
🔐 Step 2 — Access Control

Veloren does not use HTTP — it communicates over a custom TCP/UDP protocol. Caddy and Authentik forward_auth do not apply here. Access control is handled at the network layer:

  • Tailscale — port 14004 is only reachable from enrolled Tailscale nodes. No public exposure needed.
  • Server admin password — set an admin password in the Veloren server config to prevent unauthorised admin commands
  • Auth server — Veloren supports account authentication via veloren.net accounts; for a private server, this can be left as open (no account required) or restricted

🖥️ Step 4 — Client Setup (per player)
  1. Download and install the Veloren client from veloren.net/download
  2. Ensure Tailscale is running and connected to the server's tailnet
  3. In the Veloren client: select Add Server
  4. Enter the server's Tailscale IP (or MagicDNS hostname) and port 14004
  5. Create a character and connect — no account needed for a private server by default
⚠️
Client and server versions must match exactly. When the server Docker image is updated, all players must update their client to the same version before connecting.
✅ Character data and world state persist in the Docker volume. The world continues to exist between sessions — nothing is lost when players log off.

🎮 What Is This?

Veloren is an open-source multiplayer voxel RPG written in Rust, inspired by games like Cube World, Dwarf Fortress, and Minecraft. It generates vast procedural worlds filled with biomes, dungeons, caves, towns with NPCs, wild animals, and hostile creatures. The progression system covers combat skills, crafting, gliding, magic, and character builds.

The game is genuinely impressive in scope — it has been in active community development since 2018 and has grown into a polished, content-rich experience. A private server for a friend group means a persistent shared world that builds over time: shared base camps, explored regions, cooperative dungeon runs, and accumulated character progression.

The one caveat for this hub: Veloren is a native application, not a browser game. Clients must be installed on each player's machine. This page documents the server setup and is registered in the hub for discoverability, but "Play" directs to the download page rather than launching an iframe.

🔧 Server Management
# Check status:
docker compose -f ~/services/veloren/docker-compose.yml ps

# View logs (world gen progress on first boot):
docker compose -f ~/services/veloren/docker-compose.yml logs -f

# Update (coordinate with players — version must match client):
docker compose -f ~/services/veloren/docker-compose.yml pull
docker compose -f ~/services/veloren/docker-compose.yml up -d

# Back up world data:
docker run --rm -v veloren_data:/data -v ~/backups:/backup \
  alpine tar czf /backup/veloren_$(date +%Y%m%d).tar.gz /data
