⚠️
Quizmaster is a Scala/Play Framework application. Building from source requires SBT (Scala Build Tool). The Docker approach below is strongly recommended to avoid a complex JVM build environment on Windows.
📁 Step 1 — Clone Repository
cd D:\Server
git clone https://github.com/nymanjens/quizmaster.git quizmaster
cd quizmaster
🐳 Step 2 — Build & Run with Docker

Check the repository for a Dockerfile or docker-compose.yml. If present, use it directly:

docker compose up -d

If no Dockerfile is provided, build using SBT inside a Scala Docker image:

quizmaster:
  image: sbtscala/scala-sbt:eclipse-temurin-21.0.1_12_1.9.7_3.3.1
  working_dir: /app
  volumes:
    - D:\Server\quizmaster:/app
  command: sbt run
  ports:
    - "9090:9000"
  restart: unless-stopped
💡
First run may take several minutes as SBT downloads dependencies and compiles the Scala code. Subsequent starts are fast if the build cache is preserved in the volume.
🌐 Step 3 — Caddy Reverse Proxy
quiz.{$DOMAIN} {
  forward_auth authentik:9000 {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy quizmaster:9000
}

📝 Step 5 — Create Questions

Quizmaster uses a structured question format defined in the application. Check the repo's conf/ or data/ directory for example question files. Questions can include text, images, and audio. Build your own quiz files before hosting your first game night.

Refer to the GitHub README for the exact question file format and how to load custom quizzes.

🧠 What Is This?

Quizmaster is a self-hosted web application for running structured quiz nights. There are two distinct interfaces: a master screen (for the quiz host, typically shown on a TV or shared display) and a player screen (on each participant's phone or laptop).

The quizmaster controls question pacing — advancing slides, revealing answers, and managing scores — while players type their answers on their own devices. The application handles scoring and displays a running leaderboard.

Unlike Digibuzzer (which is purely buzzer-based), Quizmaster supports typed free-text answers, making it suitable for traditional quiz night formats with specific factual questions. It also supports multimedia questions with images and audio clips.

Questions are fully custom — you write them yourself in a structured format before hosting. This makes it perfect for themed quiz nights (e.g., server lore, franchise trivia, music rounds) tailored to your friend group.

👥 Roles
RoleInterfaceResponsibilities
QuizmasterMaster screen (/master)Advances questions, reveals answers, manages scoring and round flow

🎙️ Recommended Setup
  • Quizmaster uses the master screen URL on a TV/projector or shared screen
  • Players navigate to the player URL on their own phones/laptops
  • Voice chat (Discord, etc.) recommended for the quizmaster to read questions aloud
  • All devices must be on the same Tailscale network to reach the self-hosted instance
