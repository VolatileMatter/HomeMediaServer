🔔 V1 Goal: Digibuzzer running as a Docker container, accessible at games./buzzer/ (or a dedicated subdomain) via Tailscale, gated by Authentik forward_auth. Multiple players can join a room from any Tailscale device.
⚠️
Digibuzzer requires a Node.js backend (Vite SSR + Socket.io). It cannot be served as static files. It needs its own Docker container with an exposed port, routed through Caddy like other services.
📁 Step 1 — Clone & Build
cd D:\Server
git clone https://codeberg.org/ladigitale/digibuzzer.git digibuzzer
cd digibuzzer
npm install
npm run build
💡
Or use the Docker approach in Step 2 to avoid needing Node.js on the host directly.
🐳 Step 2 — Docker Compose

Add to your Docker Compose stack:

digibuzzer:
  image: node:20-alpine
  working_dir: /app
  volumes:
    - D:\Server\digibuzzer:/app
  command: sh -c "npm install && npm run build && npm run start"
  ports:
    - "3100:3000"
  restart: unless-stopped

Or build a proper Dockerfile once the setup is stable. The app listens on port 3000 by default inside the container.

🌐 Step 3 — Caddy Reverse Proxy

Add a Caddy block to route a subdomain or path to the container. A subdomain is cleaner because Socket.io WebSocket upgrade works more reliably at the root path.

buzzer.{$DOMAIN} {
  forward_auth authentik:9000 {
    uri /outpost.goauthentik.io/auth/caddy
    copy_headers X-authentik-username X-authentik-groups
  }
  reverse_proxy digibuzzer:3000
}

Then register it as an Application + Provider in Authentik pointing to buzzer..

🔔 What Is This?

Digibuzzer is a web-based virtual buzzer system for quiz games, developed by La Digitale — a French collective creating free digital tools for education. The app lets a game master create a room with a shareable code, and players join on any device with a browser.

When a question is posed, players press a large on-screen buzzer button. The server captures response times to the millisecond and determines who buzzed in first — displayed on the game master's screen in real time. Points can be awarded or deducted. Multiple rounds are supported.

The interface is clean and works well on both desktop and mobile. Documentation and UI are in French, but the interaction model is intuitive enough to use without reading it. Ideal for quiz nights, family games, or classroom use.

👥 How Rooms Work
RoleWhat they do
Game MasterOpens the app, creates a room, gets a room code. Controls rounds, awards points, sees who buzzed first.
PlayerOpens the app on their device, enters the room code and a username. Sees a big buzzer button.

Rooms are ephemeral — they exist only while the server session is active. No persistent storage of game history.

🔌 Protocol

Digibuzzer uses Socket.io over WebSockets for realtime buzzer events. This means:

  • Caddy must pass WebSocket upgrade headers through to the container correctly
  • All players must be on the same Tailscale network (all traffic flows through the self-hosted instance)
  • No external relay — your server is the source of truth for buzzer timing
⚠️ Known Limitations
IssueDetail
French-only docsOfficial documentation is in French. UI labels are mostly French too, but interaction model is intuitive.
No persistent roomsRooms reset if the server restarts. No built-in session persistence.
