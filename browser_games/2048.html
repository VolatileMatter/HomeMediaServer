🎮 What Is This?

2048 was created by Gabriele Cirulli over a weekend in March 2014 and became an instant viral sensation. The premise is elegantly simple: a 4×4 grid, tiles numbered in powers of two, and a single mechanic — slide all tiles in one direction. When two equal tiles collide they merge into their sum. The goal is to create a tile with the value 2048.

What makes 2048 enduring is the planning required. Every move affects every tile on the board simultaneously, which means thinking two or three moves ahead is essential. The game is theoretically solvable through careful corner-anchoring strategies, but easy to derail with a single careless move. The tension between tight spaces and the right merge is quietly addictive.

The repository on GitHub is the original canonical codebase — not a clone. It's entirely static HTML/CSS/JS with no dependencies and requires no build step to run.

📖 How to Play
  • Use arrow keys (or swipe on mobile) to slide all tiles in one direction
  • Tiles with the same number merge when they collide — their sum becomes a new tile
  • A new tile (2 or 4) spawns each turn in a random empty cell
  • Reach the 2048 tile to "win" — but you can keep playing to chase higher tiles
  • The game ends when no moves remain
  • Classic strategy: keep your highest tile in a corner and build chains toward it

🏆 Leaderboard

The original 2048 codebase stores only a single best score in the browser's localStorage. There is no server-side score submission, no account-linked tracking, and no leaderboard API.

If a shared leaderboard is desired in the future, the recommended approach is to hook the game's Scoreboard object (in js/game_manager.js) to post scores to the hub's score API on game over. This would require a small patch to the source. Document that work here if undertaken.
