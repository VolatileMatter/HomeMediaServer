🐳 Step 1 — Clone & Docker Compose
cd ~/services
git clone https://github.com/Greghi2000/slotter-fullstack-app.git slotter
cd slotter

Create ~/services/slotter/docker-compose.yml (or adapt the one in the repo if present):

services:
  slotter:
    build: .
    container_name: slotter
    restart: unless-stopped
    ports:
      - "8780:3000"
    environment:
      - MONGODB_URI=mongodb://mongo:27017/slotter
    depends_on:
      - mongo

  mongo:
    image: mongo:7
    container_name: slotter-mongo
    restart: unless-stopped
    volumes:
      - slotter_mongo:/data/db

volumes:
  slotter_mongo:
docker compose up -d --build

Check the repo's README and existing docker-compose.yml (if present) for the correct environment variable names and port. The above is a reasonable starting point but may need adjusting to match the project's actual config.

🎮 What Is This?

Slotter is a fullstack fruit machine simulation that recreates the aesthetic and mechanics of 1980s British-style pub fruit machines. The key distinguishing mechanic is hold: after a spin, you can hold one or more reels in place and spin only the remaining ones. This creates genuine strategic depth — if two reels show matching fruit, holding them and spinning the third is a real decision with meaningful odds.

The fullstack architecture means each player has a registered account with a server-persisted virtual credit balance. Your balance survives closing the browser, switching devices, and server restarts (as long as the MongoDB volume is intact). This makes it more immersive than the static slot machine where the balance is browser-local.

The 1980s aesthetic — classic fruit symbols, retro UI chrome, the characteristic sound design — gives it a distinct character compared to modern video slots.

📖 How to Play
  • Register an account to get a starting virtual credit balance
  • Insert credits and pull the lever (or press Spin) to spin all reels
  • After a spin, use the Hold buttons to freeze individual reels
  • Spin again — only non-held reels spin; held reels stay in place
  • Matching symbols across the payline pay out virtual credits
  • Your balance is stored in your account — it persists between sessions
