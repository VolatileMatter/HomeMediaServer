Dashboard Setup Guide

// How to Deploy · Stats Agent · Control Server · Public Page //

Everything needed to get the live dashboard running: the Python stats agent, the control server for start/stop buttons, the public read-only status page, and Caddy routing.

🐉 The Dashboard tab shows a simulation of what the dashboard looks like. This tab explains how to make it actually work with live data from your server.
🏗️ Architecture: Two Dashboards
🌍 Public Status Page (read-only)

A stripped-down version — service status lights, recent downloads, player counts. No controls. Served publicly at status.yourdomain.com. File: monitor-public.html.

Anyone on the internet can see if your services are up, without being able to touch anything.

🔒 Private Control Panel

Full controls for starting/stopping services. Only accessible from your local network (or Tailscale). File: monitor-control.html. Caddy rule blocks external access to this page.

📊 Step 1 — Stats Agent (Port 9001)
stats_server.py — serves live CPU/RAM/disk/uptime data

Create D:\Server\Dashboard\stats_server.py:

from flask import Flask, jsonify
import psutil, time, json, os

app = Flask(__name__)

@app.route('/stats')
def stats():
    d = psutil.disk_usage('D:\\')
    return jsonify({
        'cpu':          psutil.cpu_percent(interval=1),
        'ram':          psutil.virtual_memory().percent,
        'disk_used':    d.percent,
        'disk_free_gb': round(d.free / 1e9, 1),
        'uptime':       int(time.time() - psutil.boot_time()),
        'active_sessions': load_sessions()
    })

def load_sessions():
    """Load active sessions from a JSON file that services write to."""
    path = 'D:/Server/Dashboard/sessions.json'
    try:
        if os.path.exists(path):
            return json.load(open(path))
    except: pass
    return []

if __name__ == '__main__':
    app.run(port=9001, host='0.0.0.0')

🎛️ Step 2 — Control Server (Port 9002, localhost only)
control_server.py — handles start/stop/restart commands

Create D:\Server\Dashboard\control_server.py:

from flask import Flask, request, jsonify
import subprocess

app = Flask(__name__)
ALLOWED = ['minecraft-vanilla','minecraft-modded','ark','ss13','audiobookshelf','calibre']

@app.route('/control/<service>/<action>', methods=['POST'])
def control(service, action):
    if service not in ALLOWED:
        return jsonify({'error': 'unknown service'}), 400
    if action in ('start','stop','restart'):
        subprocess.Popen(f'schtasks /run /tn "{service}-{action}"', shell=True)
        return jsonify({'ok': True, 'service': service, 'action': action})
    return jsonify({'error': 'unknown action'}), 400

@app.route('/control/user/block', methods=['POST'])
def block_user():
    data = request.json
    # Write to users.json and Windows Firewall
    return jsonify({'ok': True})

@app.route('/control/ip/block', methods=['POST'])
def block_ip():
    data = request.json
    ip = data['ip']
    subprocess.run(['netsh','advfirewall','firewall','add','rule',
                    f'name=BLOCK_{ip}','dir=in','action=block',f'remoteip={ip}'])
    return jsonify({'ok': True})

if __name__ == '__main__':
    app.run(host='127.0.0.1', port=9002)  # localhost ONLY — never expose this

nssm install ControlServer python "D:\Server\Dashboard\control_server.py"
nssm set ControlServer AppDirectory "D:\Server\Dashboard"
nssm start ControlServer
⚠️
Binds to 127.0.0.1 only. The control server is intentionally not accessible from the network. The dashboard page calls it from the browser, which only works when you're on the server itself (or via Tailscale where Caddy forwards to localhost).
👥 Step 3 — Active Sessions Tracking
sessions.json — track who is using what service

Each service that supports webhooks can write to this file when a user starts/stops a session. The dashboard reads it to show "Alice is watching Jellyfin."

# Example sessions.json:
[
  {"user": "alice",   "service": "Jellyfin",       "since": "8:42 PM"},
  {"user": "bob",     "service": "Minecraft",       "since": "7:15 PM"},
  {"user": "charlie", "service": "Audiobookshelf",  "since": "9:00 PM"}
]

# Jellyfin webhook setup (Settings → Webhooks → Add):
# Event: PlaybackStart / PlaybackStop
# URL: http://localhost:9001/session/update
# Body template: {"user":"{{NotificationUsername}}","service":"Jellyfin","action":"{{Event}}"}

# Add to stats_server.py:
@app.route('/session/update', methods=['POST'])
def session_update():
    data = request.json
    sessions = load_sessions()
    user = data.get('user')
    if data.get('action') == 'PlaybackStart':
        sessions = [s for s in sessions if s['user'] != user]
        sessions.append({'user': user, 'service': data.get('service','?'), 'since': time.strftime('%I:%M %p')})
    elif data.get('action') == 'PlaybackStop':
        sessions = [s for s in sessions if s['user'] != user]
    with open('D:/Server/Dashboard/sessions.json', 'w') as f:
        json.dump(sessions, f)
    return jsonify({'ok': True})

☀️ Step 6 — Sunshine Session Detection

Sunshine logs active connections. Poll the Sunshine API to detect when streaming is active and surface a warning on the home page and user homepage:

# In stats_server.py, add to the /stats endpoint:
import requests as req

def sunshine_active():
    try:
        r = req.get('https://localhost:47990/api/sessions',
                    verify=False, timeout=2,
                    auth=('admin', 'your-sunshine-password'))
        sessions = r.json()
        return len(sessions) > 0, sessions
    except:
        return False, []

# Add to stats response:
# 'sunshine_active': sunshine_active()[0]

The home tab and user homepage both poll http://localhost:9001/stats and check sunshine_active to show/hide the streaming-in-use warning banner.

🎛️ Step 7 — Owner News Feed

The user homepage pulls owner news from a simple JSON file you edit directly:

# D:\Server\Dashboard\news.json
[
  {
    "title": "ARK Server Up This Weekend",
    "body": "Spinning up the ARK server Saturday evening. Let me know if you want to join.",
    "date": "Mar 15, 2025",
    "pinned": false
  }
]

# Add new items at the top of the array — the homepage shows the most recent 3-5.
# The user homepage fetches: GET http://localhost:9001/news
# Add to stats_server.py:
@app.route('/news')
def news():
    try:
        return jsonify(json.load(open('D:/Server/Dashboard/news.json')))
    except:
        return jsonify([])

@app.route('/news', methods=['POST'])  # admin-only post
def post_news():
    items = json.load(open('D:/Server/Dashboard/news.json')) if os.path.exists('D:/Server/Dashboard/news.json') else []
    items.insert(0, request.json)
    json.dump(items[:20], open('D:/Server/Dashboard/news.json','w'), indent=2)
    return jsonify({'ok': True})
