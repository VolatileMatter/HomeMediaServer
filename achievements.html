⚙️ How the Achievement System Works
Architecture

The achievement server runs on port 9003 and maintains a JSON database of achievement definitions and per-user unlock records. Services send a webhook to it when a user hits a milestone. The homepage polls it every 60 seconds to display the community feed.

Service (Calibre-Web, ABS, EmulatorJS, etc.)
  → POST http://localhost:9003/api/achievement/trigger
    { "username": "alice", "trigger": "book_finished", "metadata": {...} }
  → Achievement server checks definitions
  → If trigger matches and requirements met → unlock for user
  → Stored in D:/Server/Achievements/unlocks.json
  → Homepage feed updates on next poll
Achievement server — achievement_server.py
# D:\Server\Achievements\achievement_server.py (port 9003)
from flask import Flask, request, jsonify
import json, os, time

app = Flask(__name__)
DEFS_FILE    = 'D:/Server/Achievements/definitions.json'
UNLOCKS_FILE = 'D:/Server/Achievements/unlocks.json'
USERS_BASE   = 'D:/Users'

def load(f): return json.load(open(f)) if os.path.exists(f) else {}
def save(f, d): json.dump(d, open(f,'w'), indent=2)

@app.route('/api/achievement/trigger', methods=['POST'])
def trigger():
    data     = request.json
    username = data.get('username')
    trigger  = data.get('trigger')
    meta     = data.get('metadata', {})
    if not username or not trigger: return jsonify({'error':'missing fields'}),400

    defs    = load(DEFS_FILE)
    unlocks = load(UNLOCKS_FILE)
    if username not in unlocks: unlocks[username] = {}
    newly_unlocked = []

    for ach_id, ach in defs.items():
        if trigger in ach.get('triggers', []) and ach_id not in unlocks[username]:
            # Check any required metadata thresholds
            req = ach.get('requires', {})
            ok = all(meta.get(k,0) >= v for k,v in req.items())
            if ok:
                unlocks[username][ach_id] = {
                    'date': time.strftime('%Y-%m-%d %H:%M'),
                    'meta': meta,
                    'public': True  # user can make private in settings
                }
                newly_unlocked.append({'id':ach_id, 'name':ach['name'], 'icon':ach.get('icon','🏅'), 'pts':ach.get('pts',10)})

    save(UNLOCKS_FILE, unlocks)
    # Also write to user's personal folder for Nextcloud visibility
    user_ach_path = f'{USERS_BASE}/{username}/achievements.json'
    save(user_ach_path, unlocks.get(username, {}))
    return jsonify({'ok':True, 'newly_unlocked': newly_unlocked})

@app.route('/api/achievements/recent')
def recent():
    unlocks = load(UNLOCKS_FILE)
    defs    = load(DEFS_FILE)
    all_recent = []
    for user, user_unlocks in unlocks.items():
        for ach_id, unlock_data in user_unlocks.items():
            if not unlock_data.get('public', True): continue
            ach = defs.get(ach_id, {})
            all_recent.append({'user':user,'name':ach.get('name',ach_id),'icon':ach.get('icon','🏅'),'date':unlock_data.get('date',''),'pts':ach.get('pts',10)})
    all_recent.sort(key=lambda x: x['date'], reverse=True)
    return jsonify(all_recent[:20])

@app.route('/api/achievements/user/<username>')
def user_achievements(username):
    unlocks = load(UNLOCKS_FILE)
    defs    = load(DEFS_FILE)
    user_data = unlocks.get(username, {})
    result = []
    for ach_id, ach in defs.items():
        unlock = user_data.get(ach_id)
        result.append({'id':ach_id,'name':ach['name'],'icon':ach.get('icon','🏅'),'desc':ach.get('desc',''),'pts':ach.get('pts',10),'earned':unlock is not None,'date':unlock.get('date') if unlock else None})
    return jsonify(result)

if __name__ == '__main__':
    app.run(port=9003, host='0.0.0.0')

🎮 RetroAchievements Integration
🐉
RetroAchievements (RA) is a community-run service that adds achievement systems to thousands of classic games — NES, SNES, GBA, N64, PS1, and more. EmulatorJS supports RA natively when given user credentials. Achievements unlock on the RA website AND in the local achievement feed.
User setup (in User Settings tab)

Each user adds their RetroAchievements.org username and API key in their User Settings → RetroAchievements section. These are stored as Authentik user attributes and passed to EmulatorJS by the save server via the RA credentials lookup:

# When EmulatorJS loads a game, the save server
# fetches the user's RA credentials from Authentik:
GET /api/v3/core/users/?username=[name]
→ user.attributes.ra_username
→ user.attributes.ra_token

# Passed to EmulatorJS as:
EJS_core = "genesis_plus_gx";
EJS_retroachievements = true;
EJS_retroachievementsUserName = "[ra_username]";
EJS_retroachievementsToken    = "[ra_token]";

Bridging RA unlocks → local achievement feed

Add a webhook handler so when RA unlocks happen in EmulatorJS, they also post to the local achievement server:

# In EmulatorJS save server (server.py) — add RA achievement relay:
@app.route('/api/ra_unlock', methods=['POST'])
def ra_unlock():
    data     = request.json
    username = request.headers.get('X-authentik-username', 'anonymous')
    # Relay to local achievement server:
    requests.post('http://localhost:9003/api/achievement/trigger', json={
        'username': username,
        'trigger':  'ra_achievement',
        'metadata': {
            'game_id':  data.get('gameId'),
            'ach_id':   data.get('achievementId'),
            'ach_name': data.get('achievementTitle'),
            'points':   data.get('points', 0)
        }
    })
    return jsonify({'ok': True})
📋 Adding Achievements to a New Service
Step 1: Define the achievement

Add an entry to D:\Server\Achievements\definitions.json. Choose a trigger name, set any metadata requirements, assign point value.

Step 2: Add the webhook call to the service

When the milestone is reached, the service (or its connected notification system) posts to the achievement server. For Jellyfin and ABS, use their built-in webhook systems. For custom apps, call it directly from the Python/Flask code.

# In any service's webhook handler or Python backend:
import requests
requests.post('http://localhost:9003/api/achievement/trigger', json={
    'username': current_user,     # from X-authentik-username header
    'trigger':  'trigger_name',   # matches definitions.json
    'metadata': { 'extra': 'data' }  # optional threshold data
})
Step 3: Test the unlock

Manually post to the trigger endpoint, then check that the achievement appears in the recent feed on the home tab and in the user's personal D:\Users\[username]\achievements.json.

curl -X POST http://localhost:9003/api/achievement/trigger \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","trigger":"book_finished","metadata":{}}'
Step 4: Add the achievement card to this tab

Add a card to the showcase section at the top of this page so all users can see what achievements are available. Include: icon, name, description, source service, and point value.

🔒 Privacy Controls

Users control their achievement privacy in User Settings → Privacy. They can:

  • Set all achievements to private (won't appear in community feed)
  • Set individual achievements to private (e.g. hide "Marathon Listener" but show gaming achievements)
  • Toggle "Show earned achievements publicly" (if off, only the icon shows, not which game/book)

The achievement server checks the public flag on each unlock record before including it in the /api/achievements/recent endpoint. The homepage feed only shows public achievements.
