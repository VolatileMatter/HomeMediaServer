Excalidraw — Collaborative Whiteboard

// Docker · WSL2 · Port 8086 · Authentik forward_auth //

Excalidraw is a virtual whiteboard for sketching diagrams, flowcharts, and freehand drawings. The official image is stateless — drawings are saved as .excalidraw files to each user's folder in D:\Users\[username]\diagrams\. Real-time collaboration rooms are ephemeral (in-memory only).

🐉 Excalidraw is stateless by design. Persistence is handled by exporting .excalidraw files and saving them to the user's Nextcloud-synced folder. There is no server-side auto-save.
📁 User Data Storage → SSO TAB

Excalidraw itself stores nothing on the server. Users export their work as .excalidraw files and save them to:

D:\Users\[username]\diagrams\     ← per-user diagram files
D:\Users\[username]\obsidian\     ← if using Obsidian integration

These folders are visible in Nextcloud and count toward each user's storage quota. See the SSO tab → User Data Storage for the full architecture.

💡
Tip: Teach users to use File → Save to disk to export their diagram as a .excalidraw file, then drag it into their Nextcloud folder. They can reload it later with File → Open.
🐳 Deployment — Docker in WSL2
Create the compose file

In your WSL2 Ubuntu terminal:

mkdir -p ~/services/excalidraw && cd ~/services/excalidraw

cat > docker-compose.yml << 'EOF'
services:
  excalidraw:
    image: excalidraw/excalidraw:latest
    container_name: excalidraw
    ports:
      - "8086:80"
    restart: unless-stopped
EOF
Start the container 
docker compose up -d
docker compose logs excalidraw   # confirm started

Excalidraw is now running at http://localhost:8086.

Open Windows Firewall for port 8086 
netsh advfirewall firewall add rule name="Excalidraw" dir=in action=allow protocol=TCP localport=8086
Auto-start on Windows boot

Task Scheduler → Create Task → Action:

wsl -d Ubuntu -- bash -c "cd ~/services/excalidraw && docker compose up -d"

Trigger: At startup, delay 2 minutes. Run whether logged on or not: ☑

🔐 Authentik — forward_auth Gate
Caddy configuration

Excalidraw has no built-in authentication. Protect it entirely via Caddy forward_auth:

# Add to your Caddyfile:
draw.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:8086
}
💡
Collaboration rooms use URL fragments like /#room=.... These work fine behind forward_auth — the room ID is never sent to the server, so Authentik does not interfere with the real-time sync.
Authentik Application entry
# Authentik admin → Applications → Create:
Name:       Excalidraw
Slug:       excalidraw
Launch URL: https://draw.yourdomain.com
Provider:   Proxy Provider (forward_auth single application)
  External host: https://draw.yourdomain.com
Access:     authentik-family + authentik-admins groups
💾 Saving & Loading Diagrams
📤 Saving a diagram
  1. Draw your diagram in Excalidraw
  2. File → Save to disk (or Ctrl+S)
  3. Save the .excalidraw file to D:\Users\[you]\diagrams\
  4. Nextcloud picks it up automatically and syncs it
📥 Loading a diagram
  1. Open Excalidraw in the browser
  2. File → Open
  3. Select your .excalidraw file
  4. Or drag the file directly onto the canvas
🤝 Real-time Collaboration

Click the Live collaboration button in the top bar. Share the generated room URL (contains a #room=... fragment) with anyone who has access to the server. All participants must be authenticated through Authentik first. Rooms are in-memory only — they vanish when all participants leave. Save your diagram before closing.

⚠️
Room data is not persisted on the server. If the last person leaves a room, all unsaved changes are lost. Everyone should export before closing.
🔧 Updating
# Pull latest image and restart:
cd ~/services/excalidraw
