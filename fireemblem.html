⚙️ Setup for In-Browser Games
Same architecture as Pokémon and Castlevania libraries

The Flask save server, EmulatorJS frontend, and per-user save architecture is identical. Extend the same server.py:

FIRE_EMBLEM_ROMS = 'D:/Emulation/FireEmblem'

@app.route('/fe/roms/<path:filename>')
def serve_fe_rom(filename):
    if 'user' not in session: return redirect('/login')
    return send_from_directory(FIRE_EMBLEM_ROMS, filename)

# Save IDs: fe-blazing, fe-sacred, fe-shadow-dragon, etc.

🎮 GameCube / Wii Games via Dolphin + Sunshine
Path of Radiance & Radiant Dawn

These run in Dolphin (GameCube/Wii emulator). Use the Sunshine streaming setup from the GameCube tab. Add each game as a Sunshine application:

# In Sunshine → Applications → Add:
Name:    Fire Emblem: Path of Radiance
Cmd:     "D:\Emulators\Dolphin\Dolphin.exe" -e "D:\Emulation\FireEmblem\gc\path-of-radiance.iso"

Name:    Fire Emblem: Radiant Dawn
Cmd:     "D:\Emulators\Dolphin\Dolphin.exe" -e "D:\Emulation\FireEmblem\wii\radiant-dawn.iso"

Save files for Dolphin are stored in %USERPROFILE%\Documents\Dolphin Emulator\GC\USA\ (GameCube) and \Wii\ (Wii) — these are shared by whoever accesses the Dolphin instance. For per-user saves over streaming, you'd need multiple Dolphin profiles, which is complex. Simpler: agree that Path of Radiance and Radiant Dawn are single-player experiences with one save each.
