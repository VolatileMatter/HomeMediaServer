Pokémon Game Library

// All Generations · Browser-Based · Per-User Saves · EmulatorJS //

Play every mainline Pokémon game in order, directly in the browser. Per-user save files so different players keep separate progress. No installation needed on the client side.

🐉
Legal note: EmulatorJS is open source. You need to own the games you emulate. ROMs are legal to create from cartridges you own; obtaining them from elsewhere is a grey area. This guide assumes you have your own ROMs.
🐉 V1 Goal: A web page at http://[server-ip]:8085/pokemon where users log in, pick a game, and play — with their save files tied to their username and persisting between sessions.
🧱 Architecture Overview

The stack is:

  • EmulatorJS — open-source browser emulator supporting GB, GBC, GBA, NDS, N64, and more. Runs entirely in the browser via WebAssembly.
  • Nginx or Caddy — serves the static HTML/JS files and the ROM files
  • A tiny backend (Python/Flask) — handles user login, save file upload/download, and per-user save storage
💡
EmulatorJS save files: EmulatorJS stores saves in the browser's IndexedDB by default. For per-user server-side saves, we intercept the save events and POST them to the backend, then restore them on load. This is the key to cross-device, per-user saves.

Create D:\Emulation\Pokemon\server.py:

from flask import Flask, request, jsonify, send_from_directory, session, redirect
from werkzeug.security import check_password_hash, generate_password_hash
import os, json

app = Flask(__name__, static_folder='static')
app.secret_key = 'change-this-to-a-random-string'

SAVES_DIR = 'D:/Emulation/Pokemon/saves'
ROMS_DIR  = 'D:/Emulation/Pokemon'
USERS_FILE = 'D:/Emulation/Pokemon/users.json'

def load_users():
    if os.path.exists(USERS_FILE):
        with open(USERS_FILE) as f: return json.load(f)
    return {}

def save_users(users):
    with open(USERS_FILE, 'w') as f: json.dump(users, f)

@app.route('/api/login', methods=['POST'])
def login():
    data = request.json
    users = load_users()
    u = users.get(data['username'])
    if u and check_password_hash(u['password'], data['password']):
        session['user'] = data['username']
        return jsonify({'ok': True})
    return jsonify({'ok': False, 'error': 'Invalid credentials'}), 401

@app.route('/api/logout', methods=['POST'])
def logout():
    session.pop('user', None)
    return jsonify({'ok': True})

@app.route('/api/me')
def me():
    return jsonify({'user': session.get('user')})

@app.route('/api/save/<game_id>', methods=['POST'])
def save(game_id):
    if 'user' not in session: return jsonify({'error': 'not logged in'}), 401
    user_dir = os.path.join(SAVES_DIR, session['user'])
    os.makedirs(user_dir, exist_ok=True)
    data = request.get_data()
    with open(os.path.join(user_dir, f'{game_id}.sav'), 'wb') as f:
        f.write(data)
    return jsonify({'ok': True})

@app.route('/api/save/<game_id>', methods=['GET'])
def load_save(game_id):
    if 'user' not in session: return jsonify({'error': 'not logged in'}), 401
    path = os.path.join(SAVES_DIR, session['user'], f'{game_id}.sav')
    if os.path.exists(path):
        with open(path, 'rb') as f:
            return f.read(), 200, {'Content-Type': 'application/octet-stream'}
    return '', 404

@app.route('/roms/<path:filename>')
def serve_rom(filename):
    if 'user' not in session: return redirect('/login')
    return send_from_directory(ROMS_DIR, filename)

@app.route('/')
@app.route('/login')
@app.route('/play')
def serve_static():
    return send_from_directory('static', 'index.html')

if __name__ == '__main__':
    app.run(port=8085, host='0.0.0.0')

For each game card, clicking "Play" calls a function that:

  1. Checks if user is logged in (redirect to login if not)
  2. Fetches the save file from /api/save/[game_id]
  3. Initializes EmulatorJS with the ROM path, core, and the fetched save data
  4. Sets up a save callback that POSTs save data to /api/save/[game_id] on every in-game save
🎛️ EmulatorJS Save Integration
<!-- In your play page -->
<div id="game"></div>
<script src="/emulatorjs/data/loader.js"></script>
<script>
const gameId = 'emerald'; // from URL param
const systemCore = 'mgba';

// Load existing save from server
async function initGame() {
    let saveData = null;
    try {
        const r = await fetch(`/api/save/${gameId}`);
        if (r.ok) saveData = await r.arrayBuffer();
    } catch(e) {}

    EJS_player = '#game';
    EJS_gameUrl = `/roms/gba/pokemon-emerald.gba`;
    EJS_core = systemCore;
    EJS_gameID = gameId;
    EJS_language = 'en-US';

    // Pre-load save file if exists
    if (saveData) {
        EJS_saveStateOnExit = false;
        EJS_onGameStart = function() {
            EJS_emulator.gameManager.setSaveFile(new Uint8Array(saveData));
        };
    }

    // Auto-save every 30 seconds + on manual save
    EJS_onSaveState = async function(saveData) {
        await fetch(`/api/save/${gameId}`, {
            method: 'POST',
            body: saveData,
            headers: { 'Content-Type': 'application/octet-stream' }
        });
    };
}

initGame();
</script>
🔐 SSO Integration → SSO TAB

V1 uses the simple username/password system above. For SSO, the Flask session can be replaced with Authentik forward auth — when a user authenticates via Authentik, the username header is passed to Flask which uses it to look up the correct save directory. The X-authentik-username header replaces the session-based login entirely.
