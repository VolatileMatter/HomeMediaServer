Nextcloud

// Self-Hosted Google Drive · Port · XAMPP on Windows · OIDC Native //

Nextcloud is the gold standard for self-hosted cloud storage — your own Google Drive for file sync, sharing, calendar, contacts, and photos, all on hardware you control. It also serves as the unified storage quota system for every per-user app on this server.

⚙️ Step 1 — Install XAMPP & Nextcloud NATIVE WINDOWS · NO DOCKER
🔐 Install XAMPP

Download from apachefriends.org. During install, select: Apache, MySQL, PHP, phpMyAdmin. Install to C:\xampp.

🔐 Start Apache and MySQL — enable as Windows Services

Open XAMPP Control Panel → click Start next to Apache and MySQL. Check the "Svc" checkbox for both so they auto-start on boot.

🔐 Create the Nextcloud database

Open http://localhost/phpmyadmin → Click "New" → database name nextcloud, collation utf8mb4_general_ci → Create.

Go to User accounts → Add user account → username nextcloud, host localhost, strong password → check "Grant all privileges on database nextcloud" → Go.

🔐 Configure PHP for large file uploads

Edit C:\xampp\php\php.ini:

memory_limit = 512M
upload_max_filesize = 16G
post_max_size = 16G
max_execution_time = 3600
date.timezone = America/Los_Angeles

Enable extensions (remove the leading semicolon):

extension=intl
extension=gd
extension=exif

Restart Apache after saving.

🔐 Download and install Nextcloud

Get the zip from nextcloud.com/install. Extract the nextcloud folder to C:\xampp\htdocs\nextcloud.

🔐 Run the installer at http://localhost/nextcloud 
Admin username:   your-admin-name
Admin password:   (strong password)
Data folder:      D:\NextcloudData   ← your large internal drive
Database:         MySQL
DB user:          nextcloud
DB password:      (from step above)
DB name:          nextcloud
DB host:          localhost

Click Install — takes 1–2 minutes.

🔐 Open Windows Firewall for LAN access 
netsh advfirewall firewall add rule name="Nextcloud" dir=in action=allow protocol=TCP localport=80

Access from other devices on the LAN: http://[server-ip]/nextcloud

⚡ Step 3 — Performance Tuning

After initial setup, run these database maintenance commands to improve performance for large libraries:

cd C:\xampp\htdocs\nextcloud
php occ maintenance:mode --on
php occ db:add-missing-indices
php occ maintenance:mode --off

For large file transfers, enable chunked uploads in the desktop client: Settings → Advanced → Chunked uploads.

🗂️ Step 4 — External Storage Mount ()

Every per-user app on this server stores data in \[username]\. Mount this into Nextcloud so users can see their app data in the file browser and it counts toward their quota.

Enable the External Storage app
cd C:\xampp\htdocs\nextcloud
php occ app:enable files_external
Add the per-user mount using the %user% template
# Creates an "App Data" folder in every user's Nextcloud
# that maps to D:\Users\[their-username]\ on disk:
php occ files_external:create \
  --config=datadir="\%user%" \
  --apply-to-all-users \
  "App Data" local null

# Verify:
php occ files_external:list
⚠️
Username matching: Nextcloud resolves %user% to the user's Nextcloud login name. This must exactly match the folder name in \, which in turn must match their Authentik username (Gmail prefix). All three must be consistent: Authentik username = alice, Nextcloud username = alice, folder = \alice\.
Enable quota counting for external storage
# In Nextcloud admin → External Storage → edit the App Data mount:
# ☑ Enable sharing
# ☑ Enable encryption
# ☑ Check changes: "Once every direct access"

# Force quota to include external storage:
php occ config:app:set files_external allow_user_mounting --value=yes
php occ config:system:set quota_include_external_storage --value=true --type=boolean
Set per-user quotas
# Set quota for individual users (covers Nextcloud files + App Data):
php occ user:setting alice files quota "50 GB"
php occ user:setting bob   files quota "10 GB"
php occ user:setting dave  files quota "1 GB"
php occ user:setting john  files quota "5 GB"

# Check current usage for all users:
php occ user:report
🔐 Step 5 — SSO via Authentik (OIDC Native)

Nextcloud uses the user_oidc app for native OpenID Connect integration with Authentik. This is a full OIDC flow — not Caddy forward_auth — so users get a proper "Sign in with Authentik" button.

In Authentik: create an OAuth2/OIDC provider
# Authentik admin → Applications → Providers → Create
# Type: OAuth2/OpenID Provider
# Name: Nextcloud
# Client type: Confidential
# Redirect URIs: /apps/user_oidc/code
# Scopes: openid, profile, email
In Authentik: create an Application
# Applications → Create
# Name: Nextcloud
# Slug: nextcloud
# Provider: (the one you just created)
# Launch URL:
In Nextcloud: install and configure user_oidc
# Install the app:
php occ app:enable user_oidc

# Or install via: Nextcloud admin → Apps → search "OpenID Connect user backend"

Then go to Admin → OpenID Connect → Add provider:

FieldValue
Identifierauthentik
Client ID(from Authentik provider)
Client Secret(from Authentik provider)

Use unique UID☑ (maps to Authentik username)
⚠️
Username mapping: Ensure the "Unique identifier claim" is set to preferred_username so Nextcloud usernames match Authentik usernames, which must also match \[username]\ folder names.
Test SSO login in an incognito window

Open in a private browser tab → click "Log in with Authentik" → sign in with a Google account that has been provisioned in Authentik. You should land in Nextcloud as an authenticated user.

Verify user provisioning and quota

Confirm the user account was auto-created in Nextcloud with the correct username (matching their Authentik username). Check in Authentik admin at → Directory → Users if needed. Then confirm their App Data external mount shows their \[username]\ folder.

Verify the Caddyfile entry 
caddy reload --config

Confirm loads over HTTPS with a valid certificate.

How to update Nextcloud

Nextcloud's built-in updater is the safest upgrade path for XAMPP installs:

# Option A — Web updater (easiest):
# Go to Admin → Overview → there will be an update notification
# Click "Open updater" and follow the wizard

# Option B — CLI updater:
cd C:\xampp\htdocs\nextcloud
php updater\updater.phar

# After any update, run:
php occ upgrade
php occ db:add-missing-indices
php occ maintenance:mode --off
⚠️
Never skip major versions. Nextcloud must be upgraded sequentially through each major version (e.g. 27 → 28 → 29, not 27 → 29 directly). Check the release notes before upgrading.
⚠️ Known Breakpoints
🔴 PHP version mismatch after XAMPP update
If you update XAMPP independently of Nextcloud, the new PHP version may not be compatible with the currently installed Nextcloud version. Always check Nextcloud's PHP requirements before updating XAMPP. Recovery: downgrade PHP in XAMPP or upgrade Nextcloud to a compatible version.
🟡 External Storage mount breaks after folder restructure
If \ is ever moved or renamed, the External Storage mounts will silently fail — users will see an empty "App Data" folder with no error. Re-run php occ files_external:list and update the datadir config to the new path.
🟡 SSO redirect URI breaks after domain change
If you change your domain, update the redirect URI in both the Authentik provider ( → Applications → Nextcloud → Provider) and in Nextcloud's user_oidc settings (Admin → OpenID Connect → Discovery endpoint). The callback URI must be exactly: /apps/user_oidc/code
🟢 Background jobs warning in admin Overview
Nextcloud may warn that background jobs haven't run recently. Set up a Windows Scheduled Task to run php C:\xampp\htdocs\nextcloud\cron.php every 5 minutes under the SYSTEM account. This handles thumbnail generation, notifications, and file scans.

🎯 How Nextcloud Achievements Work

Nextcloud does not have a native webhook system for file events, so achievements are tracked via polling — the achievement server periodically queries the Nextcloud database or occ CLI to check quota usage and file counts per user. There are no real-time triggers; achievements are awarded on the next polling cycle after a threshold is crossed.

💡
Quota data source: Use php occ user:report or query the oc_accounts_data table directly in MySQL for per-user used/quota values. The achievement server can run this on a schedule and fire award events when thresholds are met.

💾 Where Data Lives

Nextcloud data is split across two locations: the app itself in XAMPP, and user files on the data drive. The External Storage mount adds a third logical location — but it's just a pointer to \ which is already the source of truth for all per-user app data.

App files
C:\xampp\htdocs\nextcloud
PHP + XAMPP
C:\xampp\
Nextcloud data
D:\NextcloudData
MySQL database
C:\xampp\mysql\data\nextcloud
User app data (ext. mount)
\[username]\
💡
External Storage is read-through, not a copy. The "App Data" folder visible in each user's Nextcloud is a live mount of \[username]\. Files aren't duplicated. Backing up \ covers all app data for all users.

🛠️ Common Issues
🔴 "Service unavailable" / 503 after Windows restart
Apache or MySQL didn't start automatically. Check that both are registered as Windows Services (the "Svc" checkbox in XAMPP Control Panel). Fix:
  • Open XAMPP Control Panel → start Apache and MySQL manually
  • If the "Svc" checkbox is unchecked, check it to re-register as a service
  • Or from PowerShell: Start-Service Apache2.4; Start-Service mysql
🔴 Large file upload fails / times out
PHP limits weren't applied or Apache didn't restart after the php.ini change. Verify C:\xampp\php\php.ini has upload_max_filesize = 16G, post_max_size = 16G, and max_execution_time = 3600. Then restart Apache from the XAMPP Control Panel and try again.
⚠️ SSO login fails / redirect error
Almost always a redirect URI mismatch. Check:
  • Authentik provider → Redirect URIs matches exactly: /apps/user_oidc/code
  • Nextcloud user_oidc → Discovery endpoint is reachable from the server
  • Restart Apache after any config change: Stop-Service Apache2.4; Start-Service Apache2.4
⚠️ "App Data" external mount shows empty for a user
Either the user's \[username]\ folder doesn't exist yet, or there's a username mismatch. Check:
  • Verify the folder exists: Test-Path "\[username]"
  • Verify Nextcloud username matches Authentik username exactly (case-sensitive)
  • Run php occ files:scan [username] to force a rescan
⚠️ Nextcloud not accessible from outside the LAN
Check in order:
  • Is Apache running? → XAMPP Control Panel or Get-Service Apache2.4
  • Is Caddy running and routing to localhost:80?
  • Is the Cloudflare DNS record correct for ?
  • Check Caddy logs: Get-Content C:\caddy\caddy.log -Tail 50
💡 Admin overview shows warnings about missing indices or background jobs
Run php occ db:add-missing-indices for index warnings. For background job warnings, create a Windows Scheduled Task to run php C:\xampp\htdocs\nextcloud\cron.php every 5 minutes as SYSTEM.
