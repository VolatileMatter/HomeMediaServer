📦 Step 1 — Install Calibre Desktop NATIVE · REQUIRED
Why Calibre Desktop is required

Calibre-Web is a browser frontend — it does not manage the library itself. Calibre Desktop creates and maintains metadata.db, which is the database Calibre-Web reads from. You need both installed.

Download and install Calibre

Download from calibre-ebook.com. Install normally. When it asks where your library is, point it at your internal drive: D:\Books. This is where metadata.db will live.

Import your existing ebooks

In Calibre: Add books → select your ebook folder. Calibre will copy/link them into the library structure and update metadata.db. Run Edit Metadata → Download Metadata and Covers to auto-fill author, description, and cover art from online sources.

Set Calibre to auto-start minimized to tray

Calibre Preferences → Behavior → check "Minimize to system tray". Then add a shortcut to shell:startup so it launches on boot. Calibre-Web needs Calibre's library to remain accessible — keeping Calibre open ensures metadata.db doesn't get locked.

💡
Alternative: If Calibre Desktop in the background bothers you, just close it after any library changes. Calibre-Web reads metadata.db directly — Calibre doesn't need to be running for Calibre-Web to serve books. Just open Calibre whenever you want to add or edit books.

Calibre-Web's native OAuth integration uses "Login with Google" as the OAuth flow. Authentik serves as the OAuth provider. The redirect URI must match exactly what's set in Authentik.
Alternative: Caddy forward_auth (simpler)

If the native OAuth approach is fiddly, put Calibre-Web behind Caddy with Authentik forward auth instead. This gates access at the proxy layer without touching Calibre-Web's internal auth:

# Caddyfile:
books.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:8083
}

With this approach, users are already authenticated by the time they reach Calibre-Web. Calibre-Web's own login page is bypassed. Create matching Calibre-Web accounts for each user with their Google username.

Link existing Calibre-Web accounts to Authentik users

If users already have Calibre-Web accounts (with reading progress), match them to their Google email so progress is preserved when they switch to SSO login:

🐉 What Is Calibre, and How Does It Work?
The two-piece setup: Calibre Desktop + Calibre-Web

This server runs two separate programs that work together. Calibre Desktop is the library manager — it's the app you open to add books, fix metadata, convert file formats, and keep everything organized. It stores everything about your library in a single file called metadata.db. Calibre-Web is the web server — it reads that same metadata.db file and turns your library into a website that anyone on the server can browse, read, and download from. You manage books in Calibre Desktop; everyone reads them through Calibre-Web.

Calibre Desktop — what it does
FeatureWhat it means in plain language
Library managementKeeps all your books in an organized folder structure with a searchable database. You add a book once and it's tracked forever — author, title, series, tags, rating, publication date, cover art, description.
Metadata editingYou can fix any book's info — correct a misspelled author name, add series info (e.g. "Book 3 of The Stormlight Archive"), add genre tags, swap out the cover art. Changes apply immediately across the whole library.
Download metadata & coversRight-click any book → "Download metadata and covers" and Calibre will look it up online (Amazon, Goodreads, OpenLibrary, etc.) and fill in everything automatically. Works for most books instantly.
Format conversionConvert any book from one format to another — EPUB to MOBI for Kindle, PDF to EPUB for better reading, etc. Calibre handles ~20 input and output formats. Quality varies but it works well for most ebooks.
Send to KindleRight-click a book → "Send to device" or "Send by email" and it goes wirelessly to a connected Kindle. Calibre handles the conversion to the right format automatically.
Catalog & searchFull-text search across your entire library, filter by tag/series/author/rating/format, build smart saved searches. You can find "all unread Brandon Sanderson books tagged fantasy that I've rated 4+ stars" in seconds.
News downloadCalibre can download news articles from hundreds of sources (NYT, BBC, etc.) and package them as an ebook to send to your Kindle daily. Optional feature.
Plugin systemCalibre has hundreds of community plugins — DRM removal (for books you own), better metadata sources, store integrations, reading progress sync, and more.
Content server (built-in)Calibre Desktop has its own basic web server built in, but it requires the GUI to stay open. This is why we use Calibre-Web instead — it runs as a proper background service.
Calibre-Web — what it adds on top
FeatureWhat it means in plain language
Browser-based readingRead EPUBs and PDFs directly in any web browser — no app download needed. The built-in reader supports bookmarks, progress tracking, font size changes, and dark mode.
Per-user accountsEach person has their own account with their own reading progress, bookmarks, and shelves. Your "currently reading" list doesn't mix with anyone else's.
ShelvesPersonal reading lists that live in Calibre-Web only — they don't touch the master library. You can make a "Want to Read" shelf, a "Favorites" shelf, anything. Shelves can be public (visible to others) or private.
OPDS catalogExposes your library as an OPDS feed — a standard format that e-reader apps (Moon+ Reader, KyBook, Librera, etc.) can connect to directly and browse/download from, without ever opening a browser.
Kobo syncKobo e-readers can sync directly with Calibre-Web over the network — reading progress, bookmarks, and sideloaded books stay in sync automatically.
Send to KindleAny user can click "Send to Kindle" on any book and it gets emailed to their Kindle address automatically. Calibre-Web handles the conversion and delivery.
User permissionsEach user account has granular permission settings — you can allow browsing and shelves but prevent uploading or editing, which is exactly how the family accounts are configured on this server.
Reading statsTracks when you started a book, your current progress, and when you finished. This is the data the achievement polling script reads to award XP.
Virtual library restrictionsYou can limit what a user sees to a subset of the library — e.g. show only books tagged "Family Friendly" or only a specific author's works. Used on this server to prevent users from being overwhelmed by the full library.
🐉
The key thing to understand: Calibre Desktop owns the library. Calibre-Web only reads it. If you add a book in Calibre Desktop, it shows up in Calibre-Web after a database reconnect. If someone "adds a book to a shelf" in Calibre-Web, that only changes Calibre-Web's own app.db — the master library and metadata.db are never touched.
🔐 Secure Multi-User Architecture
How the family setup works — the full picture

The goal is to let family members browse the library and keep personal reading lists without any risk of accidentally changing or deleting books. This is achieved through three layers of protection working together:

LayerWhat it doesWhere it's configured
1. Read-only filesystemThe D:\Books\ folder is mounted into Calibre-Web's service account with read-only permissions. Even if someone found a way to trigger a file operation, the OS would reject it. The master library files cannot be deleted or modified by Calibre-Web.Windows folder permissions / NSSM service account
2. Authentik SSO + reverse proxy headersAuthentik authenticates the user (via Google). Caddy passes the verified username to Calibre-Web as a trusted HTTP header (X-Remote-User). Calibre-Web trusts this header instead of its own login form — users never see a Calibre-Web password prompt.Caddy → copy_headers X-authentik-username; Calibre-Web Admin → Reverse Proxy Authentication
3. Calibre-Web user permissionsEach user account has a specific permission set that allows reading and shelf management but blocks everything destructive.Calibre-Web Admin → User Management
User permissions — what's enabled and disabled for family accounts
PermissionFamily usersWhy
View books / browse library✅ EnabledCore function
Download books✅ EnabledUsers can save books to their devices
Read in-browser✅ EnabledCore function
Access to Shelves✅ EnabledPersonal reading lists
Add / remove books to Shelves✅ EnabledUsers can curate their own lists
Send to Kindle✅ EnabledEach user sets their own Kindle address
Edit book metadata❌ DisabledWould alter the shared library for everyone
Delete books❌ DisabledPermanent and irreversible
Upload books❌ DisabledKeeps the master library owner-controlled
Admin panel access❌ DisabledAdmin is owner-only
Virtual Libraries — keeping the library manageable

The full library may have 1,000+ books. Rather than showing every user everything, each user's Calibre-Web profile can be restricted to a Virtual Library — a filtered view of the library based on tags, authors, or any other Calibre metadata. This is set per-user in Admin → Users → [user] → Restriction.

Example restrictionWhat the user sees
tags:"Family Friendly"Only books tagged "Family Friendly" in Calibre Desktop
series:"Stormlight Archive"Only books in that specific series
rating:>=4Only books rated 4 stars or higher
(no restriction)Full library — for admin / owner accounts

Users can always toggle between their restricted view and their personal Shelves in the Calibre-Web sidebar. Shelves are completely personal and don't affect the library view or anyone else's data.

The master library is always safe. Every action a family member can take either reads from the library (no changes) or writes only to their personal area in app.db. The book files and metadata.db are never written to by Calibre-Web under any normal operation.

🔄 Updates
How to update Calibre-Web

Calibre-Web does not auto-update. Check periodically, especially if the login flow or reading experience breaks.

Check current version and stop the service 
nssm stop CalibreWeb

# Check current version in the Calibre-Web Admin panel:
# Admin → About
Update via pip 
pip install --upgrade calibreweb
Restart the service and verify 
nssm start CalibreWeb

Open http://localhost:8083 and confirm the new version is running. Test SSO login with an incognito window.

How to update Calibre Desktop

Calibre shows an update notification in the toolbar when a new version is available. Click it or download from calibre-ebook.com. Calibre updates preserve your library and metadata.db completely — no migration needed.

How to update Readarr

Readarr has a built-in updater: System → Updates → Check for Updates. It will download and apply the update, then restart itself automatically.

⚠️ Potential Breakpoints — Watch For These
🔴 SSO / OAuth login breaks after a Calibre-Web update
Calibre-Web's OAuth implementation can shift between versions. After any update, test the SSO login from an incognito window before assuming everything is fine. If it breaks, the fallback is the local admin account. Check the Calibre-Web GitHub issues for OAuth-related regressions in the release notes.
🟡 Calibre-Web loses track of the library after a Calibre update
If Calibre Desktop updates and moves or rebuilds metadata.db, Calibre-Web may fail to find the library. Go to Admin → Edit Basic Configuration and re-confirm the library path is still D:\Books. If metadata.db moved, point Calibre-Web to the new location.
🟡 Readarr downloads land in the library but don't appear in Calibre-Web
Calibre-Web doesn't watch the filesystem in real time. After Readarr downloads a book, you need to trigger a library scan in Calibre-Web: Admin → Reconnect Database. Or set up the Readarr post-download script (see First Time Setup) to trigger this automatically via API.
🟡 Authentik redirect URI mismatch after a domain change
If you change your domain name, update the Redirect URI in both Authentik (the provider config) and Calibre-Web's OAuth settings simultaneously. The exact URI format must match: https://books.yourdomain.com/oauth/authorize/google/callback. Mismatches cause a silent redirect error.
🟡 metadata.db gets locked when both Calibre Desktop and Calibre-Web try to access it
SQLite has limited concurrent write support. If Calibre Desktop is actively doing a large metadata update while Calibre-Web is serving requests, you may see brief errors. The solution is to close Calibre Desktop before doing bulk edits, or do large library operations during off-hours. Calibre-Web is read-mostly so this is rarely a problem in practice.
🟢 app.db grows over time with reading history
Calibre-Web's app.db accumulates reading progress, bookmark data, and activity logs. It stays under 50 MB in almost all cases and doesn't need manual maintenance. If you ever need to clean it, export user reading lists first, then use the admin panel's database tools.

🎯 How Book Achievements Work

Calibre-Web does not have a built-in webhook system like Jellyfin. Reading progress and completion events must be detected by polling Calibre-Web's SQLite database (app.db) or by adding a lightweight middleware layer that watches for status changes. The recommended approach is a scheduled polling script that runs every few minutes and calls the achievement server when milestones are hit.

Polling script checks app.db every 5 minutes
  → Finds user "alice" has read_status=1 (finished) for a new book
  → POST http://localhost:9003/api/achievement/trigger
    { "username": "alice", "trigger": "book_finished", "metadata": { "title": "..." } }
  → Achievement server grants "Bookworm" achievement + 10 XP
  → Stored in D:\Users\alice\achievements.json
  → Appears in homepage community feed
🔌 Polling Script Setup
Step 1 — Create the polling script
# D:\Server\Achievements\calibre_poll.py
import sqlite3, requests, json, os, time
from datetime import datetime

DB_PATH      = 'D:/Books/app.db'
ACH_ENDPOINT = 'http://localhost:9003/api/achievement/trigger'
STATE_FILE   = 'D:/Server/Achievements/calibre_state.json'

def load_state():
    if os.path.exists(STATE_FILE):
        return json.load(open(STATE_FILE))
    return {}

def save_state(state):
    json.dump(state, open(STATE_FILE, 'w'))

def poll():
    state = load_state()
    conn  = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row

    rows = conn.execute('''
        SELECT u.name, b.title, b.author_sort, brl.read_status, brl.last_modified
        FROM book_read_link brl
        JOIN books b ON b.id = brl.book_id
        JOIN user  u ON u.id = brl.user_id
        WHERE brl.read_status = 1
    ''').fetchall()

    for row in rows:
        key = f"{row['name']}::{row['title']}"
        if key not in state:
            # New completion — trigger achievement
            requests.post(ACH_ENDPOINT, json={
                'username': row['name'],
                'trigger':  'book_finished',
                'metadata': {
                    'title':  row['title'],
                    'author': row['author_sort'],
                }
            }, timeout=5)
            state[key] = str(datetime.now())
            print(f"Triggered: {row['name']} finished '{row['title']}'")

    save_state(state)
    conn.close()

if __name__ == '__main__':
    while True:
        poll()
        time.sleep(300)  # poll every 5 minutes

What lives where

The book files themselves (D:\Books\) are shared — every user reads from the same library. Calibre-Web's app.db is also shared but stores data per user account inside it (reading progress, bookmarks, shelves). Per-user personal data (exports, backups, uploaded books) lives in each user's own folder under D:\Users\.

D:\Books\
  metadata.db            ← Calibre's library index (~50–500 MB depending on library size)
  app.db                 ← Calibre-Web users, progress, bookmarks (~1–50 MB)
  [Author]\[Book]\       ← The actual ebook files (as large as your library is)

D:\Users\alice\calibre\
  progress_export.json   ← Reading progress snapshot (exportable for backup)
  uploads\               ← Books alice has personally uploaded to the shared library
  reading_lists\         ← Exported shelves / reading lists
📊 Per-User Data Breakdown

A power user who reads 500 books with full bookmarks and shelves will have under 5 MB of personal data in app.db. The database as a whole is tiny relative to the book library.

Counts toward Nextcloud quota?

The shared app.db and metadata.db do not count toward any individual user's Nextcloud quota — they're server-side shared files. What does count toward a user's quota is anything they store in D:\Users\[name]\calibre\: personal progress exports, uploaded books, and reading list exports. These are typically negligible in size unless a user is uploading large PDF books.

💾 Exporting Reading Progress to the User Folder
Progress export endpoint (add to stats_server.py)

This lets reading progress be backed up alongside everything else in the user's Nextcloud-synced folder:

# Add to D:\Server\Dashboard\stats_server.py:
@app.route('/calibre/export/<username>')
def export_calibre_progress(username):
    import sqlite3, json
    conn = sqlite3.connect('D:/Books/app.db')
    conn.row_factory = sqlite3.Row
    progress = conn.execute('''
        SELECT b.title, b.author_sort, brl.read_status, brl.last_modified,
               brl.read_pos_epub_cfi, brl.kobo_reading_state_id
        FROM book_read_link brl
        JOIN books b ON b.id = brl.book_id
        JOIN user  u ON u.id = brl.user_id
        WHERE u.name = ?
    ''', (username,)).fetchall()

    out_path = f'D:/Users/{username}/calibre/progress_export.json'
    os.makedirs(os.path.dirname(out_path), exist_ok=True)
    with open(out_path, 'w') as f:
        json.dump([dict(r) for r in progress], f, indent=2, default=str)
    return jsonify({'ok': True, 'books': len(progress)})

Call this endpoint from a nightly scheduled task to keep each user's progress export up to date: http://localhost:[stats-port]/calibre/export/alice

🛠️ Common Issues
❌ Calibre-Web shows a blank library or "No books found"
  • Verify the library path is correct: Admin → Edit Basic Configuration → confirm the path is D:\Books (or wherever metadata.db lives)
  • Confirm metadata.db exists in that folder and is not zero bytes
  • Check that the CalibreWeb service account has read permission on D:\Books
  • Try Admin → Reconnect Database to force a re-read
❌ "Sign in with Authentik / Google" button doesn't appear or fails
Check in order:
  • Admin → Edit Basic Configuration → Feature Configuration — confirm OAuth is enabled and the Client ID/Secret are filled in
  • Confirm the Redirect URI in Authentik exactly matches https://books.yourdomain.com/oauth/authorize/google/callback
  • If using Caddy forward_auth instead of native OAuth, the "Sign in" button won't appear — users are redirected automatically by Caddy
  • Restart the CalibreWeb service after any OAuth config change: nssm restart CalibreWeb
❌ Calibre-Web service won't start
Check the NSSM service log: 
nssm status CalibreWeb
# Look in: C:\ProgramData\nssm\ or wherever NSSM writes output
# Or check Windows Event Log → Application → source: CalibreWeb
Common causes: Python path changed after an update (re-check the NSSM service path with where cps), or a pip dependency conflict (pip install --upgrade calibreweb to fix).
⚠️ New books downloaded by Readarr don't appear in Calibre-Web
Calibre-Web doesn't auto-scan. Trigger a manual refresh:
  • Admin → Reconnect Database — reloads metadata.db
  • Or call the API: POST http://localhost:8083/api/admin/libraries/scan with your API key
  • Note: Readarr downloads to D:\Books, but Calibre-Web reads from metadata.db. The file being on disk isn't enough — Calibre Desktop must also have imported/indexed it first. Run Calibre Desktop → Add Books to pick up the file, then reconnect in Calibre-Web.
⚠️ Reading progress is lost after SSO migration
When switching from local Calibre-Web accounts to Authentik SSO, progress is tied to the old username. To preserve it:
  • In Admin → Users, edit each existing user and set their Email to their exact Gmail address
  • When they log in via Authentik, Calibre-Web matches on email and uses the existing account
  • If the username format changed, use the SQLite migration script in the SSO tab → Save Migration section
⚠️ Send-to-Kindle fails silently
Check in order:
  • Verify the Gmail App Password is still valid (Google occasionally revokes these)
  • Confirm the sender Gmail address is in the Amazon "Approved Personal Document E-mail List" for the user's Kindle account
  • Check that the user's @kindle.com address is set correctly in their Calibre-Web profile
  • Test SMTP manually: Admin → Edit E-Mail Settings → Test E-Mail
🟢 In-browser EPUB reader is slow or doesn't load
The built-in reader (Read Online) works best for EPUBs under ~50 MB. Very large EPUBs with heavy images may time out. For large books, download the file and read in a dedicated app instead. PDFs always perform better downloaded than read in-browser.
