Calibre Book Library Server

// Calibre-Web · Content Server · No Docker Required //

Set up a self-hosted ebook library accessible from any device on your network (and beyond). Supports EPUB, MOBI, PDF, and syncs with Kindle.

🐉 V1 Goal: Calibre running as a background service, accessible from your browser at http://[your-PC-IP]:8083. SSO integration documented in the SSO tab.
📖 Option 1 — Calibre Desktop Built-in Content Server NATIVE · EASIEST
What This Is

Calibre's desktop app includes a built-in content server that works in any browser. It's the simplest option — no extra software needed. The downside is that the Calibre GUI needs to stay open (or you set it to minimize to tray).

Install Calibre

Download from calibre-ebook.com. Install normally. When it asks, point your library to your new internal drive, e.g. D:\Books.

Enable the content server

In Calibre: Preferences → Sharing → Sharing over the net. Check "Start the content server when Calibre starts." Set port to 8083. Optionally set a username/password here.

Open Windows Firewall for port 8083 
netsh advfirewall firewall add rule name="Calibre Server" dir=in action=allow protocol=TCP localport=8083
Access from any device

Open a browser on your phone/laptop/TV and go to http://[your-server-IP]:8083. You'll see your full library.

Auto-launch Calibre on Windows startup

Press Win + R → type shell:startup → drag a shortcut to Calibre into that folder. Calibre can be configured to start minimized: right-click the desktop shortcut → Properties → add --hide-windowed to the target.

Target example:
"C:\Program Files\Calibre2\calibredb.exe" --with-library "D:\Books"
🌐 Option 2 — Calibre-Web (Better UI, More Features) NATIVE · RECOMMENDED
What Calibre-Web Adds

Calibre-Web is a separate web application that reads your Calibre library folder directly and provides a much nicer UI with user accounts, reading progress sync, Kindle send-to-email, and OPDS catalog support. It does NOT require Docker.

Install Python 3.11+ (if not already)

Download from python.org. During install, check "Add Python to PATH."

Install Calibre-Web via pip 
pip install calibreweb
Run it for the first time 
cps

This starts Calibre-Web. Open http://localhost:8083 in your browser. You'll see first-time setup.

Point it at your Calibre library

In setup, enter the path to your Calibre library folder (the folder that contains metadata.db), e.g.: D:\Books. Set an admin username/password.

Run Calibre-Web as a Windows Service (auto-start)

Use NSSM (Non-Sucking Service Manager) — a free tool that turns any EXE into a Windows service.

# Download nssm from nssm.cc/download
# Extract to C:\tools\nssm\

# Run in admin PowerShell:
C:\tools\nssm\win64\nssm.exe install CalibreWeb

# In the dialog:
# Path: C:\Users\[you]\AppData\Local\Programs\Python\Python311\Scripts\cps.exe
# Start Type: Automatic
# Then: nssm start CalibreWeb
Open the firewall 
netsh advfirewall firewall add rule name="Calibre-Web" dir=in action=allow protocol=TCP localport=8083
📱
OPDS for e-readers Calibre-Web exposes an OPDS catalog at http://[IP]:8083/opds. Apps like Moon+ Reader, KyBook, or Kyōbō can subscribe to this and browse/download books directly.
📬 Kindle Send-to-Email Setup
Send Books Wirelessly to Kindle

In Calibre-Web: Admin → Edit E-Mail settings. Enter your Gmail SMTP settings, then enable "Send to Kindle" per user. Each user sets their @kindle.com address in their profile. You'll also need to whitelist your Gmail address in your Amazon account's "Approved Personal Document E-mail List."

SMTP Server: smtp.gmail.com
SMTP Port:   587
From:        [email protected]
(use a Gmail App Password, not your main password)
🔐 SSO Integration (See SSO Tab) → SSO TAB

Calibre-Web supports OAuth via Google natively. You can enable "Login with Google" in Admin settings and restrict access to specific Gmail accounts. Full walkthrough is in the SSO tab.

It also supports LDAP/Active Directory and Authentik OIDC — connect via the user_oidc app. When a user authenticates via Authentik with their Google account, Calibre-Web matches on email address and logs them into their existing account (preserving all reading progress). See the SSO tab → Save Migration for the script that links existing Calibre-Web accounts to Gmail addresses.

💾 User Data Storage → SSO TAB
Where Calibre-Web stores per-user data

Calibre-Web stores reading progress, bookmarks, and shelf data in its own SQLite database (D:\Books\app.db), keyed per user account. This is distinct from the book files themselves (which are shared). The database is lightweight (~1–10 MB) and does not need to live in the user's personal folder — it stays where it is.

What lives in D:\Users\[username]\calibre\:

  • Exported reading progress snapshots (JSON exports for backup/portability)
  • Custom book uploads the user adds to the shared library
  • Personal reading lists exported from Calibre-Web

The reading progress inside Calibre-Web's own database counts toward quota via Nextcloud External Storage, but it's negligible in size. The main quota item here is any books they upload.

Calibre-Web + Nextcloud External Storage

The stats server exposes an endpoint that exports a user's Calibre-Web reading progress to their personal folder, enabling it to be backed up with everything else:

# Add to stats_server.py:
@app.route('/calibre/export/<username>')
def export_calibre_progress(username):
    import sqlite3, json
    conn = sqlite3.connect('D:/Books/app.db')
    conn.row_factory = sqlite3.Row
    progress = conn.execute('''
        SELECT b.title, b.author_sort, brl.read_status, brl.last_modified,
               brl.read_pos_epub_cfi, brl.kobo_reading_state_id
        FROM book_read_link brl
        JOIN books b ON b.id = brl.book_id
        JOIN user u ON u.id = brl.user_id
        WHERE u.name = ?
    ''', (username,)).fetchall()
    conn.close()
    export = [dict(r) for r in progress]
    # Write to user's personal folder:
    out_path = f'D:/Users/{username}/calibre/reading_progress.json'
    os.makedirs(os.path.dirname(out_path), exist_ok=True)
    json.dump(export, open(out_path, 'w'), indent=2, default=str)
    return jsonify({'ok': True, 'books': len(export)})

Call this nightly via Task Scheduler per user to keep their D:\Users\[username]\calibre\ folder current. Counts toward their Nextcloud quota.
