Physical Server & Network
Hardware, storage, static IP, Ethernet — nothing works without this
START HERE
  • Assess open SATA ports, drive bays, PCIe slots in your gaming desktop
    Count free SATA ports and 3.5" bays. If you need more, a PCIe SATA card adds 4 ports for ~$20.
    🖧 Hardware →
    30 min
  • Buy and install internal drives — shucked WD Elements 14TB+ (~$15/TB on sale)
    Move all media off USB to internal SATA. Pool with Windows Storage Spaces. This is the single most impactful upgrade.
    🖧 Hardware →
    2–4 hrs
  • Assign server a static local IP in router (e.g. 192.168.1.100)
    DHCP reservation by MAC address. Every tunnel, firewall rule, and bookmark depends on this never changing.
    15 min
  • Wire server to router via Ethernet (not WiFi)
    Powerline adapter (~$40) if the PC is far from the router. WiFi causes intermittent failures.
    30 min

🌐 Cloudflare Domain & Zero Trust
Register domain → Cloudflare proxy → Tunnel (zero open ports) → WAF
BEFORE ANYTHING GOES ONLINE
  • Register a .com domain via Cloudflare Registrar (~$9/yr)
    dash.cloudflare.com → Registrar. At-cost pricing. Automatically uses Cloudflare's nameservers.
    🌐 Domain →
    20 min
  • Enable Cloudflare proxy (orange cloud) on all public subdomains
    Your home IP is never in public DNS. Cloudflare's IPs are shown instead. Enable for every subdomain you create.
    🌐 Domain →
    10 min
  • Set up cloudflared Tunnel — eliminates all router port forwarding
    Install cloudflared on the server. Create a tunnel. Map subdomains to local ports in the tunnel config. No inbound ports are ever opened on your router.
    🛡️ Security →
    1 hr
  • Enable Cloudflare WAF, Bot Fight Mode, DDoS protection (all free)
    Security → Bot Fight Mode: ON. Security Level: Medium. Under Attack Mode available if needed.
    🛡️ Security →
    15 min
  • Set up DDNS PowerShell script (updates Cloudflare when home IP changes)
    🌐 Domain →
    15 min

🔐 Authentik SSO — The Identity Foundation
Install Authentik → connect Google → create groups → every service uses it
BEFORE ANY OTHER SERVICE
  • Enable WSL2 + install Docker in Ubuntu (one-time)
    wsl --install → reboot → open Ubuntu → sudo apt install docker.io docker-compose. This also enables Immich, RomM, LinguaCafe, Mealie.
    🔐 SSO →
    30 min
  • Install Authentik via Docker Compose on port 9000
    Create admin account. This is the central login page. All users will be redirected here when they access any service.
    🔐 SSO →
    1 hr
  • Add Google as an OAuth source in Authentik
    Google Cloud Console → create OAuth credentials → paste into Authentik's "Sources" → Google. Users now see "Sign in with Google" on the Authentik login page. No new passwords needed.
    🔐 SSO →
    45 min
  • Create Authentik groups: Admins, Users, Family, Friends
    Groups map to service access levels. Admins see all internal tools. Users get media and games. Friends might only get Jellyfin. Nextcloud storage quotas are also driven by group.
    🔐 SSO →
    30 min
  • Expose Authentik at auth.yourdomain.com via Cloudflare Tunnel
    Add to cloudflared config. Then add a Cloudflare Access policy to additionally protect this URL — only allow your specific Gmail addresses to reach it.
    🌐 Domain →
    15 min
  • Configure user nuke/block workflow in Authentik admin
    Authentik admin → Users → deactivate instantly kills all access. The SSO tab has the full nuke procedure including Cloudflare IP block.
    🔐 SSO →
    10 min
Why Authentik first? Install it before any other service and you configure each service correctly the first time. Install services first and you're doing double work to retrofit SSO later.

Caddy, IPBan, Vaultwarden, Tailscale, Uptime Kuma
BEFORE GOING LIVE

    45 min
  • Migrate watch history from Plex → Jellyfin with Plex2Jellyfin

    15 min
