Generative Art Toys

// Static Files · Port 5052 · Python http.server · Authentik forward_auth //

A curated collection of open-source generative art, interactive simulations, and creative coding toys — all self-hosted, all running in the browser. No accounts, no tracking, no external calls. Served as static files behind Authentik.

🎲 All toys are client-side only — they run entirely in the browser with no server communication after the initial page load. Nothing is stored per-user. The server just serves the files.
📁 User Data Storage → SSO TAB

No user data is stored. These are stateless browser toys. No D:\Users\ folder needed for this service.

🗂️ Folder Structure & Setup
Create the toys folder 
mkdir -p "D:\Server\GenArt"
# Each toy gets its own subfolder:
# D:\Server\GenArt\
#   index.html          ← launcher page (create this — see below)
#   sandspiel\
#   particle-life\
#   tiny-islands\
#   town-builder\
#   ... etc

Install as a Windows service via NSSM 
# Run as Administrator:
nssm install GenArt python
nssm set GenArt AppParameters "-m http.server 5052"
nssm set GenArt AppDirectory "D:\Server\GenArt"
nssm start GenArt

netsh advfirewall firewall add rule name="GenArt" dir=in action=allow protocol=TCP localport=5052

GenArt is now serving at http://localhost:5052.

🔐 Authentik — forward_auth Gate
Caddy configuration
# Add to your Caddyfile:
art.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:5052
}
Authentik Application entry
# Authentik admin → Applications → Create:
Name:       Generative Art
Slug:       genart
Launch URL: https://art.yourdomain.com
Provider:   Proxy Provider (forward_auth single application)
  External host: https://art.yourdomain.com
Access:     authentik-family + authentik-admins groups

➕ Adding More Toys

Clone or copy the toy's built output into a new subfolder under D:\Server\GenArt\. Add a link card to index.html. No restart needed — Python's http.server serves files live.

💡
Some toys need to be built from source (npm run build) before serving. Check each project's README. The simplest ones are pure HTML+JS and work immediately without a build step.
