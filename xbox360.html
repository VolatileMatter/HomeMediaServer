Xbox 360 Emulation

// Xenia · Windows Only · Sunshine Streaming //

Xbox 360 emulation via Xenia — Windows-only, GPU-accelerated. Games are played locally on the server and streamed to remote devices via Sunshine+Moonlight.

🟢
Compatibility note: Xenia is still maturing. Compatibility varies significantly by game. Many games run well; some have graphical glitches, crashes, or don't boot at all. Check the Xenia compatibility tracker before expecting a specific game to work perfectly. Popular titles (Halo 3, Red Dead Redemption, Skyrim, Dark Souls, etc.) generally run well.
🐉 V1 Goal: Xenia running on your Windows server, games loaded from an internal drive, accessible via Sunshine streaming. Each game added as a Sunshine app for easy one-click launch.
⚙️ Installation
Check system requirements

Xenia requires:

  • Windows 10 64-bit (or Windows 11)
  • A GPU with Vulkan support (most Nvidia GTX 1000+ and AMD RX 400+ series)
  • 8GB+ RAM (16GB recommended)
  • A modern multi-core CPU (the 360 had 3 cores with SMT — modern CPUs handle this fine)
Download Xenia

Get Xenia Canary (most up-to-date version) from github.com/xenia-canary/xenia-canary/releases. Download the latest xenia_canary.zip. Extract to D:\Emulators\Xenia\.

💡
Xenia vs Xenia Canary: Canary is the actively maintained fork with more frequent updates and better compatibility than the original Xenia master branch. Always use Canary.
Organize your game library 
D:\Emulation\Xbox360\
  ├── Halo3\
  │   └── default.xex
  ├── RedDeadRedemption\
  │   └── default.xex
  ├── DarkSouls\
  │   └── default.xex
  └── ...

# Or use ISO/XISO format:
D:\Emulation\Xbox360\
  ├── Halo3.iso
  ├── RedDead.xex
  └── ...

Xbox 360 games can be in XEX format (extracted) or ISO/XISO format (disc image). Both work with Xenia. XEX is slightly faster to load.

First launch and configuration

Run xenia_canary.exe. Go to File → Open → navigate to a game's default.xex or ISO. On first run, Xenia creates a xenia-canary.config.toml config file in the same folder.

Configure for best performance

Edit xenia-canary.config.toml:

[GPU]
# Use Vulkan for best compatibility
gpu = "vulkan"

# Resolution scale (1 = native 720p/1080p, 2 = double res for HD)
# Higher = better visuals but more GPU load
resolution_scale = 2

[Storage]
content_root = "D:/Emulation/Xbox360/Content"  # saves, DLC

[CPU]
# Leave at default unless experiencing crashes
threads = 0  # 0 = auto
Controller setup

Xenia uses XInput automatically — just plug in an Xbox controller (or use DS4Windows/BetterJoy for PS/Switch controllers). When streaming via Sunshine+Moonlight, the remote controller is forwarded as XInput device 0.

☀️ Sunshine Integration
Add Each Game as a Sunshine App

For the cleanest streaming experience, add each Xbox 360 game as its own Sunshine application. This lets you launch directly into a specific game from Moonlight:

# In Sunshine Web UI → Applications → Add:

Name:    Halo 3
Cmd:     "D:\Emulators\Xenia\xenia_canary.exe" "D:\Emulation\Xbox360\Halo3\default.xex"

Name:    Red Dead Redemption
Cmd:     "D:\Emulators\Xenia\xenia_canary.exe" "D:\Emulation\Xbox360\RedDeadRedemption\default.xex"

# Or add Xenia itself as a launcher:
Name:    Xenia (Xbox 360)
Cmd:     "D:\Emulators\Xenia\xenia_canary.exe"
For Streaming Quality

Xbox 360 games often target 720p or 1080p with complex lighting. Configure Sunshine to stream at 1080p60 for best results. If your GPU is encoding at the same time as rendering the game, reduce the game's resolution scale in Xenia's config to free up GPU headroom.

# In Sunshine config or web UI:
Resolution: 1920x1080
FPS:        60
Bitrate:    30–50 Mbps (higher for fast-paced games)
Encoder:    nvenc (Nvidia) or amf (AMD)
💾 Save Files

Xenia stores save files in the content_root folder you configure — by default D:\Emulation\Xbox360\Content\. Each save is tied to a virtual "profile" that Xenia manages. Unlike the Pokémon/Castlevania setup, Xenia doesn't have a multi-user save system — all saves are in one location.

For per-user saves over Sunshine streaming, the simplest approach is to maintain separate Xenia installation folders per user and configure each Sunshine session to use the appropriate one. This is complex — for V1, one shared save location is fine.

🐧 Linux Note

Xenia does not run on Linux. The Linux port is experimental and most games don't boot. This is one of the primary reasons to keep a Windows machine — see the Linux tab for the recommended hybrid architecture.

🌐 Xbox Live Emulation — Playing Together Online
⚠️
Honest status: Xbox Live emulation for Xenia is still maturing. True seamless Xbox Live (the way it worked on a real 360) isn't available yet in Xenia. What IS available are several workarounds that cover different use cases — from asynchronous sharing to real-time co-op.
Option 1: System Link via Virtual LAN (Best for co-op)

System Link is the 360's LAN multiplayer mode — it was designed for local network play. Using a virtual LAN tool makes players on different machines appear to be on the same LAN, enabling System Link for games that support it.

Tools: Radmin VPN or ZeroTier (both free)

All players install Radmin VPN (or ZeroTier)

Download from radmin-vpn.com — free for up to 100 devices on a network. All players join the same network using a network ID and password you share.

Create a shared network

One person (the host — you) creates the network: Radmin VPN → Network → Create Network. Share the network name and password with players. Everyone joins and gets a virtual 26.x.x.x IP.

In Xenia: enable System Link

Edit xenia-canary.config.toml:

[XAudio2]
# These settings help with System Link connectivity:

[Live]
# Not yet supported — use SystemLink instead

In most games, go to the multiplayer menu → LAN Party / System Link → Create or Join a session. Players on the Radmin network will see each other's sessions.

🎮
Games with confirmed System Link support in Xenia: Halo 3, Halo: ODST, Halo: Reach, Gears of War 1/2/3, Left 4 Dead (partially), CoD Black Ops (partially). Check each game's Xenia issue tracker for the current status.
Option 2: Insignia — Real Xbox Live Replacement (Real Hardware Only)

Insignia (insignia.live) is a fan-run replacement for Xbox Live that works on actual Xbox 360 hardware. It supports real-time online play for games like CoD: Black Ops, Halo 3, Left 4 Dead, etc.

Xenia does not currently support Insignia. The Insignia authentication system requires kernel-level Xbox Live calls that Xenia's HLE (high-level emulation) doesn't fully implement yet. This may change — the Xenia team is actively working on Live support.

🐉
If you have a real Xbox 360: Insignia is the definitive way to play 360 games online with friends. You need a real console and the ability to modify it slightly (RGH/JTAG or a specific dashboard version). Insignia is completely free and has a growing community of players.
Option 3: Asynchronous / Package Sharing (Viva Piñata style)

Some 360 games had asynchronous online features — sending gifts, sharing buildings, trading items between players without being online at the same time. Viva Piñata is the best example: you could send pinatas as gifts to friends even when they weren't playing.

These features relied on Xbox Live's cloud storage ("Marketplace" items and friend sharing), which is now offline. In Xenia, there's currently no equivalent — the save data is local only. However:

  • You can manually copy save files between players and have them import them (share a piñata by sending someone your save file)
  • A community-built tool could theoretically host a simple "item exchange" server — but none exists specifically for Viva Piñata yet
  • The Xenia save data for Viva Piñata is at D:\Emulation\Xbox360\Content\[profile_id]\4D5308AB\00000001\ — shareable as a file
Option 4: Parsec for Same-Screen Co-op

For games that support local co-op (2-player on the same screen), use Parsec to share your Xenia session with a friend. They connect and use their controller as Player 2. This works for any game with local co-op including Halo, Gears, Castle Crashers, etc. See the Mario Party tab for Parsec setup details.
