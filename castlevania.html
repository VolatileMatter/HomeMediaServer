📖 Recommended Story Playing Order

This order is designed for narrative coherence — not release date. The Castlevania timeline is non-linear, so playing in release order means jumping back and forth across the story era. This order threads the lore together and inserts outside media (anime, etc.) at the natural story beats.

⚙️ Setup (Shares Architecture with Pokémon Library)
If you already set up the Pokémon server

You can extend the same Flask server to serve Castlevania too. Just add a /castlevania route and point it at a separate ROM folder. The save files are stored per-user, per-game-ID, so there's no conflict.

# Extend server.py with Castlevania routes:
CASTLEVANIA_ROMS = 'D:/Emulation/Castlevania'
# Saves go directly to D:/Users/[username]/emulation/castlevania/
# username = request.headers.get('X-authentik-username', 'anonymous')
# save_dir = f'D:/Users/{username}/emulation/castlevania'
# See SSO tab → User Data Storage for full architecture

@app.route('/cv/roms/<path:filename>')
def serve_cv_rom(filename):
    if 'user' not in session: return redirect('/login')
    return send_from_directory(CASTLEVANIA_ROMS, filename)

# Save/load endpoints use the same /api/save/<game_id> routes
# — just prefix game IDs with 'cv-' to avoid conflicts:
# cv-sotn, cv-aria, cv-nes1, etc.

⚠️
PS1 games need a BIOS file for the pcsx_rearmed core. Place scph1001.bin (PS1 BIOS) in your EmulatorJS bios directory. You must dump this from a PS1 you own.
🎮 UI Notes for Castlevania

The game selector UI works identically to the Pokémon library. Sort by release year and label each card with its platform. For Symphony of the Night, note that the PS1 core has slight load times on first boot — this is normal.

Controller mapping for Castlevania: NES games need D-pad + A/B + Start/Select. GBA/DS games need an analog shoulder button for the sub-weapon system. EmulatorJS handles this automatically with a standard Xbox or PlayStation controller, or you can use the on-screen touch controls on mobile.

💾 User Data Storage → SSO TAB

Save files are written directly to D:\Users\[username]\emulation\castlevania\ by the Flask save server. The Authentik X-authentik-username header identifies each user — no login prompt inside the game page. Each user's saves are completely isolated and count toward their Nextcloud storage quota.

# In server.py, saves go directly to the user's path:
username = request.headers.get('X-authentik-username', 'anonymous')
save_dir = f'D:/Users/{username}/emulation/castlevania'
os.makedirs(save_dir, exist_ok=True)
save_path = os.path.join(save_dir, f'{game_id}.sav')

See the SSO tab → User Data Storage for the full architecture, Nextcloud quota integration, and the session manager service.
