Offline Wikipedia & WikiHow

// Kiwix · Local Hosting · One-Click Updates //

Host your own offline copies of Wikipedia (English) and WikiHow using Kiwix — a FOSS offline wiki reader. Accessible from any browser on your network.

🐉 V1 Goal: Kiwix server running on port 8888, serving both Wikipedia and WikiHow from your local drive. Accessible from phones, TVs, tablets — no internet needed. One-command updates.

⚙️ Setup with Kiwix FOSS · Windows Native
Download Kiwix Tools (the server component)

Go to download.kiwix.org/release/kiwix-tools/ and download the Windows 64-bit zip (e.g. kiwix-tools_win-x86_64.zip). Extract to D:\Server\Kiwix\.

You'll mainly use kiwix-serve.exe — it's a self-contained web server for ZIM files.

Create a ZIM storage folder 
D:\Server\Kiwix\zims\

ZIM files are Kiwix's compressed archive format. One file = one entire website.

Download Wikipedia ZIM

Browse the Kiwix library at library.kiwix.org and search for "Wikipedia English." Download to your ZIM folder. Options:

# Option A: Download via kiwix-manage (command line):
kiwix-manage.exe D:\Server\Kiwix\library.xml add "https://download.kiwix.org/zim/wikipedia_en_all_nopic_2024-11.zim"

# Option B: Direct download (large file — use a download manager):
# wikipedia_en_all_nopic_YYYY-MM.zim  ← no pictures (~22GB)
# wikipedia_en_all_maxi_YYYY-MM.zim   ← with pictures (~90GB)
Download WikiHow ZIM 
# Search library.kiwix.org for "wikihow"
# Download: wikihow_en_all_YYYY-MM.zim (~25GB with images)
Start kiwix-serve pointing at your ZIM folder 
D:\Server\Kiwix\kiwix-serve.exe --port=8888 --library D:\Server\Kiwix\library.xml

# Or simply point at the folder (serves all ZIM files in it):
D:\Server\Kiwix\kiwix-serve.exe --port=8888 D:\Server\Kiwix\zims\

Open http://localhost:8888 — you'll see a library index with all your ZIM files listed. Click any to browse.

Run Kiwix as a Windows Service (NSSM) 
nssm install Kiwix "D:\Server\Kiwix\kiwix-serve.exe" "--port=8888 D:\Server\Kiwix\zims\"
nssm start Kiwix
Open firewall 
netsh advfirewall firewall add rule name="Kiwix" dir=in action=allow protocol=TCP localport=8888

Access from any device: http://[server-ip]:8888

🔄 One-Command Updates
Update Script — updates both ZIM files

Kiwix releases updated ZIM snapshots monthly. Create D:\Server\Kiwix\update.ps1:

# update.ps1 — downloads latest ZIM files, replaces old ones
$zimDir = "D:\Server\Kiwix\zims"
$baseUrl = "https://download.kiwix.org/zim"

# Latest Wikipedia no-pics (check current filename at library.kiwix.org)
$wikiUrl  = "$baseUrl/wikipedia/wikipedia_en_all_nopic_2024-11.zim"
$wikihowUrl = "$baseUrl/wikihow/wikihow_en_all_2024-11.zim"

Write-Host "Stopping Kiwix service..."
nssm stop Kiwix

Write-Host "Downloading Wikipedia..."
Invoke-WebRequest -Uri $wikiUrl -OutFile "$zimDir\wikipedia_en.zim" -Resume

Write-Host "Downloading WikiHow..."
Invoke-WebRequest -Uri $wikihowUrl -OutFile "$zimDir\wikihow_en.zim" -Resume

Write-Host "Restarting Kiwix..."
nssm start Kiwix
Write-Host "Update complete."

Run this monthly or quarterly. The -Resume flag means if the download is interrupted, it resumes from where it left off. Use a scheduled task to run it monthly, overnight.

💡
Check for new filenames: Kiwix ZIM URLs include the year-month. Update the $wikiUrl and $wikihowUrl variables in the script monthly, or automate this by scraping the library.kiwix.org API.
Auto-detect Latest Version (Advanced)
# Automatically find the latest ZIM URL:
$apiUrl = "https://library.kiwix.org/catalog/v2/entries?lang=eng&name=wikipedia_en_all_nopic"
$latest = (Invoke-RestMethod $apiUrl).entries[0].link.href
Write-Host "Latest Wikipedia ZIM: $latest"
Invoke-WebRequest -Uri $latest -OutFile "$zimDir\wikipedia_en.zim" -Resume
🏠 Home Page Integration
Add to Dashboard & Monitor

Add Kiwix to the dashboard's quick-launch buttons and service status checks:

// In monitor-control.html SERVICES array, add:
{ id:'kiwix', name:'Offline Wiki', url:'http://localhost:8888', port:8888, ctrl:false },

// In QL (quick launch) array:
{ icon:'📖', label:'Wikipedia', url:'http://localhost:8888/wikipedia_en/A/Main_Page' },
{ icon:'❓', label:'WikiHow',   url:'http://localhost:8888/wikihow_en/A/Main-Page' },

Direct deep links to Wikipedia article search:

http://[server-ip]:8888/wikipedia_en/A/Special:Search?search=YOUR_QUERY

# Example bookmark for instant Wikipedia search from any device:
# Set as a browser search engine with keyword "w" pointing to:
# http://192.168.1.100:8888/wikipedia_en/A/Special:Search?search=%s
🌍 Remote Access

Kiwix can be made publicly accessible (read-only, no auth needed for public reference material):

# Caddyfile:
wiki.yourdomain.com {
    reverse_proxy localhost:8888
    # No auth needed — it's just reference content
}

Or keep it private via Tailscale and access with http://[tailscale-ip]:8888 from anywhere.
