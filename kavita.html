Kavita — Comics & TTRPG Guides

// One Instance · Two Libraries · Port · FOSS · Windows Native · Forward-Auth //

Kavita is a self-hosted reading server for comics (CBZ/CBR), manga, PDFs, and ebooks. A single instance runs two libraries — Comics and TTRPG Guides — with per-library access control, separate reading progress, and a native comic/PDF reader experience.

💡 One Instance, Two Libraries

Kavita supports multiple Libraries within a single instance, each with its own format type, folder root, and per-user access permissions. This is the recommended approach when your user base overlaps — the same people accessing comics and TTRPG guides share one login, one admin panel, and one reading progress database.

💡
Library-level access control: In Kavita's admin panel you can restrict a Library to specific users. If you want your TTRPG group to see the guides but not the full comic collection (or vice versa), you can lock down access per-library without running separate instances.
📦 Step 1 — Download & Extract Kavita FOSS · Windows Native
Download the latest Kavita release

Go to github.com/Kareadita/Kavita/releases and download kavita-win-x64.tar.gz. Extract to D:\Server\Kavita\.

Configure the port

Open D:\Server\Kavita\appsettings.json and confirm the port:

{
  "Port": ,
  "IpAddresses": "",
  "LoggingLevel": "Information",
  "AllowStatCollection": false

▶️ Step 3 — First Run & Add Libraries
Run Kavita for the first time 
D:\Server\Kavita\Kavita.exe

Open http://localhost: and create your admin account.

Add the Comics library

Admin Panel → Libraries → Add Library. Set:

  • Name: Comics
  • Type: Comic
  • Folder: D:\Comics

Save and let the initial scan complete.

Add the TTRPG Guides library

Admin Panel → Libraries → Add Library. Set:

  • Name: TTRPG Guides
  • Type: Book
  • Folder: D:\TTRPGs

The "Book" type enables Kavita's PDF reader with page-turning and bookmarking. Save and let it scan.

⚙️ Step 4 — Install as a Windows Service (NSSM)
Register with NSSM 
nssm install KavitaComics "D:\Server\Kavita\Kavita.exe"
nssm set KavitaComics AppDirectory "D:\Server\Kavita"
nssm start KavitaComics
Open the firewall 
netsh advfirewall firewall add rule name="Kavita" dir=in action=allow protocol=TCP localport=
Verify auto-start

Reboot and confirm Kavita is reachable at http://[server-ip]: before configuring the reverse proxy.

🌍 Step 5 — Reverse Proxy (Caddy)

Add a single block to your Caddyfile — one subdomain covers both libraries since they're in the same instance:

 {
    reverse_proxy localhost:
}

Reload Caddy:

caddy reload --config
🔐 Step 6 — SSO / Forward Auth → SSO TAB

Kavita does not yet support OAuth/OIDC natively. Authentication is handled at the Caddy layer via Authentik forward auth — users log in through Authentik once and are passed through to Kavita. Kavita's own user accounts still exist underneath; create matching usernames in Kavita for any user who needs reading-progress tracking and shelves.

⚠️
🔐 Admin step: After SSO is wired up, verify that unauthenticated requests to are blocked at the Caddy/Authentik layer before sharing the URL with users.

📚 Libraries at a Glance
📕 Comics

Reads CBZ, CBR, and image-based formats. Kavita auto-detects series, volumes, and chapters from filenames and embedded metadata. ComicInfo.xml inside a CBZ is fully supported.

  • Page-by-page reader with zoom and fit modes
  • Reading direction (L→R, R→L for manga)
  • Progress tracked per issue
  • Reading lists and collections
⚔️ TTRPG Guides

Kavita's "Book" library type gives PDFs a clean two-page reader with bookmarks, highlights, and progress tracking per book. Great for rulebooks you dip in and out of.

  • PDF reader with two-page spread mode
  • Bookmark any page mid-session
  • Per-user reading progress
  • Organise by series (e.g. D&D 5e, Pathfinder 2e)

Last confirmed working version

Kavita does not auto-update. Check GitHub releases periodically, especially after any scanner or reader breakage.

nssm stop KavitaComics

# Back up the database before updating:
Copy-Item "D:\Server\Kavita\config" "D:\Backups\Kavita\config_$(Get-Date -Format 'yyyy-MM-dd')" -Recurse
Download and extract the new release

Download the new kavita-win-x64.tar.gz from GitHub. Extract to a staging folder, then copy all files except the config/ folder into D:\Server\Kavita\. Do not overwrite config/ — that's your database.

Restart and verify 
nssm start KavitaComics

Open http://localhost:, confirm both libraries are visible and reading progress is intact.

⚠️ Known Breakpoints
  • Filename changes break progress: Kavita tracks series by folder/filename structure. Renaming files outside of Kavita will reset reading progress for those items.
  • Large PDF scan times: TTRPG PDFs with hundreds of pages slow the initial scan. This is normal — let the scan complete fully before testing the reader.
  • CBR support requires an extraction tool: Kavita extracts CBR (RAR archives) at read time. If RAR extraction fails, convert to CBZ using ComicTagger or 7-Zip.
🎯 How Kavita Achievements Work

Kavita exposes a REST API that the achievement server can poll on a schedule to check reading progress, completion status, and chapter counts per user. There are no native webhooks in Kavita, so all tracking is pull-based via the API.

💡
Get your API key from Kavita: User menu (top-right) → User Settings → API Key. This key is used to authenticate poll requests from the achievement server.

What lives where

The comic and TTRPG files themselves are shared — every user reads from the same folders. Kavita's config/ folder holds the SQLite database (kavita.db) which stores all per-user data: reading progress, bookmarks, shelves, and user accounts. This database is small (typically under 50 MB) but is the most critical thing to back up.

💡
All per-user data is in the Kavita database. No per-user folder in is needed for this service. Back up D:\Server\Kavita\config\kavita.db to preserve all reading progress and user accounts.

🛠️ Common Issues
❌ Kavita won't start / port already in use
Check whether the port is already bound by another process: 
netstat -ano | findstr :
If something else is on that port, either stop it or change Kavita's port in appsettings.json. Also check NSSM status: 
nssm status KavitaComics
If it shows SERVICE_STOPPED with an error code, check D:\Server\Kavita\config\logs\ for the startup error.
⚠️ Library scan runs but no series appear
Kavita is strict about folder structure. Comics must be nested at least one folder deep: Publisher\Series\Issue.cbz or Series\Issue.cbz. PDFs in the TTRPG library should be in a named series folder. Loose files at the library root won't be detected. Check Admin → Libraries → [library name] → Scan for error messages.
⚠️ CBR files fail to open in the reader
Kavita needs to extract CBR (RAR) archives at read time. On Windows, this sometimes fails if the RAR version is too new. Convert problematic CBRs to CBZ using 7-Zip (extract, then re-zip as .cbz) or ComicTagger. CBZ is the recommended format going forward.
⚠️ Reading progress reset after a library rescan
This happens when file or folder names change between scans. Kavita uses the file path to anchor progress. Never rename files outside of Kavita. If you restructured your library, reading progress for affected items will be lost — it cannot be migrated automatically.
🟢 Metadata (covers, descriptions) is wrong or missing
Kavita reads metadata from embedded ComicInfo.xml (CBZ) or embedded PDF metadata. Run Admin → Libraries → Refresh Metadata after adding or correcting metadata files. For comics, ComicTagger can bulk-write correct ComicInfo.xml from ComicVine or other sources.
