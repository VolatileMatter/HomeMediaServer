Kavita — Comics & TTRPG Guides

// Comic & PDF Reader · Port · Native Windows · Caddy forward_auth //

Kavita is a self-hosted reading server for comics (CBZ/CBR), manga, PDFs, and ebooks. One instance runs two libraries — Comics and TTRPG Guides — with per-library access control, shared reading progress, and a native comic/PDF reader experience.

💡 One Instance, Two Libraries

Kavita supports multiple Libraries within a single instance, each with its own format type, folder root, and per-user access permissions. This is the right approach when your user base overlaps — the same people accessing comics and TTRPG guides share one login, one admin panel, and one reading progress database.

💡
Library-level access control: In Kavita's admin panel you can restrict a Library to specific users. If you want your TTRPG group to see guides but not your full comic collection (or vice versa), you can lock down access per-library without running separate instances.

▶️ Step 3 — First Run & Add Libraries
Run Kavita for the first time 
D:\Server\Kavita\Kavita.exe

Open http://localhost: and create the admin account.

🔐 Add the Comics library

Admin Panel → Libraries → Add Library. Set Name: Comics · Type: Comic · Folder: D:\Comics. Save and let the initial scan complete.

🔐 Add the TTRPG Guides library

Admin Panel → Libraries → Add Library. Set Name: TTRPG Guides · Type: Book · Folder: D:\TTRPGs. The "Book" type enables Kavita's PDF reader with page-turning and bookmarking. Save and let it scan.

⚙️ Step 4 — Install as a Windows Service (NSSM)
🔐 Register with NSSM and start 
nssm install KavitaComics "D:\Server\Kavita\Kavita.exe"
nssm set KavitaComics AppDirectory "D:\Server\Kavita"
nssm start KavitaComics
🔐 Open the firewall 
netsh advfirewall firewall add rule name="Kavita" dir=in action=allow protocol=TCP localport=
Verify auto-start

Reboot and confirm Kavita is reachable at http://[server-ip]: before configuring the reverse proxy.

🔐 Step 5 — SSO / Authentik Integration SSO TAB
Pattern B — Caddy forward_auth (proxy-level)

Kavita does not support OAuth/OIDC natively. Auth is handled at the Caddy layer via Authentik forward auth. One subdomain covers both libraries since they share one instance:

# Add to Caddyfile ():
 {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username
                     X-authentik-groups
                     X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:
}

# Reload Caddy:
caddy reload --config
⚠️
🔐 Admin step: Kavita's own user accounts still exist underneath the forward-auth layer. Create matching usernames in Kavita (Admin → Users) for anyone who needs reading-progress tracking or library access restrictions. The Authentik username must match the Kavita username exactly.

📚 Libraries at a Glance
📕 Comics

Reads CBZ, CBR, and image-based formats. Kavita auto-detects series, volumes, and chapters from filenames and embedded ComicInfo.xml.

  • Page-by-page reader with zoom and fit modes
  • Reading direction (L→R or R→L for manga)
  • Per-issue progress tracking
  • Reading lists and collections
⚔️ TTRPG Guides

Kavita's "Book" library type gives PDFs a clean two-page spread reader with bookmarks and progress tracking per book.

  • Two-page spread PDF reader mode
  • Bookmark any page mid-session
  • Per-user reading progress
  • Organised by series (e.g. D&D 5e, Pathfinder 2e)

⚠️ Known Breakpoints
  • Filename changes break progress: Kavita anchors reading progress to file paths. Renaming files outside Kavita resets progress — no migration path exists.
  • Large PDF scan times: TTRPG PDFs with hundreds of pages slow the initial scan. Normal behaviour — let the scan complete before testing the reader.
  • CBR extraction failures: Kavita extracts CBR (RAR) archives at read time. Convert problem files to CBZ via 7-Zip (extract → re-zip as .cbz). CBZ is the recommended format going forward.
🎯 How Kavita Achievements Work

Kavita exposes a REST API that the achievement server polls on a schedule to check reading progress, completion status, and chapter counts per user. There are no native webhooks — all tracking is pull-based via the API.

💡
Get your API key from Kavita: click your avatar (top-right) → User Settings → API Key. This key authenticates poll requests from the achievement server. Each user has their own key.
# Test the API is reachable and the key is valid:
Invoke-WebRequest `
  -Uri "http://localhost:/api/library" `
  -Headers @{ "Authorization" = "Bearer [api-key]" }

What lives where

The comic and TTRPG files are shared — every user reads from the same folders. Kavita's config\ folder holds the SQLite database (kavita.db), which stores all per-user data: reading progress, bookmarks, shelves, and user accounts. The database is typically small (under 50 MB) but is the most critical thing to back up.

💡
All per-user data is in the Kavita database. No per-user folder in is needed for this service. Backing up D:\Server\Kavita\config\kavita.db preserves all reading progress and user accounts.

🛠️ Common Issues
❌ Kavita won't start / port already in use
Check if the port is bound by another process: 
netstat -ano | findstr :
If something else is on that port, stop it or change Kavita's port in appsettings.json. Also check NSSM status and logs: 
nssm status KavitaComics
Get-ChildItem "D:\Server\Kavita\config\logs\" | Sort LastWriteTime -Desc | Select -First 3
⚠️ Library scan completes but no series appear
Kavita requires files to be nested at least one folder deep: Publisher\Series\Issue.cbz or Series\Issue.cbz. PDFs need a named series folder too. Loose files at the library root are not detected. Check Admin → Libraries → [library] → Scan for error messages.
⚠️ CBR files fail to open in the reader
Kavita extracts CBR (RAR) archives at read time — this can fail with newer RAR versions. Convert problem files to CBZ: open with 7-Zip, extract the images, then re-zip as .cbz. CBZ is the recommended format going forward.
⚠️ Reading progress reset after a library rescan
Kavita uses file paths to anchor progress. Renaming or moving files outside Kavita resets progress for those items — it cannot be recovered. Always use Kavita's admin panel to rename series, not the filesystem directly.
🟢 Cover art or metadata is wrong / missing
Kavita reads metadata from embedded ComicInfo.xml (in CBZ) or PDF metadata. Run Admin → Libraries → Refresh Metadata after correcting metadata files. For comics, ComicTagger can bulk-write correct ComicInfo.xml from ComicVine.
