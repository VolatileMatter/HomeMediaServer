Kavita — Comics & DnD Books

// Two Separate Instances · Comics (5000) · DnD (5001) · FOSS //

Kavita is a self-hosted reading server for comics (CBZ/CBR), manga, PDFs, and ebooks. Running two separate instances keeps your DnD rulebooks completely separate from your comics library.

🐉 Best practice: Two Kavita instances on separate ports with separate data folders. Each has its own users, shelves, and reading progress. They share the same executable but nothing else.
🤔 Why Two Instances?

Kavita supports multiple "Libraries" within one instance, but they share users, settings, and the admin panel. Two separate instances means:

  • Completely separate user accounts (your DnD group doesn't see comics)
  • Separate reading history and progress
  • Independent appearance and metadata settings
  • One can be updated/restarted without affecting the other
💡
Alternatively: If your user base is the same people for both, use one instance with two Libraries (Comics + DnD Books). Libraries in Kavita are fully separate with per-library access permissions. But if you want different people to have access to different content, two instances is cleaner.
📁 Folder Structure
D:\Comics\
  ├── Marvel\
  │   ├── The Amazing Spider-Man\
  │   │   ├── The Amazing Spider-Man 001.cbz
  │   │   └── The Amazing Spider-Man 002.cbz
  │   └── X-Men\
  └── DC\
      └── Batman\

D:\DnD\
  ├── Core Rules\
  │   ├── Players Handbook 5e.pdf
  │   ├── Dungeon Masters Guide 5e.pdf
  │   └── Monster Manual 5e.pdf
  ├── Supplements\
  └── Adventures\
📖
CBZ/CBR for comics: These are just zip/rar archives of images. If you have comics in other formats, use ComicTagger (free) to convert and tag them. Kavita reads CBZ natively and gives a proper manga/comic reader experience.
⚙️ Instance 1 — Comics (Port 5000) FOSS · Windows Native
Download Kavita

From github.com/Kareadita/Kavita/releases — download the latest kavita-win-x64.tar.gz. Extract to D:\Server\Kavita-Comics\.

Configure the Comics instance port

Open D:\Server\Kavita-Comics\appsettings.json and set the port:

{
  "Port": 5000,
  "IpAddresses": "",
  "LoggingLevel": "Information",
  "AllowStatCollection": false
}
Run Kavita Comics for the first time 
D:\Server\Kavita-Comics\Kavita.exe

Open http://localhost:5000 — create admin account. Add a Library → type "Comics" → folder D:\Comics.

Install as Windows Service (NSSM) 
nssm install KavitaComics "D:\Server\Kavita-Comics\Kavita.exe"
nssm set KavitaComics AppDirectory "D:\Server\Kavita-Comics"
nssm start KavitaComics
Open firewall 
netsh advfirewall firewall add rule name="Kavita Comics" dir=in action=allow protocol=TCP localport=5000
⚙️ Instance 2 — DnD Rulebooks (Port 5001)
Extract a second copy to a separate folder

Extract the same Kavita release zip to D:\Server\Kavita-DnD\ — completely separate from the comics instance.

⚠️
Do NOT share the same Kavita folder for two instances. Each instance needs its own copy of the Kavita executable and its own config/ folder so their databases don't conflict.
Set the DnD instance to port 5001

Edit D:\Server\Kavita-DnD\appsettings.json:

{
  "Port": 5001,
  "IpAddresses": "",
  "LoggingLevel": "Information",
  "AllowStatCollection": false
}
First-time setup at http://localhost:5001

Create a separate admin account. Add a Library → type "Book" → folder D:\DnD. Kavita's "Book" library type supports PDF with a nice reader interface.

Install as a second Windows Service 
nssm install KavitaDnD "D:\Server\Kavita-DnD\Kavita.exe"
nssm set KavitaDnD AppDirectory "D:\Server\Kavita-DnD"
nssm start KavitaDnD

netsh advfirewall firewall add rule name="Kavita DnD" dir=in action=allow protocol=TCP localport=5001
🌍 Remote Access & Reverse Proxy
# Caddyfile entries:
comics.yourdomain.com {
    reverse_proxy localhost:5000
}

dnd.yourdomain.com {
    reverse_proxy localhost:5001
}

Or access both via Tailscale: http://[tailscale-ip]:5000 and http://[tailscale-ip]:5001.

🔐 SSO Integration → SSO TAB

Kavita does not yet support OAuth/OIDC natively. Put each instance behind Caddy with Authentik forward auth for a unified login experience. Alternatively, Kavita's own user system (username/password per instance) is sufficient for V1.
