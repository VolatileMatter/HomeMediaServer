Virtual Tabletop

// VirtualTabletop.io · Self-Hosted · TTRPGs Online //

VirtualTabletop.io (VTT) is a self-hostable virtual tabletop for TTRPGs — think Foundry VTT or Roll20, but designed for self-hosting. Run it on your server so your D&D group can play without paying subscription fees.

🐉 Two strong options: Foundry VTT (one-time $50 license, best in class, most popular) and Owlbear Rodeo (free, simpler). This tab covers Foundry VTT as the primary recommendation, with notes on alternatives.
🐉
Why self-host a VTT? Roll20 and D&D Beyond require ongoing subscriptions and put your game data in someone else's cloud. A self-hosted Foundry VTT instance gives you: your own persistent world data, no player subscription required, full mod support, and a server your group can access 24/7 from any browser — no app install needed.

⚙️ Foundry VTT — Installation on Windows
Port: 30000

Foundry VTT runs as a Node.js server. Players connect from any browser — no software install needed on their end.

Step 1: Purchase Foundry VTT

From foundryvtt.com → Purchase ($50 USD, one-time). You get a license key and can download the software. The server license allows self-hosting — players connect for free.

Step 2: Install Node.js 20 LTS

From nodejs.org → LTS release → Windows installer. Run with all defaults.

Step 3: Download and extract Foundry VTT

From your Foundry account → My Purchased Licenses → download Node.js version (not the Windows app). Extract to D:\Server\FoundryVTT\.

Step 4: Create a data directory 
mkdir D:\FoundryData

All world data, assets, and modules go here. Keep it on your large media drive, not the OS drive.

Step 5: First launch and license setup 
cd D:\Server\FoundryVTT
node resources\app\main.js --dataPath="D:\FoundryData"

Open http://localhost:30000. Enter your license key. Foundry verifies online once, then works offline. Create your admin password.

Step 6: Install as a Windows service (NSSM) 
nssm install FoundryVTT node "D:\Server\FoundryVTT\resources\app\main.js"
nssm set FoundryVTT AppParameters "--dataPath=D:\FoundryData"
nssm set FoundryVTT AppDirectory "D:\Server\FoundryVTT"
nssm start FoundryVTT

netsh advfirewall firewall add rule name="FoundryVTT" dir=in action=allow protocol=TCP localport=30000
Step 7: Add to Caddy (Tailscale)

Add a block to your Caddyfile so Foundry is reachable at from any Tailscale device:

# In C:\caddy\Caddyfile:
 {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:
}

# Reload:
caddy reload --config

Players visit — Tailscale must be running on their device. No download, no account — they just enter the game password you set.

⚠️
Foundry + Cloudflare proxy note: Cloudflare's free tier has a 100 MB file upload limit. If players are uploading large asset files (maps, audio), use cloudflared tunnel with Cloudflare's proxy disabled (grey cloud) for the VTT subdomain to avoid this limit, or use Cloudflare's bypass cache rules.
🔐 SSO — Authentik Integration

Foundry VTT has its own user/password system. For your home server, the simplest approach is to use Caddy forward_auth in front of it to require Authentik login before players can even reach the Foundry login page:

# Caddyfile:
vtt.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:30000
}

This means players need a Google login (via Authentik) AND then a Foundry game password. This adds a layer that prevents random internet visitors from even seeing your Foundry instance.

🗺️ Map & Asset Storage

Map images and tokens can be very large. Store them on your media drive:

D:\FoundryData\
  ├── worlds\           ← Active game worlds
  │   └── myworld\
  │       ├── world.json
  │       └── scenes\
  ├── Data\
  │   ├── maps\         ← Battle maps (100–500 MB each for 4K maps)
  │   ├── tokens\       ← Token art
  │   ├── music\        ← Ambient audio
  │   └── modules\      ← Installed modules
💡
Free map sources: r/battlemaps, Dungeondraft (map making software), and 2minutetabletop.com all have free CC-licensed maps ready for Foundry.

💾 Backups

World data is irreplaceable after months of play. Back it up automatically:

# D:\Server\Scripts\backup_foundry.ps1
# Run weekly via Task Scheduler

$src  = "D:\FoundryData\worlds"
$dst  = "D:\Backups\FoundryVTT"
$date = Get-Date -Format "yyyy-MM-dd"
$dest = "$dst\worlds_$date"

New-Item -ItemType Directory -Force -Path $dest | Out-Null
Copy-Item -Path "$src\*" -Destination $dest -Recurse
Write-Host "Foundry backup complete: $dest"

# Keep only last 4 backups:
Get-ChildItem $dst -Directory | Sort-Object CreationTime | Select-Object -SkipLast 4 | Remove-Item -Recurse -Force
🔗 Connection to DnD Rulebooks (Kavita)

Your Kavita DnD instance (port 5001) holds all your PDFs. When players need to reference a rulebook mid-session, they can open a browser tab to dnd.yourdomain.com alongside the VTT. A Foundry journal entry can link directly to specific rulebook PDFs stored in Kavita:

# In a Foundry Journal Entry (HTML mode):

  📖 Players Handbook — Chapter 7: Combat
</a>
