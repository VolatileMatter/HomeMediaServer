Browser Emulation

// EmulatorJS · Port · Native Windows (NSSM) · Authentik forward_auth //

A Netflix-style retro game library running entirely in the browser via WebAssembly. Play NES, SNES, GBA, NDS, N64, PS1, and a dozen more systems. Per-user save files, RetroAchievements integration, GLSL shader browser, on-screen controller overlays, and completionist playlists — all accessible from any device including mobile.

Complete?
Mark done when: library loads at games.yourdomain, saves persist across devices, RA linked, shaders load, overlays work on mobile.
🍓
Pi Sentry Compatibility RUNNING ON PI

This service is already deployed on the Pi Sentry.

EmulatorJS is already deployed on the Pi Sentry at port 8085. ROMs are served from /mnt/archive/Roms/. Per-user save files are managed by the Flask save server. All emulation runs client-side in the browser — the Pi only serves static assets.

⚡ V1 Goal: A browser-accessible game library at where users log in via Authentik, browse games in a Netflix-style grid, play with per-user cloud saves, and can enable RetroAchievements tracking. Mobile-playable with on-screen overlays.

🐍 Step 3 — Install the Flask Save Server
Install dependencies

Create D:\Emulation\BrowserGames\server.py
from flask import Flask, request, jsonify, send_from_directory, send_file
from flask_cors import CORS
import os, json, mimetypes

app = Flask(__name__, static_folder='.')
CORS(app)

ROMS_DIR    = 'D:/Emulation/BrowserGames/roms'
COVERS_DIR  = 'D:/Emulation/BrowserGames/covers'
SHADERS_DIR = 'D:/Emulation/BrowserGames/shaders'
USERS_BASE  = 'D:/Users'
GAMES_FILE  = 'D:/Emulation/BrowserGames/games.json'

def get_user():
    # Authentik injects this header via Caddy forward_auth
    return request.headers.get('X-authentik-username', 'anonymous')

def user_save_dir(username):
    path = f'{USERS_BASE}/{username}/emulation/browser'
    os.makedirs(path, exist_ok=True)
    return path

# ── Game catalogue ──────────────────────────────────────────────
@app.route('/api/games')
def games():
    with open(GAMES_FILE) as f:
        return jsonify(json.load(f))

# ── Save file API ───────────────────────────────────────────────
@app.route('/api/save/<game_id>', methods=['GET'])
def load_save(game_id):
    username = get_user()
    path = os.path.join(user_save_dir(username), f'{game_id}.sav')
    if os.path.exists(path):
        return send_file(path, mimetype='application/octet-stream')
    return '', 204  # No save yet

@app.route('/api/save/<game_id>', methods=['POST'])
def store_save(game_id):
    username = get_user()
    path = os.path.join(user_save_dir(username), f'{game_id}.sav')
    with open(path, 'wb') as f:
        f.write(request.data)
    return jsonify({'ok': True})

# ── Save state slots (1–5 per game) ─────────────────────────────
@app.route('/api/state/<game_id>/<int:slot>', methods=['GET'])
def load_state(game_id, slot):
    username = get_user()
    path = os.path.join(user_save_dir(username), f'{game_id}.state{slot}')
    if os.path.exists(path):
        return send_file(path, mimetype='application/octet-stream')
    return '', 204

@app.route('/api/state/<game_id>/<int:slot>', methods=['POST'])
def store_state(game_id, slot):
    username = get_user()
    path = os.path.join(user_save_dir(username), f'{game_id}.state{slot}')
    with open(path, 'wb') as f:
        f.write(request.data)
    return jsonify({'ok': True})

# ── ROM serving (requires auth header to be present) ────────────
@app.route('/roms/<path:filename>')
def serve_rom(filename):
    if get_user() == 'anonymous':
        return 'Unauthorized', 401
    return send_from_directory(ROMS_DIR, filename)

# ── Cover art ───────────────────────────────────────────────────
@app.route('/covers/<path:filename>')
def serve_cover(filename):
    return send_from_directory(COVERS_DIR, filename)

# ── Shader listing ──────────────────────────────────────────────
@app.route('/api/shaders')
def list_shaders():
    shaders = []
    for root, dirs, files in os.walk(SHADERS_DIR):
        for f in files:
            if f.endswith('.glslp') or f.endswith('.glsl'):
                full = os.path.join(root, f)
                rel  = os.path.relpath(full, SHADERS_DIR).replace('\\', '/')
                category = rel.split('/')[0]
                shaders.append({'path': rel, 'name': f, 'category': category})
    return jsonify(shaders)

# ── Static files (index.html, JS, etc.) ─────────────────────────
@app.route('/')
@app.route('/<path:filename>')
def static_files(filename='index.html'):
    return send_from_directory('D:/Emulation/BrowserGames', filename)

if __name__ == '__main__':
    from waitress import serve
    serve(app, host='0.0.0.0', port=8085)

🎬 Step 5 — Deploy the Library Frontend (index.html)

The frontend is a single-page HTML file that fetches /api/games and renders the Netflix-style grid. Key features built into it:

  • Category tabs: By Platform · By Developer · By Franchise · Playlists
  • Search bar: Live filter across all fields
  • Game detail modal: Cover art, description, achievements count, play button
  • Emulator launcher: Bootstraps EmulatorJS in an overlay, loads user's save
  • Overlay controls: On-screen D-pad + buttons (on by default, toggle-able)
  • Shader picker: Fetches /api/shaders, previews applied to cover art via CSS filter approximation; actual shader is passed to EmulatorJS

🏅 Step 8 — RetroAchievements Integration
How it works

EmulatorJS has native RetroAchievements support. Users link their own free RA account once via the settings panel. After that, RA tracks their achievements automatically per game, synced to the global RA leaderboard.

// In the EmulatorJS initialization block (inside index.html):
EJS_retroarch = true;
EJS_cheats    = false;     // disable if desired
// RA credentials: user enters once, stored in localStorage per-device
// EJS_raUsername and EJS_raToken are set from the settings panel

// In settings panel HTML (one-time setup per device):
// User enters RA username + RA API token (from retroachievements.org/settings)
// Store with: localStorage.setItem('ra_username', val)
//             localStorage.setItem('ra_token', val)
// Then pass to EmulatorJS: EJS_raUsername = localStorage.getItem('ra_username')
//                           EJS_raToken   = localStorage.getItem('ra_token')
⚠️
RA only works with unmodified ROMs — ROM hacks will not trigger RA achievements since their checksums don't match the RA database. For hacks, RA integration is silently skipped (set ra_game_id: 0 in games.json).

🌊 EmulatorJS Initialization Snippet

The play page bootstraps EmulatorJS like this (simplified — full version in index.html):

<div id="game"></div>
<script src="/emulatorjs/data/loader.js"></script>
<script>
async function launchGame(game) {
    // 1. Load user's save file from server
    let saveData = null;
    const saveRes = await fetch(`/api/save/${game.id}`);
    if (saveRes.status === 200) saveData = await saveRes.arrayBuffer();

    // 2. Configure EmulatorJS
    EJS_player      = '#game';
    EJS_gameUrl     = game.rom;
    EJS_core        = game.core;
    EJS_gameID      = game.id;
    EJS_language    = 'en-US';
    EJS_pathtodata  = '/emulatorjs/data/';
    EJS_retroarch   = true;   // RetroAchievements support

    // 3. RA credentials (user sets once in settings panel)
    EJS_raUsername  = localStorage.getItem('ra_username') || '';
    EJS_raToken     = localStorage.getItem('ra_token') || '';

    // 4. Shader (user-selected, persisted to localStorage)
    const shader = localStorage.getItem('active_shader');
    if (shader) EJS_shaders = `/shaders/${shader}`;

    // 5. On-screen overlay
    EJS_VirtualGamepadSettings = {
        enabled: localStorage.getItem('overlay_hidden') !== '1',
        opacity: 0.65,
        buttonColor: 'rgba(200,168,75,0.5)',
        stickColor:  'rgba(200,168,75,0.35)',
    };

    // 6. Pre-load save file if one exists
    if (saveData) {
        EJS_onGameStart = function() {
            EJS_emulator.gameManager.setSaveFile(new Uint8Array(saveData));
        };
    }

    // 7. Auto-save on in-game save + every 60s
    EJS_onSaveState = async (data) => {
        await fetch(`/api/save/${game.id}`, {
            method: 'POST', body: data,
            headers: {'Content-Type': 'application/octet-stream'}
        });
    };
    EJS_saveStateOnExit = true;
}
</script>

  1. Place the ROM file in D:\Emulation\BrowserGames\roms\[system]\
  2. Add cover art (ideally 300×400px JPG) to D:\Emulation\BrowserGames\covers\ named [game-id].jpg
  3. Add an entry to games.json with all required fields (see Setup tab)

  5. Refresh the browser — the library auto-rebuilds from games.json on each load
⚠️ Known Breakpoints
  • EmulatorJS major version change: Core file paths inside data/ occasionally move between major releases. After pulling, verify your existing games still launch before pushing to production.
  • WASM memory limits on mobile: N64 and PSX titles use substantial WASM memory. Low-RAM mobile devices (<4GB) may crash when launching these. GBA/SNES are safe on any device.
  • GLSL shader format: EmulatorJS only supports .glslp preset files (not .slangp which are Vulkan/D3D). The libretro glsl-shaders repo has both — only serve the .glslp files.
  • PS1 BIOS requirement: If users report a black screen on PSX games, the BIOS file is missing or misnamed. Required: scph5501.bin in emulatorjs/data/bios/.
  • RetroAchievements token expiry: RA tokens don't expire but can be revoked. If a user reports RA not working, have them re-generate their API token at retroachievements.org/settings.
  • NDS touch controls on iOS: The melonDS core's touch input for the bottom screen requires enabling "Desktop site" in Safari, or using the dedicated overlay touch area. iOS Safari in mobile mode may misroute touch events.

🏆 RetroAchievements Setup
🏅 RetroAchievements

RA is a free community-run achievement system for retro games. EmulatorJS supports it natively. Each user signs up at retroachievements.org, gets an API token from their settings page, and enters it once in the game library's settings panel. After that, every supported game auto-submits achievements to their RA profile.

What users need to do (one-time per device)
  1. Create a free account at retroachievements.org/createaccount.php
  2. Go to retroachievements.org/settings and copy your API key
  3. In the game library, open ⚙ Settings → RetroAchievements tab
  4. Enter your RA username and API key — saved to localStorage for this device
  5. Launch any supported game — achievements will track automatically
💡
RA credentials are stored per-device in localStorage, not server-side. Users need to re-enter them on each device they play on (phone, tablet, desktop). This is by design — RA tokens should not be stored server-side.
🏆 Server-Side Achievement Triggers

The Flask save server fires these events to the local achievement server via POST when they occur:

# server.py — fire achievement trigger on save:
import requests

def fire_achievement(username, trigger, metadata={}):
    try:
        requests.post(
            f'http://localhost:{ACHIEVEMENT_PORT}/api/achievement/trigger',
            json={'username': username, 'trigger': trigger, 'metadata': metadata},
            timeout=2
        )
    except: pass   # Never block a save for an achievement

# Examples:
# After storing a save: fire_achievement(user, 'game_played', {'game': game_id})
# After 10 saves:       fire_achievement(user, 'ten_games_played')
# After playlist completion: fire_achievement(user, 'playlist_complete', {'playlist': name})

🤖 Feeding the AI Recommendation Engine

Every completed save event is a strong taste signal. Games where a user has substantial save file growth (indicating real play time) are even stronger. The recommendation engine receives game completion events to suggest next games in a franchise or similar titles across franchises.

What lives in D:\Users\[username]\emulation\browser\

Each game creates up to 6 files per user: one SRAM save (.sav) and up to 5 named save state slots (.state1 through .state5). Save files are tiny — typically 8KB–512KB each depending on the system. Even a library of 100 games totals well under 50 MB per user.

💡
This folder is inside D:\Users\[username]\ which is mounted as "App Data" in Nextcloud and counts toward the user's storage quota. Save data is negligible — it won't meaningfully affect quotas.

Game shows black screen or hangs on "Loading…"
1. Open browser DevTools → Console — look for 404s on ROM or core WASM files.
2. Verify the ROM path in games.json matches the actual file path under roms/ (case-sensitive on the Flask server).
3. For PSX games: check that the BIOS file exists at emulatorjs/data/bios/scph5501.bin.
4. Verify the core name in games.json matches an actual .wasm file in emulatorjs/data/cores/.
Save file not loading — game starts fresh every time
1. Check the browser console for a failed GET /api/save/[game-id] request.
2. Verify the Flask server is running: curl http://localhost:8085/api/games.
3. Confirm X-authentik-username header is being injected by Caddy — check with curl -v https://games.yourdomain.com/api/games and look at request headers.
4. Check D:\Users\[username]\emulation\browser\ exists and contains [game-id].sav.
5. Check D:\Logs\browsergames-err.log for Python errors.
RetroAchievements not triggering in-game
1. RA only works with unmodified ROMs — confirm the game has ra_game_id > 0 in games.json.
2. Confirm EJS_retroarch = true and EJS_raUsername / EJS_raToken are non-empty before launch.
3. Test the RA token is valid by checking retroachievements.org with it.
4. Some games need a "hardcore mode" off — EmulatorJS does not enforce hardcore mode by default.
On-screen overlay not appearing on mobile
1. Check if overlay_hidden is set in localStorage — open DevTools → Application → Local Storage.
2. Verify EJS_VirtualGamepadSettings.enabled is true in the launch script.
3. The overlay auto-hides when a gamepad is detected — disconnect any paired controllers and reload.
Shader applied but no visible effect
1. Only .glslp preset files work — not .slangp (Vulkan) or .cgp (Cg). Filter the shader list to *.glslp only.
2. Some shaders require EJS_color or specific core support. CRT shaders in shaders/crt/ are the most reliably supported.
3. Check the browser console for GLSL compile errors if a specific shader fails silently.
NSSM service starts but /api/games returns 500
1. Check D:\Logs\browsergames-err.log for the Python traceback.
2. Most common: games.json has a syntax error (trailing comma, missing quote). Validate with python -m json.tool games.json.
3. Verify all path constants in server.py exist on disk.
4. Run manually to see live errors: python D:\Emulation\BrowserGames\server.py
