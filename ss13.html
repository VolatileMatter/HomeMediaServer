Space Station 13 Server

// BYOND · Off by Default · /tg/station or Bay12 //

Self-hosted SS13 server using the BYOND engine. Off by default, spun up on demand. Run any open-source codebase — /tg/station13, Baystation, or Paradise Station.

🐉 V1 Goal: A BYOND-based SS13 server running on demand, accessible to players who have the BYOND client installed, password-protected. Off by default.
🧠 Understanding SS13 / BYOND

Space Station 13 runs on the BYOND engine — a proprietary game engine by Dantom/BYOND LLC. Players need the free BYOND client installed to connect. The server-side BYOND Dream Daemon is free to download and run.

All major SS13 codebases are open source (MIT or similar). The most popular:

⚙️ Installation
Install BYOND

Download from byond.com/download. The installer includes both the client and the Dream Daemon server. Install to default location.

Clone your chosen codebase 
# Install Git if needed: https://git-scm.com
# For /tg/station:
git clone https://github.com/tgstation/tgstation.git D:\SS13\tgstation

# For Paradise:
git clone https://github.com/ParadiseSS13/Paradise.git D:\SS13\paradise
Compile the server code (.dmb file)

Open the BYOND Dream Maker application. File → Open → browse to your codebase folder and open the .dme file (e.g. tgstation.dme). Press Compile (or Build → Compile). This creates a .dmb file — the server binary.

⚠️
Compilation can take 5–15 minutes for large codebases like /tg/. The first time is slowest. Some codebases have Python/Rust build steps — check their README for prerequisite installations.
Configure the server

Copy the example config:

cd D:\SS13\tgstation\config
copy example_config\*.txt .
# Edit config.txt — set server name, admin ckeys, etc.
# Edit admins.txt — add your BYOND username as an admin

In config.txt, set:

server_name = My SS13 Server
port = 1337
map = boxstation  # or whatever map you want
hub = 0           # 0 = don't list on BYOND hub (private server)
Create a launch script

Create D:\SS13\start_ss13.bat:

@echo off
title SS13 Server
cd /d D:\SS13\tgstation
"C:\Program Files (x86)\BYOND\bin\DreamDaemon.exe" tgstation.dmb 1337 -safe -invisible -logself
echo Server stopped.
Open firewall port 
netsh advfirewall firewall add rule name="SS13 BYOND" dir=in action=allow protocol=TCP localport=1337
netsh advfirewall firewall add rule name="SS13 BYOND UDP" dir=in action=allow protocol=UDP localport=1337
Connect with BYOND client

Open BYOND client → File → Open Location → enter byond://[your-server-IP]:1337. Log in with your BYOND account. You should see the map loading.

🎛️ Dashboard Integration
On-Demand Start/Stop via Scheduled Tasks
# Create start task:
schtasks /create /tn "ss13-start" /sc once /st 00:00 /tr "D:\SS13\start_ss13.bat" /ru SYSTEM /f

# Stop task:
schtasks /create /tn "ss13-stop" /sc once /st 00:00 /tr "taskkill /IM DreamDaemon.exe /F" /ru SYSTEM /f

The dashboard Start/Stop buttons then call schtasks /run /tn "ss13-start" / ss13-stop".

🔄 Keeping the Server Updated
# Update code from GitHub:
cd D:\SS13\tgstation
git pull

# Recompile (do while server is stopped):
"C:\Program Files (x86)\BYOND\bin\dm.exe" tgstation.dme

Most codebases have update notes in their Discord servers or CHANGELOG files. Don't update mid-round — wait for a round end and a server restart.

🔐 Access Control

SS13 uses BYOND's account system for identity. Access control is done via config files:

# config/admins.txt — one BYOND username per line with rank:
YourBYONDUsername = Headmin

# To make the server invite-only (whitelist):
# config/whitelist.txt — list of allowed BYOND usernames (if whitelist is enabled in config.txt)
# Set in config.txt: whitelist = 1
💡
Setting hub = 0 in config.txt prevents your server from appearing on the public BYOND hub. Players can only connect if they know your IP and port directly.
