Transitioning to Linux

// Pros & Cons · Best Distros · Per-Service Considerations //

Should you run Linux on your home server? This tab covers the real trade-offs, which distro to pick, how each service fares, and how to make the transition.

🐉 Summary: Linux is better for everything except game streaming and Xbox 360 emulation. A hybrid approach (Linux for always-on services, Windows for gaming) is the best long-term architecture.
⚖️ Overall Pros & Cons
✅ Advantages of Linux for a Home Server
  • Better performance — no background Windows Update, telemetry, or GUI overhead
  • All server software designed for Linux first (Docker, Nextcloud, etc.)
  • More stable — Linux servers run for years without rebooting
  • Better security posture — smaller attack surface, no SMB exposed by default
  • Free forever, no licensing issues
  • Cron jobs replace Windows Task Scheduler (simpler, more powerful)
  • Better CLI tools for automation
  • Systemd services > NSSM for service management
❌ Disadvantages / Challenges
  • Steam and gaming: Most Windows games work via Proton/Wine but not all, and setup is more complex
  • XENIA (Xbox 360 emulator): Linux port is incomplete — Xbox 360 emulation basically requires Windows
  • Sunshine game streaming: Works on Linux, but Nvidia GPU setup is more complex
  • No GUI: Headless Linux is more efficient but requires CLI comfort
  • Driver quirks: Some older hardware has better Windows support
  • Learning curve if you're not already familiar with Linux

🏗️ The Hybrid Architecture (Recommended)
Keep Windows for Gaming, Run Linux for Everything Else

The ideal long-term setup is two machines (or dual-boot on one):

🐧 Linux Server (always on)
  • Plex, Calibre-Web, Audiobookshelf
  • Nextcloud, qBittorrent, Sonarr
  • Minecraft, ARK servers
  • All emulation (except Xbox 360)
  • Dashboard, Caddy, Vaultwarden
⊞ Windows PC (on-demand)
  • Sunshine game streaming
  • Xbox 360 emulation (XENIA)
  • Steam gaming
  • Can be shut down when not gaming
🚀 How to Transition: Step-by-Step
Install Ubuntu Server 24.04 LTS

Download from ubuntu.com. Boot from USB. Choose "Ubuntu Server (minimized)" for a lean install. Set up SSH during install so you can manage it headlessly.

Replace NSSM with systemd service files

Every service you ran with NSSM on Windows gets a .service file in /etc/systemd/system/. Example for Audiobookshelf:

[Unit]
Description=Audiobookshelf
After=network.target

[Service]
Type=simple
User=www-data
ExecStart=/usr/bin/audiobookshelf
Restart=always

[Install]
WantedBy=multi-user.target

Then: sudo systemctl enable --now audiobookshelf

Replace Task Scheduler with cron

Edit cron jobs with crontab -e:

# Minecraft 2am shutdown, 3am restart:
55 1 * * * /home/server/minecraft/shutdown.sh
0  3 * * * /home/server/minecraft/start.sh

# DDNS update every 5 minutes:
*/5 * * * * /home/server/ddns.sh
Nextcloud on Linux (better than XAMPP)

Use the official Nextcloud snap package:

sudo snap install nextcloud
sudo nextcloud.manual-install admin yourpassword

Runs with its own Apache and PHP — no XAMPP needed.

Migrate your data with rsync

From Windows to Linux (run on Linux, pulling from Windows share):

rsync -avz --progress "//192.168.1.100/Media" /mnt/media/

Or from an external drive connected directly to the Linux box.
