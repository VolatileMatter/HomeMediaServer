Linux Contingency Plan

// If Windows 10 Support Dies · What to Do · What Gets Lost //

Windows 10 mainstream support ends October 2025. Extended security updates run until October 2028. This page is for the worst-case scenario: you can't or won't upgrade to Windows 11, and you need to migrate everything to Linux.

⚠️
This is a contingency page — not the current plan. Right now, run Windows 10. Extended security updates cover you until 2028. This is the playbook if that changes and you need an exit strategy.

🖥️ Virtual Machine Option for Xenia
Run Windows in a KVM/QEMU VM with GPU Passthrough

This is the gold standard solution for keeping Windows-dependent software running on a Linux host. Your Linux server runs Xenia in a Windows VM that has direct access to your GPU (GPU passthrough), so gaming performance is near-native.

Requirements

CPU must support AMD-Vi or Intel VT-d (most modern CPUs do). You need two GPUs — one for the Linux host and one to pass through to the Windows VM. A cheap second-hand GPU (~$50 for a GT 1030) handles the Linux desktop; your gaming GPU passes through to Windows.

Install KVM/QEMU + VFIO on Ubuntu 
sudo apt install -y qemu-kvm libvirt-daemon-system virt-manager ovmf
sudo systemctl enable libvirtd

# Verify CPU supports virtualization:
egrep -c '(vmx|svm)' /proc/cpuinfo  # should be > 0
Configure GPU passthrough (VFIO)

Add your gaming GPU's PCI IDs to VFIO. This binds the GPU to the VFIO driver on boot, making it available for the Windows VM:

# Find GPU PCI IDs:
lspci -nn | grep -i nvidia  # example: 10de:2484

# Add to /etc/modprobe.d/vfio.conf:
options vfio-pci ids=10de:2484,10de:228b

# Update initramfs:
sudo update-initramfs -u
sudo reboot
Create Windows VM in virt-manager

Create a new VM using a Windows 10/11 ISO. During setup, add the GPU PCI device to the VM. Windows sees the real GPU and installs proper Nvidia/AMD drivers. Xenia runs in this VM with near-native performance.

Access the Windows VM via Sunshine

Install Sunshine inside the Windows VM. It sees the passed-through GPU and can stream games. Moonlight clients connect to the VM's IP (or the host's Tailscale IP with port forwarding) exactly as they do today.

💡
Complexity level: GPU passthrough is advanced. It works extremely well once configured, but the setup is finicky and documentation is scattered. Budget a full weekend to get it working. The Arch Wiki has the best GPU passthrough guide in existence, even for Ubuntu users.
Simpler Alternative: Windows VM Without GPU Passthrough

If you only have one GPU, run a Windows VM without GPU passthrough. Xenia won't run well (GPU emulation is too slow), but other Windows-only software (like specific games that work on software rendering) may be usable.

For Xenia specifically, if no GPU passthrough: accept the loss and use the Xbox Game Pass / cloud streaming as a replacement for Xbox 360 gaming. It's not self-hosted, but it's better than nothing.

🚀 Migration Guide: Windows → Ubuntu Server
Install Ubuntu Server 24.04 LTS

Download from ubuntu.com. Boot from USB. Choose "Ubuntu Server (minimized)". Enable SSH during install. Choose static IP matching your current Windows static IP.

Replace Task Scheduler with cron 
crontab -e

# Minecraft 2am shutdown, 3am restart:
55 1 * * * /home/server/minecraft/shutdown_all.sh
0  3 * * * /home/server/minecraft/start_all.sh

# DDNS every 5 minutes:
*/5 * * * * /home/server/ddns.sh
Replace NSSM services with systemd 
# Example for Navidrome:
sudo nano /etc/systemd/system/navidrome.service

[Unit]
Description=Navidrome
After=network.target

[Service]
Type=simple
User=navidrome
ExecStart=/opt/navidrome/navidrome
Restart=always

[Install]
WantedBy=multi-user.target

sudo systemctl enable --now navidrome
Nextcloud: use Snap (much simpler than XAMPP) 
sudo snap install nextcloud
sudo nextcloud.manual-install admin yourpassword
# Everything configured automatically
Docker services (Authentik, Immich, etc.) become first-class

All the services currently running in WSL2 (Authentik, Immich, RomM, Mealie) move to native Docker on Linux. Remove the WSL2 wrapper — everything is cleaner, faster, and uses less memory.

Data migration 
# Mount Windows NTFS drives in Ubuntu:
sudo apt install ntfs-3g
sudo mount -t ntfs-3g /dev/sdX /mnt/media

# Or migrate with rsync over network:
rsync -avz --progress "//192.168.1.100/Media" /srv/media/
