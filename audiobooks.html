Audiobookshelf

// Audiobooks & Podcasts · Port · Native Windows · Caddy forward_auth //

Audiobookshelf is the gold standard for self-hosted audiobook and podcast management. It streams to mobile apps, tracks per-device listening progress, and syncs chapters across sessions. Runs as a native Windows Service — no Docker required.

⚙️ Part A — Install Audiobookshelf NATIVE · NO DOCKER
What You Need
  • Windows 10 64-bit
  • Your audiobooks already on an internal drive at
  • Port free (default — do not change)
🔐 Download the Windows installer

Go to github.com/advplyr/audiobookshelf/releases and download the latest .exe installer (e.g. audiobookshelf-win-x64-installer.exe).

🔐 Run the installer as Administrator

Right-click → Run as administrator. The installer sets up Audiobookshelf as a Windows Service automatically — it starts on boot with no extra configuration.

🔐 Verify the service is running 
# In PowerShell (admin):
Get-Service -Name "AudioBookshelf"

# Or: Task Manager → Services tab → look for "AudioBookshelf"
🔐 Open the web interface and create an admin account

Navigate to http://localhost:. On first launch you'll be prompted to create an admin account.

🔐 Open Windows Firewall for port 
netsh advfirewall firewall add rule name="Audiobookshelf" dir=in action=allow protocol=TCP localport=
🔐 Add your library

In the web UI: Settings → Libraries → Add Library. Set type to Audiobooks and point it at . Repeat for Podcasts if you have a podcast folder (e.g. D:\Podcasts).

🔐 Part B — SSO via Authentik (OIDC)

Audiobookshelf supports OpenID Connect (OIDC) natively. Once configured, users click "Sign in with Google" (via Authentik) on the Audiobookshelf login page — no separate ABS passwords needed.

🔎 Part D — Metadata & Cover Art

Go to Settings → Item Metadata Utils → Match Library Items to batch-fetch metadata from Audible, Google Books, Open Library, and others. You can also right-click individual books → "Match" to search manually.

✨
If you use tone (a free CLI audiobook tagger), pre-tag files before adding them to ABS for perfect metadata from the start.

⚠️ Known Breakpoints
🔴 Windows installer fails silently if an older version is still running
Stop the service before running the installer: Stop-Service AudioBookshelf. Then run the installer, then Start-Service AudioBookshelf.
🟡 OIDC redirect URI breaks after a domain change
If you change your domain, update the Redirect URI in both: (1) Authentik provider settings → Redirect URIs, and (2) ABS Settings → Authentication → Redirect URI. Both must match exactly.
🟢 Podcast episode metadata occasionally fails to fetch
Right-click the podcast → "Re-fetch RSS" to force a refresh. This is cosmetic and doesn't affect playback.

🎯 How Audiobookshelf Achievements Work

Audiobookshelf does not have native outbound webhooks. Achievements are driven by database polling — the achievement server periodically queries the ABS API for listening progress and fires triggers when thresholds are crossed (e.g. a book reaches 95% finished).

💡
ABS Admin Token required: Generate a token in ABS Settings → Users → [admin user] → API Token. Store it as ABS_ADMIN_TOKEN in stats_server.py. The server polls http://localhost:/api/users to map usernames to progress data.
API Polling Pattern (stats_server.py)
# Poll ABS for per-user listening progress:
import requests

ABS_ADMIN_TOKEN = "your-admin-token-here"
ABS_BASE = "http://localhost:"
HEADERS  = {"Authorization": f"Bearer {ABS_ADMIN_TOKEN}"}

def get_abs_users():
    r = requests.get(f"{ABS_BASE}/api/users", headers=HEADERS)
    return r.json().get("users", [])

def check_audiobook_progress(username, user_token):
    headers = {"Authorization": f"Bearer {user_token}"}
    r = requests.get(f"{ABS_BASE}/api/me/progress", headers=headers)
    for item in r.json():
        if item.get("progress", 0) >= 0.95:
            trigger_achievement(username, "audiobook_finished", {
                "title": item.get("libraryItemId")
            })

# Run this on a schedule (e.g. every 30 minutes via Task Scheduler)

👤 Per-User Data
What Audiobookshelf stores per user

Audiobookshelf keeps all listening progress, bookmarks, and playback position in its own SQLite database (C:\ProgramData\audiobookshelf\absdatabase.sqlite), keyed per user account. The audio files are shared from .

What lives in \[username]\audiobooks\:

  • listening_progress.json — nightly export of all listening positions and bookmarks, generated by the stats_server API export route
  • Any personal podcast feeds added by the user (if per-user custom RSS is enabled)
💡
This folder is visible in Nextcloud as "App Data" and counts toward the user's storage quota. See the SSO tab → User Data Storage for the full architecture.

🛠️ Common Issues
❌ Service won't start after Windows update or installer upgrade
Check the Windows Service Control Manager for an error code:
  • Get-Service AudioBookshelf — if status is "Stopped", try Start-Service AudioBookshelf
  • Check logs at C:\ProgramData\audiobookshelf\log\ for startup errors
  • If the port is in use: netstat -ano | findstr : — kill the conflicting PID
  • Re-run the installer as Administrator to repair the service registration
⚠️ SSO login fails / OIDC redirect error
Almost always a redirect URI mismatch. Check:
  • Authentik provider → Redirect URIs must exactly match: /auth/openid/callback
  • ABS Settings → Authentication → Redirect URI must match the same value
  • Restart the ABS service after any config change: Restart-Service AudioBookshelf
⚠️ Mobile app can't connect to server
  • Confirm the service is running: Get-Service AudioBookshelf
  • Confirm the firewall rule is active: netsh advfirewall firewall show rule name="Audiobookshelf"
  • Try connecting to http://[server-LAN-IP]: from the mobile app — use the LAN IP, not localhost
  • For remote access outside the home, ensure VPN is connected before opening the app
⚠️ Books show as unrecognised / metadata not found
ABS is strict about folder naming. Check:
  • Each audiobook should be in its own folder: Author - Title\ or Title\
  • Go to Settings → Item Metadata Utils → Match Library Items to retry metadata fetching
  • Right-click a specific book → "Match" to search manually
💡 Listening progress not syncing between devices
Progress syncs to the server continuously over the network. If a device is offline, it will sync on next connection. Force a sync by closing and reopening the mobile app with an active connection.
