Audiobookshelf Server

// Self-Hosted Audiobooks & Podcasts · No Docker Required //

Audiobookshelf is the gold standard for self-hosted audiobook and podcast management. It streams to mobile apps, tracks listening progress, and syncs across devices.

🐉 V1 Goal: Audiobookshelf running as a Windows Service on port 13378, serving your audiobook collection from an internal drive. Mobile apps connecting over local network (and Tailscale/VPN for remote).
⚙️ Installation — Windows Native NATIVE · NO DOCKER
What You Need
  • Windows 10 64-bit
  • Your audiobooks on an internal drive (e.g. D:\Audiobooks)
  • A free port — default is 13378
Download the Windows installer

Go to github.com/advplyr/audiobookshelf/releases and download the latest .exe installer (e.g. audiobookshelf-win-x64-installer.exe).

Run the installer as Administrator

Right-click → Run as administrator. The installer sets up Audiobookshelf as a Windows Service automatically — it will start on boot without any extra configuration.

Verify the service is running 
# In PowerShell (admin):
Get-Service -Name "AudioBookshelf"

# Or in Task Manager → Services tab → look for "AudioBookshelf"
Open the web interface

Navigate to http://localhost:13378. On first launch you'll be prompted to create an admin account.

Open Windows Firewall for port 13378 
netsh advfirewall firewall add rule name="Audiobookshelf" dir=in action=allow protocol=TCP localport=13378
Add your library

In the web UI: Settings → Libraries → Add Library. Set type to "Audiobooks" and point it at your folder, e.g. D:\Audiobooks. Do the same for Podcasts if you have a podcast folder.

📁 Recommended Folder Structure

Audiobookshelf works best when each audiobook is in its own folder, named Author - Title or just Title.

D:\Audiobooks\
  ├── Fiction\
  │   ├── Brandon Sanderson - The Way of Kings\
  │   │   ├── part1.mp3
  │   │   ├── part2.mp3
  │   │   └── cover.jpg
  │   └── Terry Pratchett - Guards! Guards!\
  │       └── guards_guards.m4b
  └── Non-Fiction\
      └── Matthew Walker - Why We Sleep\
          └── why_we_sleep.mp3

D:\Podcasts\
  └── My Favorite Murder\
      ├── episode_001.mp3
      └── episode_002.mp3
💡
.m4b files (audiobook format) store chapters, cover art, and metadata internally. Audiobookshelf reads all of this automatically. For multi-file audiobooks, add a cover.jpg in the folder.
📱 Mobile Apps
📱 Official Audiobookshelf App

Available on iOS and Android. Free. Connect it by entering your server URL: http://[server-IP]:13378. It tracks progress across devices and supports background playback, sleep timers, and variable speed.

🎙️ Podcast Management

Audiobookshelf also acts as a full podcast manager. It can auto-download new episodes of any RSS feed you subscribe to. Configure under Settings → Podcasts → add RSS feed URL.

🔎 Metadata & Cover Art
Auto-fetching Metadata

Audiobookshelf can fetch metadata from Audible, Google Books, Open Library, and others. Go to Settings → Item Metadata Utils → Match Library Items to batch-update everything. You can also right-click individual books → "Match" to manually search.

✨
Tone2 / Tonal integration: If you use tone (a free CLI tool for audiobook tagging), you can pre-tag all your files before adding them to Audiobookshelf for perfect metadata from the start.
🌍 Remote Access (Outside Your Home)
Option A: Tailscale (Easiest, Free) RECOMMENDED

Install Tailscale on your server and your phone. Your server gets a permanent private IP like 100.x.x.x. The mobile app connects to http://100.x.x.x:13378 from anywhere, securely, without opening ports.

Download: tailscale.com — free for personal use, up to 100 devices.

Option B: Port Forward (Router Config Required)

Log into your router (usually 192.168.1.1), find "Port Forwarding," and forward external port 13378 to your server's local IP on port 13378. Then access via your public IP. Recommend adding a username/password in Audiobookshelf admin settings before doing this.

💾 Backups

Audiobookshelf stores its database and metadata in C:\ProgramData\audiobookshelf (the default config path). Back this up regularly — it contains all your listening progress, user accounts, and podcast subscriptions. Your actual audio files are stored wherever you pointed the library (e.g. D:\Audiobooks).

# Simple robocopy backup to another drive:
robocopy "C:\ProgramData\audiobookshelf" "E:\Backups\abs-config" /E /MIR
🔐 SSO Integration → SSO TAB

Audiobookshelf supports OpenID Connect (OIDC) authentication. This means you can integrate it with Google OAuth or a self-hosted SSO like Authentik. See the SSO tab for full setup. Once configured, users click "Sign in with Google" on the Audiobookshelf login page.
