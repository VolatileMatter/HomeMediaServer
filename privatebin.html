PrivateBin — Zero-Knowledge Pastebin

// Docker · WSL2 · Port 8089 · Per-User Data · Authentik forward_auth //

PrivateBin is a self-hosted pastebin where the server never sees the paste content. Everything is AES-256 encrypted in the browser before being sent — the decryption key is only ever in the URL fragment, which is never transmitted to the server. Safe for sharing passwords, keys, and private notes within the family.

🔒 Zero-knowledge design: PrivateBin's server stores only encrypted blobs. It mathematically cannot read what you paste. The key lives in the URL after the # — share the full link only with the intended recipient.
📁 User Data Storage → SSO TAB

PrivateBin is a shared family service — all users share one instance. Paste data is stored encrypted at rest:

D:\Users\common\privatebin\data\   ← encrypted paste blobs (server can't read these)
D:\Users\common\privatebin\cfg\    ← conf.php configuration file

There is no meaningful per-user data here (the server only stores ciphertext). The common folder is not user-quota-tracked. See the SSO tab → User Data Storage for the full architecture.

⚠️
WSL2 path mapping: Create D:\Users\common\privatebin\ on Windows first, then reference it as /mnt/d/Users/common/privatebin/ in the Docker compose file.
🐳 Deployment — Docker in WSL2
Create data directories (Windows) 
New-Item -ItemType Directory -Force "D:\Users\common\privatebin\data"
New-Item -ItemType Directory -Force "D:\Users\common\privatebin\cfg"

# Set permissions so the container user (1000:1000) can write:
# In WSL2 terminal:
chmod 755 /mnt/d/Users/common/privatebin/data
Create conf.php

Save as D:\Users\common\privatebin\cfg\conf.php:

<?php
; PrivateBin configuration
[main]
; Allow users to create pastes
discussion = true
opendiscussion = false
password = true
fileupload = false
burnafterreadingselected = false
defaultformatter = "plaintext"
syntaxhighlightingtheme = "sons-of-obsidian"
sizelimit = 10485760
template = "bootstrap5"
languageselection = false
languagedefault = "en"

[expire]
default = "1week"

[expire_options]
5min = 300
10min = 600
1hour = 3600
1day = 86400
1week = 604800
1month = 2592000
1year = 31536000
never = 0

[formatter_options]
plaintext = "Plain Text"
syntaxhighlighting = "Source Code"
markdown = "Markdown"

[traffic]
limit = 10
header = "X-Forwarded-For"

[purge]
limit = 300
batchsize = 10

[model]
class = Filesystem

[model_options]
dir = PATH "srv/data"
?>
Create the compose file 
mkdir -p ~/services/privatebin && cd ~/services/privatebin

cat > docker-compose.yml << 'EOF'
services:
  privatebin:
    image: privatebin/nginx-fpm-alpine:latest
    container_name: privatebin
    restart: always
    read_only: true
    user: "1000:1000"
    volumes:
      - /mnt/d/Users/common/privatebin/data:/srv/data
      - /mnt/d/Users/common/privatebin/cfg/conf.php:/srv/cfg/conf.php:ro
    ports:
      - "8089:8080"
    tmpfs:
      - /tmp
      - /run
      - /var/run/nginx
EOF
💡
read_only: true mounts the container filesystem as read-only for security. The tmpfs entries give nginx and PHP-FPM the writable temp space they need without compromising the read-only stance.
Start the container 
docker compose up -d
docker compose logs privatebin   # confirm started

PrivateBin is now running at http://localhost:8089.

Open Windows Firewall for port 8089 
netsh advfirewall firewall add rule name="PrivateBin" dir=in action=allow protocol=TCP localport=8089
Auto-start on Windows boot

Task Scheduler → Create Task → Action:

wsl -d Ubuntu -- bash -c "cd ~/services/privatebin && docker compose up -d"

Trigger: At startup, delay 2 minutes. Run whether logged on or not: ☑

🔐 Authentik — forward_auth Gate
Caddy configuration
# Add to your Caddyfile:
paste.yourdomain.com {
    forward_auth localhost:9000 {
        uri /outpost.goauthentik.io/auth/caddy
        copy_headers X-authentik-username X-authentik-groups X-authentik-email
        trusted_proxies private_ranges
    }
    reverse_proxy localhost:8089
}

🔒 How Zero-Knowledge Works

When you create a paste, the browser generates a random encryption key. Your text is encrypted locally in the browser using AES-256-GCM before any data is sent to the server. The server receives and stores only the ciphertext. The decryption key is appended to the URL after a # — browsers never include URL fragments in HTTP requests, so the server never sees it.

Example paste URL:
https://paste.yourdomain.com/?abc123#DECRYPTIONKEY

The server stores:  abc123 → [encrypted blob]
The server never sees: DECRYPTIONKEY

Share the full URL with your recipient.
Share just the base URL (without the #) and it's useless.
💡
Burn after reading: Enable "Burn after reading" when sharing a password or secret. The paste is deleted from the server the moment it's opened for the first time.
